# 工作者/服务

工作者/服务是一个抽象类，主要负责数据库信息检索操作、API 请求和文件下载：

- 每个工作者/服务应具有单一功能（单一职责原则）。
- 一个交互器可以调用一个或多个工作者/服务。
- 它们必须是通用的，以便可以被不同的交互器使用。

##### 模型

在 VIP 循环中，信息沿一个方向流动：从视图到交互器（以请求的形式），从交互器到展示器（作为响应），再回到视图（作为 ViewModel）。因此，使用与场景相关并包含（清单 6-6）的模型是非常成熟的：

- **请求模型**：包含允许交互器执行操作的参数。
- **响应模型**：包含工作者获取并传递给展示器的适当数据。
- **ViewModel 模型**：通常是原始类型的数据，传递给视图以更新它。

```swift
enum HomeModel {
enum Fetch {
struct Request {}
struct Response {}
struct ViewModel {}
}
}
```
