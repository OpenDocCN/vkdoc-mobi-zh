# 8. 结论

## 整洁架构的重要性

我们已到达本书的结尾。我希望通过您的阅读，您已经深刻认识到，一方面，软件架构在我们应用开发中的重要性，另一方面，该架构应尽可能遵循整洁架构的原则。

我们较为深入地研究了那些在今天可以被认为是 iOS 操作系统应用开发中使用最广泛或需求最迫切的架构：MVC、MVP、MVVM、VIPER 和 VIP。

随后，我们以不那么具体的方式了解了一些架构，它们虽然不那么广为人知或常用，但为我们提供了应用开发的其他观点或方法：RIB、TEA、Redux 和 TCA。

正如我们在研究本书中展示的不同架构时所看到的，并且我们一直在强调在应用开发中使用整洁架构的重要性，使用整洁架构将使我们能够：

*   将业务逻辑隔离在一个层中，使其易于测试。
*   隔离用户界面，使其仅负责显示提供给它的数据。此外，我们必须能够修改它而不影响其余代码。
*   独立于所使用的任何库，这应允许我们更换一个库而不必大幅修改应用程序代码。
*   独立于数据源，这样我们就可以更换数据源而不影响业务逻辑。

简而言之，采用整洁架构开展工作应使我们能够开发出可扩展且可测试的应用程序，任何开发者都可以在这样的应用中工作。
