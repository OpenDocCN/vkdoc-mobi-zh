# 理解 Objective-C 中变量的作用域与生命周期

## 1. 变量的作用域

作用域指的是变量在程序中的可见性和可访问性。在 Objective-C 中，根据变量声明的位置不同，它们拥有不同的作用域：

- **全局变量**：在任何函数或方法外部声明的变量，可以在程序的任何位置被访问。
- **局部变量**：在函数、方法或代码块内部声明的变量，仅在该特定作用域内可访问。
- **实例变量**：实例变量（也称为成员变量）在类接口中声明，并且可被该类的所有实例方法访问。
- **类变量**：类变量（也称为静态变量）在类接口中使用 `static` 关键字声明，并在该类的所有实例之间共享。

## 2. 变量的生命周期

生命周期指的是变量在内存中存在的时间长度。

在 Objective-C 中：

- **局部变量**：仅存在于声明它们的代码块、函数或方法中。在进入代码块时创建，在退出代码块时销毁。
- **实例变量**：只要对象（类的实例）存在，它就存在。在通过 `alloc` 和 `init` 方法创建对象时被初始化，在对象被销毁时（通常当引用计数降至零时）被释放。
- **类变量**：在程序执行的整个期间都存在。当类被加载到内存时被初始化，在程序终止时被销毁。

**示例 - 变量的作用域与生命周期：**

```
@interface MyClass : NSObject {
    int instanceVariable; // 实例变量
}
@end

@implementation MyClass
- (void)exampleMethod {
    int localVar = 10; // 局部变量
    // 访问实例变量
    instanceVariable = 20;
    // 打印局部变量
    NSLog(@"局部变量: %d", localVar);
}
@end
```

在此例中，`instanceVariable` 是一个实例变量，可被 `MyClass` 的所有实例方法访问。`localVar` 是一个局部变量，其作用域仅限于 `exampleMethod` 方法。

Objective-C 中的方法对于将行为封装在对象内部至关重要，有助于实现面向对象编程原则，例如封装、抽象和继承。理解变量的作用域和生命周期有助于你在程序中高效地管理数据，确保合理的内存管理和逻辑结构。通过掌握方法、作用域和变量生命周期，你可以开发出模块化且可扩展的健壮高效的 Objective-C 应用程序。
