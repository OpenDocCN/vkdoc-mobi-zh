## 前言

我第一次接触 Cocoa 是在 1989 年左右，那时我还是一名大学生。当然，那是在 iPhone 出现之前，在 Mac OS X 出现之前，甚至在它被称为 Cocoa 之前。当时，今天 Cocoa 的种子是 NeXTStep 的一部分，那是 NeXT 电脑的核心操作系统。NeXTStep 领先于它的时代数年，虽然那个装满 NeXT 工作站的实验室在计算机科学课程中未被充分利用，但我的学生系统管理员工作让我每天都使用它们。作为一名用户，我彻底着迷了。我不想在这里过多赘述 NeXT 的用户体验，但我要说明的是，Mac OS X 的许多最佳特性与其说来自古老的 Mac，不如说来自 NeXTStep。

那时，还没有万维网，NeXTStep 开发者社区也不大，除了 NeXT 随其早期机器附带的那些晦涩难懂的巨著之外，关于开发环境的文献非常稀少。我时常试图弄懂 Objective-C 和 AppKit，但由于身边没有专家，也几乎没有可供参考的示例代码（更不用说我的实际学业有时会分散我玩弄有趣项目的精力），我基本上是一筹莫展。

大学毕业后，一件完全出乎意料的事情发生了。一位朋友向我介绍了我们城市的一家咨询公司，这家公司正在为一些大客户构建定制的 NeXTStep 应用，我有幸加入了该公司。突然间，我身边有了一群同事，他们不仅已经使用 NeXTStep 编程了一段时间，有些人甚至在 NeXT 工作过！其中一位就是 Peter Clark。只需他们一点专业帮助就能让我入门，那些多年来显得如此神秘的东西突然之间就变得清晰了。在几周内，我学到了很多东西，甚至能够开始领导一些 NeXTStep 开发方面的培训和指导工作。

我这里要说的重点不是我有多天才或学得有多快。而是我们现在称之为 Cocoa 的那套技术真的非常强大——而且相当容易学习和应用——但你在学习过程中很可能需要一些帮助。我希望这本书能帮你指明正确的方向，并帮助你学习 Cocoa 编程的精髓，这样当你读完它时，你将拥有足够的 Cocoa 知识，能够推动自己前进，并编写你梦想中的 Mac 应用程序。简而言之，这就是那本我希望 20 年前就能找到的书。

——杰克·纳廷

## 关于作者

**杰克·纳廷** 早在 Cocoa 被命名为 Cocoa 之前的旧时代就开始使用它。他使用 Cocoa 及其前身为各种行业和应用开发软件，包括游戏、平面设计、在线数字发行、电信、金融、出版和旅游。当他不从事 Mac 或 iOS 项目、不用 Unity3D 制作炫酷游戏、或者不和他的朋克摇滚乐队一起摇滚时，他通常会与家人共度时光。杰克在 Twitter 上是 `@jacknutting`，并偶尔在 `www.nuthole.com` 上写博客。（照片由 Alison De Mars von Blixen 提供。）

**彼得·克拉克** 大约在 1990 年大学期间开始接触 NeXTStep。这段经历为他职业生涯的前半段定下了基调，当时他与杰克在双子城的一家咨询公司共事。他与湾区的公司、芝加哥和双子城的公司合作过 NeXTStep 和 WebObjects 项目，写过一本关于 OpenStep 开发的书，随后又转向了 Java、.NET、Ruby 和 Scala 开发以及管理工程团队。他很高兴 NeXTStep 的许多设计模式至今仍然有用，并发现 Cocoa 开发既让人感到熟悉舒适，又常有令人愉悦的惊喜。彼得目前是明尼苏达大学一个研究项目的软件发展总监。他偶尔在 `http://blog.pclark.net` 上写博客，也可以在 Twitter 上找到他，用户名为 `@pclark`。彼得想让你知道，他至少比杰克早一两个月开始使用 NeXTStep。（照片由 Wade Stebbings 提供。）
