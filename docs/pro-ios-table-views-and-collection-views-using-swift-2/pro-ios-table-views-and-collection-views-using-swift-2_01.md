# 引言

如果你是 iOS 应用开发者，那么在开发项目中你很可能会用到表格视图和集合视图。它们是 iOS 应用的基础与核心。借助它们，你可以创建从最简单列表到功能完备用户界面的各种内容。

表格视图和集合视图是 UIKit 中较为复杂的两个组件。将它们用于（可能乏味的！）标准用户界面相当简单，但自定义它们则更具挑战性。

本书以任务为导向，旨在帮助你在实现自定义表格视图和集合视图时获得支持。尽管本书深入探讨了 API，但你始终可以选择自己想要的详细程度。本书旨在同时作为参考手册和自定义指南，对初学者和中级开发者均适用。


## 本书涵盖内容

第 1 章《表格视图快速入门》通过一些当前技术水平的示例介绍了表格视图。在展示了一些可能实现的效果之后，我们将从一个简单的基于表格视图的 iPhone 应用程序开始，这将向您介绍`UITableView`及其主要元素。该应用程序还将作为后续版本的基础，并作为一个可工作的原型，供您在自己的实验中使用。

在第 2 章《表格的组成结构》中，您将了解表格视图各部分如何协同工作。您将看到`UITableViews`的主要类型及其结构。您将学习如何使用 Interface Builder 和代码创建它们，以及如何将`UITableViewController`类用作模板。

第 3 章《集合视图快速入门》将焦点转移到`UICollectionView`，展示这种强大且无比灵活的控件所能实现的功能。与第 1 章类似，我们将从一个简单的示例开始，该示例可作为更高级主题的起点，并提供一个可工作的原型作为您实验的基础。

第 4 章《集合视图的组成结构》探讨了集合视图各部分如何协同工作。您将看到它们与`UITableViews`相比有多相似——但更加灵活——以及其结构组件。您将学习如何使用 Interface Builder 和代码创建它们，以及如何将`UICollectionViewController`类用作模板。

第 5 章《向视图提供数据》介绍了表格和集合视图从何处获取数据以及如何将数据传入。它展示了这些视图如何跟踪分区和行，并涵盖了`UITableView`和`UICollectionView`类所利用的一些软件设计模式。

第 6 章《表格单元格的组成结构》聚焦于构成表格的单元格。您将了解单元格的内部结构，以及它们是如何创建和重用的。此外，它还涵盖了`UITableView`类免费提供的标准单元格类型。

在第 7 章《改善单元格外观》中，您将开始探索超越标准单元格类型、自定义表格视图外观和感受的过程。本章涵盖了一些让单元格呈现所需外观的最快捷方法。

第 8 章《使用子类创建自定义单元格》介绍了在表格和集合视图中自定义单元格各个方面最强大、最灵活的方法。能力越大，责任越大——但值得坚持，以便能够完全掌控应用程序的外观、感受和行为。

第 9 章《改进交互》将引导您完成在表格和集合视图中嵌入交互控件、实现下拉刷新、利用手势识别器以及实现视图内搜索的过程。

第 10 章《使用表格进行导航》涵盖了 iOS 用户界面中几乎无处不在的一个功能，并展示了如何以简单一致的方式使用表格浏览分层数据。

当需要呈现大量数据时，iOS 用户界面受限的尺寸会带来一些挑战。第 11 章《索引、分组和排序》提出了一些在表格中排列数据的方法，以帮助用户找到所需内容。

第 12 章《选择和编辑内容》展示了如何使用表格和集合视图来管理数据。它涵盖了如何添加、删除和重新排列信息，以及由此带来的一些界面方面的问题。

第 13 章《静态表格》介绍了表格视图一个常被忽视的用途——以静态方式呈现控件和信息。这些表格通常可作为简化复杂布局的捷径。

第 14 章《WatchKit 中的表格》探讨了在苹果首次涉足可穿戴技术时，基于表格的界面所面临的挑战。Apple Watch 的小尺寸和低功耗带来了一些挑战，但有效利用表格可能是实现引人入胜的用户交互的关键。

第 15 章《集合视图流式布局》介绍了`UICollectionView`家族中最有用的组件之一，它允许您以极少的复杂计算，非常快速地构建复杂的线型布局。

当基于线的布局不足以实现您所需的界面时，可以完全自定义集合视图。第 16 章《集合视图自定义布局》探讨了如何完全掌控集合视图外观和感受的每个方面。

第 17 章《动画与交互式集合视图》将集合视图推向了交互界面的可能极限。将集合视图与手势识别器和自定义布局相结合，您可以创建出仅受想象力限制的界面。

## 本书风格

我试图在两种书籍风格之间架起一座桥梁——一种是深入探讨每一个微小细节的风格，另一种是提供特定解决方案的“烹饪手册”风格。两者都各有用处，但有时我发现，那些描述非常详细、功能丰富且优雅的解决方案可能会掩盖我正在尝试解决的问题的细节。同样，有时“烹饪手册”式的解决方案又过于具体，不易适应我的特定情况。

在随后的代码示例中，我尝试平衡这两种风格。视觉上的修饰和无关紧要的功能被控制在最低限度，这有望产生既演示如何构建解决方案，又能作为您自己代码构建块的示例。

## 关于第二版

技术世界没有任何东西会长期停滞不前，苹果的框架也不例外。第一版于 2012 年出版，那时 iPad、集合视图、iOS 设计语言的彻底革新以及软件发展领域长期来最激动人心的进展之一都尚未出现。

随着 Swift 在 2014 年的推出，苹果吸纳了现有无数编程语言和范式中所有最精华的部分，并将其与传奇般的框架和 API 相结合，创造出了一种完全配得上用于描述它的所有赞誉之词的产品。Swift 在 2015 年底的开源将在未来数月乃至数年内带来一些非常有趣的发展。

我多年来一直愉快地使用 Objective-C 进行开发，但随着 Swift 的出现，我们已经实现了巨大的飞跃。本书专门使用 Swift，因为我猜想在可预见的未来，我和许多其他 iOS 开发者都将这样做。

本书撰写时使用的是 Swift 2.0、WatchKit 2.0 和 Xcode 7.1。在涵盖一个动态且快速变化的世界时，本书已尽可能做到最新和与时俱进——当情况发生变化时，更新和勘误表将从 Apress 网站以及本书在 GitHub 上的源代码中获取。



## 本书源代码

你可以从 Apress 网站（`www.apress.com/9781484212431`）或 GitHub 上的 `https://github.com/timd/ProiOSTableCollectionViews` 下载每一章示例的源代码。

虽然这是快速上手的最直接方式，但我还是鼓励你花点额外时间，在学习过程中亲自输入代码。借助 `Xcode` 的代码补全功能，这并不会花费太多时间，而且经过你眼睛和大脑处理、再由指尖输出的代码，更有可能被你深刻理解并掌握。

## 更多资源

除了本书内容，网上还有大量其他信息可供参考（更不用说 Apress 出版的一系列精彩著作了）：

- 作为总体概述，苹果的“表格视图编程指南”和“集合视图编程指南”是覆盖本书大部分主题的详细指南。这两份资料都可以在 Apple 开发者门户或 `Xcode` 的文档中找到。
- Apple 的 iOS 开发者库提供了所有 Cocoa Touch 库以及 Swift 语言的完整文档。虽然文档本身通常不包含示例，但该库是查找每个类、协议、库和语言详细参考的一站式资源。
- 在线论坛是极好的资源。诸如 Stack Overflow（`www.stackoverflow.com`）这类网站是寻求实用建议的好去处。很可能已经有很多人遇到过并解决了你正在面对的问题，答案就在那里。Stack Overflow 的习俗和惯例最初可能有点令人生畏，但值得坚持。毕竟，没有愚蠢的问题，只有尚未得到解答的问题。
- 在谷歌上做一般搜索，通常能从博客中找到答案。网上有一些极具天赋的开发人员，他们会定期发布关于如何使用 iOS 和 Objective-C 实现各种功能的文章，其中很多人还会在自己的网站、GitHub 或类似平台上提供源代码。
- Apple 也提供了一些相当详细的源代码示例。这些示例的质量可能参差不齐。我有时会发现它们可能有点过于复杂，反而掩盖了我试图掌握的核心技术。但你不应忽视它们，哪怕仅仅是因为它们是由对框架有深入理解的工程师编写的。
- 像斯坦福和麻省理工这样的大学，会在其网站和 iTunes U 上发布整个学期的讲座模块。他们的技术教育是全球顶尖的，有些在线讲座还是由现任或前任 Apple 工程师讲授的。这些内容绝对值得一看。
- 世界各地都有本地用户组定期聚会。软件行业有一条铁律：总有人在某个主题上比你懂得更多，当你和他们边喝饮料边讨论时，问题也总显得不那么棘手。
- 邮件列表在过去几年里经历了复兴。一些优秀的例子（它们通常不仅涵盖编码主题，还涉及设计和商业问题）包括 `http://iosdevweekly.com`、`http://natashatherobot.com` 和 `http://ios-goodies.com`

最后，如果你曾与某个棘手问题作斗争并解决了它，那么请亲自发布出来，无论是在自己的博客还是像 Stack Overflow 这样的网站上。即使该主题之前已被讨论过无数次，对于同一个问题，也总有空间容纳另一种见解。你独特的视角也许正是别人所需要的。
