# 在树莓派上搭建网页服务器

在万维网（即你今天所知的基于浏览器的*互联网*）早期，网页服务器的概念对大多数开发者而言是透明的。你只需安装一个程序来托管你的 HTML 文件（或者在 GeoCities 等服务平台免费注册一个账号），然后大部分时间都在思考文字该选多大字号、哪里能找到最炫的 GIF 图片。对多数人来说，网页服务器不过是上传文件的地方。

随着网页开发技术持续进步，越来越多人开始需要自主掌控网站托管，这使得人们对网页服务器技术的认知随之提升。与其依赖托管服务商承诺的安全性和在线时长保证，你完全可以亲手配置服务器，将主动权掌握在自己手中。

此外，电子商务和电子病历系统等更先进的应用程序开始兴起，它们对用户个性化、数据库集成和安全性都提出了各自的要求。为满足这些需求，PHP 和 Ruby 等脚本语言逐渐流行起来，让程序员能够创建页面，在用户通过浏览器请求页面后立即完成数据处理和静态页面生成的全部逻辑。人们开始开发*网页应用程序*，而不再局限于静态网页。

在物联网的世界里，你可以利用网页应用程序技术，通过*HTTP*（超文本传输协议）公开物联网设备的数据。每当你使用网页浏览器或连接服务器的应用程序时，它都在使用 HTTP 传输数据。正如你可能从前几章了解到的，Bluetooth 或 HomeKit 等协议针对特定场景进行了优化，有时需要大量的领域专业知识才能成功部署。而网页技术是一个众所周知的平台，拥有强大的开发社区和较低的硬件要求——你只需要一台能运行网页服务器的设备即可。

在本章中，你将重温前几章关于树莓派和 Node.js 开发环境的内容，并对其进行扩展：使用 Node.js 的 Express 模块，通过 HTTP 和 HTTPS 发布相同的数据。由于 Node 的二进制文件体积小、可扩展性强，且拥有庞大的开发者群体，它已成为一等公民式的网页应用程序脚本语言。

## 学习目标

在本章中，你将学习如何在你一直使用的树莓派上搭建网页服务器，通过 HTTPS 端点发布传感器数据，并从 iOS 版 IOTHome 应用连接该服务器。你将使用 Node.js 创建网页应用程序，并使用 Swift 开发 iOS 应用。虽然已有一些项目正致力于让 Swift 在树莓派上运行，但在当前环境下，Python 和 Node.js 是在树莓派上构建基于网页的应用程序时文档最完善、支持度最好的选择。

在本章中，你将学习如何使用 Node.js 的 Express 模块来完成监听 HTTPS 连接并路由到 Node.js 的任务，而不是依赖 Apache 这样的完整网页服务器应用程序。如果你计划在同一服务器上运行多个网页应用程序，可能需要考虑使用 Apache 或 NGINX 来更好地管理连接。不过，本章的项目旨在模拟单一用途物联网设备或单一用途网页微服务的用例，例如你在 Heroku 或 Amazon Web Services Lambda 上托管的服务。

在构建树莓派网页服务器的过程中，你将学习 iOS 物联网应用开发的以下核心概念：

-   网页应用程序开发核心概念
-   设置 Express 以通过 Node.js 公开网页服务
-   在 Node.js 应用程序中读取温度传感器的数据
-   在 Node.js 应用程序中读取蓝牙传感器的数据
-   使用 HTTPS 提供安全性保障

本章的项目主要基于 Node.js 的 Express 模块。如果你想了解更多关于该模块的信息或获得更多支持，请访问其官方网站：[`www.expressjs.com`](http://www.expressjs.com)。

## 创建用于通过 HTTPS 共享数据的网页服务器

在本节中，我将重点介绍使用 Node.js 和 Express 搭建网页服务器的过程，以及如何使用 Node 模块通过 HTTPS 公开 IOTHome 传感器（温度、湿度、开关状态）生成的数据。本节中的项目假设 Node.js 和第 7 章的温度传感器已正确设置。如果你对任一方面仍感觉不熟悉，或者你是直接开始本章的学习，我强烈建议你在继续本章之前，先回顾第 7 章的相关设置章节。

在本节中，你将首先实现一个通过 HTTP 传输数据的网页服务器，然后学习如何为服务器应用 SSL 证书，使其成为 HTTPS 服务器——这是网页应用程序新的安全标准。成功搭建网页服务器后，你将使用它为 IOTHome 应用程序的主屏幕提供数据。

### 使用 Express 公开网页服务

与 HomeBridge 项目（主要任务是安装和配置由第三方开发的 Node 应用程序）不同，在本章中，你将从头开始开发自己的 Node 应用程序。要开始开发项目，首先确定项目的位置。打开树莓派并开启终端后，在你的主目录下创建两个新目录：`sites` 和 `iothome`。这些目录分别对应大多数托管 Linux 发行版用作网页应用程序*文档根目录*（起始点）的 `sites` 或 `/var/www/` 目录，以及应用程序本身的位置。如列表 8-1 所示，创建新目录，然后使用 `cd` 命令将工作目录切换到 `iothome` 目录。

```
mkdir ~/sites
mkdir ~/sites/iothome
cd ~/sites/iothome
```

接下来，你必须使用 `npm` 在 `iothome` 项目的目录中安装 Express。在第 7 章中，你全局安装了 HomeBridge 的模块，这意味着树莓派上所有基于 Node 的应用程序都可以使用它们。然而，对于应用程序开发，我建议仅在你当前正在处理的项目中安装模块。在多应用程序环境中，这样可以防止对其他项目产生副作用。要为 IOTHome 项目安装 Express，运行不带全局标志的 `npm install`。

```
npm install express
```

为了验证安装，你可以编写一个简单的 Node 应用程序，使其通过 HTTP 回显 "Hello World"。要开始开发过程，使用 nano 或你喜欢的文本编辑器创建一个名为 `app.js` 的新 JavaScript 文件。

```
nano app.js
```

与 iOS 或 Arduino 编程（应用程序从非常具体的方法开始执行，例如 iOS 的 `viewDidLoad`、Arduino 的 `setup`）不同，Node 应用程序会立即开始执行。由于 Node 实现了面向对象编程（OOP）概念，你应该使用这些概念以可预测的方式控制程序流程。要使用 Express，你必须先引入该模块，然后实例化它的一个对象。在 Express 运行的同时，你可以继续其他操作，但所有 HTTP 请求都必须通过 Express 对象进行处理。要创建你的空白 Express 应用程序，实现列表 8-2 中的代码示例。

```javascript
var express = require('express');
var app = express();
```
