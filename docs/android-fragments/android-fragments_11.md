# 第 2 章：响应配置变更

当应用程序在设备上运行且设备配置发生变化时（例如，旋转 90 度），应用程序需要相应地做出响应。新配置很可能看起来与之前的配置不同。例如，从竖屏模式切换到横屏模式意味着屏幕从又高又窄变为又矮又宽。UI 元素（按钮、文本、列表等）需要重新排列、调整大小甚至移除，以适应新的配置。

在 Android 中，默认情况下，配置变更会导致当前 Activity 被销毁并重新创建。应用程序本身会继续运行，但它有机会根据配置变更来改变 Activity 的显示方式。在极少数情况下，如果需要在不销毁和重新创建 Activity 的情况下处理配置变更，Android 也提供了相应的方法。

请注意，配置变更的形式多种多样，不仅仅是设备旋转。如果设备连接到扩展坞，这也是一种配置变更。改变设备的语言同样如此。无论新的配置是什么，只要您为相应配置设计了 Activity，Android 就能处理大部分过渡工作，为用户提供无缝体验。

本章将从 Activity 和 Fragment 两个角度，带你了解配置变更的全过程。我们将向你展示如何为这些过渡设计应用程序，以及如何避免可能导致应用程序崩溃或行为异常的陷阱。
