# Xamarin.Forms：全面指南

Xamarin 本来就已经非常出色了。我非常热爱 MonoTouch（`Xamarin.iOS` 的前身产品名称），以至于在 Xamarin 成为一家公司之前（后来，我与 Nat、Miguel de Icaza 以及 Joseph Hill 共同推动其发展，并成为了执行团队的一员），我就已经为此写了一本书。能用同一种语言 `C#`（这个星球上我最喜欢的语言）来为 iOS 和 Android（当时的 Windows Mobile 还处于起步阶段）构建原生移动应用，并共享所有后端和逻辑代码，这一切还能在 Mac（我那时钟爱的操作系统）上完成，这难道不酷吗？这非常酷。但这也意味着一个巨大的机遇：虽然你可以用 Xamarin 共享大量代码，但即便使用了 MVVM 框架，统一 UI 代码仍然是一个巨大的挑战。

“这是一张画布，它使用 iOS 和 Android 上的原生 UI 工具包绘制了一个方框；这是代码，”Jason 说。果然，同一个 iOS 和 Android 项目都使用相同的代码来渲染这个方框。于是，`Xamarin.Forms` 诞生了。在确定 `Xamarin.Forms` 这个品牌之前，我们使用了 Duplo 这个代号，当时这很贴切：`Xamarin.Forms` 就像一个儿童积木玩具。你可以用它搭建一些基础的玩具应用，谈不上复杂，但即便如此，Jason 所创造的价值已显而易见。`Xamarin.Forms` 迅速演进，如今已几乎找不到两年前那个深夜 Jason 向我展示的原型框架的影子。

如今，`Xamarin.Forms` 是一个极其强大的框架。通过自定义渲染器扩展它，以提供原生、平台特定的功能，在 UI 层面几乎没有什么是你做不到的。

现在，你手中（无论是实体书还是电子书）拿着的是第一本非 Xamarin 官方出版的关于 `Xamarin.Forms` 的书（别担心，它也涵盖了 `Xamarin.iOS` 和 `Xamarin.Android`，让你可以深入接触原生的底层 UI）。

我对这本书感到非常兴奋。一年前 Dan 第一次联系我谈及这本书时，他告诉了我他的愿景：一本内容详尽、深入，并且叙事质量上乘的书，其精神与我那本（现在已严重过时的）MonoTouch 处女作一脉相承。一本能公正地展现这个平台的书。在阅读完本书后，我相信你会认同 Dan 独特的视角和对平台清晰的理解，以一种非常实用的方式呈现出来；他创建了一本关于使用 Xamarin 的可靠手册，我相信，无论你是 Xamarin 开发新手还是老手，都会欣赏它。

这本厚重的书提供了清晰的指导，将帮助你构建高质量的应用程序，从架构考量开始，然后深入到实用的代码策略。它以 `Xamarin.Forms` 为主导，但也教你如何穿透到原生 UI 工具包，并且不仅解释了事情如何做，还解释了为什么这样做。

那么，我将你交到能干的作者手中，并祝你在 Dan 的得力指导下，通过 Xamarin 构建应用的旅程愉快。

> ——Bryan Costanich  
> Xamarin 公司副总裁

## 简介

本书是一本实用的 `Xamarin.Forms` 入门指南，也是使用 `C#` 和 .NET 构建原生 Android、iOS 和 Windows Phone 应用的跨平台参考书。

如果你将 Xamarin 平台视为一个金字塔，`Xamarin.Android` 和 `Xamarin.iOS` 是它的基座，`Xamarin.Forms` 在其顶端，那么这正是本书使用 `C#` 所涵盖的内容。移动 UI 占据了金字塔的绝大部分，本书探索了使用 Xamarin 布局、控件和列表的重要概念、元素和实用方案。

许多新 Xamarin 项目面临的燃眉之问是：`Xamarin.Forms` 适合我的项目吗？本书涵盖了在选择 `Xamarin.Forms` 方案与使用 `Xamarin.Android` 或 `Xamarin.iOS` 的平台特定方案进行对比时，需要考虑的关键因素。

当你遇到了 `Xamarin.Forms` 开箱即用功能的限制时，你将希望通过使用自定义渲染器来定制 `Xamarin.Forms` 控件，从而利用平台特定的功能。

你还会学习所有关键的 Xamarin UI 导航模式：分层导航和模态导航、下钻列表、选项卡、导航抽屉等。你可以使用提供的导航代码，为几乎任何业务应用搭建出基本骨架。

本书是一本 SQLite 数据访问指南。我们将介绍在 Xamarin 应用中访问 SQLite 数据库的最常见方法，以及如何构建数据访问层（DAL）。一旦建立了数据库，你将希望将数据绑定到 UI。你可以手动绑定，或者使用 `Xamarin.Forms` 的数据绑定将 UI 元素绑定到数据源。我们将介绍多种读写数据绑定的技术，既包括绑定到数据模型，也包括为模型-视图-视图模型（MVVM）架构绑定到视图模型。

构建一个应用不仅仅需要 UI 和数据访问；你还需要将代码组织成专业级的架构。我们将探索从入门级到企业级的解决方案构建技术，以帮助你解耦功能层、管理平台特定代码，并共享跨平台类，以实现最佳的代码复用性、可测试性和可维护性。

## 章节内容

本书所有章节均跨平台，但内容侧重于 Xamarin.Forms。UI 章节（第 3 章至第 6 章）在章节开头以小型索引的形式并列展示了 Xamarin.Forms、Xamarin.Android 和 Xamarin.iOS 的元素，以便于理解跨平台概念，并在需要时更容易考虑自定义渲染器。这些 UI 章节的第一部分为 Xamarin.Forms，第二部分为 Xamarin.Android，第三部分为 Xamarin.iOS。

第 1 章 — 使用 Xamarin 进行移动开发

介绍 Xamarin 平台，涵盖本书所有关键主题。

第 2 章 — 构建移动用户界面（Xamarin.Forms 入门）

Xamarin.Forms 入门指南，并对比 Xamarin.Forms 与特定平台方法（如 Xamarin.iOS 和 Xamarin.Android）。涵盖 Xamarin.Forms 的页面、布局和视图。

第 3 章 — 使用布局进行 UI 设计

布局帮助我们组织控件的位置和格式，使我们能够构建和设计移动应用的屏幕。

第 4 章 — 使用控件（视图）进行用户交互

选择器、滑块、开关和其他移动 UI 控件可促进用户交互和数据输入，这些交互和输入对于移动用户界面来说是独特的，并且由于手势的使用而与 PC/鼠标界面不同。

第 5 章 — 创建可滚动列表

列表是移动应用中最简单、最强大的数据显示和选择方法之一。

第 6 章 — 导航

导航允许用户遍历应用、在不同屏幕之间切换以及访问功能。分层导航、模态导航、导航抽屉、向下钻取列表和其他关键模式构成了移动 UI 导航的核心。状态管理是指在用户导航应用时处理屏幕之间传递的数据。

第 7 章 — 使用 SQLite 和数据绑定进行数据访问

`SQLite` 是许多 Xamarin 开发人员的首选数据库。使用 `SQLite.NET` 或 `ADO.NET` 在本地存储和检索数据。使用 Xamarin.Forms 数据绑定，将 UI 元素与数据模型融合。通过绑定到视图模型来使用 MVVM 模式。

第 8 章 — 自定义渲染器

扩展标准的 Xamarin.Forms 控件，利用特定平台的 UI 功能集，同时通过自定义渲染器保持跨平台方法。

第 9 章 — 跨平台架构

通过管理特定平台的代码来构建跨平台应用的架构。Xamarin 中的项目级别选项包括可移植类库 (PCL) 和共享项目。跨平台编码技术包括条件编译、依赖注入 (DI) 和文件链接。
