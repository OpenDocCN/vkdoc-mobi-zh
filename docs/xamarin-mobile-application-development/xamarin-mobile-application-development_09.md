# 摘要

商业应用正经历着硬件转型的巨大变革；自个人电脑商业化以来，我们从未见过如此规模的变化。消费级移动设备的势头已达到临界点，这影响着商业应用现在必须在其上运行的设备。随着移动操作系统之间的竞争持续激烈，仅仅让一款移动应用在单一平台上运行并正常工作已经不够了。

我们必须从一开始就考虑跨平台。

在 .NET 的世界里，Xamarin 为我们提供了工具，使跨平台开发成为常态而非特例，因此我们没有任何借口。如果采用恰当的方法，业务逻辑、数据访问层，甚至越来越多的 UI，都基本上是平台无关的。因此，无论你是构建 Android、Windows Phone 还是 iOS 应用，其许多组件的开发方法在很大程度上可以是相同的。

让我们来看看代码！
