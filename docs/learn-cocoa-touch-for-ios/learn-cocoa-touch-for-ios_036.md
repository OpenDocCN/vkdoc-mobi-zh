# 使用 Blocks 编写现代代码

如果说苹果在开发 Cocoa Touch 相关开发工具和语言时有一种趋势的话，那就是它正竭尽全力让开发者能轻松创建引人入胜、易于使用的应用。似乎每一个 iOS 新版本发布，都会为 Cocoa Touch 开发者带来更多便利。其中一个例子出现在 iOS 4.0 和 Mac OS X Snow Leopard 中，它是苹果在 C 语言中新增的一个特性：blocks。C 语言的发展通常缓慢如冰河，因此任何新特性都因其稀有而成为新闻。而 Blocks 带来的兴奋远不止于此：它赋予代码更大的自由度、更合理的逻辑分组以及更好的封装性。在本章中，我们将讨论什么是 blocks、它们如何工作，以及你为何会想使用它们。我们还会介绍苹果要求使用 blocks 的一些新 API，你在第 6 章的 Twitter 示例中已经略见一斑。最后，我们将重新审视那个示例，扩展其功能，把它变成一个更好的应用。首先，我们来聊聊 blocks 究竟是什么。

## 什么是 Blocks？

简单来说，block 是一段自包含的代码。与函数或方法类似，block 可以接受参数并返回值，但与函数或方法不同的是，它们是匿名的；也就是说，你可以在不为其命名的情况下创建并执行一个 block。通常，block 代表一个离散的任务，无论是比较两个对象、执行动画，还是调整图像大小。从这个角度来看，blocks 在为代码引入多线程处理方面扮演着重要角色；本章我们将初步体验这一点，并在下一章进行更深入的探讨。

Blocks 也常用于延迟代码执行；除了传递目标-动作对、或对象与方法选择器之外，你还可以传递一个 block，在其他任务完成后执行它。这一切既便捷又高效，而苹果还为 blocks 添加了一些内存管理魔法，使其使用起来更加轻松。
