# 第 9 章：用户界面设计

代表了视图内部的坐标系（与`frame`属性相反，后者表示视图在其父视图坐标系中的位置和大小），上文那个子视图的`bounds`属性，其`size`成员的`width`值为`100`，`height`值为`100`，但它的`origin`成员将是点`(0, 0)`。事实上，由于所有视图的坐标系都是从`(0, 0)`开始的，每个`UIView`对象的`bounds`属性默认将具有相同的原点。你可以更改`bounds`属性的原点，以调整视图将要显示的部分。

**注意：** 尽管`UIView`类有一个`center`属性，它返回视图的中心点，但你应该将其视为只读属性。如果你试图在尺寸为奇数的视图上设置`center`属性，可能会得到一个不在偶数像素边界上的原点，例如`(55.5, 102.5)`。这会由于渲染缺陷而导致你的视图看起来模糊。同样，在设置视图的`frame`属性时，始终使用整数以避免渲染问题，或者使用`floorf()`或`roundf()`将你使用的数值强制转换为整数。

### 视图显示属性

`UIView`类定义了几个属性，你可以用它们来自定义视图的外观和行为，而无需创建`UIView`的子类。有些属性很直观，例如`backgroundColor`，它是一个`UIColor`对象，表示视图的背景颜色；以及`alpha`，它是一个`CGFloat`类型，用于表示对象的透明度，取值范围从`0`（完全透明）到`1`（完全不透明）。要给视图设置蓝色背景并使其半透明，你可以设置属性如下：

```
[myView setAlpha:0.5f];
[myView setBackgroundColor:[UIColor blueColor]];
```

**注意：** `UIView`类中定义的属性之一是`opaque`，它是一个`BOOL`值，作为给渲染引擎的一个提示。如果视图完全用不透明的内容（即没有任何透明度的内容）填充其边界，那么它应该被视为不透明，因此其`opaque`属性应设置为`YES`（默认值）。

对于任何半透明或未完全填充其边界的视图，你应该将其`opaque`属性设置为`NO`。正确设置`opaque`属性可以改善绘制性能，并避免不可预测的副作用。
