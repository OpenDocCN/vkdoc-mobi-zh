# 第 5 章：处理用户触摸

iPhone 与其之前的每一个计算平台相比，在许多方面都发生了巨大变化，但最显著的变化莫过于它没有鼠标或键盘。iPhone 不再响应键盘按键、鼠标点击和鼠标悬停事件，而是响应触摸。围绕触摸设计用户界面不仅仅是简单地将点击转换为轻触。iPhone 提供了直接操控：当你将手指放在“邮件”中的一条消息上并向下移动一英寸时，消息内容也会随之向下移动一英寸。直接操控有效地消除了用户与内容之间的一层抽象，这层抽象可以追溯到最初的图形用户界面（GUI）：滚动条。因此，iOS 和其他触摸屏操作系统比它们的桌面前辈更易用、更直观。本章将向你展示如何在应用中使用触摸，将枯燥的用户体验转变为用户喜爱的丰富、沉浸式应用。我们将讨论多种使应用响应用户触摸的方法，包括自定义视图和手势识别器，并更新我们的 MyStuff 示例应用，添加一些新的用户界面元素。

## 响应者链

如果你查看 `UIView` 的文档，你会发现 `UIView` 继承自 `UIResponder`。要查找文档，请在 Xcode 中按 `⌘+Option+2` 或选择“窗口”➤“管理器”来打开管理器窗口。选择工具栏上的“文档”图标，你将进入文档浏览器。要查找 `UIView` 的文档，我们将在文档库中进行搜索。要打开搜索面板，请点击左侧窗格中的放大镜图标，按 `⌘+Option+2`，或选择“帮助”➤“文档和 API 参考”。在搜索框中输入 `UIView` 即可开始搜索。如果找不到任何结果，你可能没有搜索 iOS 库；点击搜索字段文本框中的沙漏图标，然后选择“显示查找选项”。确保在“文档集”旁边出现的下拉列表中启用了某个 iOS 库。
