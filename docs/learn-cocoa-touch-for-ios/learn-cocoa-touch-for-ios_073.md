要查看`Xcode`如何为您重新整理文件，您可以右键点击`Xcode`文件浏览器中的文件，并选择**在访达中显示**。您应该会看到一个名称以`.lproj`结尾的文件夹，其中包含您的文件。要添加更多语言，请再次点击加号按钮；您将看到支持的语言列表。选择您想要为此文件提供另一个版本的语言，`Xcode`会将当前版本复制到相应的文件夹中。只需修改或替换该复制文件，即可为该语言提供独立的资源。



那么，这一切是如何运作的呢？针对每种语言和区域设置，iOS 会构建一个首选语言列表。假设一个虚构国家有两种官方语言：法语和中文。再假设一个应用程序支持法语和英语。对于设备设置为法语的用户，iOS 会自动从 `fr.lproj` 目录加载资源，而不是 `en.lproj` 目录。由于这个虚构国家的首选语言列表同时包含法语和中文，因此如果某用户设备设置为中文，但未找到中文资源时，系统会使用法语资源。要获得这种内置的文化敏感性，你只需提供多种语言的资源即可，只要 iOS 包含对该国家的支持。

## 第十二章：本地化与国际化

使用 iOS 内置本地化工具的最大好处之一，就是你可以发布一个应用面向全球用户。无需分别发布英语版、法语版、西班牙语版、中文版和德语版，只需发布一个包含所有这些语言本地化的版本。这样做有诸多优势：首先，如果用户在其设备上切换语言（例如为了学习第二语言），你的应用会随之切换。其次，在更新应用时，你无需费心更新多个版本并等待苹果逐一审核。第三，如果只有一个应用，追踪下载数据、使用分析以及排查问题都会容易得多。

当你使用苹果的内置控件时，部分本地化工作会自动完成。你使用的任意系统按钮（例如“完成”按钮）都会自动翻译成用户的语言：英语的“Done”在法语中是“OK”，在德语中是“Fertig”——至少根据“完成”按钮的显示如此。类似地，为屏幕键盘上的“返回”键指定不同的值，也会使用苹果提供的翻译值。这虽然很棒，而且据我所知，苹果的翻译质量一流，但这里存在一个小风险：如果你需要在应用的其他地方翻译“完成”一词，你的翻译可能与苹果的不一致，这可能会让用户困惑。让母语使用者测试应用，并告知你翻译是否准确，这一点很重要。此外，除非你在应用中有针对目标语言的本地化资源，否则系统按钮不会自动翻译。纯英语应用会显示纯英语的按钮，但如果你提供了本地化文本，按钮也会随之本地化。

我们首先来详细看看最容易本地化的内容：文本。

### 本地化文本

文本本地化是通过一种名为字符串文件（strings file）的方式实现的。这是一种可添加到项目中的特殊资源，能让你的应用根据用户首选语言加载文本。要在 Xcode 中为应用添加字符串文件，选择 File → New → File... 或按下 `⌘+N`，在左侧栏的 iOS 下选择 Resource，然后从右侧列表中选择 Strings File。这个文件的常规名称是 `Localizable.strings`。在此文件中，你需要提供一系列字符串，这些字符串被分配给键，键本身也是字符串。在代码中，你将引用这些键，并根据需要加载对应的字符串。考虑以下创建标题为“提交”的按钮的示例：

```objc
UIButton *submitButton = [UIButton buttonWithType:UIButtonTypeRoundedRect];
[submitButton setTitle:@"Submit"
              forState:UIControlStateNormal];
```

注意，“Submit”这个词是硬编码的英文。在本地化过程中，你可以提取这一行并将其放入字符串文件中，或许使用标题 `SubmitButtonTitle`：

```
"SubmitButtonTitle" = "Submit";
```

现在，当我们在代码中引用 `SubmitButtonTitle` 这个键时，就会得到“Submit”（提交）这个词。
