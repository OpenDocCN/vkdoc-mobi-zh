# iOS 开发入门指南

虽然 iPhone 应用程序算是相对较新的产物，但它们基于已存在数十年的技术，这些技术同样出现在 Mac 上。`Mac OS X` 引入了一套新的 API 和框架，统称为 `Cocoa`。虽然 `iOS` 与 `Mac OS X` 共享许多底层系统框架和 API，但涉及触控用户界面、电话功能以及 iOS 专属功能的 API 则位于 `Cocoa Touch` 层中，这是移动设备上相当于 `Cocoa` 的对应层。`Cocoa Touch` 与其桌面版的一个相似之处是开发工具，其中包括相同的 IDE——`Xcode`。实际上，当你下载开发者工具时，iOS 和 Mac OS X 开发所需的 SDK 都已包含在内。在本章中，我们将更详细地了解这些工具并开始使用它们。

## 安装 Xcode

在开始编写你的应用程序之前，你需要安装苹果的开发者工具。虽然在应用开发过程中会用到许多独立的应用程序、库和实用工具，但你主要使用的还是苹果的 IDE——`Xcode`。

**注意：** 与 iPhone 和其他苹果产品不同，`Xcode` 中的首字母 `X` 采用大写形式。

安装 `Xcode` 有两种方法。最简单、支持最好且最及时的方法是从 Mac App Store 下载 `Xcode`。下载完成后，`Xcode` 将位于你的 `/Applications` 目录中，无需进一步安装。

**注意：** 默认情况下，`Xcode` 安装程序会将开发者工具安装到硬盘的 `/Applications` 文件夹中。虽然也可以将 `Xcode` 安装到其他位置，但较新版本的安装程序并未向用户提供该选项。我建议将 App Store 版本的 `Xcode` 安装到 `/Applications` 目录，并将你可能使用的任何测试版安装到其他文件夹。

第二种安装 `Xcode` 的方法是从苹果开发者网站下载安装程序。虽然苹果并不总是通过这种方式发布每个最终正式版的 `Xcode`，但这是安装预发布版本工具集的方式。使用你的开发者凭据登录后，你将下载一个包含开发者工具安装程序包的磁盘映像。运行该包即可安装 `Xcode`。在撰写本文时，`Xcode` 的最新版本是 `4.3`；虽然旧版本也能在你的 Mac 上运行，但 `4.0` 之前的版本存在显著差异，以至于可能难以跟随本书中的教程进行操作。

无论采用哪种方式，你都应该知道苹果的工具集是一个大型下载包，通常超过几个 GB。虽然在将各个组件分离为 `Xcode` 可以更新而无需重新下载整个工具集方面取得了一些进展，但初始下载很可能无法在你附近的咖啡店完成。

## 开发者工具

你已安装的开发者工具以 `Xcode` 为核心，但本书中还会大量使用其他一些组件：

- `Instruments` 可用于检查应用程序的性能，查找内存泄漏，发现计算瓶颈，甚至轻松分析游戏的 3D 渲染过程。

- `iOS Simulator` 可在模拟环境中运行你的 iOS 应用程序。需要注意模拟器（Simulator）与仿真器（Emulator）之间的区别。在模拟器中，你的代码是为模拟器运行的平台编译的。对于 iOS 应用而言，代码是为你的 Mac 编译的，并运行在一个类似 iPhone 的虚拟环境中。而在仿真器中，代码的编译方式与仿真器和你要开发的平台完全相同。目前没有可用的 iOS 仿真器。但如果有的话，为仿真器编译的代码将与为目标设备编译的代码完全相同。这一点在测试中很重要，因为不同平台的处理器架构不同；你的 Mac 使用 Intel 处理器，而 iPhone 使用 ARM 处理器。
