# 2. 超越服务

曾几何时，一家 IT 咨询公司需要储备硬件库存，例如内存条。工程师们通过销售并安装这些内存条来加快电脑速度，从而为客户创造更多价值，并延长与客户的合作时间。虽然我们现在很少为大多数电脑（尤其是苹果电脑）安装内存，但这却是一个很好的范例：既能让雇佣你的人生活更便利，又能额外赚点利润。不妨考虑一下附加额外的云服务，比如数据备份。并专注于那些能让支持系统更好用，或者在出现问题时让你更省心的事情。

同时也要留意竞争。我曾有一个客户解雇了我们公司，因为我们向他们出售了一台打印机，而他们随后在网上其他地方找到了便宜 30 美元的同款打印机。最糟糕的是，我相当肯定我们出售那台打印机本身就已经亏本了。你永远不可能以比任何人都低的价格出售你卖的大部分东西。但你可以尝试与某个供应商达成协议，让他们为你提供比所有可能的硬件销售总和更有价值的客户推荐。

话虽如此，如果你将商品和服务打包在一起提供，那么客户就可以通过一张采购订单同时获得两者，一次性完成交易，最终拥有更好的体验。在本章中，我们将探讨利用第三方产品的各种方法，这些方法有助于扩大你在客户中的影响力，将客户引入一个标准化（因此也更容易维护）的环境，并让客户更满意，这意味着更多的口碑推荐业务（在我看来这是最好的业务类型）。

#### 注意

如果你只是兼职做咨询，我强烈建议你不要向客户销售或转售任何东西。

## 经销商账户

你听说过英迈国际或 TechData 吗？它们是分销商，或者说总代理商。如果你打算销售各种硬件和软件，你很可能需要与这些分销商中的一家或两家建立账户关系。你也可以选择与 Synnex、Avnet、Arrow、Jabil 或 CDW 等公司合作。很可能，当你要购买硬件和软件并销售给客户时，你会发现需要建立多种不同的关系。这些关系将超越分销商，直接延伸到其他供应商。

#### 注意

申请经销商账户通常伴随着对企业信用的审查。如果你正从银行获取融资，请注意不要过度透支你的信用，也不要让信用审查次数过多。

根据你想要打造的公司的类型，考虑向客户提供以下生态系统：

- **云解决方案**：我们将在本章详细讨论这一点。
- **硬件和软件**：我们也会在本章后面讨论销售硬件和软件。
- **带宽**：如果你要销售带宽，请准备一家电信运营商和一家有线电视运营商的选项，并让他们相互竞争，以便为每个客户争取最优价格。你所在地区会有各自的本地供应商，因此可以直接联系他们。联系时，别忘了询问他们是否能向你推荐业务。
- **基础设施服务**：这包括监控、备份、网络（无线、交换机等）和电话。是的，我提到了电话。

除了构建一个能满足所有客户需求的全方位服务产品组合之外，还要考虑其他因素。这些因素包括：

- **你能从这些公司获得什么样的付款条件？** 考虑一下你需要多长时间来支付他们款项，以便确定要与你的客户签订什么样的付款条件。
- **采购流程是怎样的？** 你需要一个简单流畅的采购流程，最好是拥有一个网络门户以及授权/支持电话号码。如果你需要注册交易才能获得额外折扣，那么你也希望该流程能在线上完成。
- **退货流程是怎样的？** 一件有问题的设备退货可能会浪费大量时间。在开始销售之前，就要弄清楚供应商如何处理退货，以及每次退货会耗费你多少时间。
- **你能赚多少钱？** 我把这个问题留到最后，因为正如你可能从本书中领会到的，我认为大部分这些东西的经济利益都是辅助性的。重要的是客户满意度等等。但是，如果你会亏损，或者赚不到足以使其可行的利润，那就不要销售。相反，在这种情况下，要与那些能为你提供良好推荐利益的组织建立关系。

归根结底，如果你方法不当，销售东西可能会很糟糕。但是，根据你业务的性质，你也可能从销售中获得一笔非常可观的额外收入。在开始销售之前，务必小心谨慎，确保你了解自己在做什么，以及销售行为带来的税务/会计方面的影响。



## 销售移动性

我常去健身房旁边的一家小店铺买冰沙。这家店由一位健身教练经营。她对技术了解不多，但非常非常聪明，而且生意一直在成长。每次我去买冰沙时，她都会问我很多关于她设备的问题，有时这些问题会将我工作中涉及的概念置于一个全新的背景中。它们帮助我理解小企业主是如何看待技术的。她的问题改变了我为小企业提供咨询的方式。

她没有电脑，也不想要电脑。她也不怎么用手机。她使用的是 iPad。有时，当员工在 iPad 上乱翻时，她会在浏览器历史记录里看到一些她可能不想看到的东西。不过，因为公司规模较小，她很乐意在初期尽可能保持开放，不限制员工行为以降低成本。每年都有成千上万的新企业以这种方式起步（请记住，兼职的苹果顾问并非唯一在零工经济中成功的人）。

我有时喜欢给她一些有用的建议（把这当作免费咨询吧）。这些建议通常是“找个会计”或者“这里有个能标注 PDF 的应用”之类的。随着业务发展，她想永远用 iPad 来经营。其他一些成长型的企业也类似，比如苹果在其营销文档中作为典型引用的“冰沙王”（Smoothie King），它有超过 500 家连锁店使用苹果设备来加快结账速度。还有我常去的本地咖啡馆，现在已有 7 家分店。不过，我也给了她一些关于如何使用第三方工具来更好地部署苹果设备的建议。

并非所有企业，也绝非所有工作岗位（即使是在最小的企业里）都能用平板电脑完全替代电脑。但许多企业都希望这样做，因为平板电脑比台式机和大多数笔记本电脑更便宜、更易管理、所需培训更少，并且能带来更高的客户满意度。由于大多数移动设备的屏幕空间有限，其上运行的应用通常只针对特定的工作职能。在许多环境中，要串联起一个工作流程可能需要多个应用，正如我们在本书中一直讨论的那样。但即使你在一个设备上安装了许多应用，其总花费也往往低于市场上许多流行的传统软件包。不过，在某些时候，许多工作确实需要更复杂的硬件，必须使用笔记本电脑。

似乎每年都有整个行业将大量台式机和笔记本电脑换成了移动设备。这为精明的苹果新顾问带来了巨大的潜在新客户源。苹果率先在零售业掀起了移动设备潮流，但自那以后，我们在现场服务、医疗甚至宗教活动领域都看到了移动设备的普及，这带来了源源不断的客户。有时，只需精通一个应用，或在苹果零售店或专注 Windows 的咨询公司交到一个朋友，就能对你的业务产生翻天覆地的影响。

随着移动设备在这些环境中获得如此强大的动力，一些关于安全性和可扩展性的有趣问题开始出现。设备可能会丢失，需要被擦除数据。当你拥有大量设备时，你希望它们能以相似的方式设置，并希望设备配置能实现自动化。供应商已经听取客户的需求。例如，苹果提供了以下你在与客户沟通时需要了解的技术：

-   **移动设备管理 (MDM)**：例如 Jamf Now ([`www.jamf.com`](http://www.jamf.com)) 等解决方案，允许管理员远程管理设备：管理应用、内容、设置，甚至对设备执行操作，例如远程擦除和锁定那些脱离组织控制（因丢失和/或被盗）的设备。如果你大量涉及移动设备部署，你将需要与 Jamf、AirWatch、SimpleMDM 或市面上其他 MDM/管理平台建立合作关系。

-   **批量采购计划 (VPP)**：允许用户批量购买应用，然后将购买的应用自动部署到设备上。有关批量采购计划的更多信息，请参阅 `deployment.apple.com`。

-   **设备注册计划 (DEP)**：强制设备加入 MDM 解决方案，在设备激活时自动加入该解决方案。有关 DEP 的更多信息，请与你在苹果的客户团队或苹果设备经销商合作。

#### 注意

有关苹果术语和技术的更全面列表，请参阅附录 A。

许多应用仅适用于单一平台。例如，iPad 应用商店中的许多应用在安卓设备上没有对等产品，反之亦然。曾几何时，供应商锁定被视为坏事。但如今，它有助于降低成本。如果顾问或企业主只需要学习一个平台，他们就能更深入地了解如何管理设备。随着业务增长，可能会希望迁移到另一个平台，但到那时，企业应该已经发展到需要聘请外部组织来实施新解决方案，而这将是一笔不小的开支。

企业使用的许多应用会与网站通信来存储数据。几乎没有数据存储在移动设备上。因此，备份变得比以往更不重要，因为软件即服务 (SaaS) 解决方案也提供了应用数据的集中管理，初创公司办公室无需实体服务器。

随着定制应用和应用商店应用覆盖了企业需要执行的越来越多任务，越来越多的小企业选择完全依赖移动设备来完成以前由电脑完成的工作。只要能够安全地做到这一点，他们就应如此。这降低了创业成本。与我创办第一家公司、并实现能够接受信用卡付款的阶段所需的成本相比，那家为我做冰沙的店铺是在极低的预算下起步的。希望更低的风险和前期成本能让越来越多的初创企业成功，并带来更多的创新。帮助人们成功应该是我们始终如一的的首要目标。

应用、云服务、硬件和其他工具可以帮助你的新企业为你的客户提供帮助。在下一节中，我们将重点讨论云服务的销售与协作。



## 是否应当销售云服务？

云服务适合你的客户吗？答案是：很可能。是否每一种云解决方案都适合每一家企业？显然不是——否则我们也不会问这个问题（这点是我在某堂课上学到的）。但云计算在商业领域风靡一时，因为从长远来看，自行购买服务器并支付其支持与备份费用，往往比向云服务商付费并直接使用其提供的服务更昂贵。

根据客户独特的业务需求和特点，帮助他们找到合适的解决方案至关重要。审视基于云的工具有很多不同角度。这可能意味着部署类似以下某一种方案（或它们的混合方案）：

-   使用`CrashPlan`将文件备份到云端——也可能备份到现场服务器，或办公室的服务器。
-   使用`Dropbox`、`Google Drive`、`Office 365`或`Box.com`，将文件实际迁移到云环境中。
-   将物理服务器或本地服务器迁移到云环境，让客户获得一条经过改进且具备冗余能力的互联网连接，而在其办公室部署这样的连接在经济上并不划算。
-   实施各种工具，例如作为 SaaS 的`Salesforce.com`，从而减少对物理服务器、虚拟服务器或云端服务器的需求。

以上绝非你可能需要精通或合作的全部工具清单。如需更全面的列表，请参阅第 5 章的“技术点滴”部分。但如何选择正确的解决方案呢？我们来审视几个问题与一些恰当的回答。

### 你的文件有多大？

大型图像或媒体文件可能体积过大，无法在远程地点处理。任何了解设计师或视频剪辑师的人都知道，他们每天早上花半小时等待每个项目文件打开会多么痛苦。在将这些文件迁移到云端之前，先弄清楚某客户使用的每种文件类型需要多久才能打开。也许最终更好的做法是将它们同步到本地服务器，或者寻找替代的工作流程。

### 你是否有合规性问题？

许多环境有法律、医疗、政府、银行或其他合规性要求。每个云环境都有自己的一套合规标准。从（欧盟的）安全港协议到 SOC2、HIPAA，再到仅仅是在客户的安全审计文档上勾选一个框，都会涉及大量合规性问题。在你开始与客户合作进行集成之前，请确认你找到或销售的任何解决方案都满足每位客户的具体要求。

### 你的业务逻辑是否有特定需求需要处理？

业务需求通常包括客户关系管理（CRM）、会计软件、专业服务自动化（PSA）、适用于各类企业的工具以及其他选项，所有这些都需要相互连接。确保工具能完成客户所需，包括一些小任务。处理这个问题的最佳方式通常是观察他们工作，并记录他们如何与原有工具交互。

### 某个特定工具是否需要与其他工具交互？

在一个以应用为中心的世界里，很少有工具能完成客户的所有需求。客户通常更喜欢简单易用、上手快的工具，而不是花费数十万美元购买一个什么都做不好、却又样样都揽的单一工具。但请确保这些工具在需要时能够连接到其他工具，以避免繁琐的双重录入。

### 你是否需要轻松地将数据移入和移出云解决方案？

许多云解决方案只提供向环境添加数据的功能，却让导出数据变得困难。在你将某个（无论出于何种原因）可能不适合客户的解决方案强加给他们之前，请确保他们能在数据存入特定工具后将其取出。

### 成本是多少？

仔细阅读细则。如果客户需要以编程方式与数据交互，他们可能必须付费购买高级账户类型。通常会有表格列出诸如 API 访问或“专业功能”等特性与选项。在为客户评估解决方案成本时，请确保你了解这些层级和费用，否则他们可能会支付比你报价高出三倍的费用，并为此恨你（你自己可能也会恨自己）。

### 是否有公司外部的人员需要访问数据？

如果有外部方需要连接，你可能必须为每个人创建（并支付）额外的账户。你不希望有任何意外。与客户合作，确定外部方是否会连接到云工具，弄清楚他们的访问权限是否可以受到限制，并确定客户能否为供应商、客户和承包商提供适当的访问级别。

### 你是否有合适的网络？

如果客户依赖互联网来访问文件、数据库、联系人和其他关键业务资产，那么你需要确保与外界的连接绝对不能中断，否则一旦（不是“如果”）中断，员工就会无所事事。冗余、快速的互联网连接是绝对必要的，与员工闲置无法工作的成本相比，这点投入微不足道。

归根结底，云解决方案很可能实际上适合大多数客户。可能需要稍加研究，才能找到符合每个客户独特需求和预算的正确解决方案。但如果你有计划将本地解决方案迁移到云端，或迁移到某个特定的 SaaS 解决方案，考虑到云服务商领域如此广阔，这样做是完全可行的。既然你已经掌握了一些在按服务逐一推进这类计划时可以提出的问题，那就享受这个过程吧——并希望借此机会与客户合作，重塑他们开展所有精彩业务的方式。如果成功，并且你真正捕捉并改进他们的工作流程，你将拥有一个终身客户！



### 如果你销售云服务

许多供应商提供的服务能让你的客户受益，其中大部分还提供经销商或推荐账户类型。`Google Apps`、`Microsoft`、`Box` 和 `Dropbox` 等文件共享服务允许你转售或关联账户。这些服务可以带来稳定的额外收入和附带福利，使你能更轻松地支持客户。你甚至可能不打算获得全部新增收入流，这样做或许是为了向客户收取更少的服务费用，同时还能获得独家权限。

在选择新的云服务（或至少对你的产品组合而言是新的服务）时，需要评估以下几点：

-   客户能否直接联系供应商？
-   作为推荐方，你能否获得更高级别的支持？
-   推荐方是否会获得额外培训？
-   你能否从一个统一管理面板中管理多位客户？
-   客户能否在不产生额外费用的情况下，为你创建一个委托管理账户？
-   他们支付给供应商的费用中，你的“佣金”占比是多少？
-   客户能否根据需要按年支付服务费用？
-   如果客户停止与你合作，他们将其服务转移到其他供应商的流程是否简便？

这些都是重要的问题，而且你可能还会遇到更多问题。每个答案的优先级将取决于客户类型以及数据类型或服务类型。我建议你制作一个电子表格，以便选择想要合作的供应商，避免措手不及。要审慎地进行选择。为了帮助你，我还在附录 E 中整理了一系列客户可能需要的技术。

将时间分散在许多不同的工具上可能会令人沮丧。虽然为了给客户提供尽可能全面的覆盖，与尽可能多的供应商合作看起来很有吸引力，但请记住，围绕“好、更好、最佳”的方法，甚至只与一家供应商合作进行集中化管理，能让你更加专注。你可以非常擅长围绕该供应商构建解决方案，可能从供应商那里获得潜在客户，并且在客户遇到问题时需要访问的工具也更少。

你正在销售解决方案这一事实可能会改变你的方法。例如，如果你在销售 `Dropbox`，那么可能在对话中根本不会提及 `Box.com` 或 `Office 365`。只需确保每位客户的体验都很棒，并且他们不会为了让你多赚钱而牺牲流畅的工作流程。

### 评估云解决方案的安全性

现在的观点是一切都必须上云，而且要立刻上云！有时，寻找工具关乎工作流程。工作流程应该合理且出色。

但也有一种观点认为，你甚至不应该保存大量数据，除非这些数据是保密的，并且因此得到了妥善的保护。保存他人信息及其行为的责任过于重大，以至于其风险超过了你可能使用这些数据所能获得的好处。

安全性很重要。工作流程也很重要。考虑到有大量服务可用于任何特定任务，如果某些服务不够安全，那么很可能还有其他十种服务可供选择。那么，你为什么可能会选择使用某个特定的服务呢？

-   **合规性**：确认在线服务能安全保管你的数据的最佳方法之一，是检查服务各方面的治理方式。许多云解决方案会列出其所遵循的控制措施，通常采用行业标准化控制措施，例如 SOC 2（`https://en.wikipedia.org/wiki/Service_Organization_Controls#SOC_2_overview`）、FedRAMP（`https://www.fedramp.gov`）等。深入了解某个组织遵循这些标准的哪些部分，可以让你很好地了解你可能使用的潜在云服务对你的隐私有多重视，以及他们允许哪些人访问存储在他们云中的各类数据。

-   **身份验证**：一些服务在登录时提供的不仅仅是密码。例如，他们可以在你登录时发送包含验证码的短信。这就是所谓多因素身份验证，它利用你知道的信息（你的密码）和你拥有的物品（你的手机）来允许访问在线服务。许多供应商提供一种称为“单点登录”的功能，而不是强制最终用户在多个服务中重新输入密码。如果你是一家小企业（比如说少于 10 人），你实际上可以免费使用大多数这类工具，直到业务增长需要升级到付费层级。

-   **静态加密**：你的数据在云服务中是如何存储的？你选择的任何云服务都应始终保持所有数据加密。你可以问他们一些问题，比如：“你们的数据库是否存储在加密的文件系统上？”、“你们在数据库中存储的我们公司的哪些数据是未加密的？”、“你们具体如何存储我上传到你们数据库的文件？”以及“谁具体拥有解密这些数据的密钥，以及这些密钥是如何存储的？” 对于大多数合规形式，需要记住的是，只要公司对客户公开其做法有多么不安全，他们就可以有一些相当不安全的做法。所以，对于你关心的领域要多提问。

-   **证书**：当我们询问那些可用于解密数据的密钥时，还有一些密钥可以在数据传输过程中保护你的数据。确保所有流量都受到使用强加密证书的安全协议的保护（如果你不确定，可以询问每家供应商），并且他们用于这些证书的私钥得到了妥善保管且快要过期。

-   **隐私政策**：合规的组织通常应有隐私政策。当你阅读政策时，它不应该是这样的：“你如何回应对我数据的请求？”

-   **地理因素**：世界各地的文化规范各不相同。大多数美国人不关心这个问题。然而，欧洲人通常不希望他们的数据存储在欧洲联盟以外的服务器上（尤其是美国的服务器）。这是因为系统管理员和客户都熟知欧盟（以及欧盟以外的国家）的法律，并且希望强制推行在不同地区被视为可接受的隐私和安全水平。



## 数据安全与云服务基础

*   **服务等级协议（SLA）**：如果你无法访问数据，那数据还有什么价值？SLA 是承诺达到特定服务等级的协议。一个真正优秀的 SLA 包含“五个九”的正常运行时间，即 99.999%的时间你的数据都是可用的。请记住，99%的时间意味着你的数据可能会离线数天（实际上每年 3.65 天），而供应商仍然能满足其正常运行时间的要求。另外，要关注如果供应商未达到 SLA 会面临什么处罚；如果没有处罚，那么签订 SLA 还有什么意义呢？

*   **备份**：备份通常是许多人最后才考虑的安全方面。然而，如果数据丢失了，那你还不如把它们记在纸上。备份需要大量空间，耗费时间，并且通常需要非常昂贵的设备。事实上，不必自己备份服务器，往往是许多人想要将系统从办公室迁移到云端的主要原因之一。因此，请确保你了解云服务商保留你数据的时间、在发生故障时恢复这些数据所需的时间，以及完成此操作需要什么流程。此外，看看你是否可以拥有自己数据的备份副本。

*   **安全扫描**：你可以免费运行自己的扫描，看看是否会被阻止访问产品，或者是否发现供应商尚未意识到的自身漏洞。这里有几个注意事项：不要扫描大型供应商（如`Dropbox`或`Google`）——这主要适用于你的供应商在其云基础设施中为你提供了专用服务器的情况。

在当今快节奏的商业环境中，大型和小型公司似乎随时都有新服务上线。这可能是一个快速的列表制作应用，也可能是一个运行整个组织的巨大新工具。你不需要穿锡箔帽（指过度疑神疑鬼），但你应该采取一些基本的安全预防措施，以免显得疏忽。我建议将这样的清单转化为一个实际的核对表，并审计你拥有的每一个应用，以验证每个应用都满足你提出的要求。这并非多疑，而是良好的商业实践。

## 推荐与销售

IT 行业有句老话：“没有人因为买了思科而被解雇”，或者更早的：“没有人因为买了 IBM 而被解雇”。销售你从未听说过的产品时要小心。销售像`Salesforce`这样的大品牌是有保障的，而推荐一家可能倒闭的小公司或专业服务则存在风险。永远不要让佣金左右你的判断。从为客户找到正确解决方案开始，然后反向建立关系，这种关系能帮助你管理多个客户，并通过重复使用相同的服务（并磨练该服务的技能）为客户提供附加价值。

当你在每个垂直市场中都有明确的市场领导者时，这最容易实现。尤其是当你在这些市场拥有足够多的客户，成为这些市场工作流程的专家时。这些被称为细分市场。如果你正在与一个需要这些领导者之一的客户合作，但解决方案无法快速掌握，那么考虑抓住机会寻找另一家公司，你可以与之交换推荐客户，如果客户需要他们提供的解决方案。这能避免你显得不专业，并且可能对你更有利（从长远来看，为客户做最好的事总是更有利可图的）。

## 提供硬件和软件

云解决方案并不能解决所有问题。客户可能仍然需要各种只能通过供应商或经销商订购的软件。此外，客户还需要计算机和移动设备。如果你身处苹果咨询领域，这些设备将来自苹果。与其尝试销售苹果硬件，尤其是在起步阶段，不如与当地的苹果零售店建立良好关系，帮助客户获得硬件，并确保当他们购买设备时，这些设备是`DEP`兼容的。

苹果还销售其他与苹果平台配合良好的硬件。从 USB 驱动器到打印机再到保护壳，苹果的产品价格会稍高一些，但你和你客户都知道苹果认可这些硬件。那么，你可以向客户销售什么呢？所有这些产品你都可以卖！或者，你可以让苹果店的业务代表来处理这些。

以下是你可能选择销售硬件的一些原因：

*   你可以为客户争取到更便宜的价格。
*   你需要确保他们为大型项目获得正确的设备。
*   如果你在销售硬件，你可以获得更好的支持。
*   你可以更快地获得设备。
*   你可以获得更专业或更高度定制的硬件。

但销售硬件也有一些负面因素。这些包括：

*   如果出了任何问题，客户会生你的气。
*   客户可能会找到更优惠的价格，并因此对你感到不满。
*   在给定的客户付款周期内，你需要垫付产品资金。
*   订购和采购硬件可能非常耗时，特别是如果你需要向供应商注册每笔交易才能获得最优价格时。

最终，如果你打算转向全职咨询，我建议尝试向客户销售东西。即使你不将其作为业务的主要部分，了解转售涉及的内容以及渠道如何运作，将使你能够代表你的客户更好地管理其他供应商。



## 提供基础设施服务

还记得我们在本章前面提到的电话服务吗？如果你的客户确实需要在其办公室获得帮助，为了完善你的服务提供，以下是一些需要考虑的事项：

-   **监控服务**：监控服务可以在计算机或网络设备出现问题时向你发出警报。像 `Watchman Monitoring` 这样的解决方案可以监控 Mac（以及 Mac 上常见的第三方工具），并在各种故障正在发生或即将发生时通知你。例如，如果硬盘即将损坏，你会收到警报，有望在客户现场的用户发现问题之前修复它。网络设备可以通过针对客户所有设备类型的工具进行监控，也可以通过一个集中式工具（这类工具很多）进行监控。无论从客户满意度还是客户安全性来看，这项服务都价值连城，而且无论是你承担成本还是将其转嫁给客户，它都能收回成本。

-   **互联网服务**：前面我们提到过销售互联网服务。如果你这样做，可以考虑为每个客户销售两条线路：一条是高服务等级协议（SLA）的主线路，另一条是住宅级但通常速度更快的线路。许多路由器都支持将负载分担到两条线路上。拥有两条线路意味着如果其中一条出现故障，客户仍然在线，同时也意味着客户只需额外支付很少的费用就能获得更快的速度。

-   **无线接入点（WAP）**：客户需要在办公室使用无线网络。你为他们提供的互联网服务可能捆绑了无线路由功能，但这通常无法满足安全要求或统一管理的要求。像 `Meraki` 这样的工具允许经销商通过单一门户管理多个客户。其他工具如 `SonicWall` 或 `Cisco` 则提供更高的可扩展性，同时也提供了大量企业级功能。

-   **交换机**：交换机允许你连接那些显然还有人仍在使用的所有以太网设备。销售这些设备时，要考虑哪些设备能够使用虚拟局域网（VLAN）进行网络分段，哪些设备能与苹果设备良好兼容，以及哪些设备能提供客户所需的吞吐量。

-   **路由器**：对于较小的客户，你真正想要的是融合网络，也就是多功能设备。例如，一个五人公司可能只需要一个 `LinkSys` 设备就能充当 WAP、交换机和路由器。随着客户规模扩大，他们原先的设备可能就不够用了。我喜欢保留旧设备以备不时之需，但值得一提的是，对于任何网络设备，都要留意客户何时可能超出你经济高效地为他们提供服务的范围——或者你的公司何时可能需要在某项技术上做得更好，并在此期间以折扣价提供一些服务。

-   **外围设备**：没错，有些人仍然会打印。有些人需要摄像头、网络摄像头、多功能一体机等。这就是你需要开始划定销售范围界限的地方。硬盘、大容量存储设备（RAID 和 `Drobo`）以及其他外围设备很容易证明其价值，但相机滤镜或 iPad 保护壳就有点偏离主业了。

-   **电话服务**：我要提到的最后一点是电话。许多小企业让员工使用个人手机（或公司提供的 iPhone）作为工作电话。至少，大多数拥有实体办公室的客户至少需要一条固定电话线路。随着业务增长，他们可能会选择使用 IP 语音（VoIP）解决方案或电话系统。如果你打算销售这类服务，尽量保持其以应用驱动、基于云端且尽可能无需实际硬件。如果这些开始变得有挑战性，那么考虑一个专门处理大型传统电话系统和/或视频会议的合作推荐伙伴。

-   **布线**：所有顾问都需要一个可靠的布线合作推荐伙伴。但很少有人应该自己动手布线。

最终，你不需要成为所有客户的万能专家。如果客户需要你不知如何实现的某项服务，你有一个选择：寻找合作推荐伙伴或把它提供给客户。如果它远远超出你的技能范围，这通常很容易做决定。但如果它恰好在你技能范围内，那么请务必意识到它可能让你偏离真正核心业务有多远。但最重要的是，要时刻留意银行账户状况，因为你通常需要稍微偏离原定路线才能找到持久的成功。
