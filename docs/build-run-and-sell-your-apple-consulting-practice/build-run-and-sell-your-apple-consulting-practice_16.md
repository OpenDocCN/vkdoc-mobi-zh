# 16. 在大公司内部运营咨询业务

许多公司都设有服务部门。在苹果生态圈中，这些主要是软件供应商。一些硬件提供商也这么做，例如大型磁带库制造商。大多数面向企业销售硬件和软件的公司，其产品都需要客户引入配套服务。一些较小的软件供应商也设有服务团队。

客户可能希望从硬件或软件公司购买服务的原因包括以下几点：

-   服务是必需的。当需要大量定制化时，情况往往如此。
-   客户没有时间实施硬件或软件。
-   客户需要培训以快速掌握实施硬件、软件或更新的技能。
-   客户需要为项目的一部分引入服务，并且希望这些服务能通过单一采购订单进行管理。

我提及这些的原因是，你可以围绕满足这些客户需求来构建服务包或捆绑方案。但首先，让我们看看你所在的公司希望你做什么（这通常与客户希望你做的有很大不同）。

## 公司希望你做什么？

我们许多人会采取这样的方式：希望销售服务并发展服务业务。这样我们就可以雇佣更多人，帮助更多客户。但这真的是我们公司希望我们做的吗？当然，钱总是钱。但许多公司更希望你能推动核心产品的销售，而非服务销售。差不多是这样。

这是一个难以平衡的问题，而找到平衡点的关键是真正去和你的老板以及老板的老板进行沟通（没错，用你的“人话”去谈）。我事先告诉你，大多数小型组织会说“去赚所有能赚的钱”，但大型组织可能会谈论构建供应商生态系统之类的事情。重要的是找到增长、利润和社区赋能之间的正确组合，并且让所有人达成共识。

### 找到正确组合

正如我之前所说，最终总会有人问及利润率和盈利情况。如果你没有一个好的说辞（并且是书面形式的），那你可能就要找新工作了。那么，正确的组合是什么？

本章剩余部分将重点阐述其中的一些内容，但在你开始推行某个议程——甚至可能在做与公司要求相悖的事情——之前，请考虑以下几点（并应与管理层讨论）：

-   服务组织需要达到多少利润率？
-   为了推动软件和硬件销售，有多少服务会免费提供？
-   免费服务是否会以费用返还的形式计入服务团队的账目？
-   如何处理服务的销售赋能？
-   你是否计划将人员扩充作为服务套餐的一部分？
-   服务团队是要参与赋能社区（经销商、顾问、托管服务提供商、系统集成商等）来交付服务，还是两者相互竞争？

#### 注意

这些问题并不多，考虑到对利润的潜在影响，即使是首席执行官也乐于回答。你应每季度与你的经理回顾这些问题，鉴于你的经理可能并非始终与他们的上级保持一致，或者你的管理层发生变动，最好每年再向上级汇报一次。

一旦你明确了利润目标和销售目标，就可以设定价格。我会采用之前章节中相同的定价方式。毕竟，生意就是生意。不同之处在于，你需要为免费服务预留出相应的时间。不过，提醒一句：不要过于人为地摆弄数字。如果你应该达到 15%或 40%的利润率，那就是你的关键指标。当你知道需要赚取多少时，就可以构建服务包并准备好出售你的服务了。



## 构建服务包

捆绑销售是在大型组织中提升服务销售额的绝佳方式。由于易于解释，它们还能减少销售过程中的摩擦。一旦你知道了必要的利润率，就来看看你的成本、可计费百分比与组织为促成销售而付出的时间百分比，减去转售商利润，然后算出必要的每日数字。很简单，对吧？让我们从日均价格与成本结构中提取利润率：

> `((年计费天数 * 建议零售价) * 平均转售商利润率) / 年度部门成本`

如果你拉取一份销售总商品报告，就能相当清晰地了解与团队运营相关的成本以及可能出现的复杂性。上述公式可快速填入电子表格或数据库。而且它只会变得越来越复杂。一旦你知道了每日成本、必要利润率和实际目标，现在我们就来构建一个服务包。想知道为什么我之前说要在做这个练习之前（在你执行此操作之前）先确定定价吗？这是为了进行合理性检查。

### 提示

只要有可能，我倾向于与大型组织内的服务团队按日增量工作。如果你正在开会或需要快速做出决定，了解每日成本和每日利润能为你提供一个很好的基准。即使你处理的是固定成本的《工作说明书》，你也会根据完成可交付成果所需的时间来进行计算，以构建服务包。

服务包旨在让购买服务变得更简单。这能让团队更轻松地沟通客户将获得什么。有两种处理捆绑销售的方式：按天或按可交付成果。然后你需要决定是否将服务与组织产品组合中的其他产品捆绑。例如，假设你的组织销售培训课程。你可以将 5 天的培训课程和 5 天的定制服务捆绑在一起，对其应用 10%或 20%的折扣，这样客户更有可能购买该服务包，从而可能为你带来原本无法获得的额外销售。

按天捆绑意味着需要编写的《工作说明书》更少（“更少”与“没有”截然不同）。这会影响你的成本，并且客户总是期望在批量购买时获得折扣。只需确保在叠加转售商利润后，你仍能达到目标利润率。

既然你已经构建了服务包以及可应用于它们的适当折扣，那么是时候建立支持业务服务组件的流程，或对其进行优化以匹配新的服务包了。

### 变更控制

一旦你构建了计划推向市场的产品和服务，它们应该接受审查，然后进入某种变更控制流程。每个组织的审查方式会有所不同，但请对此心怀感激，而不要有任何抵触情绪。

如果没有人愿意提供审查意见，那就去找产品管理团队。所有产品管理团队都乐于对你可能询问的任何事情提供反馈，包括产品本身、产品相关的所有方面、制作花生酱果酱三明治的最佳方法，以及关于齐柏林飞艇乐队和平克·弗洛伊德乐队哪个对摇滚乐影响更深的无休止辩论。

### 将服务运营化

每个公司都会有不同的领域需要检查确认，然后你才能认为你的新服务包和运营可以自动运行，这实际上是任何业务功能的真正目标。根据公司的规模，你需要与一些团队会面，解释你将要做的事情及其原因。其中一些团队甚至还会给你布置“家庭作业”。

以下是一些你应该提出的问题，如果你的组织中有处理这些问题的团队，请安排会议以确认你的假设。

-   **法务团队**：提供服务最重要的方面之一是确保你与客户签订的合同涵盖了向客户提供的所有商品和服务。任何时候你更改或添加一个`SKU`，都要与你的法务/合同团队预约，核实这些内容是否涵盖了你正在做的事情。如果你的法务团队管理公司保险单，也请一并审查这些保险单；否则，请同时与保险/风险团队和法务团队合作，确保保险单涵盖你将提供的服务范围。

-   **销售运营团队**：大多数销售团队需要一个`SKU`。我们稍后会讨论销售赋能，但你需要向销售运营团队提供产品名称和成本清单。尽早安排这次会议，因为许多组织有特定的变更窗口期，在这些窗口期内可以更改`SKU`，这是因为分销商和转售商也有类似的窗口期。通常，你所做的更改至少需要 60 天才能对转售商生效。此外，还要敲定与合作伙伴进行沟通和销售赋能的任何细节。

-   **财务团队**：你已经协调好了`SKU`，但现在需要弄清楚它们被销售后会发生什么。需要解决的一些细节包括：
    -   何时确认收入（例如，在服务交付时，还是在服务合同期限内平均分摊）？财务部门将如何报告？
    -   财务成本对已确认的收入有何影响，包括佣金、成本、未分配费用、设施成本、分包成本等？
    -   一旦收入被确认，从这些`SKU`中产生的收入实际如何分配，以及你将如何对此进行报告？

-   **工单系统**：这是一个重要的组成部分。你需要以这样一种方式开立工单，即不要使支持客户的工作比原本更加困难。因此，你的工单需要使用与你的支持团队相同的系统。但工单系统可能还需要跟踪差旅，并支持对基于时间计费的顾问使用情况的报告。此外，如果你根据工单关闭来确认收入，请确保与财务部门协调好如何获取这些报告。

一旦你解决了所有这些细节，就把它们写下来。如果你的公司有内网或维基平台，请将信息放在那里。尽可能详尽地记录，然后让你会面过的团队确认你做的笔记，这些笔记随后会变成文章。这些资料还可以作为未来有人询问服务运营如何运作时的常见问题解答。

既然你已经解决了一大堆细节问题，要做好准备迎接更多我无法提前为你准备的问题。一旦这些问题也解决了，你就可以着手开展营销和销售赋能工作了。在开始与销售团队合作之前，首先要确保你的营销团队已经提供了良好的“空中掩护”来支持他们的工作。



## 市场营销

一旦你有了产品，就该制定营销计划了。这与你之前书中提到的如何营销自己的咨询业务类似，区别在于，你不是花自己的钱，而是很可能要尝试说服某个市场部门的人动用他们的预算。

争取到预算最轻松的时机是年底。如果你在七月份入职一家公司，而他们已经规划好了全年的广告和营销支出，那么很可能你无法做成太多事情。但这也正是开始谈论下一年计划以及你届时想要达成什么目标的时候。

在此期间，你需要尽可能多地积累资产。并且要确保这些资产保持最新状态。这其中既包括内部销售与营销赋能资产，也包括团队能够通过邮件活动、会议等渠道对外分发的资产。这些资产包括：

*   **客户成功案例**：客户购买了服务，从而更快地进入了市场。效果非常棒。你看到其中的吸引力了吧？
*   **一页纸简介**：理想情况下，每个产品 SKU 都准备一份，可以在会议上分发。
*   **素材图片**：如果你还不能投放广告，不妨先搜集一些素材，为后续制作广告做准备。
*   **为其他营销资产做贡献**：没错，事实就是，你越擅长与人合作，他们就越可能愿意帮助你。

一旦市场推广方面有了不错的起色，就该开始与销售团队合作进行赋能了。

## 销售，销售，再销售

销售团队既是你的最佳盟友，也是你的头号劲敌。他们销售你的服务，从某种意义上说，这也是你存在于这个组织的原因。但他们也经常想提供折扣。给服务和其他辅助 SKU 打折通常比给核心产品打折要容易得多。老实说，公司是靠销售核心产品来运营的，所以这本身也合理。

但你是有利润率目标的。为了赢得销售而免费提供服务，会让你的血压飙升。如果大量免费服务被送出去，最终肯定会有人问，为什么服务业务的利润率如此糟糕。即使他们嘴上说不会问，也迟早会问。当这种情况发生时，能够讲出一个有说服力的故事至关重要。最简单的办法就是在实际的损益表（P&L）上展示你正在盈利（说实话，做服务怎么可能不盈利呢？！？！）。但如何处理免费服务的问题呢？答案就是：收费冲销（Charge-backs）。

对收费冲销最简单的描述是：将打折的服务天数计费到另一个团队头上。如果销售团队因为给你的 SKU 打折或免费赠送而“受到责罚”，那么他们这样做的可能性就会大大降低，即使收费冲销使用的是软成本而非你本应收到的机会成本。

但如前所述，销售可以成为你的最佳盟友。他们是组织的命脉，带来新业务，并促成你负责交付的服务的销售。你给予他们的关注越多，服务就越容易占据他们的头脑，他们销售得也越多。以下是一些在组织中用来驱动理想销售行为的杠杆。

务必确保销售团队和客户团队拥有他们所需的所有资料。如果他们需要什么，就去制作。在他们开口之前，你最好先准备好以下内容：

*   一份可对外分发、描述每个 SKU 的一页纸简介。
*   描述服务或捆绑包的邮件模板。
*   围绕服务和捆绑包的销售话术。
*   每个 SKU 的工作说明书（SOW），提供给客户。
*   概述服务常见问题及解答的常见问题解答 FAQ 或维基 Wiki。

这能移除任何阻碍销售人员安心销售你服务产品的障碍，从而让他们更有信心。而信心往往是销售中最重要的方面。如果你想提高服务的销售额，就专注于让销售团队充满信心，并保持与他们的沟通，可以举办午餐学习会、赋能会议、送些甜甜圈，当然，还要征求他们的反馈。

一旦服务开始销售，就该组建团队来交付服务了。除非你已经组建了团队。那么，就该管理那个团队了。

## 人员配置

咨询公司员工的职业发展路径相当直接，我们在本书中用了一整章来讲述。但在大型公司里，服务员工的职业发展则略有不同。请记住，你手下员工最好的结局就是离开你。或者取代你。当那些应得的人取代你时，那种感觉是最好的。

所有的培训，所有的工作，当他们最终离开服务团队时，处境反而更好了。这是为什么？嗯，我们先说说员工最初为什么会离开。

*   **为了更接近产品**：有多少人能像那些直接面对客户并交付工作的人一样，在真实场景中接触到那么多客户？这些往往是构建产品路线图的最佳时机。在某个节点，许多服务人员会厌倦抱怨优先级，转而希望对公司生产的硬件或软件的未来有更积极的参与。
*   **为了更贴近产品**：服务工程师通常能成为出色的质量保证（QA）测试员（毕竟，他们在产品发布后一直在与客户一起测试产品），或者，如果他们掌握了相关技能，也容易转入开发团队。
*   **为了转入销售**：服务人员中很少有人愿意承认这一点，但在硬件或软件公司，他们是你所能找到的最好的系统工程师之一。那些真正尝试转入销售的人，不仅能说会道，还能付诸实践。
*   **出差太多**：前面的原因是服务工程师可能转岗的其他职位。而如果你找不到解决方案，这个原因则可能导致某人直接离开公司。许多团队成员一开始热爱出差，但随着年龄增长，我们会想要建立关系、生儿育女、安定下来。

当然还会有其他原因。但就我目前所见，这些是主要原因。有些组织选择建立非常小的团队，或者根本不建团队，而是依赖分包商，我们将在下一节中讨论这一点。



## 作为服务的专业服务：分包

专注。软件公司常常需要专注于自己擅长的事情：开发软件。但你却在这里，努力组建一个服务团队，甚至可能与自己的客户竞争。

把所有事情都外包出去，对吧？！？！首先，我们通过租赁方式外包了硬件的所有权。然后，我们将硬件的支持外包给托管服务提供商。接着，我们把自己的服务器外包给硬件即服务提供商。之后，我们决定更倾向于使用软件即服务，并转向由供应商托管的网站。在这个将旧概念重新包装为"即服务"的时代，许多软件公司选择雇佣其他组织来充当自己的专业服务部门，而不是自行组建。这使得它们能够专注于核心业务（例如，开发软件、制造冰箱或铺设太阳能电池板）。

大多数已经拥有专业服务部门的组织对此类做法不会有兴趣。你们已经投入精力构建了流程，招聘了"在路上奔波"的顾问，这已经是你们预算的一部分。但如果你正在创办一家新公司，或者试图应对超高速增长，那么选择在哪里投入时间和资源就至关重要。如果服务不是你的核心业务，那么不涉足服务领域，你或许能以更明智的方式加速发展。

总的来说，如果你的产品需要不时地搭配服务，那么将整个服务部分外包是一种低风险的服务交付方式，同时还能获取收入。但在你寻求外包服务时，有几件事需要向供应商问清楚。

-   **利润空间**：我们这里说的是多少钱？如果只涉及 3 到 5 个百分点的利润，那除非你想简化核心产品或服务的销售流程，否则没有太大理由开启这样一段旅程。
-   **范围界定**：确保明确约定，如果客户需要的服务不在标准服务目录内，范围界定该如何进行。
-   **横向支持**：客户直接向供应商付款时，期望获得专业级的服务，所以务必与第三方安排好支持事宜。
-   **折扣**：当公司运营自己的服务部门时，为了达成销售，通常会对服务提供标准折扣。毕竟，从账面上看，你总希望尽可能多地销售核心产品。因此，务必清楚允许的折扣幅度（包括百分比和年收入影响），并且让所有相关方了解这将给运营服务部门的利润空间带来何种影响。这在分包时至关重要，因为如果不小心，服务销售可能出现净亏损。
-   **可见性**：项目是如何处理的？你应该能够看到服务交付的日期和时间，更重要的是，能将其集成到会计系统中，以便你的财务团队能正确确认收入。
-   **客户满意度问题**：客户遇到问题时，通常会打电话给他们的销售团队。你希望销售团队专注于销售，而不是为了应对不满客户而疲于解释。所以，要有一个处理客户不满的标准话术。
-   **客户调查**：客户满意度不仅仅是在愤怒客户来电时做出反应。积极主动地进行客户满意度调查，是掌握客户对所提供服务感受的好方法。
-   **后续支持**：分包商能否直接与客户沟通？如果不能，则应制定严格的规则。如果可以，那么销售和客户管理团队需要了解这对未来服务销售的潜在影响。

"风险"是这类尝试中最重要的方面。服务总是存在出问题的风险。从差旅延误导致资源不足，到错过预约时间，再到意外关闭客户的网络，导致他们在一年中最繁忙的几天中断业务，这些情况我都见过。如果你建立自己的服务部门，那么你就在承担一些风险。如果你的团队足够大，这些风险将分摊到员工身上，可能不会对利润空间造成太大影响。

## 结论

你该做什么？是发展服务组织？专注于利润？从组织中挤出利润空间？建立更强的销售漏斗？帮助达成交易？改进产品？还是以上所有？对于拥有服务团队的公司来说，答案多种多样。总的来说，较小的组织会追求服务收入，而随着组织规模扩大和成熟度提高，这些讨论需要反复进行。

拥有自己的公司可能感觉很不错，但风险也很大。如果你一直在经营自己的咨询公司，并且准备换个新工作，那么在一家更大的组织里带领服务团队，无疑是你可能寻找下一个职位的地方之一。或者，如果你在公司里做过其他职位，现在转到服务部门，希望我们这一章能为你提供一些见解，了解一些该做的事情以及看待业务的方式。

大多数向企业销售硬件和软件的公司都会有一个服务团队。并且，大多数公司对此会有不同的看法。在某些情况下，比如郭士纳接手 IBM 时，一家软件公司可以转变为更侧重于服务的组织。但那是刻意的转型。

归根结底，在成长型公司，职位需求名额是很难获得的。如果你的产品需要太多配套服务，那么也许部分资源应该转向用户体验岗位。
