# 了解你的用户

决定应用程序发展方向的最大因素，取决于你对用户了解多少。这些用户可能是现有用户，也可能针对全新应用设定的假想用户。你对用户了解得越多，就越清楚如何制定用户体验策略的细节。

在着手开发应用程序之前，精心进行一些关于用户本质的基础研究，能带来巨大益处。在许多情况下，最佳的起点是尝试定义用户画像。你可能对应用的本质已有构想，其中包含一些关于该应用将吸引谁、为何会受到欢迎的假设。利用这些假设来构建一个假想用户的画像。画像应包含足够的信息，让你和设计团队成员能在某种程度上与用户产生共鸣。年龄、性别、收入水平和职位是重要细节，但你还应包含家庭与亲属信息，以完善整体图景。生活方式信息也很关键；你越能明确其兴趣、爱好和社交活动，画像就越精准。作为拼图的最后一块，你还需要包含一些关于其动机与目标的信息。对目标和动机的概括性陈述即可，但你或许希望将其应用于特定用例。如果试图从零开始捏造用户画像，这听起来像是艰巨的任务，但如果你以某个你认识的人为基础来构建画像，就可以抢占先机。你可以尽可能准确地描绘那个人，但也始终可以选择修饰细节、补充信息，确保画像是你所针对用户类型的典型代表。

你可能会对自己勾勒出的一个或多个画像能准确反映所寻求的用户类型充满信心。如果你缺乏信心，或者觉得细节不够充分，那么你可能需要考虑在某种程度上验证这些画像。有几种不同的方法可以做到这一点。如果你想保持流程更非正式，可以邀请团队其他成员来确认你的假设或补充更多细节。你也可以选择更正式地验证用户画像，在这种情况下，你需要招募并筛选具有相同基本特征的个人进行访谈。你的提问方式应能帮助你判断关于画像的假设是否准确。你很可能会从这些访谈对象那里学到许多额外细节，这些细节对于完善已建立的画像将非常有价值。

一旦确立了画像，你就可以利用这些信息来模拟与应用程序使用相关的不同类型场景。将画像映射到场景中，让你能够进行角色扮演，并通过目标受众的视角体验假想的应用程序。要有效做到这一点，你需要根据你所记录的特定用户的信息来思考和行动。这说起来容易做起来难，但如果你对自己设计的画像有很好的把握，就能理解他们会看重何种用户体验。详细记录这些场景可能很重要。场景可以采用简短的书面叙述形式，也可以在一系列故事板中可视化呈现。将场景与特定功能或功能集被突出的特定用例对齐，也可能很有价值。
