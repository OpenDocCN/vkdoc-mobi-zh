# 初识设备运动接口

`attitude`属性结合了陀螺仪的角度变化信息、加速度计的重力方向信息，并且可能还会使用磁力计来校准方向（绕 Z 轴旋转）并防止漂移。

`userAcceleration`属性关联了加速度计和陀螺仪的数据，排除了重力和姿态变化的影响，以提供精确的加速度测量值。

`magneticField`属性会调整设备偏差，并尝试补偿磁干扰。

总之，设备运动接口（Device Motion Interface）信息更丰富、更智能。如果说它有缺点，那就是它需要更多的处理能力，这会消耗应用性能并缩短电池寿命。如果你的应用只需要了解运动或旋转的大致情况，那么加速度计或陀螺仪的原始数据就足够了。但如果你真的想了解设备的位置、方向或朝向，那么设备运动接口已经为你计算好了。

> **注意**
> 设备可能需要以“8”字形移动来帮助校准磁力计。如果你将`CMMotionManager`的`showsDeviceMovementDisplay`属性设置为`YES`，iOS 会自动显示一个提示用户进行此操作的界面。

## 测量变化

如果你的应用需要了解任何运动测量值的变化率，它就需要时间信息。例如，要测量角度旋转的变化，你需要用当前速率减去上一次速率，再除以两个样本之间的时间差。

但你在哪里能找到这些测量的时间呢？在前面的章节中，我写道：“`CMAccelerometerData`唯一的属性是`acceleration`”，以及关于`CMGyroData`和`CMMagnetometerData`的类似陈述。这并不完全正确。

`CMAccelerometerData`、`CMGyroData`、`CMMagnetometerData`和`CMDeviceMotion`类都是`CMLogItem`的子类。`CMLogItem`类定义了一个`timestamp`属性，上述所有类都继承了该属性。

`timestamp`属性记录了测量发生的精确时间，允许你的应用精确地比较样本、计算变化率、为后续记录或任何你想到的其他目的。

> **提示**
> 如果你需要计算姿态变化（减去两个`CMAttitude`对象的值），`-multiplyByInverseOfAttitude:`方法可以为你完成计算。
