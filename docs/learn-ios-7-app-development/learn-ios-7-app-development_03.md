# 老派风格 vs. 潮到没朋友

我是一名老派程序员。我是从最底层（字面意义上）开始学习编程的。我写的第一段程序是在一个 4 位微控制器上，通过拨动开关输入机器指令。因此，在我开始使用 `BASIC` 和 `C` 这类“高级”语言编程之前，我几乎已经掌握了关于机器代码的一切。在涉足 `C++` 之前，我对 `C` 语言了如指掌；而在为（革命性的）Macintosh 计算机编写第一个图形用户界面（GUI）应用之前，我已是 `C++` 专家。

尽管我珍视这些积累的知识，且其中大部分仍然有用，但我认识到，如今要开发优秀的 iOS 应用，并不需要这种“自底向上”的方法。过去几十年软件开发领域的许多进步，都在于将开发者——也就是你——从 CPU 指令、硬件接口和软件设计的繁琐细节中解放出来。这让你能够专注于利用这些技术将想法变为现实，而不是把所有时间都花在操心寄存器分配和内存管理上。

所以，令人振奋的消息是，你只需具备最基础的计算机编程知识，或是对底层技术有最起码的了解，就能直接上手创建功能完备的 iOS 应用。这正是本书前几章要做的事情——教你如何无需任何传统的编程方式就能创建一个 iOS 应用。

这并非意味着你不需要这些技能就能掌握 iOS 开发。恰恰相反，你的编程技能越强，你的开发效率就会越高。变化在于，这些技能不再是过去那种必须事先掌握的先决条件了。现在，你可以在探索 iOS 开发新路径的同时，并行学习它们。
