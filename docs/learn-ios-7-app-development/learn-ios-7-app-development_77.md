# 获取路线指引

你的应用可以调用的另一个资源是“地图”应用。没错，就是 iOS 自带的那个标准地图应用。有一些方法可以让你的应用启动地图应用来协助用户。这是一种简单的方式，可以为你的用户提供地图、位置、路线和导航服务，而无需将这些功能集成到你的应用中。

你通过 `MKMapItem` 对象来使用地图应用。你可以从当前位置（`+mapItemForCurrentLocation`）或地理编码地标对象（`-initWithPlacemark:`）创建一个或多个 `MKMapItem` 对象。

创建完成后，向地图项对象发送 `-openInMapsWithLaunchOptions:` 消息（针对单个地图项），或者将一个地图项数组传递给 `+openMapItems:launchOptions:`。启动选项是一个字典，可以用来告诉地图应用要显示地球的哪个区域、要在地图上高亮显示哪个区域、是否需要提供前往指定地点的驾车或步行路线、使用哪种模式（地图、卫星、混合）、是否显示交通信息等等。

使用 `MKMapItem` 的代码示例位于《位置感知编程指南》的“提供路线指引”一章中。

在掌握 iOS 应用开发的旅程中，你已经走了很长的路。你跨越了许多里程碑，而学会使用位置服务是其中重要的一步。你现在知道如何出于各种目的获取用户的位置，并将其显示在地图上。

说到地图，你还学到了很多关于地图的知识。你现在知道如何在你的应用中展示地图，用兴趣点进行标注，以及自定义这些标注的外观。你学会了如何跟踪并在地图上显示用户的位置，以及如何向界面添加你自己的视图控件。
