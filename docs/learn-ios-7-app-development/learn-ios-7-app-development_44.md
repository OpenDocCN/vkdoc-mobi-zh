# 文本视图

文本视图（`UITextView`）对象是标签和文本字段的结合体。它并非两者的子类，但本质上继承了两者的功能，并添加了一些额外特性。文本视图是否可编辑，由其`editable`属性控制。

当`editable`设置为`NO`时，文本视图的行为类似于多行标签。它能够以多种字体、字号和样式显示多行文本，并控制换行方式。在此基础上，它还增加了一些额外功能：滚动、选择和**数据检测器**。

与标签不同，如果文本超出视图的垂直空间，用户可以通过滚动文本视图来查看剩余内容。你在 Surrealist 应用中就使用过这一功能。

用户可以在文本视图中选择文本（通过长按文字）。选中的文本可以复制到剪贴板，或用于在字典中查询单词。此外，你还可以启用数据检测器。数据检测器是一种技术，能够识别特定文本的用途（例如电话号码或某人的电子邮件地址）。用户随后可以点击该文本执行相关操作（拨打电话、撰写新邮件等）。

当`editable`属性设置为`YES`时，文本视图会变成一个（迷你）文字处理软件。用户可以随心所欲地输入、选择、剪切、复制和粘贴。所有在“文本字段”一节中描述的编辑功能和选项都适用于文本视图。唯一缺少的可能是边框；文本视图不会绘制边框。

文本视图还能够编辑带样式（属性字符串）的文本，但你需要提供额外的用户界面元素，让用户能够选择字体、字号、样式、颜色等。文本视图会处理将这些样式应用到用户输入内容的机制，但你的控制器必须告知文本视图具体是哪些样式。

在以下情况中，应使用文本视图，而非标签或文本字段：

- 用户需要编辑多行文本
- 文本内容超出视图范围，且希望实现滚动
- 希望用户能够选择、复制文本，或查询定义
- 希望使用数据检测器
