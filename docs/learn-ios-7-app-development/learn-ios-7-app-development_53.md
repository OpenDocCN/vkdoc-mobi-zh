# 练习

如果 Shapely 的界面有一个重大缺陷，那就是它允许用户创建大到覆盖整个界面的形状，并且允许他们将形状移动到屏幕边缘之外，或者覆盖按钮视图。如果 Shapely 能够温和地滑动或缩小用户拖动的形状，从而避免这种情况发生，那岂不是很棒？我也这么认为。

你的挑战是：向 Shapely 添加代码，使得形状无法被移动到屏幕边缘之外，也无法覆盖添加形状的按钮。解决这个问题有多种方法。你可以简单地阻止用户在拖拽或捏合手势期间过度移动或调整形状大小。另一种解决方案是允许他们将形状移动到任何位置，然后在之后温和地“纠正”它。无论你选择哪种解决方案，都要让用户清楚地了解正在发生的事情，这样用户就不会仅仅认为你的应用出了故障。

你可以在 `Learn iOS Development Projects` ➤ `Ch 11` ➤ `Shapely E1` 文件夹中找到我的解决方案。（提示：我在 `SYViewController.m` 中添加了一个 `-corralShape:` 方法。）
