# 推拉之间

你可能觉得自己还没有完成获取运动数据的第三步，但实际上你已经完成了。当你发送 `-startAccelerometerUpdates` 消息时，这一步骤就已经隐含在内。这个方法会开始收集运动数据，但需要由你的应用定期去查询这些数值。这被称为拉取模式；`CMMotionManager` 对象会持续更新运动数据，而你的应用则按需从中拉取数据。

另一种方式是推送模式。要使用这种方式，请发送 `-startAccelerometerUpdatesToQueue:withHandler:` 消息。你需要传入一个操作队列（我将在第 24 章中解释）以及一个在运动数据更新时立即执行的代码块。这种实现方式要复杂得多，因为代码块会在单独的线程上执行，因此你所有处理运动数据的代码都必须保证线程安全。只有当你的应用必须绝对、立即处理最新的运动数据时，才真正需要这种模式。属于这类情况的应用非常少。
