# 目标的唯一性

最好的类应该具有单一目标。一个设计良好的类应当只代表一样事物或概念，封装与该事物相关的所有信息，而不掺杂其他内容。类中的每个方法应只执行一项任务。软件工程师将此称为单一职责原则。

一个启动定时器的按钮对象功能有限。诚然，如果你需要一个能启动定时器的按钮，它确实能完美胜任。但如果你需要重置分数的按钮或翻页的按钮，它就毫无用处了。相比之下，一个`UIButton`对象具有无限用途，因为它只做一件事：呈现一个可供用户点击的按钮。当用户点击时，它会向另一个对象发送消息。那个对象可以启动定时器、重置分数或翻页。

优秀的对象就像乐高积木™。创建执行简单、独立任务的对象，并将它们与其他对象连接起来解决问题。不要创建能够解决完整问题的对象。我将在第 8 章中进一步讨论这一点。
