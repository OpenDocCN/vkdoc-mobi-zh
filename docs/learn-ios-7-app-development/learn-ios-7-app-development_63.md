# 15. 如果你构建了它……

## 摘要

Interface Builder 是 Xcode 的“秘密武器”。它让创建复杂界面变得轻而易举：将界面元素拖拽到画布上，将它们连接起来，按下一个按钮，它们就变成了你应用中的工作对象。这就像魔法一样。

> 任何足够先进的技术都与魔法无异。——亚瑟·C·克拉克

“魔法”常常被用来描述我们不理解的事物。Interface Builder 有时也符合这个标准；它有效，但你不知道它是如何做到的。好吧，走到幕布后面，准备学习这些秘密吧。在本章中，你将：

-   了解 Interface Builder 文件究竟是什么
-   弄清楚 Interface Builder 文件中的对象如何变成应用中的对象
-   发现与 Interface Builder 功能等效的编程方式
-   理解占位对象是如何工作的
-   以编程方式加载你自己的 Interface Builder 文件
-   提供你自己的占位对象

通常，Interface Builder 文件由你的视图控制器对象加载，这隐藏了许多细节。这没问题。事实上，这非常好。如果你正在使用视图控制器对象，你应该让它处理细节——这正是它的作用。但有时你会遇到无法使用 Interface Builder，但又需要以代码方式完成类似功能的情况。或者，如果你想加载一个不由视图控制器管理的 Interface Builder 文件又该怎么办？如果你首先理解了 Interface Builder 的工作原理，这些都很容易实现。

## Interface Builder 文件的工作原理

一个 Interface Builder 文件包含一个序列化的对象图。在第 14 章中，你学到了一点关于序列化的含义以及所涉及的一些挑战。在第 18 章和第 19 章中，你将学习如何序列化对象以及如何创建可以被序列化（归档）的对象。但现在，你只需要知道，“序列化一个对象”意味着将其属性转换为可传输的字节数组，并最终逆转这个过程以恢复它们。

> **注意：** 我仍然使用通用计算机工程意义上的“序列化”一词。在 Cocoa Touch 的语言中，Interface Builder 文件是对象的归档。加载一个 Interface Builder 文件包括解归档这些对象。

那么什么是对象图？对象图是一个由对象、该对象引用的所有对象、这些对象引用的所有对象等组成的一个集合。启动这个图的一个或少数几个对象被称为根对象或顶层对象。

序列化从根对象开始。该对象将其属性转换为序列化的字节数组。如果它的任何属性引用了其他对象，这些对象就会被要求将其属性值序列化到同一个字节数组中，依此类推，直到整个图中的所有对象和属性值都被转换。最终的字节数组描述了整个对象集合、它们的属性以及它们之间的关系。
