# 位置监控

Pigeon 是一款需要即时、精确（尽可能）且持续监控用户位置的应用程序。因此，它需要一台具备 GPS 功能的 iOS 设备，并持续收集位置数据。并非所有应用都如此。许多应用并不需要精确的位置信息、持续监控，或立即收到移动通知。

对于位置需求不那么高的应用，Core Location 框架提供了多种信息和传递方法。每种方法都涉及不同级别的硬件和 CPU 参与，这意味着它们会以不同方式影响 iOS 设备的电池续航和性能。

作为一条原则，你应只收集应用运行所需的最少位置信息。假设你正在编写一个旅行应用，需要知道用户何时离开一个城市并到达下一个城市。不要启动 GPS 硬件（像 Pigeon 那样）并开始监控他们的一举一动。为什么？因为你的应用永远收不到用户到达目的城市的通知——因为用户的电池早就耗尽了！而用户充电后要做的第一件事，就是删除你的应用。来看一些不需要那么多电量就能获取位置信息的方法。

## 近似位置与非 GPS 设备

没有 GPS 硬件的 iOS 设备也能获取位置信息。这些设备利用从 Wi-Fi 基站、手机信号塔及其他来源收集的位置数据。精度可能很粗略——有时是公里级而非米级——但足以将用户定位到一个城镇。这对于推荐附近的餐厅或正在上映的电影来说，信息已经绰绰有余。

因此，即使你的应用没有`gps`硬件要求，你仍然可以请求位置信息，并且有可能获取到。请查看`CLLocation`对象的`horizontalAccuracy`属性，了解所报告位置的不确定性（以米为单位）。如果该值很大，那么设备可能没有使用 GPS，或者处于 GPS 不准确的环境中。

**注意**

配备 GPS 的 iOS 设备也会使用这些替代位置信息来提高 GPS 三角定位的速度（GPS 三角定位本身相当慢）并降低功耗。该系统称为辅助 GPS（Assisted GPS）。

如果你的应用只需要近似位置信息，请通过向`CLLocationManager`发送`-startMonitoringSignificantLocationChanges`消息（而非`-startUpdatingLocation`消息）来收集位置数据。这种方法仅获取用户位置的粗略估计，并且只在位置发生重大变化时通知你的应用，从而节省大量的处理能力和电池续航。

## 监控区域

回到那个旅行应用，某些 iOS 设备能够监控位置的重大变化，即使设备处于待机状态。这是通过区域监控实现的。区域监控允许你在地图上定义一个区域，并在用户进入或离开该区域时收到通知。这是一种极其高效（低功耗）的确定用户是否移动的方法。

例如，你可以创建两个`CLRegion`对象：一个环绕用户当前所在城市，另一个环绕他们下一个要前往的城市。然后为每个区域（最多 20 个）向位置管理器对象发送`-startMonitoringForRegion:`消息。这样，你的应用只需等待，直到委托对象收到`-locationManager:didEnterRegion:`或`-locationManager:DidExitRegion:`消息。

使用区域监控，当用户到达工作地点或家庭聚会时，你的应用会收到通知。要了解更多关于区域监控的信息，请查阅 Xcode 的“文档与 API 参考”窗口中“位置感知编程指南”的“监控基于形状的区域”一节。“位置感知编程指南”还描述了如何在后台接收位置数据——即当你的应用不是前台活跃应用时，这是本书尚未讨论的内容。

## 减少位置变化消息

延长电池续航的另一种方法是减少应用接收到的位置信息量。我已经讨论过仅接收重大变化或监控区域，但在这两种极端情况之间，还有既非极端又不用每秒更新用户位置的方法。

第一种方法是设置位置管理器的`distanceFilter`和`desiredAccuracy`属性。`distanceFilter`减少了应用接收的位置更新次数。它会等待设备移动了设定的距离后，才再次更新你的应用。`desiredAccuracy`属性告诉 iOS 应该花费多大努力来确定用户的精确位置。放宽该属性意味着定位硬件不必那么辛苦工作。

你可以提供的另一个提示是`activityType`属性。这告诉管理器你的应用用于汽车导航，而非个人健身应用。位置管理器将利用这个提示来优化硬件使用。例如，如果用户在较长时间内没有移动，汽车导航应用可能会暂时关闭 GPS 接收器。

## 移动与方向

你的应用可能更关心用户移动的方向和速度，而非具体位置。如果方向是你关注的重点，请查看从`CLLocationManager`的`location`属性中获取的`CLLocation`对象的`speed`和`course`属性。

如果你只想知道用户的方向，可以通过向位置管理器发送`-startUpdatingHeading`消息（而不是`-startUpdatingLocation`消息）来仅获取该信息。确定用户方向比确定其精确位置更为高效。

要了解更多关于方向信息的内容，请阅读“位置感知编程指南”中的“获取与方向相关的事件”章节。

## 地理编码

如果你的应用对地图上的地点感兴趣怎么办？它可能想知道某个商家的位置。或者它有一个地图坐标，想知道那里有什么。

将关于地点的信息（商家名称、地址、城市、邮政编码）转换为地图坐标，以及反向转换的过程，称为地理编码。地理编码是苹果提供的一项网络服务，它能够将地点的信息字典（例如一个地址）尽可能准确地转换为经纬度，反之亦然。将地点信息转换为地图坐标称为正向地理编码。将地图坐标转换为对该位置的描述称为反向地理编码。

地理编码通过`CLGeocoder`对象执行。`CLGeocoder`会将描述性信息字典或地图坐标转换为`CLPlacemark`对象。地标对象是地图坐标与该坐标位置描述的集合。这些信息将包括坐标所在的国家、地区、城市、街道地址、邮政编码（如果适用），甚至包括它是否是一片水域。
