# 文档概览

“文档”一词有多种含义，但在此上下文中，文档是指包含用户生成内容的数据文件（或数据包），你的应用程序可以打开、修改并最终将其保存到持久化存储中。我们在桌面计算机系统上已经习惯了文档的存在。在移动设备中，文档的概念退居次要地位，但它仍然存在，并且形式大致相同。在少数应用程序中，比如 Pages 文字处理应用，熟悉的文档隐喻会直接呈现在用户面前；你启动应用，看到一系列有名称的文档，选择其中一个进行编辑，文档打开后你就可以开始输入了。在其他应用中，你使用单个文档的感觉可能不那么明显，有些应用甚至完全隐藏了文档机制。你可以为你的应用选择任何一种方法。iOS 提供了所需工具来支持你想要的任何界面，但它不会强制规定某一种。

这种灵活性使你能够在后台完全隐藏地添加文档存储和管理功能，可以松散地耦合到用户界面，也可以效仿桌面计算机系统的传统文档隐喻。无论你对界面做何决定，起点都是 `UIDocument` 类。以下是在你的应用中使用 `UIDocument` 的基本步骤：

*   创建 `UIDocument` 的自定义子类
*   设计一个用于选择、命名和共享文档的界面（可选）
*   将应用的数据模型转换为适合永久存储的数据，或从其中转换回来
*   处理文档的异步读取
*   将文档移动到云端（可选）
*   观察来自共享文档的变更通知，并处理冲突（可选）
*   实现撤销/重做功能，或至少跟踪文档的变更
