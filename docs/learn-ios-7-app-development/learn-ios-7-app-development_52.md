# 从绘图创建位图

你也可以反向操作——将图像或绘图转化为一堆像素——这有两种技术，具体取决于你希望如何处理结果。

最简单且推荐的技术是调用`UIGraphicsBeginImageContext`函数，通过一块内存（它会方便地为你分配）创建图形上下文。你只需指定所需的绘图区域大小。

然后你可以立即开始在该上下文中绘图，就像响应`-drawRect:`消息一样。所有绘图函数都有效，其结果会被写入临时内存缓冲区。完成绘图后，调用`UIGraphicsGetImageFromCurrentImageContext`，iOS 将返回一个新的`UIImage`对象，其中包含你刚刚绘制的内容。你将在第 13 章中使用此技术。

> **注意：** 此技术称为**离屏绘制**，因为你是在一个不会显示在屏幕上、或非响应`-drawRect:`消息的图形上下文中绘图。你可以在任何时候启动离屏绘制来渲染任何内容，然后将其保存到`UIImage`对象中。甚至可以在后台线程中执行绘制。

完成后，调用`UIGraphicsEndImageContext`来销毁上下文并丢弃临时缓冲区。

虽然此技术对于将任何绘图转换为图像很有用，但它仍然无法让你访问绘制内容的单个像素；你无法从上下文或`UIImage`对象中获取这些信息。如果你要追踪像素，就需要使用更低级的函数`CGBitmapContextCreate`。

`CGBitmapContextCreate`创建一个绘图上下文（就像`UIGraphicsBeginImageContext`一样），但缓冲区是你提供的字节数组，正如之前在`CMColorView`中所做的那样。上下文创建后，你执行的任何绘图都会直接写入该数组。绘制完成后，你可以对生成的像素进行任何操作：统计黑色像素的数量、查找最暗和最亮的像素，你能想到的都可以。

所有这些技术以及支持的众多像素格式，都在 *Quartz 2D 编程指南* 中有所描述，你可以在 Xcode 的文档和 API 参考中找到。

## 高级图形

哦，还有更多。在你被这些图形话题搞得头晕目眩之前，请允许我简要介绍几个可能派上用场的技巧。

### 文本

你也可以直接在自定义视图中绘制文本。基本技巧如下：

1. 创建一个描述文本字体、样式和大小的`UIFont`对象。
2. 设置绘制颜色。
3. 向`NSString`对象发送其任一`-drawAtPoint:...`或`-drawInRect:...`消息。

你还可以使用各种`-sizeWithFont:...`方法获取字符串绘制时的大小（以便计算其占用的空间）。

在第 4 章中编写的 Touchy 应用以及之后第 12 章中的 Wonderland 应用里，都能找到相关示例。`-drawAtPoint:...`和`-drawInRect:...`方法只是底层文本绘制函数的封装，这些函数在 *Quartz 2D 编程指南* 的“文本”章节中有所描述。如果你需要对文本进行精确控制，请阅读 *Core Text 编程指南*。

### 阴影、渐变和图案

你已经学会了绘制实心形状和实线。Core Graphics 还能实现更多功能。它可以绘制图案和渐变，还可以在你绘制的形状背后自动绘制“阴影”。

你可以通过创建各种图案、渐变和阴影对象，然后像设置颜色一样将它们设置到当前图形上下文中来实现这一点。在 *Quartz 2D 编程指南* 中可以找到丰富的示例和样例代码。

### 混合模式

图形上下文以及许多绘图函数的另一个属性是混合模式。混合模式决定了正在绘制的像素如何影响上下文中已有的像素。通常，混合模式为`kCGBlendModeNormal`。这种模式绘制不透明像素，忽略透明像素，并混合半透明像素的颜色。

还有大约二十多种其他混合模式。你可以执行“乘法”和“加法”，仅在现有图像的不透明部分绘制，仅在现有图像的透明部分绘制，使用“硬光”或“软光”绘制，仅影响亮度或饱和度——列表很长。你可以使用`CGContextSetBlendMode`函数设置当前混合模式。某些绘图方法接受一个混合模式参数。

可用的混合模式在 *Quartz 2D 编程指南* 的两个地方都有文档记录和示例。对于绘图操作（形状和填充），请参考“路径”章节中的“设置混合模式”部分。对于图像混合的示例，请查找“位图图像和图像蒙版”章节中的“将混合模式用于图像”部分。

### 上下文堆栈

所有这些设置可能会让你的图形上下文难以处理。假设你需要绘制一个复杂形状，包含渐变、投影、旋转，并带有特殊混合模式。在设置所有这些属性并绘制完形状后，你现在只想绘制一条简单的线。天哪！你是否需要重置每一个设置（投影、变换、混合模式等）？

别慌，这是常见情况，有一个简单的机制可以处理。在进行大量更改之前，调用`CGContextSaveGState`函数来保存当前图形上下文几乎所有的状态。它会拍摄你当前上下文设置的快照，并将其推入堆栈。然后，你可以更改所需的任何绘图属性（裁剪区域、线宽、描边颜色等），并绘制你想要的任何内容。

完成后，调用`CGContextRestoreGState`，上下文的所有设置将立即恢复到你调用`CGContextSaveGState`时的状态。你可以根据需要任意嵌套这些调用：保存、更改、绘制、保存、更改、绘制、恢复、绘制、恢复、绘制。在复杂的绘制方法中，以调用`CGContextSaveGState`开始并不罕见，这样方法的后续部分可以获取到一个未修改的图形上下文。
