# 19. 文档，你是说存储

## 摘要

如果你的 iOS 应用需要存储超过少量信息，就需要使用文档。iOS 提供了一个强大的文档框架，将数据存储带入了 21 世纪。iOS 文档（`UIDocument`）类处理或让你轻松实现现代功能，如自动保存、版本控制和云存储。在此过程中，你最终将学会如何归档对象。在本章中，你将：

- 创建自定义文档对象
- 使用文档对象作为应用的数据模型
- 学习如何归档和取消归档数据模型对象
- 设计可增量加载或保存的文档
- 处理异步文档加载
- 管理文档更改和自动保存

你会发现大量关于如何使用`UIDocument`的“指南”，因为坦率地说，这是一个使用起来很复杂的类。它有许多活动部件，而且有相当多的细节你必须注意。这导致许多开发者忽略`UIDocument`，转而“自行开发”文档存储方案。不要这样做。掌握`UIDocument`并不难，而且回报丰厚。

`UIDocument`可能看起来令人生畏——直到你理解它为何如此运作。一旦你理解了其架构背后的一些逻辑，以及它为你实现了什么，你需要编写的代码就都会变得合理。因此，在本章中，你不仅将关注“如何做”，还将关注“为什么”。到本章结束时，你将能像专业人士一样使用`UIDocument`。
