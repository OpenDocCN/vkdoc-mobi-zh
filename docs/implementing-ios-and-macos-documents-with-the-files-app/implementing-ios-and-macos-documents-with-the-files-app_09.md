# 8. 使用共享按钮共享文档

在前面的章节中，你了解了如何在 iOS 和 macOS 上实现文档以保存数据并在你的应用之间共享数据。在本章中，你将看到另一种使用共享按钮共享数据的方式，这样你就可以动态地从一个应用向另一个应用共享数据，而无需使用文档来存储和共享数据。

本章涵盖的主要内容有：

-   使用共享按钮（作为用户）
-   使用共享按钮（作为开发者）
-   管理共享数据

## 使用共享按钮（作为用户）

要共享数据，需要一个应用来共享数据（有时这个过程称为*提供*数据）以及另一个（或多个）应用来接收数据并按接收应用的需求使用数据。请注意，就像文档及其类型的情况一样，发送者和接收者（或提供者和接收者）之间的连接是动态的。两个应用都不需要了解对方。文档类型或共享信息让每个应用能够独立运行，这样你就不需要构建需要大量开发和维护成本的大型多用途应用。

共享是 Cocoa 和 Cocoa Touch 的一部分（从操作系统的原始 NeXTSTEP 和 Rhapsody 版本开始就已经存在了）。为了共享数据，发送者和接收者都需要有一个通用的格式可用。以下是一个基本结构的例子。

## 摘要

尝试发送方与接收方的各种组合。传递给活动视图控制器的项目集合是一个类型为 `Any` 的集合。供参考，以下是 `UIActivityViewController` 中 `init` 的声明：


你会注意到，你可以将活动指定为可选类型；关于它们的更多信息可在文档中找到。

### 注意

严格来说，偏好设置是某个设置的值，例如绘制新线条时使用的颜色，其中 `lineColor` 偏好的值为蓝色。一旦设置了某个设置的值，该值就会持久化，并在每次应用运行时使用，直到被更改。在这种情况下，偏好设置也可称为默认值。在 iOS 中，内置的“设置”应用让用户可以控制偏好设置和配置。

Cocoa 中的偏好设置存储在 Cocoa 偏好设置系统（也称为用户默认值系统）中。如果你想对这些术语进行严格区分，那么在 iOS 中，内置的“设置”应用为用户控制这些值。在 Cocoa 和 Cocoa Touch 中，用户默认值系统管理一个包含这些值的数据库。有些人区分设置（用户界面的一部分）和用户默认值（Cocoa 框架的一部分）。

本章探讨了当今你可以用于这些数据工具的各种可能性。

## 审视数据结构

处理文档时，你可以控制数据的格式化和管理。你可以使用标准文档类型，该类型可能有关联的类（例如图像），但用于用户默认值、设置和偏好设置的数据则更为有限。具体来说，这些对象必须全部是属性列表元素。属性列表是一种键值结构，其中键是字符串，值是以下任何一种简单数据类型：

- 字符串
- 日期
- 整数
- 浮点数
- 布尔值

属性列表（扩展名为 `.plist`）专为高效的序列化而设计。`PropertyListSerialization` 类会为你处理此过程。属性列表在 Cocoa 中广泛使用；多年来，这段代码经过不断优化和测试，使其可以可靠地用于许多任务。

此外，`NSData` 类型可以存储在属性列表中。如果这样做，你必须处理该数据与你想要处理的对象之间的序列化和反序列化。一个非常常见的例子是将 `NSData` 类型转换为某种二进制格式，然后可以将其解释为图像或其他复杂的数据结构。

属性列表还可以包含数组和字典。这意味着你可以使用一个属性列表，该列表本身是一个包含字典的数组，而该字典又声明了另一个数组以及数字和其他简单对象。

在考虑用户默认值、设置和偏好设置时，需要牢记的另一个限制是，苹果公司的指导建议是，这些工具应仅用于有限的目的和相对少量的数据。

在考虑以这种方式存储数据时，不必过于担心可以存储多少数据作为用户默认值、设置或偏好设置，因为这并非真正的限制因素。如果你以这种方式用尽设备几乎所有的存储空间，远在你用尽空间之前，性能就会下降。

属性列表作为 `PropertyListSerialization` 类的实例存储。作为该类的实例，属性列表在读取或写入时会被完整地读取或写入。这可能意味着，为了访问一个单一的布尔值，你可能需要读取或写入一个包含几十、几百甚至上千个项目的完整属性列表。你可以通过将属性列表值存储在数组或字典中来集中读取或写入，但这并不能真正帮上什么大忙。你需要读取整个属性列表才能获取到数组或字典，然后才能精确定位到你想要的值。

存储为默认值、偏好设置和设置的数据可以是任何你想要存储的内容（前提是它是一个属性列表，并且不占用太多空间）。这些术语之间存在差异，这正是理解它们的一个很好起点。

### 提示

当你考虑使用这些工具存储自己的数据时，请记住，“设置”应用和用户默认值系统会管理这些数据。如果你希望完全控制数据、数据的存储方式以及数据的用户界面，你可能需要使用数据库或文档，由你自行控制这些特性。

## 理解用户默认值

用户默认值是应用某个方面的起始值（默认值）。例如，一个允许你存储班级中学生姓名的应用，可能会首先将学生标识为“学生 1”、“学生 2”，以此类推。用户添加的新学生可能被称为“新学生”。

在这样一个应用中，你可能希望让用户设置他们自己的默认值，例如：

- “学生”是应用中每个项目的名称，新项目被标记为“新学生”。
- 你可以将项目的默认值更改为“项目”，这样在应用运行时，你会看到“椅子”和“新椅子”。
- 稍微复杂一点的默认值可能允许用户在“新”和数字之间做选择，以标识新项目，例如“新椅子”或“椅子 3”。

像这样的默认值通常会对用户公开，以便他们可以选择。

### 提示

在这种情况下，空字符串可能是默认值。
