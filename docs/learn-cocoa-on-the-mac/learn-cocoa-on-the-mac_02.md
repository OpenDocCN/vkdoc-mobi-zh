# 必爱可可

## 摘要

欢迎！你来到这里，想必是为了给 Mac 编写程序。没错，你绝对来对地方了。（来找普拉提？右边第三个门。）读完整本书，你将掌握创建快速、高效、美观的 Mac OS X 应用所需的一切知识。

创建现代 Mac 应用的关键在于 Cocoa。根据苹果公司的说法，Cocoa 是一组面向对象的框架，为 Mac OS X 应用提供运行时环境。随着本书的深入，你将全面了解 Cocoa 框架和运行时环境。现在，不妨把 Cocoa 想象成程序员的助理，它能处理 Mac 开发中的大部分日常事务。在使用 Cocoa 编写程序时，从绘制窗口到在文本字段中闪烁光标，Mac 应用执行的几乎所有常见任务都由它代为处理，让你能专注于编写使应用独具特色的代码。

Cocoa 几乎为你所有的开发需求都提供了对应的类。Mac OS X 用户界面的每个部分，从窗口到菜单、滚动条到按钮、图像到图标，都有对应的 Cocoa 类。只要你能想到一个想添加到应用中的用户界面元素，该元素极有可能已经作为 Cocoa 类实现了。

使用 Cocoa 的另一个好处是它与 Mac OS X 紧密结合。使用 Cocoa 构建应用，你的应用就能与其他应用良好协作，并无缝对接 Finder 和 Dock 等 Mac OS X 元素。

### 25 年何止 25 年？

Cocoa 自 1986 年以来就以某种形式存在了。我们称之为 Cocoa 的这些技术源于 NeXTStep 的 `AppKit`，这是为 NeXT 平台开发的应用程序构建工具。当苹果公司在 1996 年收购 NeXT 时，他们开始构建新版本的 Mac OS（即我们现在所称的 OS X），新操作系统的大部分基础都源于从 NeXT 收购的技术。

随着 OS X 的发展，Cocoa 也在不断演进。苹果公司加入了经典 Mac OS 的技术，如 QuickTime，以及全新的技术，如 Quartz 渲染系统，它实现了 OS X 使用的所有炫目视觉效果和动画。他们还确保在此过程中保留了 Mac 著名的易用性。

归根结底：Cocoa 在不断地发展和扩展。你在本书中将学到的开发工具、库和框架，是超过 25 年实验和精炼的成果。

## 准备一台 Mac 并下载工具

在开始使用 Cocoa 创建应用之前，你需要一台 Mac。它不必是最新或最强的 Mac；事实上，过去四五年左右生产的几乎所有 Mac 都能很好地完成本书中的练习。当你编写软件的意愿变得更加强烈时，你可能会发现投资一台更新或更快的机器是值得的，但就目前而言，几乎任何基于 Intel 的 Mac 都能很好地满足你学习所需的知识。最新的 Mac 目前最高可运行 OS X 10.8，而最老的基于 Intel 的 Mac 无法运行任何高于 10.7 的 OS X 版本。但本书将使用的 Xcode 4.6 在 10.7 和 10.8 上都能良好运行。

你还需要一个 Apple ID（可能与你已在 Mac 或 iOS App Store 或其他 iTunes 购买中使用的 ID 相同），并且强烈建议你加入苹果开发者关系组织——苹果开发者连接。其中有付费会员，但基本会员是免费的。要加入 ADC，请打开网页浏览器并导航至 [`http://developer.apple.com/mac/`](http://developer.apple.com/mac/)。如果你已有 Apple ID，请点击“登录”按钮并输入你的信息。如果你是苹果在线服务的新用户，请点击注册链接并创建你的账户。

无论你是注册成为开发者，还是仅使用你的 Apple ID，获取苹果免费开发者工具集 Xcode 的最简单最佳方法是从你 Mac 上的 App Store 下载。只需启动 App Store，搜索 Xcode，然后让它安装。与 App Store 上的大多数内容一样，安装过程基本是自解释的。Xcode 的安装时间可能比大多数其他应用长得多，因为它是一个相当大的下载包。

> 注意：如果你已经为 iOS 开发安装了 Xcode，那么你已经准备好了。Xcode 包含了创建 OS X 应用和 iOS 应用所需的所有工具和框架。尽管框架存在重大差异，但工具基本相同，因此如果你做过任何 iOS 开发，在创建 Cocoa 程序时应该会感到得心应手。

## 开始前你需要了解的知识

本书假定你已经具备一些基本的编程知识。你应该熟练掌握循环、变量、指针和链表等概念。本书还假定你理解面向对象编程的基础知识，并熟悉 Objective-C 编程语言。从 Mac OS X Leopard 10.5 开始，该语言引入了新版本，称为 Objective-C 2.0。如果你不熟悉 Objective-C 语言较新的补充内容，不用担心。我们会特意强调所使用的 2.0 语言特性，并解释它们的工作原理以及为何使用它们。

### 初次接触 Objective-C？

以下资源可以帮助你入门。

- 编程新手？请查阅《在 Mac 上学习 C 语言（第 4 版）》，由我们的好友 Dave Mark 编写（[`www.apress.com/9781430218098`](http://www.apress.com/9781430218098)）。
- 刚接触 Objective-C？请查阅《在 Mac 上学习 Objective-C（第 2 版）》，这是 Mac 编程专家 Scott Knaster、Waqar Malik 和 Mark Dalrymple 编写的一本优秀且易于入门的 Objective-C 教程（[`www.apress.com/9781430241881`](http://www.apress.com/9781430241881)）。
- 然后，前往苹果的 Mac 开发者库，下载一份《使用 Objective-C 进行编程》，这是对该语言非常详细和广泛的描述，也是一本极好的参考指南（[`http://developer.apple.com/library/mac/#documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC`](http://developer.apple.com/library/mac/#documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC)）。



## 准备好了吗？

Mac 是一个卓越的计算平台，而 Cocoa 是构建 Mac 应用程序的最佳工具。如果你从其他平台转向 Mac，可能会觉得 Cocoa 和 Objective-C 有些令人摸不着头脑。尽管一开始可能会难以适应，但我们相信，随着你阅读本书的深入，你将很快掌握 Cocoa 的“做事方式”。只要稍加坚持，一切都会变得清晰明了。

有一点需要牢记：本书中的练习并非一份做完就能让你成为 Cocoa 开发大师的清单。在每一步中，务必确保你理解了自己刚刚完成的操作，然后再进入下一个项目。不要害怕修改代码。大胆实验并观察结果。这是理解在 Cocoa 这样的环境中编码复杂性的最佳方法之一。

此外，请记住我们为读者设立了一个讨论本书练习和内容的留言论坛。因此，如果你遇到困难，或者想与我们以及其他 Cocoa 开发者探讨任何 Cocoa 编程话题，请访问 [`http://learncocoa.org`](http://learncocoa.org) 并注册讨论论坛。
