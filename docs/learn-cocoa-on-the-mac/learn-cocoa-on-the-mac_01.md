## 前言

我第一次接触 Cocoa 是在 1989 年上大学的时候。当然，那是在 iPhone 之前，在 Mac OS X 之前，甚至是在它被命名为 Cocoa 之前。那时，今天 Cocoa 的种子是 NeXTStep 的一部分，即 NeXT 计算机核心的操作系统。NeXTStep 领先于时代多年，虽然那间装满 NeXT 工作站的实验室在计算机科学课程中利用率极低，但我的学生系统管理员工作让我每天都要使用它们。作为一名用户，我便深陷其中。在此我不赘述 NeXT 的用户体验，但我要说的是，Mac OS X 的许多最佳特性并非源自老 Mac，而是来自 NeXTStep。

那时，还没有万维网，NeXTStep 开发者社区也很小，除了 NeXT 随其早期机器附带的那些晦涩难懂的巨著之外，关于开发环境的文字资料少之又少。我时不时试图搞懂 Objective-C 和 AppKit，但身边没有专家，也缺少可供参考的示例代码（更不用说我实际的学习任务有时会分散我摆弄有趣项目的精力），所以我基本上是无从下手。

大学毕业后，发生了一件完全出乎意料的事。一位朋友向我介绍了我们市里一家咨询公司，该公司正在为一些相当大的客户构建定制的 NeXTStep 应用程序，我有幸加入了进来。突然间，我拥有了一群同事，他们不仅已经使用 NeXTStep 编程了一段时间，有些人甚至曾在 NeXT 工作过！其中一位就是彼得·克拉克。只需要他们一点专业的帮助就能让我起步，那些多年来一直神秘莫测的东西突然就讲得通了。短短几周内，我学到了很多，甚至能够开始领导一些 NeXTStep 开发的培训和指导工作。

这里想说的重点不是我有多聪明或是学得快。而是，我们现在称之为 Cocoa 的这一系列技术确实非常强大——而且相当容易学习和使用——但在这个过程中你很可能需要一些帮助。我希望这本书能引导你走上正确的方向，并帮助你掌握 Cocoa 编程的精髓，这样当你读完它时，你将拥有足够的 Cocoa 知识，能够推动自己前进，编写出你梦寐以求的 Mac 应用程序。简而言之，这是一本我希望自己在 20 年前就能找到的书。

## 关于作者

**杰克·纳廷** 从旧时代就开始使用 Cocoa，远早于它被命名为 Cocoa 的时候。他使用 Cocoa 及其前身为广泛的行业和应用开发软件，包括游戏、平面设计、在线数字发行、电信、金融、出版和旅行。当他不从事 Mac 或 iOS 项目、不用 Unity3D 制作精彩游戏、或不在他的朋克摇滚乐队里尽情演奏时，他通常会与家人共度时光。杰克在 Twitter 上的账号是 `@jacknutting`，并偶尔在 [`www.nuthole.com`](http://www.nuthole.com) 上写博客。（照片由艾莉森·德·玛尔斯·冯·布里森提供。）



## 彼得·克拉克

彼得·克拉克大约在 1990 年上大学时开始接触 NeXTStep。这段经历奠定了他职业生涯前半段的基调，当时他与杰克在双城的一家咨询公司共事。他曾为湾区、芝加哥和双城的公司开发 NeXTStep 和 WebObjects 项目，撰写了一本关于 OpenStep 开发的书籍，随后转向 Java、.NET、Ruby 和 Scala 开发以及管理工程团队。他很高兴 NeXTStep 的众多设计模式至今仍有用武之地，并发现 Cocoa 开发既令人感到熟悉又充满惊喜。彼得目前在明尼苏达大学担任一个研究项目的软件开发总监。他偶尔会在[`http://blog.pclark.net`](http://blog.pclark.net)上写博客，你可以在 Twitter 上通过`@pclark`找到他。彼得想让你知道，他至少比杰克早一两个月开始使用 NeXTStep。（照片由韦德·斯特宾斯提供。）
