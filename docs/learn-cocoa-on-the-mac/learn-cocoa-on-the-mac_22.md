# 11. 窗口、菜单和表单

在过去的几章中，我们一直专注于 Cocoa 编程中所谓“后端”的部分，即那些帮助处理应用程序基础设施的模型和控制器类。现在，是时候将注意力转向 Cocoa 的“前端”，更多地关注 MVC 架构中的视图部分了。

在本章中，我们将重点讨论窗口（不是 Microsoft 那种）、菜单（不是餐厅那种）、面板（不是太阳能那种）和表单（不是床单那种），这些都是高级 GUI 对象，几乎任何 Cocoa 应用程序都离不开它们。窗口为视图对象的绘制提供了基础。在 Cocoa 中，这些由 `NSWindow` 及其直接子类 `NSPanel` 以及几个专门的子类表示。菜单提供了大多数 Mac 应用程序都有的、熟悉的屏幕顶部访问系统级和特定应用用户操作的方式，由 `NSMenu` 和 `NSMenuItem` 对象的层次结构表示。表单通过将模态窗口作为覆盖层附加到现有窗口上，为经典的浮动模态窗口提供了一种替代方案，为用户提供了更连贯的界面。

我们将在一个“实验室”中探索这三个领域，这是一个没有实际用途的玩具应用程序，仅用于演示某些功能并让我们了解这些组件是如何工作的。
