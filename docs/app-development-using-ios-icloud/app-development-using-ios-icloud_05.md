# 1. 引言

感谢您阅读并喜爱我们的《使用 iOS iCloud 进行应用开发》一书。当我们在 2023 年撰写这本书时，iOS 版本是 15。自那以后，iOS 已经发布了两个主要版本和几个次要版本。本书是对前作的更新，旨在提供所有关键变更的详细信息，这些变更对于提升您的代码质量和体验至关重要。以下是本书将要涵盖的关键领域。

## 增强的异步 iCloud 数据库操作

iOS 17 引入了几个新的块方法（`recordMatchedBlock` 和 `queryResultBlock`）以提高查询效率和效果。这些块方法极大地减少了代码行数，并具有定义清晰的闭包语法。查询完成后，返回块具有基于 case 的返回语句，预定义了成功和失败等条件的默认定义。这些块还增强了其错误处理能力。

本书将有一个专门章节来介绍这一点。我们将用新的基于块的查询结构更新前作中的所有查询。

## 类扩展

类扩展对于 iOS 17 来说并不新鲜，但未在我们书的第一版中涉及。类扩展是一种属性工具。基于面向对象的原则，扩展为现有类添加新功能。

第 3 章将侧重于创建类级别的过程、定义属性和声明类级别的协议。我们的示例将创建图像选择器控制器委托和 UI 导航控制器委托作为类扩展，允许用户为图像对象设置图像。

## 模型-视图-控制器设计框架

模型-视图-控制器 (MVC) 是一种非常适合 Cocoa 应用程序的设计模式。MVC 设计模式使代码模块化，易于阅读。

我们将有一个专门章节介绍在 Xcode 中的 MVC 架构实现。我们将使用模型类来持久化数据，使用控制器进行导航，使用视图进行屏幕显示和呈现。

## 图像处理

iCloud 提供了一个选项来存储图像等二进制对象。本书将有一个专门章节介绍图像处理。

我们将学习如何从照片库上传图像，或从手机相机拍照，并将图片二进制内容存储在 iCloud 存储库（资产类型）中。我们还将了解允许用户访问相机和照片库所需的配置更改（`info.plist`）。在本章中，我们还将学习图像处理。



## 下拉列表

用户界面设计是 iPhone 应用设计中的关键要素。由于 iPhone 外形尺寸有限，开发人员需要明智且巧妙地利用可用空间。本章将重点介绍如何使用表视图创建一个包含预填充数据的下拉列表。它将允许用户从下拉列表中选择一个值，并在某个 UI 对象（按钮、标签或文本字段 UI 对象）上显示所选值。

## 创建可扩展的文本字段 UI 对象

本章将探讨如何根据文本长度扩展文本框的高度，以向用户显示完整的文本内容。当文本溢出时，它将允许文本按单词或字符进行换行。此外，它还会让文本框出现在键盘上方。

### 动画视图

在本书的第一版中，我们学习了 UI 对象动画。我们将有一章专门介绍更多的 UI 对象动画。
