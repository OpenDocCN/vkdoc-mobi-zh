# 第 8 章：内存管理

本章探讨如何使用 Objective-C 管理内存。

本章中的实例将向您展示如何：

-   理解内存管理
-   使用引用计数管理内存
-   使用自动引用计数（ARC）管理内存
-   在 Mac 上使用垃圾回收（GC）管理内存

## 8.1 理解内存管理

### 问题

您希望理解如何在 Objective-C 应用程序中有效地管理内存。

### 解决方案

内存是 Objective-C 应用程序所需的有限资源之一。您在 Objective-C 中使用的每个变量和对象都会占用一定内存。由于内存是有限资源，因此有可能耗尽所有可用内存。

像 Objective-C 这类基于 C 的编程语言的程序员需要思考如何有效管理内存。这对于许多使用 Java 或脚本语言的程序员来说是新鲜事。内存管理是我们在 Objective-C 中要处理的最棘手的问题之一。

内存管理不当的应用程序后果严重：应用程序可能变慢，甚至系统可能会关闭使用过多内存的应用程序。应用程序也可能发生内存泄漏。当对象被分配了内存，但即使应用程序代码中无法再访问该对象时，系统仍未回收这些内存，就会导致内存泄漏。

让我们回顾一下在决定如何处理内存管理时需要考虑的一些事项。

##### 对象生命周期

就像现实世界中的对象一样，Objective-C 对象被创建、存在然后消亡。这就是*对象生命周期*。

就内存管理而言，当一个对象诞生时，您为该对象预留内存供其使用。随着对象继续执行您编程设定的功能，对象会持续占用这些内存。一旦您不再需要该对象，就让它消亡。当对象消亡后，您可以回收该对象的内存。

##### 对象所有权

*对象所有权*是指一个实体对另一个实体负责的概念。当一个实体拥有一个对象时，该实体负责在使用完该对象后对其进行清理。例如，如果一个对象在 `main` 函数中被创建和使用，那么 `main` 函数就对该对象负责。这意味着 `main` 函数有责任清理该对象。

使这个概念稍显复杂的是，对象可以被多个实体拥有。因此，一个对象可能由 `main` 函数创建并拥有，也可能被另一个声称拥有该对象所有权的实体使用。

您会看到多个对象所有权的常见情况是在使用数组时。数组是对象的索引列表，当您将一个对象放入数组时，数组声称拥有该对象的所有权。因此，如果我在 `main` 函数中创建一个对象，然后将其放入数组，那么 `main` 函数和数组都会声称拥有该对象的所有权——并且两者都有责任清理该对象。

显然，如果没有某种系统辅助，跟踪所有可能的对象关系会迅速变得令人望而生畏。Objective-C 确实提供了一个称为*引用计数*的系统，可以帮助您跟踪对象生命周期和对象所有权。

##### 引用计数

引用计数是一个用于跟踪有多少个实体声称拥有对象所有权的系统。每个对象都有一个与之关联的特殊数字，称为*引用计数*。引用计数表示有多少个对象声称拥有该对象的所有权。

当一个对象被创建时，该对象的引用计数被赋值为 1。这表示刚刚创建该对象的实体的所有权声明。对于每个声称拥有该对象所有权的实体，对象的引用计数增加 1。

一旦某个实体不再需要拥有该对象的所有权，该实体可以减少该对象的引用计数。当一个对象的引用计数达到零时，系统会自动销毁该对象，并且系统能够回收其内存。

通过引用计数，Objective-C 负责实际的对象销毁。拥有者对象只需负责释放其对对象的所有权声明。

###### 自动释放

在某些情况下，这个引用计数系统会失效，因为不清楚谁应该声称拥有对象的所有权。例如，一些类通过函数提供旨在临时使用的便捷对象。在这种情况下，对象创建者不能声称拥有所有权，因为创建者类永远没有机会释放所有权，您也不能假设接收者会承担起创建对象的责任。

Objective-C 通过提供一种延迟放弃对象所有权的方式来提供帮助。因此，我可以创建一个对象，并声明在不久的将来某个时间点该对象可以被销毁。这被称为*自动释放*。
