### 第 2 步：为应用程序制作故事板

#### 配置主场景

![图片](img/9781430242727_Fig07-01.jpg)

**图 7-1.** *打开故事板。*

1.  既然已经设置好了数据模型，现在就可以开始设计界面了。打开故事板。你会立刻看到 Xcode 已经为主从应用程序创建了一个基本的层级结构。一个基本的导航控制器连接到一个主视图控制器，该控制器通过 segue 连接到详细视图控制器，如图 7-1 所示。你将使用这个模板来满足需求。

![图片](img/9781430242727_Fig07-02.jpg)

**图 7-2.** *将标题重命名为“我的图书馆”。*

2.  双击主视图控制器标题栏上的“Master”标题，将其重命名为“我的图书馆”，然后点击一次原型单元格以选中它，如图 7-2 所示。

![图片](img/9781430242727_Fig07-03.jpg)

**图 7-3.** *将单元格的行高改为 50。*

3.  选择“实用工具”视图，然后点击“大小检查器”。在“行高”属性中输入 `50`，如图 7-3 所示，然后按回车键。

![图片](img/9781430242727_Fig07-04.jpg)

**图 7-4.** *选择表格视图。*

4.  你正在处理的场景将成为应用程序中的第一个视图。在这种情况下，它将允许用户在你的图书馆的两个顶级层级中选择：书籍和作者。因此，你很清楚要在表格视图中显示多少个单元格：两个单元格。这使得它非常适合作为静态表格视图——这是故事板独有的一项出色新功能，允许你在故事板中静态地布局表格视图的内容，在运行时显示它，甚至在用户选择某个单元格时切换到其他视图，而无需编写任何代码！让我们将表格视图转换为静态格式：点击原型单元格下方的位置来选择表格视图，如图 7-4 所示。

![图片](img/9781430242727_Fig07-05.jpg)

**图 7-5.** *更改表格视图内容类型。*

5.  选择“属性检查器”，将“内容”类型从“动态原型”改为“静态单元格”。你会看到三个相同的单元格出现，如图 7-5 所示。

**注意：** 动态表格视图显示动态内容，在这种情况下，表格视图行数较多（有时甚至不限量）。动态表格视图使用可重用的动态单元格原型来高效地显示非常大的数据集。静态表格视图本质上就是“所见即所得”。对于静态内容，表格视图的行数是已知且固定的，因此你可以在界面生成器中设计所有行。请注意，静态表格视图只能通过故事板创建，并且必须由表格视图控制器的子类来控制。

![图片](img/9781430242727_Fig07-06.jpg)

**图 7-6.** *命名层级树的根节点。*

6.  点击三个单元格中的一个并将其删除，这样只剩下两个单元格。现在让我们思考一下。你将拥有两个数据树：一个按书籍查看数据库，另一个按作者查看数据库。这意味着你的根节点有两个选择：书籍和作者。将这两个单元格的标签文本从“Title”分别改为“书籍”和“作者”，如图 7-6 所示。现在，你将删除连接主视图控制器到详细视图控制器的 segue：点击该 segue，如图 7-6 所示，将其选中，然后删除。

![图片](img/9781430242727_Fig07-07.jpg)

**图 7-7.** *删除 `MasterViewController.m` 中 `@end` 和 `@implementation` 之间的代码——箭头指示了原来代码所在的位置。*

7.  在进行下一步之前，还有一件最后的事情要做，以确保你的静态表格视图能正确显示。你需要删除主视图控制器的一些现有代码。Xcode 在主从应用程序模板中，为你在 `MasterViewController.m` 中创建了一些代码。这些代码实现了 `UITableViewDelegate` 和 `UITableViewDataSource` 协议方法，如果它们存在，将会覆盖你在故事板中定义的静态表格视图内容。因此，在项目导航器中点击 `MasterViewController.m` 文件，选中 `@implementation` 和 `@end` 之间的所有代码，并将其删除，如图 7-7 所示。

**注意：** 因为这是一个静态表格，所以不需要任何额外的代码。稍后，你将分别为“书籍”和“作者”这两个静态单元格创建 segue，连接到它们各自要跳转到的视图控制器。这就是为什么目前主视图控制器和详细视图控制器之间没有 segue 连接的原因。



#### 设计顶层视图：分类场景

![images](img/9781430242727_Fig07-08.jpg)

**图 7-8.** *拖入一个新的视图控制器。*

1. 根据应用流程，当用户点击`Books`单元格时，你需要显示一个可用分类列表。因此，你需要一个额外的`View Controller`来展示书籍的各种分类。现在，将之前从`Master View Controller`中断开连接的当前`Detail View Controller`移到右侧。你可以在图 7-8 右侧看到它的左侧部分。这样做是为了给即将拖入的新`View Controller`腾出空间。一旦空间足够，从你的`Object Library`中拖入一个新的`View Controller`，如图 7-8 所示。

![images](img/9781430242727_Fig07-09.jpg)

**图 7-9.** *拖入一个表格视图。*

2. 在`Object Library`中找到一个`Table View`，并将其拖到新的`View Controller`上，如图 7-9 所示。

![images](img/9781430242727_Fig07-10.jpg)

**图 7-10.** *拖入一个表格视图单元格。*

3. 现在，将一个`Table View cell`拖入新的`Table View`，如图 7-10 所示。

![images](img/9781430242727_Fig07-11.jpg)

**图 7-11.** *将表格视图单元格样式更改为基本样式。*

4. 对于这个`Table View`，你将使用`Dynamic Prototypes`。实际上，你的`Table View`将显示三种不同类型的单元格。第一种是基本样式，仅显示一个标签，内容为数据库中每个分类的名称。这种单元格在`Table View`中的数量将与数据库中的分类数量相同。第二种单元格将显示“全部图书”标签，当用户点击该单元格时，会显示数据库中所有图书，无论分类如何。这个单元格将始终显示在分类列表的末尾。第三种单元格是“新增”单元格，让用户能够向数据库添加新的分类。这个单元格也会始终显示，并且总是`Table View`的最后一个单元格。修改你的`Table View Prototype`内容以实现此目的。首先，点击你已添加的单元格，并将其样式更改为`Basic`，如图 7-11 所示。

![images](img/9781430242727_Fig07-12.jpg)

**图 7-12.** *更改单元格的行高。*

5. 点击`Size Inspector`，如图 7-12 所示，然后将`Row Height`从你当前的高度改为`50`（我们手动调整到了`54`，但请将你的高度改为`50`）。

![images](img/9781430242727_Fig07-13.jpg)

**图 7-13.** *更改标识符和辅助功能属性。*

6. 返回`Attributes Inspector`，将`Identifier`改为一个在代码中有意义的名称——我们称之为`Cell`，因为这是你将复用以显示分类的标准单元格原型。现在点击`Accessory`选项，从下拉菜单中选择`Disclosure Indicator`，如图 7-13 所示。双击`Table View Cell`原型中的`Title`标签，将其重命名为“分类名称”。然后确保你刚刚配置的单元格被选中（高亮为蓝色），并按 ![images](img/U003.jpg) + `C` 复制它。

![images](img/9781430242727_Fig07-14.jpg)

**图 7-14.** *复制第一个单元格。*

7. 按 ![images](img/U003.jpg) + `V` 粘贴复制的“分类名称”单元格，如图 7-14 所示。双击你刚刚粘贴的新单元格，将其名称改为“全部图书”，因为该单元格将用户带到包含所有 Apress 图书的页面。而常规的“分类名称”单元格则会将用户带到属于 iOS、Android 或你选择的任何其他分类的图书。将名称改为“全部图书”后，将其`Identifier`（从第一个单元格粘贴过来后仍显示为`Cell`）改为`AllCell`。当标识符显示为`AllCell`后，继续下一步。

**注意：** 如你所见，`Storyboard`使得复制单元格并根据需要修改它们以创建原型变得异常简单。理解并充分利用这一概念至关重要。我们仍然看到有经验的程序员手工编写这些代码，仅仅是因为他们还没有“领悟”到——Storyboarding 如何让创建新原型变得如此简单。

![images](img/9781430242727_Fig07-15.jpg)

**图 7-15.** *创建“新增”单元格。*

8. 你需要再制作一个单元格。这个新单元格将允许用户添加任意名称的新分类。想一想：你将创建一个单元格，未来无需你或用户编写任何代码，它就可以被命名为“最佳 Apress 图书”、“Monkey”或其他任何名称——太棒了！按照你在图 7-14 中所做的操作：点击已有的两个单元格中的任意一个以选中它，按 ![images](img/U003.jpg) + `C` 复制，再按 ![images](img/U003.jpg) + `V` 粘贴，并将单元格的标签重命名为“新增”。为了使其外观不同，让我们找点乐子，更改几个属性。将`Accessory`类型设置为`Detail Disclosure`，如图 7-15 所示。你需要将`Identifier`重命名为唯一的名称，因为这是一个特殊单元格，因此将其改为`AddCell`。

![images](img/9781430242727_Fig07-16.jpg)

**图 7-16.** *更改文本颜色。*

9. 更改“新增”文本的颜色，以便用户可以轻松看出“新增”与其他选项不同。单击文本“新增”一次，将其颜色改为蓝色，如图 7-16 所示。

**注意：** 请确保在操作过程中随时保存你的工作！

![images](img/9781430242727_Fig07-17.jpg)

**图 7-17.** *从“图书”单元格到“分类”创建 Push 转场。*

10. 我们现在准备创建一个从`Master View Controller`的`Books`单元格到你刚刚设置的场景的`Push`转场。因为`Master`中有一个`Static Table View`，你可以直接从单元格本身发起转场。因此，点击`Master`中的`Books`单元格，按住 Control 键从它拖动到将要显示我们分类的`View Controller`（见图 7-17 中的 1），松开，然后从弹出菜单中选择`Push`（见图 7-17 中的 2）。

![images](img/9781430242727_Fig07-18.jpg)

**图 7-18.** *设置转场标识符并更改目标视图的标题。*

11. 选择新的转场，在`Attributes Inspector`中将其`Identifier`选项设置为`ShowCategories`（见图 7-18 中的 1）。同时，双击目标`View Controller`的`Navigation Bar`，将其标题设置为`Categories`（见图 7-18 中的 2）。

**注意：** *标识符*是一个字符串，你的应用程序用它来区分不同的转场。你为每个转场分配不同的标识符，以便`Source View Controller`中的`prepareForSegue:sender:`方法能够区分它们，并为每个转场做好适当的准备。请始终为你的转场分配标识符。

![images](img/9781430242727_Fig07-19.jpg)

**图 7-19.** *创建编辑栏按钮项。*

12. 如前所述，你将允许用户编辑`Table View`并删除单元格。这意味着你需要在`Navigation Bar`上添加一个按钮，用于将`Table View`切换到编辑模式。从你的`Object Library`中拖一个`Bar button item`到`Navigation Bar`的右侧。双击其默认标题`Item`，将文本改为`Edit`，如图 7-19 所示。

#### 布局主图书列表视图：图书场景

![images](img/9781430242727_Fig07-28.jpg)

**图 7-28.** *创建自定义单元格。*

1.  在此视图中，用户会看到一个图书列表，每一行都显示书名和图书封面缩略图。你在第 6 章中拖入了一些图书图片。我们需要在这里做的，是将现在的 Basic 类型的“分类名称”单元格更改为自定义单元格，以便添加自定义 UI 元素。点击该单元格，然后在属性检查器中将样式从 Basic 改为 Custom，如图 7-28 所示。

![images](img/9781430242727_Fig07-29.jpg)

**图 7-29.** *向自定义单元格中拖入一个 UIImageView 和三个标签。*

2.  选中顶部单元格，打开尺寸检查器，将行高改为 87，以适应图书封面图片的高度。你还需要一个 `UIImageView` 来承载图书图片，因此将一个 `UIImageView` 拖入单元格。将图像视图的宽度设为 50，高度设为 66（可以使用尺寸检查器或鼠标）。你还需要显示三个关于图书的信息标签：标题、出版年份和作者。将一个标签拖到图像视图的右侧，配置好之后，将其复制并粘贴两次。图 7-29 显示了正在将第一个标签拖入单元格。

![images](img/9781430242727_Fig07-30.jpg)

**图 7-30.** *复制并粘贴第一个标签。*

3.  将第一个标签复制并粘贴两次，然后将这些标签上下对齐排列好。拖动每个标签的右侧边缘，使其占据单元格的大部分区域——或者使用尺寸检查器将它们的宽度都设为 200 点。选中顶部标签，回到属性检查器，取消选中 Autoshrink 选项，将字体改为 System Bold 17.0，并将其颜色改为蓝色。然后，将每个标签的文本分别改为“标题”、“作者”和“出版年份”，如图 7-30 所示。我们还将*出版年份*的字体大小改为了 System 15.0，因为原来的字号看起来有点太大了。保存你的工作。

**注意：** 你在原型单元格中添加文本仅为了辅助设计过程。由于它是原型单元格，所有标签标题都将在运行时动态分配，但向我们的元素中添加占位符数据有助于指导你完成布局和自定义过程。
