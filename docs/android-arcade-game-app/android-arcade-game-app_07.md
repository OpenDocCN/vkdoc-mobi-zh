# 第 1 章：Android 游戏开发入门

欢迎阅读《Android 街机游戏开发》。在本书中，你将学习如何“从项目到发布”创建一款 Android 街机风格的游戏。虽然我会带你了解一些游戏开发问题的解决方案，但本书并不适合初级开发者。

你选择这本书，是因为你对 Android 平台充满热情，并且希望开发街机风格的 Android 游戏。读完本书后，你将掌握在 Android 最新版本 Ice Cream Sandwich 上创建一款有趣且引人入胜的游戏所需的知识。这样做的好处是，你在本书中构建的游戏将同时在 Android 手机和 Android 平板电脑上运行。

坐下来玩自己写的游戏，那种满足感是无与伦比的。这对于街机游戏尤其如此，因为它们完美契合了休闲移动游戏玩家的体验需求。

## 你需要了解的知识

在阅读本书之前，你应当具备良好的 Android 开发实践知识。这意味着你精通 Java 和 Android SDK，并且已经尝试过在 Android 上构建项目和应用程序。

你还应熟悉 Eclipse IDE（集成开发环境）。虽然 Android 游戏可以在许多不同的 IDE 中开发，但本书中的所有示例都使用 Eclipse Indigo。如果你没有当前版本的 Eclipse，请从`http://eclipse.org`下载。

**注意** 虽然可以使用模拟器调试代码，但如果你认真对待游戏开发，还是建议获取一台 Android 设备。我发现模拟器在渲染精度和运行速度上并不总是能与运行同等 SDK 版本的移动设备相比。而当你编写游戏时，精度至关重要。

最后，为了从本书中获得最大收益，你至少应具备初级的游戏开发知识。你可能从未独立编写过游戏，但应当了解开发一款游戏需要什么。在实践中，这意味着你应具备`OpenGL ES`的基础知识，并了解它如何在 Android 游戏开发中使用。

在本书创建《越狱》游戏的过程中，你将使用`glOrthof()`、`GLSurfaceView`、纹理、顶点以及其他`OpenGL ES`概念。如果这些术语对你来说都很陌生，我建议你先从一本更入门的书籍开始，例如 J. F. DiMarzio 所著的《实用 Android 4 游戏开发》（Apress 出版社，2011 年）。

## 游戏发展简史

尽管移动游戏以这种或那种形式（从纸牌游戏到骰子游戏）存在了数百年，但现代流行的移动视频游戏的起源可以追溯到任天堂 Game Boy。Game Boy 可能不是第一个，也可能不是最好的，但它确实让每个孩子、青少年，甚至是一些成年人都想随身玩着*俄罗斯方块*和*超级马里奥*；在此之前，这些游戏成就只能在那些仍带有“儿童玩具”标签的大型游戏机上实现。

**注意** 普遍认为，第一款移动游戏设备实际上是 1979 年发布的 Microvision。然而，是任天堂的 Game Boy 将移动游戏带给了大众，并将其推广为一个可行的游戏平台。

任天堂 Game Boy 配备了一个小型灰度屏幕和可互换的游戏卡带。在此之前，大多数便携式“视频游戏”都是单一功能的设备，只能玩一个或一小部分预装游戏。但人们终于可以随时随地玩他们的视频游戏了，尽管是缩水版。这使移动游戏进入了大众手中，但开发者们呢？



虽然当时已经存在一定规模的休闲或独立电脑游戏开发者，但 `Game Boy` 对他们来说并不是一个实用的平台，因为它仍被视为游戏主机。游戏开发需要使用专有的 `C` 语言库，并且必须借助昂贵且需许可的开发硬件。这使得独立游戏开发者难以企及，如果他们想表达自己的想法，就只能局限在个人电脑上。

大约在 `Game Boy` 风靡一时的同时，另外两场革命也开始萌芽；正是这些革命最终将 Android 推向了游戏平台。到 20 世纪 80 年代末，手机和互联网开始被广泛使用和接受，恰逢便携式游戏兴起。

早期的手机制造成本高昂，使用费用也高。它们是富人和名人的玩具，也是成功人士炫耀的工具。这些设备的屏幕仅由单行 LCD 文本组成；刚好够显示正在拨打的电话号码。设想一位商业大亨或好莱坞制片人坐在那里，以每分钟 15 到 25 美元的价格在单行显示屏上玩 *俄罗斯方块*，这根本不现实。手机显示屏和每分钟费率要发展到足以支持游戏，还需要再过几年。

手机和互联网变得流行后不久，两者就碰撞在了一起。起初，两者的结合产生了粗糙、缩水版的网页类内容，这些内容可以轻松在手机缓慢的处理器和有限的（主要是文本）屏幕上运行。慢慢地，基于 Java 的移动内容和应用程序开始在各种手机上涌现。虽然这增加了 Java 虚拟机的开销，但这无疑是一个好的开端。首批基于 Java 的游戏也紧随其后问世。

诺基亚最终在 2003 年尝试通过其 `N*Gage` 手机将手机与移动游戏设备完全融合。尽管 `N*Gage` 普遍被认为是一款商业失败的产品，但它为手机游戏作为一种真正的多平台活动打开了大门。开发者很快意识到，专门为手机打造的游戏实际上可以与为 `Game Boy`、`PSP` 和 `DS` 等专用游戏掌机开发的游戏竞争。

手机游戏最终在 `iPhone` 上获得了消费者的认可。不可否认，`iPhone` 不仅仅是打开了手机游戏的大门，它简直是摧毁了那扇门。这并不意味着 `iPhone` 没有缺点。开发 `iPhone` 游戏需要两样东西，并非每个休闲开发者都拥有：一台 Mac 和对 `Objective-C` 的深入理解。

Android 的故事就从这里开始。

## Android 的诞生

我在 2008 年初就开始在 Android 测试版平台上进行开发。当时，还没有任何手机宣布搭载这个新操作系统，我们作为开发者，真切地感觉自己正投身于一件令人兴奋的事业的开端。Android 捕捉到了早期开源开发的所有活力和热情。在这个平台上开发，非常让人怀念凌晨两点，拿着一罐 Jolt 可乐，坐在空荡荡的学生休息室里，等待 VAX 主机时间来运行最新代码的情景。那是一个令人振奋的平台。我很庆幸自己亲眼见证了它的实现与腾飞。

随着 Android 开始发展，以及 Google 发布更多更新来巩固最终的架构，有一点变得显而易见：Android 基于 Java，并包含许多知名的 Java 包，这对于休闲游戏开发者来说将是一个容易跨越的过渡。现有 Java 开发者所掌握的大部分知识都可以在这个新平台上复用。庞大的 Java 游戏开发者群体可以利用这些知识，相对平滑地迁移到 Android 上。

那么，如何开始在 Android 上开发游戏呢？需要哪些工具？本章的下一部分将旨在回答这些问题。

## Android 游戏编程

在 Android 上开发游戏有其优点和缺点。在开始之前，你应该意识到这些。首先，Android 游戏是用 Java 开发的，但它并非完整的 Java。你之前用于 `OpenGL` 和其他图形特效的许多包都包含在 Android SDK 中。然而，很多并不代表 *全部*，一些对游戏开发者（尤其是 3D 游戏开发者）非常有帮助的包并未包含在内。因此，你之前构建游戏所依赖的包，并非全部都能在 Android 中使用。

随着新版 Android SDK 的每次发布，越来越多的包变得可用。你需要清楚了解自己可以使用哪些包。我们将在后续章节中陆续介绍这些内容。

另一个优点（以及相应的缺点）与 Android 的易用性与其性能不足有关。Android 在易用性和简易编程方面获得的好处，可能在速度和性能上有所欠缺。大多数电子游戏，比如为个人电脑或游戏主机编写的那些，都是用 C 甚至汇编这样的底层语言开发的。这能让开发者最大程度地控制处理器如何执行代码以及代码的运行环境。当然，处理器只理解机器码，你越接近它们的原生语言，就越不需要经过层层解释器就能让游戏运行起来。Android 虽然提供有限的底层编码能力，但它会通过自己的执行系统来解释和编排你的 Java 代码。这导致开发者在游戏运行环境的控制力上有所减弱。

本书不会带你走底层游戏开发的路子。为什么不呢？因为 Java，特别是其在通用 Android 开发中的呈现形式，广为人知、易于使用，并且能够制作出一些非常吸引人且富有成就感的游戏。

本质上，如果你已经是一名经验丰富的 Java 开发者，你会发现将技能应用到 Android 上时并不会因为转换而丢失。如果你还不是资深的 Java 开发者，也不必担心。Java 是一个非常适合入门的语言。基于这个原因，我选择坚持使用 Android 原生的 Java 开发环境来编写我们的游戏。

我们已经讨论了在 Android 上开发游戏的一些优点和缺点。独立和休闲游戏开发者选择在 Android 平台上创建和发布游戏的最佳理由之一，是发布游戏时所拥有的自由度。虽然一些应用商店对于可以销售什么以及以何种价格销售有非常严格的规定，但 Android Market 并非如此。任何人都可以自由地列出和销售几乎任何他们想要的东西。这给了开发者更多的创作自由。

既然我们已经快速回顾了移动游戏的历史，并讨论了为什么你可能值得将宝贵的时间和精力投入到 Android 网络上的游戏开发中，那么是时候看看成为一名成功的 Android 游戏开发者所需的工具了。
