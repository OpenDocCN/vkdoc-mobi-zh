# 面向 iOS 开发者的 Android 迁移指南

2009 年初，通过重复相同的移植流程，我将一款 Windows 移动应用翻译成了我的第一个 iOS 应用。这开启了我的 iOS 编程之旅，后来很自然地尝试将其移植到 Android 平台。大多数移动应用都与平台无关。当你为 iOS 或 Android 完整地完成解决方案后，所有业务和技术问题都已得到验证，其他交付成果和项目工件也已经可以重用。如果不将其移植到另一个平台，那将是一种浪费。今年早些时候，我写了我的第一本书《面向 iOS 开发者的 Android 迁移指南》，来分享我的信念和经验。在 Swift 于 2014 年全球开发者大会（WWDC）上推出后，我决定写这本书，因为 Java 和 Swift 之间的相似性使得移植方法更加简单，同时相同的投资回报率分析仍然成立。

本书的主要目标是帮助经验丰富的 Android 开发者轻松进入原生 iOS–Swift 移动开发领域。这比你想象的要容易，而本书将通过 Android 的类比和映射指南，使其变得更加简单。你可以立即将常见的移动用例转换为 iOS。

## 本书适合谁？

本书专为希望利用其移动知识，并在 iOS 移动平台上发布移动应用的 Android 开发者而编写。本书将使用你熟悉的 Android 术语，向你展示常见的 iOS 编程主题和框架，而无需冗长的解释，因为作为 Android 开发者，你已经了解这些移动主题。

## 本书的组织方式

在第一部分中，你将快速让 iOS 的 `Xcode` 集成开发环境（`IDE`）启动并运行。你将得到指导，创建教程项目，这些项目将成为你的移植示例项目。我相信这是你在学习编程主题的同时获得实践经验的最佳方式。

本书的第二部分将向你展示如何规划和构建你的 iOS 应用：通过创建故事板并将应用分解为模型-视图-控制器（MVC）类。你将能够重用大部分现有的软件工件，并根据其 Android 对应部分设计其余部分。接下来会介绍常见的移动主题，包括用户界面、数据管理以及与远程服务的网络连接。完成第二部分后，你将能够创建具有丰富 UI 组件的简单但有意义的 iOS 应用，并能在本地和远程处理常见的 CRUD（创建、读取、更新、删除）操作。

最后，第三部分将带你通过一个案例研究，将一个完整的 iOS 应用移植到 Android。它会回顾如何使用 第二部分 主题中的映射指南。你也可以使用本书的目录来帮助查找所需的移植指南。

完成这段旅程后，你将能够使用 `Xcode` 和 Swift 有效地将你现有的 Android 应用移植到 iOS。
