# 2. 设计用户界面

每个应用都需要一个用户界面。与 Windows 或 macOS 等桌面操作系统可以在屏幕上同时显示多个窗口的用户界面不同，iOS 应用通常一次只显示一个填充整个屏幕的窗口（称为视图）。同样，与用户可以使用键盘或鼠标操作的 Windows 或 macOS 用户界面不同，用户必须仅通过触摸屏操作 iOS 应用，尽管也可以通过可选的键盘进行控制。在 iOS 应用中，图 2-1 显示了一个用户界面由以下部分组成：

![../images/329781_5_En_2_Chapter/329781_5_En_2_Fig1_HTML.jpg](img/329781_5_En_2_Fig1_HTML.jpg)

*图 2-1.* iOS 应用用户界面的三个部分

*   **视图**
*   **对象**，如按钮、标签、文本框和开关
*   **转场**

我们用于在第一章 1 中创建应用的“单视图应用”模板由一个视图组成，并且没有对象。由于只有一个视图，因此也没有转场，因为转场用于连接两个视图，以显示应用运行时每个视图出现的顺序。

就其本身而言，一个没有对象的视图不过是一个空白屏幕。要使视图有用，它需要包含一个或多个对象。使用 Xcode，通常有两种方法可以完成完全相同的任务：

*   **编程方式**
*   **可视化方式**

编程方式意味着编写代码来完成所有操作。在创建出现在视图上的对象时，这意味着编写 Swift 代码来定义要显示的对象、其大小、在视图上的位置以及任何其他自定义设置（如颜色）。

可视化方式意味着使用鼠标将对象拖放到视图上。将对象放置在视图上后，你可以修改其大小、位置或任何其他属性，而无需编写任何代码。

你可以使用其中一种或两种方法，但为了保持一致性，通常最好使用一种方法。此外，虽然你可以通过编写代码（编程方式）来创建整个用户界面，但通常不建议这样做，因为编写的代码越少，修改和调试程序就越容易。编程的目标是用尽可能少的代码完成尽可能多的功能。

首先，让我们看看如何以编程方式创建对象，以便你了解其工作原理。让我们在视图上创建一个标签和一个按钮。

1.  打开你的 `MyFirstApp` 项目（或创建一个新的单视图应用 iOS 项目）。
2.  在导航器窗格中单击 `ViewController.swift` 文件。中间的 Xcode 窗格会显示 `ViewController.swift` 文件的内容。
3.  修改 `ViewController.swift` 文件中的代码，使其如下所示：

```
    import UIKit
    class ViewController: UIViewController {
    override func viewDidLoad() {
    super.viewDidLoad()
    let label = UILabel()
    label.frame = CGRect(x: 125, y: 125, width: 200, height: 80)
    label.text = "This is a label"
    view.addSubview(label)
    let button = UIButton()
    button.frame = CGRect(x: 125, y: 300, width: 80, height: 80)
    button.setTitle("Button", for: .normal)
    button.setTitleColor(UIColor.blue, for: .normal)
    view.addSubview(button)
    }
    }
    ```

4.  选择 `Project` ➤ `Run` 或按下 `Command + R`。Xcode 会加载模拟器程序，让你可以模拟 iOS 设备。当你的应用在模拟器中加载时，它将显示一个标签和一个按钮，如图 2-2 所示。

![../images/329781_5_En_2_Chapter/329781_5_En_2_Fig2_HTML.jpg](img/329781_5_En_2_Fig2_HTML.jpg)

*图 2-2.* 以编程方式创建的用户界面，显示一个标签和一个按钮

5.  选择 `Simulator` ➤ `Quit Simulator` 或按下 `Command + Q`。Xcode 窗口会再次出现。

让我们回顾一下这段代码，以便你理解它是如何工作的。首先，我们必须定义要创建的对象类型，例如 `UILabel` 或 `UIButton`：

```
let label = UILabel()
let button = UIButton()
```



接下来，我们需要定义对象的位置、宽度和高度。请记住，原点 `(0, 0)` 被定义为屏幕的左上角：

```
label.frame = CGRect(x: 125, y: 125, width: 200, height: 21)
button.frame = CGRect(x: 125, y: 300, width: 80, height: 30)
```

第一行代码将 `label` 定义为距离屏幕左侧 125 点、距离屏幕顶部 125 点的位置。同时，该代码还定义了 `label` 的宽度为 200，高度为 21。

第二行代码将 `button` 定义为距离屏幕左侧 125 点、距离屏幕顶部 300 点的位置。同时，该代码还定义了 `button` 的宽度为 80，高度为 30。

定义好对象的大小和位置后，下一步是自定义对象的外观。`label` 包含文本 “This is a label”，基于以下代码：

```
label.text = "This is a label"
```

`button` 通过两种不同方式进行自定义。首先，按钮标题设置为 “Button”，并将文字颜色设置为蓝色：

```
button.setTitle("Button", for: .normal)
button.setTitleColor(UIColor.blue, for: .normal)
```

在定义了要显示的对象类型、其大小和位置，并进行自定义之后，最后一步是将其放置到视图上：

```
view.addSubview(label)
view.addSubview(button)
```

如果以上步骤看起来只是为了在用户界面上创建一个对象而大费周章，那你是对的。这就是为什么大多数开发者选择以可视化的方式设计用户界面。让我们看看这是如何操作的。

    ```
    }
    ```

许多 Xcode 项目会结合两种方法来创建用户界面。虽然通常以可视化方式创建用户界面更容易，但你也需要熟悉编写代码以编程方式创建用户界面，因为你可能会在他人创建的项目中看到这种做法。



## 了解视图与类文件

任何应用用户界面最重要的组成部分都是视图，它负责在 iOS 设备屏幕上显示信息。最简单的应用只包含一个视图，但大多数应用会包含多个视图。你可以轻松地在项目中添加或删除视图。既然视图包含按钮和文本字段等用户界面对象，你可能会好奇视图是如何存储用户在界面上输入的数据的。答案是每个视图都需要一个类文件，其中包含能够存储数据和响应用户交互（例如点击按钮）的 Swift 代码。

在继续之前，请注意 Xcode 使用了一些容易混淆的术语来描述用户界面的各个部分。用户界面存储在故事板中，屏幕上显示的每个独立窗口被称为场景。大多数应用包含多个场景，因此文档大纲允许你有选择地隐藏或显示每个场景的详细信息，如图 2-9 所示。通过点击文档大纲中的每个场景，你可以快速查看构成用户界面的不同场景。

![../images/329781_5_En_2_Chapter/329781_5_En_2_Fig9_HTML.jpg](img/329781_5_En_2_Fig9_HTML.jpg)

图 2-9。 文档大纲能让您轻松查找和查看场景

每个场景内部都有一个控制器。控制器定义了该场景在屏幕上的显示方式。你可以通过点击文档大纲中的控制器名称，或者点击故事板中的控制器图标来选择控制器，如图 2-10 所示。

![../images/329781_5_En_2_Chapter/329781_5_En_2_Fig10_HTML.jpg](img/329781_5_En_2_Fig10_HTML.jpg)

图 2-10。 您可以在文档大纲或故事板中选择控制器

每个控制器内部通常包含一个视图，它定义了应用运行时屏幕上的显示内容。按钮或标签等对象出现在视图上，因此如果你删除了一个视图，你也将删除该视图上存储的所有对象。

让我们看看我们已创建的“单视图应用”项目中的两个 `.swift` 文件，并了解文档大纲的层级结构是如何运作的：

1.  在导航器面板中点击 `AppDelegate.swift` 文件。Xcode 的中间面板会显示 `AppDelegate.swift` 文件的内容。你会注意到，已经为你创建了几个不包含代码的函数。这些函数让你可以处理应用运行时的不同情况：
    *   `didFinishLaunchingWithOptions` – 在应用启动后运行
    *   `applicationWillResignActive` – 在应用不再是屏幕上处于活跃状态的应用之前运行
    *   `applicationDidEnterBackground` – 在应用被推入后台时立即运行，以便用户使用其他应用
    *   `applicationWillEnterForeground` – 在应用即将再次成为屏幕上的活跃应用之前运行
    *   `applicationDidBecomeActive` – 在应用成为屏幕上活跃的应用之后运行
    *   `applicationWillTerminate` – 在应用即将关闭之前运行

在许多情况下，你不需要为所有这些函数编写代码。例如，你可能希望在应用关闭前保存数据（`applicationWillTerminate`），然后在应用首次启动时重新加载这些数据（`didFinishLaunchingWithOptions`）。`AppDelegate.swift` 文件就是你编写 Swift 代码来控制应用行为的地方。

2.  在导航器面板中点击 `Main.storyboard` 文件。Xcode 会显示你的用户界面。确保文档大纲可见。（你可以通过选择“编辑器”➤“显示/隐藏文档大纲”或点击文档大纲图标来切换显示或隐藏文档大纲。）

3.  在文档大纲或故事板中点击控制器图标（参见图 2-10）。

4.  选择“视图”➤“检查器”➤“显示身份检查器”，或者点击检查器面板顶部的“显示身份检查器”图标。注意，“类”文本字段显示为 `ViewController`，如图 2-11 所示。此名称指向导航器面板中的 `ViewController.swift` 文件，并将控制器与名为 `ViewController.swift` 的文件关联起来。你可以随时更改文件名，但如果更改了文件名，则必须确保所有控制器都使用正确的文件名。

## 适配不同屏幕尺寸

在设计用户界面时，你需要选择 iOS 设备的屏幕和方向，例如竖屏模式的 iPhone 8 或横屏模式的 iPad。由于你的 iOS 应用可能会在不同 iOS 设备上运行，因此需要在所有不同的屏幕尺寸和方向下对应用进行测试。

`Xcode` 提供了三种不同的方式，让你能够在不同的 iOS 设备屏幕尺寸和方向下查看应用的用户界面：

*   点击 `Xcode` 窗口底部的“视图为：”文本，然后选择不同的 iOS 设备屏幕尺寸和方向。
*   为模拟器选择不同的 iOS 设备和方向进行模拟。
*   在 `Xcode` 中预览你的视图。

第一种方式（点击“视图为：”文本）可让你在不同 iOS 屏幕尺寸和方向之间快速切换。第二种方式（使用模拟器）则能让你测试应用，就像它在真实 iOS 设备上运行一样。

你可能会在设计用户界面时使用第一种方式，然后用第二种方式来验证一切是否正常。接下来，让我们看看如何在我们的单视图应用项目中测试不同的屏幕尺寸和方向：

1.  确保你已在 `Xcode` 中加载了 `MyFirstApp` 项目。
2.  在导航区域中点击 `Main.storyboard` 文件。`Xcode` 会显示项目的故事板。
3.  点击 `Xcode` 窗口底部的“视图为：”文本，显示不同的 iOS 设备和方向选项，如图 2-17 所示。

## 摘要

每个应用都需要一个用户界面。这个用户界面会被存储在故事板中，而故事板由一个或多个视图组成，其中每个视图代表在单个屏幕上显示的信息。你可以在视图上添加按钮、文本字段、标签和滑块等对象。如果你有两个或更多视图，还需要使用转场（segues）来连接这些视图，以显示它们呈现给用户的顺序。

你可以通过编程（编写代码）或可视化方式向视图添加对象。程序员通常结合使用这两种方法来设计他们的用户界面。你可以通过代码或检查器面板来自定义对象（按钮、标签、文本字段等）。故事板中的每个视图都需要链接到一个`.swift`文件。

由于 iOS 应用可以在不同类型的设备上运行，你需要确保你的应用无论屏幕尺寸或方向如何都能看起来美观。为了帮助你查看应用在不同屏幕尺寸和方向下的用户界面，你可以更改 Xcode 用于显示故事板的 iOS 设备屏幕尺寸。

在不同屏幕尺寸下测试应用用户界面的另一种方法是运行模拟器程序，并将其设置为模仿不同的 iOS 设备，例如 iPad Pro 或 iPhone 8。如果你想要比较应用在不同 iOS 设备屏幕上的显示效果，可以使用预览功能。

用户界面代表了用户所看到的内容，所以无论用户运行的是哪种 iOS 设备，你的用户界面始终保持美观这一点非常重要。
