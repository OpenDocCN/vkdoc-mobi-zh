# 第五章：事件处理与用户输入

### 用户输入

本章致力于探讨几个重要主题：处理用户输入、实现自定义事件处理器，以及构建舒适的游戏控制方案。主要目标是为你提供扎实的输入机制知识储备，并启发你打造玩家与游戏世界之间无缝且自然的交互界面。

移动设备与桌面浏览器的输入模式存在显著差异。触摸屏不仅在游戏中，在常规应用里也能提供更直观的界面控制方式。用手指触碰按钮——即便这个按钮只是屏幕上由发光像素构成的矩形区域——显得更加自然。在游戏地图上指向新建筑物应该出现的位置，或是在战场上直接划出攻击方向来派遣军队，也都更加直观。

触摸界面在直观易用的操控方面取得了巨大进步，因为它们实现了用户与 UI 之间的直接交互。即使两岁的孩子也能操作触摸屏，而鼠标和键盘则需要更多技巧。

浏览器的输入 API 围绕事件、事件发射器和事件监听器构建。每当对象发生重要事件时，将小消息（即"事件"）传递给监听器的这种思路，并非输入处理所独有。例如，负责与服务器通信的`JavaScript`类可能会利用事件来通知其他类：某聊天室刚刚发布了一条新消息。监听事件的聊天 UI 随后会更新自身，向用户展示这条新消息。在本章中，我们将深入探讨这种架构的细节，并学习如何构建适用于大多数游戏开发场景的自定义实现。

掌握基础知识后，我们就能构建更高级的输入模型，例如拖放操作、摇杆模拟等。我们将学习如何实现"拾取"功能，即通过鼠标或触摸操作选择复杂形状。

总而言之，我们的目标是掌握以下内容：

- 如何构建自定义事件发射器
- 鼠标界面与触摸界面之间的差异
- 如何构建抽象层以隐藏底层输入系统（鼠标或触摸）的细节，并使用最低公共标准
- 如何实现拖放操作
- 如何为像素级精确的对象拾取寻找合适方案
- 如何在网页上模拟摇杆

### 浏览器事件

触摸设备为用户带来的交互体验与使用鼠标和键盘操控的传统桌面电脑截然不同。因此，浏览器事件也存在差异，这并不令人意外。在本节中，我们将了解这两种控制类型的优缺点。之后，我们将尝试构建一个通用的输入处理接口，以隐藏底层系统的细节，使其对游戏的其他部分透明。

#### 桌面浏览器与安卓浏览器的输入对比

桌面用户受限于鼠标和键盘（它们充当着人与计算机之间的"中介"），而移动端玩家能做更多事：旋转和摇晃设备、触摸和滑动屏幕、在真实世界的街道上移动以完成游戏目标、拍照以及录制视频。

当然，移动端控制也并非完美：在面对键盘所擅长的操作时，它们很快暴露不足。一个典型的例子是电子邮件输入——使用实体键盘输入文本要容易得多。让我们分析移动端控制的优缺点，以便知晓如何扬长避短。

#### 移动端控制的优势：

- 移动端控制感觉更自然。用户用手指而非鼠标光标进行指向，感觉更像是在与游戏世界的对象进行交互。
- 移动设备支持多点触控，相当于拥有十个鼠标指针！
