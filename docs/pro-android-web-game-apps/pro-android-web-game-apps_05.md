# 第 1 章：入门

## 技巧

本节致力于介绍创建应用程序的下一个非常重要的方面：开发技巧——从代码规范到良好实践和重要的架构方法。在深入本节细节之前，我想给出一个简单但非常重要的建议：编写让你引以为豪的代码。这是衡量代码质量的最佳标准。

你一定听说过很多“良好编码”的建议：为变量使用有意义的名称、编写注释、使代码格式美观、避免庞大且难以管理的函数。让我在本节中添加一些特定于 JavaScript 的技巧。

我们将探讨开发的两个方面：代码和 OOP。第一部分致力于编写干净的 JavaScript 代码；第二部分是使用 OOP。JavaScript 有着一种有点不寻常的“基于原型”的继承模型，这引起了很多误解和混淆，因此值得专门用一节来澄清它。

#### 代码

如何编写良好的 JavaScript 代码？让我们暂时忘记程序结构——类、继承、耦合和内聚——而是关注普通的 JavaScript 代码。如何让它看起来不错？如何使其可预测且易于维护？本节致力于介绍有关此主题的一些基本建议。

#### 风格

风格是你的代码的外观。关于如何编写漂亮的代码，有很多通用建议。让我们看看那些特定于 JavaScript 的建议。
