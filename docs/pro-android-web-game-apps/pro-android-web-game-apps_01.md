# 开始入门

本章的目标是为开发准备一个舒适的工作空间。移动开发的环境和工具总是比常规桌面项目要复杂一些。当涉及在单个应用中同时使用 Android 和 JavaScript 时，在正确的位置使用正确的工具可以带来巨大的差异。但工作空间不仅仅关乎工具。在开发过程中设定编码规范并遵循最佳实践也至关重要。本章还将讨论编码约定和基本的架构决策。

作为一名经验丰富的开发者，你在工具和编码方法方面已经有了一定的偏好。例如，每个 Web 程序员都有自己偏好的集成开发环境（IDE）和用于基本测试的浏览器。对于如何编写良好且易于维护的 JavaScript 代码，你很可能也有自己的看法。如果你对自己的偏好感到满意，那就继续使用它们。但至少，我鼓励你尝试本章中描述的工具和技术。你可能会发现其中一些更加便捷。

本章分为两部分——工具和技术——每部分都描述了开发过程中的一个重要方面。在本章中，我们将执行以下操作：

- **工具：**
  - 安装 Java 开发工具包
  - 比较对 JavaScript 支持良好的 IDE
  - 安装 Web 服务器（本书第一部分使用 `nginx`）
  - 安装 Android SDK 并配置模拟器
  - 创建一个基本网页，并确保它能同时在桌面浏览器、真实设备和模拟器中加载

- **技术：**
  - 回顾 JavaScript 最佳编码实践
  - 实现一个简单的继承机制，该机制将在全书面向对象编程（OOP）代码中使用

### 工具

在本节中，我们将审查并设置构建 JavaScript 应用程序所需的工具。JavaScript 是一个成熟的平台，用于创建复杂的先进软件。自然，有许多软件组件可以帮助创建、测试和调试丰富的 JavaScript 页面。

JavaScript 是一种动态语言，与 Java 或 C++ 不同。静态语言和动态语言的主要区别在于，静态语言必须在运行时之前定义数据结构。例如，在静态语言中，如果程序员想要创建一个名为 `Van` 的类，就必须显式地描述其所有内容。



`properties` 和 `methods`：`color`， `maxSpeed`， `drive()` 等等。

每个 `Van` 类型的对象都具有严格定义的相同接口。这毫无意外。

像 JavaScript 这样的动态语言允许在运行时添加、删除或更改任何类或对象的结构。因此，像下面这样的技巧是可能且有效的：

```
var van = getTheRandomVan();

van.drive = racingCar.makeUTurn; // 有效的新属性赋值
```

就像这样，你可以从不同类的对象中获取方法，并用它来替换现有的方法。在这种情况下，只有 `van` 的一个实例受到影响，其余的对象保持不变！

正如你所见，很多情况都可能发生在 JavaScript 数据结构上：变量可以改变其类型，现有的对象可以通过本地代码或通过 `Ajax` 调用从远程服务器下载的代码来扩展新的方法。

动态行为赋予了语言强大的能力，但使得像 IDE 这样的工具更难预测对象的结构及其类型。JavaScript 的动态特性阻止了代码分析器像它们帮助“经典”静态类型语言那样帮助你。

环境变量

我们在本书中使用的大多数工具都遵循相同的安装模式：

- 安装工具或解压存档
- 设置环境变量 `TOOL_HOME`，例如 `JAVA_HOME` 或 `ANT_HOME`
- 将包含可执行文件的文件夹（通常是 `TOOL_HOME/bin`）添加到 `PATH` 中，这样你就可以直接从控制台或终端调用该工具，而无需键入完整路径

设置和更改环境变量取决于你的操作系统。

Windows 7

在 Windows 7 中，右键单击“我的电脑”并从下拉菜单中选择“属性”。在打开的窗口中，单击“高级系统设置” > “环境变量”。在“系统变量”下，单击“新建”。输入变量名（例如 `JAVA_HOME`）、变量值，然后单击“确定”。请注意，你必须重新打开任何已打开的控制台窗口，才能使它们“看到”这些更改。
