# 测量像素的图形编辑器

精灵表的另一个有趣用途是渲染游戏中的光栅字体。游戏可能会使用以位图形式呈现的非常精美的字体，其中每个精灵代表一个字母或字符。

### 光栅字体

构建一个能够使用位图字体渲染字符串的类，其复杂程度可能很简单，也可能相当复杂，具体取决于你所需的功能集。如果你只需要字符串的单行位图表示，那么花五分钟就能构建一个实现该功能的类。另一方面，如果考虑换行、对齐和间距等高级解决方案，则会变得非常复杂。排版是一个广阔而有趣的话题，超出了本书的讨论范围。

### 动画基础

我们正逐渐接近游戏开发中最激动人心的部分之一：动画。当画面开始移动时，你的游戏就变得“活”了起来。看到你第一个动画精灵在屏幕上移动，这是一种令人难以置信的美妙感觉。

#### 最简单的动画

如今，人人都知道动画是如何工作的。这背后并没有什么神秘之处。当你足够快地更换帧，并更新屏幕上的画面（改变其形状、位置、切换到动画序列中的下一帧，或同时执行上述所有操作）时，用户就会体验到运动的错觉。画面变得栩栩如生——宇宙飞船飞行并射击，吃豆人逃离追击，坦克转动炮塔。

要创建动画，我们必须实现三个步骤：

1.  随时间更新画面的状态（移动画面、更换帧等）。
2.  在屏幕上绘制画面。
3.  等待一段时间，重复前两个步骤，绘制新帧（不要忘记清除前一帧的内容）。

帧变化得越快，动画对用户来说就越流畅。帧变化的速度称为帧率，以每秒帧数（即 FPS）来衡量。

按照我们刚才概述的方案，我们应该创建一个渲染帧的函数，并每隔 30 到 50 毫秒调用一次。清单 4-17 展示了实现此功能的代码。

**清单 4-17.** *最简单的画布动画*

```javascript
var currentFrame = -1;

function animate(timestamp) {
  ctx.fillStyle = "white";
  ctx.fillRect(0, 0, canvas.width, canvas.height);

  currentFrame = ++currentFrame % 5;
  spriteSheet.drawFrame(ctx, currentFrame, 10, 40);

  requestAnimationFrame(arguments.callee);
}

animate();
```

让我们仔细看看这段代码，因为它是本章将要构建的整个动画框架的基础。函数的前两行在第二章中已经熟知：我们用纯白色清空整个画布。在绘制新帧之前，我们必须清除旧帧，因为画布不会自动清除自身，它会将新内容绘制在旧内容之上。这种方法的完整示例可在名为`05.sprite_sheet_animation.html`的文件以及本章的源代码中找到。

名为`current`的变量保存了动画序列中当前帧的索引。该序列总共有五帧。下一行代码`currentFrame = ++currentFrame % 5`将计数器加一，并执行取模运算（返回除以五的余数），以确保该数字永远不会大于四。这一行可以用以下更详细的方式重写：

```javascript
current++;
if (current > 4)
  current = 0;
```

它的作用与清单 4-17 中的那一行完全相同：当计数器达到 4 后，它会回到 0，并由此继续。下一行绘制帧，最后一行安排同一函数在 50 毫秒后执行。`animate()`函数的最后一行是一个重要且新颖的概念。JavaScript 定时器是一种简单但非常强大的机制，用于安排代码片段稍后执行。每隔几毫秒重绘帧...
