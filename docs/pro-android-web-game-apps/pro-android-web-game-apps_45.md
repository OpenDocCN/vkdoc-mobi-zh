# 章节：构建等距引擎

在本章中，我们探讨了如何渲染虚拟世界。我们了解到，实现多种渲染优化方法对游戏性能至关重要，尤其是在移动设备上。我们从帧率仅约 20 帧/秒的简易瓦片引擎实现起步，逐步优化，最终在章节末尾达到了稳定的 45 帧/秒。我们学习了如何向游戏世界添加物体，以及如何正确渲染它们。

我们研究了两种不同的投影方式：俯视投影和等距投影。第一种投影适合平台射击游戏，而俯视投影最适合复古风格游戏。等距投影对用户来说看起来更自然，但实现起来需要更多工作——无论是数学层面还是处理性能层面。本书下一章将完全专注于构建一个等距游戏引擎。

## 构建等距引擎

等距视图曾是渲染虚拟世界最流行的机制之一。它非常适合角色扮演游戏、策略游戏和战术游戏，并且看起来比简单的正交俯视图好得多。许多游戏开发中的杰作都使用了等距引擎，例如《辐射》、《幽浮》、《模拟城市》以及许多其他作品。80 年代的孩子仍然记得，与简单的“平面”引擎相比，那些游戏看起来是多么先进和前沿。

如今，桌面端几乎在所有领域都使用 3D 技术——从 AAA 级游戏到小型独立项目。然而，浏览器在广泛采用 3D 方面仍然相当滞后（尤其是移动浏览器）。这就是为什么等距引擎对休闲玩家仍然非常有吸引力。这类引擎在浏览器中可以看起来非常吸引人（尤其是如果你雇佣了一位优秀的艺术家）。

社交网络也促成了等距游戏的流行。《时空花园》、《模拟人生社交版》和《城市小镇》——这些 2011 年最受欢迎的 Facebook 游戏都是等距的！的确，众多农场、塔防和小型策略游戏在采用等距视图时效果最佳。等距图形是面向大众市场最具吸引力的引擎机制之一，非常适合休闲策略项目。这就是为什么我们决定用整章篇幅来介绍等距游戏引擎，并向你展示如何从零开始创建一个。

本章致力于为等距游戏构建一个框架。我们不会涉及游戏玩法，而是关注能够渲染一个大型等距世界并允许玩家与之交互的引擎。这为你自己的游戏项目打下了非常坚实的基础。

在本章中，我们将学习如何：

- 使用一组等距精灵渲染等距地形
- 优化渲染——为等距地形实现前一章描述的图像缓存策略
- 渲染建筑物和角色等游戏对象，在实践中应用第 6 章中描述的技术
- 实现脏矩形——一种最小化重绘区域并节省 CPU 周期的策略
- 处理交互——点击元素、拖动视口
- 创建简单的 UI 控件
