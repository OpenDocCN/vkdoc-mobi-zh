# 浏览器中的图形：Canvas 元素

玩家可以原谅图形上的缺陷，但绝不能容忍设计糟糕的控制。想象一下，在玩扑克时，用户不小心点击了`全押`而不是`弃牌`，那会有多懊恼。

最后一个但同样重要的组件是游戏逻辑：游戏世界的规则和法则。它负责广泛的任务：人工智能、物理引擎、随机游戏内容的生成，以及许多其他功能。处理逻辑的代码可能分布在客户端和服务器之间，或者完全在客户端运行。对于像我们在下一章中将要制作的简单游戏，根本不需要服务器；游戏所需的一切都在一个网页上。我们将在第 12 章中为`四个球`创建多人版本时，添加一些服务器端逻辑。

你可能注意到我没有提到网络。尽管网络通信在游戏开发中扮演着极其重要的角色，但我认为它并不是游戏本身的一部分。它是一种功能、一种工具，帮助将数据字节从你闪亮的 Android 设备传输到数据中心远处灰色盒子，再返回。魔法发生在别处。尽管如此，它也是一个极其重要且通常复杂的组件，我们将在第 10 到 12 章中深入学习。

我们从游戏最基本、最核心的特性——图形——开始探索网页游戏开发。以下部分介绍了简单但功能强大的 Canvas API：这是自 Ajax 以来，为网页游戏带来最大变革的 HTML5 DOM（文档对象模型）元素。

## 在浏览器中绘制

在浏览器中绘制并非游戏开发全新或独有的任务。一旦浏览器开始展现其全部能力，显示任意动态图形的能力对于广泛的富网络应用来说变得至关重要。在浏览器中显示图形有三种方式：

- 使用带有 CSS 图像背景的`div`元素进行布局
- 使用可缩放矢量图形（SVG）
- 使用`canvas`元素

第一种方式，有时被称为“浮动 div”，被认为已经过时。它曾用于在不支持`canvas`的浏览器中制作游戏。其思路是为场景中的每个对象创建一个`div`元素。然后，通过 CSS 的`background-image`属性为`div`设置样式，从而渲染图像。这种方法有一个明显的缺点：它不允许绘制任意图形，只能使用静态图像。如果开发者想在两点之间画一条线，除非服务器上某处有这条线的图像，否则无法实现。这种方法只是在`canvas`出现之前，网页游戏开发者不得不采用的一种变通方案。

第二种方式是使用 SVG，这是一种基于 XML 的矢量图形标准。SVG 在 Android 游戏开发中用途有限；标准的 Android 2.x 浏览器不支持该标准。即使浏览器支持，由于其 XML 特性，SVG 使用起来也有些困难。任何操作，甚至绘制一条简单的线条或填充一个矩形，都涉及对 XML 节点的操作，这不如使用 Canvas API 那样直观。

游戏开发的最佳选择是使用`canvas`元素，它不仅得到了现代设备的广泛支持，还拥有直观的绘图 API。本章的剩余部分将重点介绍它。

## 基础 HTML 设置

在转向`canvas`之前，我们首先需要一个 HTML 页面。本书将使用清单 2-1 所示的简单设置。

**清单 2-1.** *HTML 骨架*

```html
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8" />
<style>
</style>
<script>
function init() {
}
</script>
</head>
<body onload="init()">
</body>
</html>
```
