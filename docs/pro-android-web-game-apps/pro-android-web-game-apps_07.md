## 不要认为资源始终存在

资源（例如纹理图片、音频文件或 XML 文件）是应用程序的重要组成部分。通常，你会推迟加载这些类型的依赖项。常见的错误是认为资源总会出现在你期望的位置——比如在你最喜欢的 `res` 文件夹中。然而，资源可能无法获取的原因至少有两个：

1.  **网络连接问题。** 你可能直接与服务器断开了连接。这会导致出现一个难看的错误信息，而不是一张漂亮的图片。
2.  **简单的编码错误**，例如拼写错误或命名规范不一致。你可能会把 `castle.png` 误打成 `casle.png`，而实际上正确的文件名是 `castle_big.png`。

通常，这样丢失资源会导致用户看不到或听不到某些内容，或者 3D 模型显示为纯黑色。一个好的解决方案是记录这类错误，并使用“无图片”占位图来明确提示出现了问题。有时，缺失图片并不足以成为弹出严重错误对话框的理由，因为游戏可能仍然可以正常运行。

## 第 1 章：快速入门

### 快速总结：编写优质代码

如果遵循简单的规则，用 JavaScript 编写优质代码并非难事。优质代码必须满足：

-   **清晰。** 阅读代码的人应能快速理解其功能。
-   **可预测。** 变量的类型和结构不应在意料之外的情况下发生变化。
-   **能容忍卡顿和故障。** 现实世界中的移动数据连接远非完美，因此要确保用户知道他们需要等待多久才能完成下载。要考虑到数据可能并非始终可用，并尽可能使用异步调用。

**提示：** 如果你想了解更多关于在现实世界中测试连接问题的知识，请继续阅读第 11 章，该章解释了如何在本地环境中模拟数据包丢失和网络延迟。

##### 面向对象编程

面向对象编程（OOP）是使用最广泛的编程范式。半个多世纪以来，它一直用于构建复杂的程序。最流行的语言，如 Java 和 C#，都基于类的概念工作——类是定义对象公共结构的“蓝图”。

JavaScript 没有类；取而代之的是函数和原型，它们提供了类似的功能。本章的这一部分专门讨论如何在 JavaScript 中实现 OOP 方法。

**注意：** 最佳 OOP 实践的描述超出了本书的范围。本节仅展示如何实现一个基本的 OOP 系统，而不是如何构建基于对象的有效架构。不过，在接下来的章节中，我们将进行大量使用对象的实践，并会努力运用最佳的 OOP 方法。
