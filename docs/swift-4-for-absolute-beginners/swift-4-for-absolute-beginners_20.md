# 为什么要使用 OOP？

在本章中，我们讨论了什么是 OOP，甚至讨论了如何创建类和对象。然而，讨论为什么要在开发中使用 OOP 原则也同样重要。

如果你看看当今流行的编程语言，它们都在一定程度上使用了 OOP 原则。Swift、Objective-C、C++、Visual Basic、C# 和 Java 都要求程序员理解类和对象，才能成功使用这些语言进行开发。要想成为当今世界的一名开发者，你需要理解 OOP。但为什么要使用它呢？

## OOP 无处不在

如今你选择做的几乎所有开发工作，都需要理解面向对象的原则。在 macOS 和 iOS 中，你与之交互的一切几乎都是对象。例如，简单的窗口、按钮和文本框都是对象，并拥有属性和方法。如果你想成为一名成功的程序员，你需要理解 OOP。

## 消除冗余代码

通过使用对象，你可以减少需要重复输入的代码量。如果你编写了用于在客户结账时打印收据的代码，那么当你需要重新打印收据时，你会希望同样的代码也能用。如果你将打印收据的代码放在了 `Sale` 类中，就不必再次重写这些代码了。这不仅节省了你的时间，而且常常有助于减少错误。如果你不使用 OOP，并且发票有改动（哪怕只是图形上的简单变化），你必须确保同时在桌面端和移动端应用中做出修改。如果你遗漏了其中一处，就可能导致两个界面的行为不一致。

## 易于调试

通过将所有与图书相关的代码集中在一个类中，当图书出现问题时，你就知道该去哪里查找。对于一个小的应用程序来说，这可能听起来不是什么大事，但当你的应用程序达到数十万甚至数百万行代码时，这将为你节省大量时间。

## 易于替换

如果你将所有代码都放在一个类中，那么随着应用程序的变化，你可以替换掉这些类，并赋予新类完全不同的功能。然而，修改后的类仍然可以以与当前类相同的方式与应用程序的其他部分进行交互。这类似于汽车零件。如果你想更换汽车上的消音器，你不需要买一辆新车。如果你与发票相关的代码分散在各处，那么修改一个类的相关项就会变得困难得多。
