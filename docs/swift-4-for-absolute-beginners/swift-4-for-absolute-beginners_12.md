# 做出决策、程序流程与应用程序设计

作为一名 iOS 开发者，最棒的事情之一就是你可以精确地告诉你的设备你想要做什么，它们就会照做——你的设备会不厌其烦地重复执行任务。这是因为 iOS 设备不在乎它们昨天有多辛苦，也不会让情绪干扰工作。这些设备不需要拥抱。

而作为开发者，也有一个缺点：你必须考虑应用程序中所有可能出现的结果。许多开发者喜欢拥有这种控制权。他们乐于专注于应用程序的诸多细节；然而，需要处理如此多的细节有时会令人沮丧。正如本书引言中提到的，开发应用程序需要付出代价，这个代价就是时间。你花在开发和调试上的时间越多，就越能掌握所有细节，你的应用程序表现也越好。你必须付出这个代价才能成为一名成功的开发者。

计算机世界是黑白分明的，没有灰色地带。你的设备产生的结果，很多都基于真和假的条件。

在本章中，你将学习计算机逻辑以及如何控制应用程序的流程。处理信息并得出结果是一切应用程序的核心。你的应用程序需要基于值和条件来处理数据。要做到这一点，你需要理解计算机如何执行逻辑运算，并根据应用程序获取的信息来执行代码。
