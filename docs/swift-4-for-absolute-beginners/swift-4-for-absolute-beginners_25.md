# 8. Swift 编程基础

Swift 是一种优雅的语言。它融合了编译型语言的效率以及许多脚本语言的灵活性和现代特性。

本章介绍 Swift 中一些更常见的概念，例如属性和集合类。它还展示了在处理用户界面元素时，如何在 Xcode 中使用属性。这听起来任务繁重，但 Swift、Foundation 框架和 Xcode 工具提供了丰富的对象和方法，以及一种轻松构建应用程序的方式。

### 使用 `let` 与 `var`

如果你花了很多时间接触 Swift，会看到 `var` 一词出现在变量声明之前。你可能也在其他声明之前看到过 `let`。`var` 用于定义变量，而 `let` 用于定义常量。这意味着，如果你用 `let` 声明一个值，将无法更改该值。以下代码定义了一个常量：

```
let myName = "Brad"
```

一旦定义了常量，就无法更改该值。

注意

如果你声明了一个变量且从未更改其值，Xcode 现在会发出警告。它会建议你使用 `let` 代替 `var`。

```
myName = "John"
```

这将会报错。如果你想创建一个可变的变量，需要使用 `var`。例如，你可以这样做：

```
var myName = "Brad"
myName = "John"
```

这不会产生任何错误，因为 `myName` 现在是一个变量。这不仅适用于 `String` 和 `Int`，也可以用于集合和其他对象。

变量提供了更大的灵活性，那么为什么有人会想使用常量呢？简单的答案是性能。如果你知道某个值不会改变，编译器可以将该值优化为常量。

### 理解集合

理解集合是学习 Swift 的基础。事实上，集合对象几乎是所有现代面向对象语言库的基本构造（有时它们被称为容器）。简单来说，集合是一种能够容纳和管理其他对象的类。集合的全部意义在于它提供了一种高效存储和检索对象的通用方式。

集合有多种类型。虽然它们都满足容纳其他对象的相同目的，但主要区别在于检索对象的方式。Swift 中最常用的集合是 `Array` 和 `Dictionary`。

这些集合都可以作为常量或常规变量来创建。如果你将集合创建为常量，则必须在创建时用对象填充它。之后就无法再对其进行修改。
