# 数据类型

在讨论了计算机如何存储数据之后，我们将介绍一个名为数据类型的重要概念。人类通常只需观察数据及其使用背景，就能判断数据类型及其用途。而计算机则需要明确指令才能完成这一过程。因此，程序员需要告知计算机所给数据的类型。以下是一个示例：`2 + 2 = 4`。

计算机需要知道你要将两个数字相加。在这个例子中，它们是整数。你可能会认为，即使是最不经意的观察者也能理解加法运算，更不用说一台精密的计算机了。然而，iOS 应用的用户通常会将数据存储为一系列字符，而非一个计算式。例如，一条短信可能写着“众所周知，`2 + 2 = 4`。”

在这个例子中，该示例是一系列字符，称为字符串。数据类型就是程序中对所存储数据的声明。变量用于存储数据，并在声明时关联一个数据类型。所有数据都存储在变量中，而变量必须有一个变量类型。例如，在 Swift 中，以下是变量声明及其关联的数据类型：

```
var x: Int = 10
var y: Int = 2
var z: Int = 0
var submarineName: String = "USS Nevada SSBN-733"
```

数据类型不能混合使用。你不能执行以下操作：

```
z = x + submarineName
```

混合数据类型会导致编译器警告或错误，你的应用将无法运行。

表 3-5 给出了 Swift 中基本数据类型的示例。

表 3-5. Swift 数据类型

| 类型 | 示例 |
| --- | --- |
| `Int` | 1, 5, 10, 100 |
| `Float` 或 `Double` | 1.0, 2.222, 3.14159 |
| `Bool` | `true`, `false` |
| `String` | `"星球大战"`, "星际迷航"` |
| `ClassName` | `UIView`, `UILabel` 等 |

## 声明常量和变量

Swift 常量和变量必须在使用前声明。使用 `let` 关键字声明常量，使用 `var` 关键字声明变量。常量在初始化后永不改变，而变量可以根据需要多次更改。

声明常量和变量的类型有两种方式：显式声明和隐式声明。

以下是显式声明变量类型的语法：

```
var name: type = value
var firstNumber: Int = 5
```

然而，声明类型通常是没必要的。隐式声明类型可以缩短代码，使其更易于编写和维护。

以下是隐式声明变量类型的语法：

```
var name = value
var firstNumber = 5
```

大多数情况下你可以使用隐式声明，因为 Swift 足够智能，能够根据赋值推断出变量的类型。

如果某个变量不会改变，你应该将其声明为常量。常量永不改变。常量以关键字 `let` 开头，如下所示：

```
let secondNumber = 10
```

为了更好地理解变量和常量是如何声明的，这里有两个示例：

```
let maximumNumberOfStudents = 30
var currentNumberOfStudents = 5
```

这段代码可以解读为：“声明一个名为 `maximumNumberOfStudents` 的新常量，并将其值设为 30。然后，声明一个名为 `currentNumberOfStudents` 的新变量，并将其初始值设为 5。”

在这个例子中，最大学生人数被声明为常量，因为最大值永远不会改变。当前学生人数被声明为变量，因为该值需要在学生注册人数变化后进行增减。

程序中用到的大多数数据可以分为四种类型：布尔值、数字、字符串和对象。我们将在本章剩余部分讨论如何处理数字和对象数据类型。在第四章中，当你学习如何编写具有决策逻辑的应用时，我们将更详细地讨论布尔数据类型。

## 可选类型

Swift 引入了一个重要的概念，称为可选类型，开发者需要理解它。即使对于经验丰富的 Objective-C iOS 开发者来说，这也是一个全新的概念。可选类型并非难以理解的主题，但需要一些时间才能适应。

在某个值可能缺失的情况下使用可选类型。可选类型表达的意思是：

* 一个变量可能有值，也可能没有被赋值。

有时常量或变量可能没有值。代码清单 3-1 展示了一个名为 `Int()` 的整数初始化器的示例，它将一个 `String` 值转换为 `Int`。

```
1 let myString = "42"
2 let someInteger = Int(myString)
3 // someInteger 被推断为类型 "Int?"，即 "可选 Int"
代码清单 3-1.
将字符串转换为整数
```

常量 `someInteger` 被赋予了整数值 `42`。`someInteger` 也被赋予了类型 `Int?`。问号表示它是一个可选类型，意味着该变量或常量的值可能缺失。请参见代码清单 3-2。

```
1 let myString = "Hello World"
2 let someInteger = Int(myString)
3 // someInteger 的值现在是缺失的
代码清单 3-2.
无法将字符串转换为整数
```

代码清单 3-2 中的第 2 行有一个问题。无法将 "Hello World" 从 `String` 转换为 `Int`。因此，`someInteger` 的值被视为缺失或 `nil`，因为在第 2 行，`someInteger` 被推断为可选 `Int`。

> **注意：** Objective-C 程序员可能使用过 `nil` 从方法中返回一个对象，其中 `nil` 表示“有效对象的缺失”。这适用于对象，但在处理结构体、基本的 C 类型或枚举值时效果不佳。Objective-C 方法通常会返回一个特殊值，比如 `NSNotFound`，以指示有效对象的缺失。这要求调用该方法的调用方知道要检查的特殊值。可选类型则指示任何类型的值的缺失，而无需使用特殊常量。

`Integer Int()` 初始化器可能无法返回值，因此该方法返回一个可选 `Int`，而不是 `Int`。同样，问号表示其包含的值是可选的，意味着它可能包含某个 `Int` 值，也可能根本不包含任何值。该值要么是一个 `Int`，要么什么都没有。

Swift 的 `nil` 与 Objective-C 中的 `nil` 不同。在 Objective-C 中，`nil` 是一个指向不存在的对象的指针。而在 Swift 中，`nil` 不是指针；它是值的缺失。任何类型的可选类型都可以被设置为 `nil`，而不仅仅是对象类型。

在第四章中，你将学习如何“解包”可选类型，并检查有效对象是否存在。
