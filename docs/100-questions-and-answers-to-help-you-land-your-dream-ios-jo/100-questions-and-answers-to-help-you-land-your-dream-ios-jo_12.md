# 问题 59：@synthesize 是做什么的？

`@synthesize` 为变量创建 getter 和 setter 方法。这允许你为变量指定一些属性，当你使用 `@synthesize` 将该属性合成到变量时，你就生成了该变量的 getter 和 setter。

属性名可以和变量名相同。有时人们希望它们不同，以便在 `init` 或 `dealloc` 方法中使用，或者当参数与变量同名传递时使用。



## 问题 60：Swift 中有哪些集合类型？

在 Swift 中，集合类型有两种：`Array` 和 `Dictionary`。

*   `Array`：你可以创建一个单一类型的数组或包含多种类型的数组。Swift 通常更倾向于前者。
*   要访问数组元素，必须使用下标，例如 `println(CardNames[0])`。
*   `Dictionary`：这类似于其他编程语言中的哈希表。字典允许你存储键值对，并通过提供键来访问对应的值。

```
var cardName : [String] = [ "Robert" , "Lisa" , "Kevin"]
```

```
var cards = [ "Robert": 22, "Lisa" : 24, "Kevin": 26]
```

## 问题 61：Swift 中的自定义运算符是什么？

当你需要一个全新的运算符时（通常是因为其功能非常独特，以至于现有的任何运算符都不适用），Swift 中的自定义运算符就派上用场了。接下来，一个有趣的问题是向潜在的候选人提问：Swift 中存在哪些类型的自定义运算符？他/她应该能够列出以下几种：

*   `Prefix`（前缀）：一种一元运算符（操作一个值），位于其操作数的*前面*。
*   `Postfix`（后缀）：一种一元运算符，位于其操作数的*后面*。
*   `Infix`（中缀）：一种二元运算符（操作两个值），位于其两个操作数*之间*。

作为编码练习，候选人可以为特定问题编写自定义运算符。

## 问题 62：在使用代码块（Blocks）时，你需要注意哪些问题？

在特定情况下，iOS 中的代码块（Blocks）可能会引入循环引用（retain cycles）。尤其是在捕获 `self` 时，如果未能避免强引用循环，就很可能发生这种情况。

## 问题 63：什么是 iOS 扩展（extension）？

Swift 扩展主要为现有类型（类、结构和枚举）添加方法，只是有几种不同的“风味”。

以下代码为 `UIColor` 类创建了一个扩展。请注意，这里使用的关键字是 `extension`，而不是 `class`、`struct` 或 `enum`。

```
extension UIColor: ImportantProtocol, MoreImportantCustomProtocol {
}
```

## 问题 64：什么是应用程序沙盒（sandboxing）？

应用程序沙盒是 OS X 中提供的一种访问控制技术，在内核级别强制执行。其设计目的是，如果应用程序受到威胁，可以将损害限制在系统和用户数据范围内。通过 Mac App Store 分发的应用必须采用应用程序沙盒。使用开发者 ID 在 Mac App Store 外部签名和分发的应用也可以（并且在大多数情况下也应该）使用应用程序沙盒。

出于安全原因，iOS 会在安装时将每个应用程序（包括其偏好设置和数据）置于一个沙盒中。沙盒是一组精细的控制机制，用于限制应用程序对文件、偏好设置、网络资源、硬件等的访问。作为沙盒化过程的一部分，系统将每个应用程序安装在其独立的沙盒目录中，该目录充当应用程序及其数据的主目录。

## 问题 65：如何为 iPad 和 iPhone 开发应用程序？

如果你在创建新项目时选择设备族（Device family）为通用（universal），你将获得为 iPhone 和 iPad 准备不同 XIB 文件或故事板（storyboards）的选项。在代码层面，你可以检查设备类型，或者在分配资源时使用适当的命名。

在运行时，你可能希望根据设备是 iPhone 还是 iPad 来加载不同的资源。从编程代码的角度，你可以按以下方式操作（使用 Swift）：

```
switch UIDevice.currentDevice().userInterfaceIdiom {
case .Phone:
// 这是 iPhone
case .Pad:
// 这是 iPad
case .Unspecified:
//
}
```

## 问题 66：iOS 中有哪些不同的后台模式？

*   `播放音频`：应用程序可以在后台继续播放和/或录制音频。
*   `接收位置更新`：当设备位置变化时，应用程序可以持续获得回调。
*   `执行有限时长任务`：这是通用的“任何情况”模式，应用程序可以在后台运行任意代码，但有时间限制。
*   `处理 Newsstand Kit 下载`：专属于报刊杂志（Newsstand）应用，应用程序可以在后台下载内容。
*   `提供 VoIP 服务`：应用程序可以在后台运行任意代码。当然，苹果会限制其使用，因此你的应用也必须提供 VoIP 服务。
