# 看似 Core Data 需要大量工作；而且，对于像这样的简单应用，它并没有带来太多优势。但在更复杂的应用中，Core Data 可以大幅减少你设计和编写数据模型所花费的时间。

### 持久化的奖赏

你现在应该已经扎实掌握了在应用会话之间保存数据的四种不同方法——如果算上你在上一章学到的用户默认设置，那就是五种方法。我们构建了一个使用属性列表持久化数据的应用，然后将其修改为使用对象归档保存数据。接着，我们进行了更改，使用 iOS 内置的 SQLite3 机制来保存应用数据。最后，我们使用 Core Data 重建了同一个应用。这些机制几乎是所有 iOS 应用中保存和加载数据的基本构建模块。
