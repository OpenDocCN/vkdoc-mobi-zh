# 排版后的文本

持续集成是一个软件工程原则，其名称本身就非常直观易懂。在项目的生命周期中，你会不断将小的软件片段集成到你的项目中。然后，随着项目的增长，你需要确保这些片段确实能按照预期的方式契合在一起。这听起来像是在用一种隐喻式的方式说，你将使用自己喜欢的版本控制系统来合并分支，但实际上这正是发生的事情——无论你使用什么工具，或者你多久集成一次代码。

持续集成的原则与“自动化测试”和“软件质量保证”（SQA）的概念密不可分。确实，集成软件片段是一回事，但确保它们能契合——意味着它们不会破坏你的软件、尊重你的编码规范，或对最终产品造成任何损害——则完全是另一回事。
