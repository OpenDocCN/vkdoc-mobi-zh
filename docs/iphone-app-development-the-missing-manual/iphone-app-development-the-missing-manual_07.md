## Objective-C：iPhone 应用的螺丝与螺母

在本节中，你将探索用于编写 iPhone 应用程序的语言：Objective-C。本章不会教你如何编程，而是向你展示这门语言与你可能用过的其他语言之间的差异。

你还将遇到 Objective-C 独有的特性。

### 方括号的世界

在你创建手电筒时，你可能注意到了这行代码：

```
[window makeKeyAndVisible];
```

如果你曾用 Java、C 或 JavaScript 编程，这些方括号可能会让你摸不着头脑。你不是第一个有这种反应的开发者！

那么这些奇怪的方括号是干什么用的呢？在回答这个问题之前，先看看这门语言的名字——结尾的 *C*。你看，这门全新的语言实际上是基于一门旧语言：1972 年丹尼斯·里奇在贝尔电话实验室开发的 C 语言。与 Java 和 JavaScript 一样，Objective-C 的创建者在许多程序员熟悉的基础上构建了新的东西。一旦你学会了一些技巧，这门语言也会很快变得熟悉起来（有关 Objective-C 与其他 C 语言关系的更多信息，请参见下面的框）。

**内幕：C 语言加特性**

Objective-C 是 C 语言语法的超集。你在本章中看到的所有内容都是使用 GNU C/C++ 编译器构建的。

当源文件使用 `.m` 而不是 `.c` 时，会进行额外的文件处理（在概念上类似于使用预处理器）。这一步将你的代码连接到一个支持 Objective-C 特性的小型运行时。类和对象只是 C 结构体；运行时让它们通过内省和消息传递进行通信。

由于整个基础设施基于标准 C，因此很容易将开源库和遗留代码与你的 Objective-C 应用程序混合使用。开始使用这门新语言时，你不必放弃你珍视的 C/C++ 源代码。

话虽如此，许多开发者发现这个面向对象的环境非常令人上瘾。将 Objective-C 用作包装器，使开源库更易于使用，这种情况相当常见。一个很好的例子是正则表达式：Cocoa Touch 本身不支持它们。多亏了 John Engelhart 的工作，RegexKit 框架（`http://regexkit.sourceforge.net/`）使得 Perl 兼容正则表达式 (PCRE) 可以与你的字符串对象一起愉快地使用。

当你熟悉了这门新语言的语法和感觉后，你将在第 60 页的框中探索如何使用标准的 C 习惯用法来实现它。

## 一切皆对象

看看下面的 Objective-C 代码。你会立即对你在其他语言中使用过的熟悉表达式和控制结构感到自在：

```
int i;
for (i = 0; i < 10; i++) {
    // 检查偶数和奇数值
    if ((i % 2) == 0)
        [label setTitle:@″Even″];
    else
        [label setTitle:@″Odd″];
}
```

在这个简单示例中，很容易看出你正在一个执行 10 次的循环中检查偶数和奇数值。

这门语言名字的第一部分暗示了那些括号的用途：*Objective* 告诉你，你在处理对象。而这些方括号让你与这些对象进行通信。在第一个示例中，你在与一个 *window* 对象通信。第二个示例则与一个 *label* 对象交互。

所以现在你可能会想，“太好了，但什么是对象？” Objective-C 是计算机科学家所称的*面向对象*语言。在开发你的 iPhone 应用程序时，你将进行面向对象编程。

**提示：** 如果你是面向对象编程的新手，请花点时间阅读维基百科上关于此主题的页面。概述和历史将帮助你快速了解这种编程风格中使用的设计动机和术语：`http://en.wikipedia.org/wiki/Object-oriented_programming`。

*对象*是包含应用程序代码和数据的内存块。面向对象设计的主要原则之一是，你的应用程序应尽可能少地了解对象的内部工作机制。像*封装*和*数据抽象*这样的术语描述了这种行为。



对象应该像一个黑盒子。事实上，使用方括号作为视觉提示可能对你有所帮助：`[]` 看起来很像一个盒子。

打开 iPhone 的瞬间，内存中就会创建数千个对象。启动你自己的应用时，还会新增数千个对象。你常常会听到用*实例*来描述这些对象中的每一个。由于有这么多实例在内存中浮动，你需要变量来跟踪那些重要的实例。

## 这一切的对象

### 命令你的对象做事

引用对象的变量允许你改变对象的行为或状态。通过向对象发送*消息*来实现这一点。在 Cocoa 文档中，接收消息的对象常被称为*接收者*。

***注意：*** 你还会看到术语*目标*用于表示接收消息的对象。消息本身有时被称为*动作*，因为消息通常会促使事情发生。在处理用户界面（在 Interface Builder 中）时，你会经常遇到这些术语。

对于本节开头展示的*窗口*对象，你向它发送了消息 `-makeKeyAndVisible`。本质上，这条消息是在告诉窗口让自己在屏幕上可见。你的“label”变量收到了消息 `-setTitle:`，并附带了一个变量；它被指示将其标题更改为一个新值。不用关心具体发生什么动作——你目前的重点是理解如何使用消息来发起动作。

当你逐渐习惯这种新的编程语法时，方括号会提醒你正在与一个黑盒子进行通信。其中第一个单词指定了接收消息的对象，其余单词提供了所需的任何其他参数。以下是一些示例：

```
[myBlackBox messageWithNoParameters];
[myBlackBox messageWithFirstParameter:one andSecondParameter:two];
```

***注意：*** 如果你有 C++ 或 Java 等其他面向对象语言的背景，发送消息在功能上等同于以下形式：

```
myBlackBox->functionWithNoParameters();
myBlackBox->functionWithParameters(one, two);
```

或这样：

```
myBlackBox.functionWithNoParameters();
myBlackBox.functionWithParameters(one, two);
```

最大的区别在于，你发送给对象的消息不仅可以在应用编译时生成，也可以在运行时生成。同样，对象的方法也可以在代码运行时被修改。Objective-C 是一种动态语言，因此你可以随时改变对象的行为。

这些消息中的每一条都由对象定义的*方法*来处理。方法使用唯一的签名，让 Objective-C 能够将你的消息和任何参数路由到正确的代码以执行。你会发现，许多方法签名在你的代码中非常易读：虽然需要多打几个字，但最终你会得到更容易理解的代码。

## 大量的类

有些方法会返回值。之前，你看到了 `setTitle:` 方法更改了一个标签的标题。有一个对应的方法叫做 `title`，它允许你向对象查询当前的标题值。例如：

```
NSString *labelTitle = [label title];
```

现在你正在跳出黑盒子进行思考。但那个 `NSString` 到底是什么东西？

这是一个变量定义。就像 C 语言中的任何变量定义一样，`labelTitle` 变量是一个指向名为 `NSString` 的类的指针（因为星号）。很好，但什么是类呢？继续往下读。

### 大量的类（续）

Objective-C 中的每个对象都属于一个*类*，因为行为和内部数据都是在类中指定的。类是方法（代码）定义和实现的地方。也是你定义哪些实例变量（数据）由该类管理的地方。

`NSString` 是一个提供丰富方法集用于存储和处理文本的类。名称中的 `NS` 表示它是 Cocoa Touch 中系统基础框架的一部分。

***注意：*** 框架是你可以添加到应用程序中的类和其他资源的集合。你将在下一章全面了解提供所有有趣类的 Cocoa Touch 框架，例如 `NSString`。

另外，由于 Objective-C 缺乏命名空间，框架通常使用两个字符的前缀来防止类名冲突。最初为 NeXTSTEP 编写的基础类使用 `NS`。Cocoa Touch 中的用户界面类使用 `UI`。一些开发者使用自己的类名前缀来避免这些冲突。例如，Iconfactory 在其项目间共享的所有类中都使用 `IF`。

因此，`labelTitle` 指向一个由系统定义的字符串对象的实例。由于你在变量中拥有这个对象指针，你可以向它发送消息。例如，你可以向你的 `labelTitle` 发送 `uppercaseString` 消息，它将返回一个新的字符串对象，其中所有小写字母都已转换为大写字母：

```
NSString *moreAwesomeLabelTitle = [labelTitle uppercaseString];
```

然后，这个 `moreAwesomeLabelTitle` 变量可以用来更新原始标签的标题：

```
[label setTitle:moreAwesomeLabelTitle];
```

***提示：*** 嵌套对象及其消息是缩短代码并减少临时变量数量的有效方法。你可以将这三行代码：

```
NSString *labelTitle = [label title];
NSString *moreAwesomeLabelTitle = [labelTitle uppercaseString];
[label setTitle:moreAwesomeLabelTitle];
```

通过使用这种技术压缩成一行代码：

```
[label setTitle:[[label title] uppercaseString]];
```

只要确保正确平衡括号即可，否则编译器会抱怨！

之前你可能好奇过，label 对象的 `setTitle:` 方法中的 `@"Even"` 和 `@"Odd"` 参数是做什么的。问得好！由于字符串在编程中大量使用，Objective-C 的创建者为这些文本对象定义了一种简写形式。如果没有这种简写，`@"Even"` 需要使用以下方式实例化：

```
[NSString stringWithUTF8String:″Even″]
```

### 类的细节

至此，应该很明显，类在 Objective-C 中非常重要。它们充当着构建应用的积木，并让你避免编写大量枯燥的代码。不难想象 `uppercaseString` 方法是如何实现的，但更妙的是完全不去想它，直接让系统来完成工作。

类是在包含在你的源代码中的头文件中定义的。像 `NSString` 这样的系统级类会被默认引入。看看上一章自动为你创建的 `FlashlightAppDelegate.h` 文件的顶部：

```
#import <UIKit/UIKit.h>
```

这一行代码就引入了整个系统的类定义。预处理器首先加载用于开发 Cocoa Touch 用户界面的类（`UIKit`）。这些类又会接着加载用于管理数据的基础类以及用于绘图的 Core Graphics 类。

***注意：*** 对于有经验的 C 程序员，请注意 `#include` 和 `#import` 之间有一个微妙的区别：预处理器会跟踪导入，并且不会多次包含同一个文件。这种技术使得避免多重定义变得容易得多！
