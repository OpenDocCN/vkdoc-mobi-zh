# 在 Unity 中使用 iOS 构建目标

当你从 App Store 将`Angry Bots`安装到 iOS 设备并开始游戏后，请注意游戏中为玩家移动和射击设计了虚拟摇杆（如图 10-2 所示），这个功能此前在`Angry Bots`中并未出现。

![9781430248750_Fig10-02.jpg](img/9781430248750_Fig10-02.jpg)

*图 10-2. 带有虚拟摇杆的 iOS 版 Angry Bots*

虽然`Angry Bots`的 iOS 版本采用了不同的控制方式，但它与 Unity 内置安装的`Angry Bots`版本基于相同的源代码。因此，我们将暂缓保龄球游戏的 iOS 移植工作，在未来几章中重新熟悉`Angry Bots`，以便专注于 Unity iOS 构建与测试流程。本章你将学习如何使用 Unity Remote 和 Xcode 中的 iOS 模拟器测试`Angry Bots`的 Unity iOS 版本。

### 使用 Unity Remote 进行测试

Unity 提供了一种非常酷的解决方案来在编辑器内测试 Unity iOS 项目，那就是`Unity Remote`应用。它运行在 iOS 设备上，通过 Wi-Fi 将触屏和加速度计数据从设备转发到 Unity 编辑器。如果你没有 iOS 设备，可以跳过本节，使用下一节介绍的 iOS 模拟器进行测试。

`Unity Remote`可以在 App Store 中找到，方式与`Angry Bots`应用相同：在 App Store 搜索框中输入`"unity technologies"`（如图 10-1 所示）。输入`"unity remote"`同样有效（如图 10-12 所示）。

![9781430248750_Fig10-12.jpg](img/9781430248750_Fig10-12.jpg)

*图 10-12. App Store 中的 Unity Remote 应用*

将`Unity Remote`安装到 iOS 设备后，请确保该设备与运行 Unity 的 Mac 处于同一 Wi-Fi 网络。启动`Unity Remote`后，屏幕应显示该 Mac 的名称（如图 10-13 所示）。选择该计算机，如果 Unity 未处于`Play`模式，`Unity Remote`屏幕将保持等待状态，并显示提示让你在 Unity 编辑器中点击`Play`。

![9781430248750_Fig10-13.jpg](img/9781430248750_Fig10-13.jpg)

*图 10-13. Unity Remote 应用中的计算机选择*

当 Unity 编辑器处于`Play`模式时，`Game View`中的画面也会出现在`Unity Remote`中，你可以用设备的触屏来玩游戏！画面可能略显粗糙，因为 Unity 通过 Wi-Fi 向设备发送游戏画面的副本。你可以通过降低 Unity 编辑器中`Game View`的分辨率来改善性能（如图 10-14 所示）。

![9781430248750_Fig10-14.jpg](img/9781430248750_Fig10-14.jpg)

*图 10-14. Game View 中最小化的设备分辨率*



无论图形质量如何，你现在都可以在 iOS 设备上玩《愤怒的机器人》了，尽管你实际上仍然是在编辑器中进行游戏，只是将设备用作控制器。触摸屏输入通过 Wi-Fi 发送回 Unity 编辑器，并由在播放模式下运行的游戏接收，因此在测试运行期间，你仍然可以使用 Unity 编辑器的所有调试功能。真是两全其美！

### 安装 Xcode

在某些时候，你必须离开 Unity 编辑器的舒适区，真正开始为 iOS 进行构建。所有 iOS 应用都必须通过苹果官方 iOS 开发工具（同时也是其官方 OS X 开发工具）Xcode 进行构建，这就是为什么 Unity iOS 构建会首先生成一个 Xcode 项目，然后由 Xcode 构建该项目以创建应用。Xcode 可从苹果开发者网站免费获取，但更方便的是，它也可以在 Mac App Store 上找到。在 Mac App Store 中搜索 `xcode`，然后点击“免费”按钮进行下载（图 10-15）。

![9781430248750_Fig10-15.jpg](img/9781430248750_Fig10-15.jpg)

图 10-15. Mac App Store 中的 Xcode

安装完成后，你应该能在“应用程序”文件夹中看到 Xcode 应用。现在，你已经准备好从 Unity 开始执行 iOS 构建了！

### 自定义玩家设置

Unity iOS 构建总是需要自定义 Unity 编辑器中的玩家设置（Unity 播放器，相对于 Unity 编辑器，是 Unity 引擎的部署版本）。在 `Edit` 菜单下选择 `Player Settings`，或点击 Build 窗口中的 `Player Settings` 按钮，`Player Settings` 将显示在 Inspector 视图中（图 10-16）。

![9781430248750_Fig10-16.jpg](img/9781430248750_Fig10-16.jpg)

图 10-16. 选择玩家设置

`Player Settings` 分为顶部的跨平台设置和底部的各平台设置。影响 iOS 构建的跨平台设置只有两个：产品名称和默认图标，它们分别提供显示在设备上的应用名称和图标。

公司名称、默认光标和光标热点属性仅用于桌面和网页播放器构建，不过你也可以填写合适的公司名称或你的名字，以便在项目中留存。如果你想知道为什么跨平台设置并非完全跨平台，请记住 Unity 最初只支持网页和桌面平台，iOS 支持是几年后才加入的（而安卓支持则更晚）。

由于我们现在只关心 iOS 玩家设置，请点击看起来有点像 iPhone 的“各平台设置”标签，以查看 iOS 的设置（图 10-17）。

![9781430248750_Fig10-17.jpg](img/9781430248750_Fig10-17.jpg)

图 10-17. iOS 的玩家设置

`Player Settings` 内容丰富，因此被组织成几个可折叠部分：分辨率与呈现、图标、启动画面，以及包罗万象的其他设置。第 12 章专门讨论了呈现问题，包括图标和启动画面设置，因此现在我们主要关注分辨率与呈现以及其他设置。

#### 分辨率与呈现

大多数分辨率和呈现设置都有合理的默认值（图 10-18），但我建议始终将默认方向从“左向横屏”改为“自动旋转”，因为苹果要求 iPad 应用支持自动旋转，这意味着显示器会自动旋转以保持竖直。其他默认方向选项包括“右向横屏”、“竖屏”或“倒置竖屏”。

![9781430248750_Fig10-18.jpg](img/9781430248750_Fig10-18.jpg)

图 10-18. iOS 玩家设置中的分辨率与呈现

如果选择了“自动旋转”，则会显示“自动旋转允许的方向”。由于《愤怒的机器人》仅设计为在横屏模式下运行，请选择“左向横屏”和“右向横屏”。这样一来，当设备转动时，《愤怒的机器人》将在左向横屏和右向横屏之间切换，但绝不会进入竖屏模式。

“使用动画自动旋转”选项指定显示器是进行可视旋转还是立即切换。我始终选择动画旋转，因为它看起来很酷。状态栏选项指定应用启动时 iOS 状态栏是否可见，以及如果可见，采用何种样式。32 位显示缓冲区决定了颜色分辨率。如果未选择此选项，则使用 16 位颜色。

**注意**   32 位显示缓冲区选项标有星号。任何标有星号的设置都会与其他移动平台（即安卓）共享。

“显示加载指示器”控制应用启动时是否显示 iOS 活动指示器（屏幕中央的小旋转图形）。此设置也在第 12 章中的呈现选项中有所涉及。

#### 其他设置

许多重要设置恰好都集中在标题通用的“其他设置”下（图 10-19）。渲染设置下的静态和动态批处理选项实际上是优化选项，影响是否以及何时自动合并网格以提高性能。这将在第 16 章的优化技术中讨论。

![9781430248750_Fig10-19.jpg](img/9781430248750_Fig10-19.jpg)

图 10-19. iOS 玩家设置中的其他设置

标识部分有两个设置：包标识符和包版本。它们对于在 iOS 模拟器中运行并不重要，但当需要提交应用时，这些设置必须与 iTunesConnect 中指定的应用对应信息相匹配（这将在下一章的应用提交流程中介绍）。

包 ID 是应用唯一标识符，通常以反转 URL 的形式表示。例如，我所有应用的包 ID 都是 `com.technicat.<appname>` 这种形式，所以我会将 `com.unity.angryBots` 改为 `com.technicat.angryBots`。

包版本与应用商店中显示的应用版本号相同。它通常格式化为两个数字 `<major>.<minor>`，或者有时是三个数字 `<major>.<minor>.<revision>`。每当你在应用商店提交应用更新时，这个数字都必须递增。

在配置下，大多数设置都有合理的默认值，但你确实需要通过选择目标设备来决定你的应用是运行在 iPhone、iPad 还是两者都支持。大多数 3D 应用通常在各种屏幕宽高比上都能良好运行，至少比 2D 应用要好，但 iPhone 和 iPod Touch 的屏幕比 iPad 更窄。有时，出于业务原因，你可能只想为 iPad 或仅为 iPhone 发布应用。

对于目标分辨率，通常选择“原生”即可（图 10-20）。它等同于“最佳质量”，而“最佳性能”则指定较低屏幕分辨率（在 iPhone 3GS 及其对应的 iPod touch 上不可用，因为它们拥有最低分辨率的屏幕）。其余选项是固定的仅限 iPhone 和仅限 iPad 的分辨率。

![9781430248750_Fig10-20.jpg](img/9781430248750_Fig10-20.jpg)

图 10-20. iOS 玩家设置中的目标分辨率选项



##### 脚本定义符号

`Scripting Define Symbols` 允许你定义自己的预处理器定义，就像上一章 `FuguPause` 脚本中引用的 `UNITY_EDITOR` 和 `UNITY_WEBPLAYER` 定义一样。例如，如果你有一些代码调用了仅适用于 Unity Pro 的某些函数，那么你可以将这些代码包裹在 `#if UNITY_PRO ... #endif` 中，然后在运行 Unity Pro 的项目的 `Scripting Define Symbols` 中添加 `UNITY_PRO`。

可以使用分号作为分隔符来添加多个定义。如果你还有一个 `USE_ADS` 预处理器定义来控制广告是否显示，你可以在 `Scripting Define Symbols` 字段中写入 `USE_ADS;UNITY_PRO`。

### 优化设置

一些 `Optimization`（优化）设置实际上与优化无关。`Api Compatibility Level`（不用在意 API 奇怪的字母大小写）指定了您所需的 `.NET` API 的版本和范围。通常你不需要更改它。

`AOT Compilation Options` 指的是在 Unity iOS 构建期间发生的提前编译（ahead-of-time compilation），这与即时（JIT）编译相对。这就是为什么 Unity iOS 脚本中需要 `#pragma strict` 的原因。它为 AOT 编译器提供了足够的信息来完成其工作。

#### SDK 版本

`SDK Version` 有两个选项：`Device SDK`（设备 SDK）和 `Simulator SDK`（模拟器 SDK）。在下一章中会提到的，为测试设备或应用提交进行构建时，默认值 `Device SDK` 是正确的选择，但在本章中，应选择 `Simulator SDK` 以便在 iOS 模拟器中运行我们的游戏。

> **注意** `SDK Version` 这个属性名称用于区分模拟器和硬件构建，这有点奇怪，但它是旧版 Unity iOS 的遗留用法，当时需要指定随 Xcode 安装的 iOS SDK。

#### 目标 iOS 版本

`Target iOS Version` 指定了应用可运行的最低 iOS 版本。保留其默认和最低值 iOS 4.0，可以最大化兼容设备的数量，从而增加潜在用户。然而，如果目标 iOS SDK 版本低于 iOS 4.3，Unity 将会发出可能导致应用商店审核被拒的警告（图 10-21）。

![9781430248750_Fig10-21.jpg](img/9781430248750_Fig10-21.jpg)

图 10-21. 提交时目标 SDK 版本低于 4.3 的警告

这只是一个潜在的苹果拒绝问题，仅当你碰巧要更新一个已在 App Store 上架、并支持 ARMv6 处理器的应用时才会出现，因为 Unity 4 已放弃 ARMv6 支持，仅支持更新的 ARMv7 系列。正如 Unity 警告所建议的，将目标版本设为 iOS 4.3 或更高是安全的，因为苹果也从 iOS 4.3 开始放弃了 ARMv6 支持（这意味着他们也放弃了对早于 iPhone 3GS 的设备的支持）。

> **提示** 当我第一次尝试向 App Store 提交仅支持 ARMv7 的更新时（2012 年），应用被拒了，最后，将目标版本设为 iOS 4.3 或更高版本才避免了这个问题。此外，一旦仅支持 ARMv7 的更新在 App Store 上获得批准，我就能够再次将目标版本设为更早的 iOS 版本，因为我不再需要更新一个支持 ARMv6 的应用了！

#### 剥离级别、脚本调用优化和优化网格数据

最后，`Stripping Level`（剥离级别）、`Script Call Optimization`（脚本调用优化）和 `Optimize Mesh Data`（优化网格数据）确实是优化设置，这些将在第 16 章与其他优化技术一同介绍。不过，简要地说，`Stripping Level` 是 Unity iOS Pro 的一个选项，用于移除未使用的代码（如果不走运，也可能移除正在使用的代码，这也是为什么它是可选的），`Script Call Optimization` 通过消除异常处理（异常本质上是可被调用代码捕获的错误或“异常”情况）来加速脚本执行，而 `Optimize Mesh Data` 则移除未使用的网格数据（例如，使用无光照材质时不需要法线）。

#### 玩家设置总结

如你所见，`Player Settings` 中有许多选项会影响 Unity iOS 构建的外观和性能。更改 `Player Settings` 后，最好通过 `File` 菜单中的 `Save Project` 或 `Save Scene`（此操作也会隐式执行 `Save Project`）来保存这些更改。虽然 `Player Settings` 中的许多（如果不是大部分）设置在某些时候都很重要，但要让 Angry Bots 在 iOS 模拟器上运行，你唯一需要做的更改就是将 `Device SDK` 设置为 `Simulator SDK`。

### 使用 iOS 模拟器测试

现在你已经安装了 Xcode，并将 `Device SDK` 设置为 `Simulator SDK`，你就可以在 iOS 模拟器上构建并运行 Angry Bots 了。`File` 菜单中的 `Build and Run` 命令和 `Build Settings` 窗口中的 `Build and Run` 按钮都会执行 `Build` 阶段，生成一个 Xcode 项目，然后执行 `Run` 阶段，在 Xcode 项目中调用构建和运行操作。通常情况下，这非常方便，但在使用 iOS 模拟器时，如果你手动在 Xcode 中执行构建和运行，你将能更好地控制 iOS 模拟器。那么，让我们再次打开 `Build Settings` 窗口（从 `File` 菜单），然后点击 `Build` 按钮。

> **提示** 在执行构建之前，请在 `Build Settings` 窗口中检查是否包含了正确的场景集合。

Unity 会弹出一个 `Save As` 窗口，提示你选择即将构建的 Xcode 项目的位置和文件名（图 10-22）。

![9781430248750_Fig10-22.jpg](img/9781430248750_Fig10-22.jpg)

图 10-22. 构建提示：选择 Xcode 项目的位置和名称

它通常默认指向 Unity 项目文件夹的顶层。让我们将这个 Xcode 项目命名为 `AngryIOS`。

> **警告** 注意不要选择 `Assets` 文件夹内的构建目标位置，否则 Unity 会尝试将整个 Xcode 项目作为项目资源导入。

如果这不是你第一次在此位置构建此项目，Unity 将检测到已存在同名 Xcode 项目，并询问你是要完全替换该项目还是追加项目更改（图 10-23）。

![9781430248750_Fig10-23.jpg](img/9781430248750_Fig10-23.jpg)

## 第 11 章

为真实设备构建：设备测试与应用程序提交

运行 Angry Bots 的 iOS 版本竟然出奇地简单。只需将构建目标切换为 iOS，然后立即在编辑器中使用 Unity Remote 应用程序作为输入设备来玩游戏；或者，要在 iOS 模拟器中运行该游戏，只需在“播放器设置”中将目标 SDK 设置为模拟器 SDK 并进行构建即可。



这看起来可能太简单了，以至于让人觉得不可能仅此而已，但事实确实如此。接下来的步骤才是令人头疼的部分：构建 Unity 项目，使其能够安装并在真实的 iOS 硬件上运行。本流程中 Unity 的部分很简单，几乎与构建一个在 iOS 模拟器上运行的项目完全相同。但是，要将应用程序安装到测试设备上，需要使用移动设备预置描述文件来构建应用，该描述文件也必须安装在 Xcode 和测试设备中。在此之前，必须使用与测试设备及应用包标识符相关联的开发证书，在苹果的预置门户中创建该描述文件。因此，在此之前，测试设备和包标识符必须在预置门户中注册，并且在创建证书签名请求之后生成开发证书。

而这一切还仅仅是为了测试。在构建用于提交到 App Store 的应用时，也需要类似的流程，但使用的是独立的发布预置描述文件和相应的证书。此外，向 App Store 提交应用还需要将应用添加到 iTunes Connect，这是一个用于在 App Store 上管理应用的独立网站。

如果处理预置描述文件听起来很麻烦，你可能会想跳过本章，稍后再回来看。事实上，我们保龄球游戏在接下来几章中的 iOS 适配，可以使用 Unity Remote（用于第 13 章的输入代码）和 iOS 模拟器（第 12 章中设置的图标、启动画面和屏幕尺寸）进行测试，就像我们在上一章中对 Angry Bots 所做的那样。此外，访问预置门户和 iTunes Connect 以及相关的非公开文档，需要成为 iOS 开发者计划的成员，所以如果你还没有获得该资格，那么无论如何你都得等待。

但是，你需要 iOS 开发者会员资格才能设置 Game Center 排行榜和成就（我将在第 14 章中介绍），以及 iAd 横幅广告和插页式广告（我将在第 15 章中介绍），而性能测量（我在第 16 章中介绍）如果不能在实际硬件上运行，也会变得索然无味。所以，至少，最好尽快开始 iOS 开发者注册流程！

### 注册成为 iOS 开发者

正如你所料，苹果开发者网站 (`http://developer.apple.com/`) 是查找大多数官方苹果开发者资源（iTunes Connect 是个例外）的地方，包括所有苹果开发者计划——Mac 开发者计划、Safari 开发者计划，以及我们感兴趣的 iOS 开发者计划。你可以以个人或公司身份注册 iOS 开发者计划。只要你有一张信用卡并准备好支付每年 99 美元的费用，以个人身份注册就相当直接。以公司身份注册也需要这些，但它还需要一个实际的法律公司实体以及关于公司的支持信息，包括一个 DUNS（数据通用编号系统）号码，这是一个由邓白氏公司分配给公司的唯一标识符。个人和公司账户都必须经过苹果公司批准，但公司账户的批准时间可能比个人账户长（我的公司 Technicat, LLC 的批准花了三个月）。

公司账户唯一额外的功能是账户管理员可以添加团队成员，因此在本书中，我们将假设我们使用的是个人账户，或者等效地，作为公司账户的管理员。

一旦你被批准成为 iOS 开发者，你就可以使用你的 Apple ID 登录访问苹果开发者网站以及 iTunes Connect (`http://itunesconnect.apple.com/`)。iTunes Connect 是管理你在 App Store 上或准备上架的所有应用的网站。

但在 iTunes Connect 中工作之前，我们将登录苹果开发者网站，并在预置门户中工作，以创建开发和发布预置描述文件，这些文件分别用于测试和发布构建。

### 预置门户

假设你拥有 iOS 开发者会员资格（如果没有，你可以跳过本章，稍后再回来），登录 `http://developer.apple.com/` 并进入预置门户。

**注意** 苹果开发者网站有几个你可以登录的区域——会员中心、iOS 开发中心和开发者论坛——你可以从会员中心或 iOS 开发中心进入预置门户（目前，这种组织方式有点令人困惑）。

一旦你登录，网站上会有详细的文档（预置门户是苹果没有太多公开的屏幕截图式文档的地方之一），但我会简要概述涉及的步骤以及你需要提供的信息。

#### 注册测试设备

开发预置描述文件仅对指定的测试设备集有效，因此首先要做的是在预置门户中注册你想要用于测试的 iOS 设备。你需要提供每台设备的唯一设备标识符（UDID）。设备的 UDID 可以通过 iTunes 或 Xcode 找到。在 iTunes 中，当 iOS 设备连接到你的 Mac 时，可以通过点击设备的序列号来找到 UDID，点击会将序列号显示切换为 UDID 显示。但该显示无法复制粘贴，因此更好的选择是从 Xcode 中获取该号码。

当 iOS 设备连接到你的 Mac 时，UDID 会显示在 Xcode 管理器窗口中。要调出 Xcode 的管理器窗口，如果 Xcode 尚未运行，请启动它，然后从窗口菜单中选择管理器（图 11-1）。

![9781430248750_Fig11-01.jpg](img/9781430248750_Fig11-01.jpg)

图 11-1. 从 Xcode 窗口菜单调出管理器

管理器窗口弹出后，所有已连接的 iOS 设备都应列在左侧面板中。选择你想要注册用于测试的设备，设备信息（包括 UDID，列为标识符）将显示出来（图 11-2）。

![9781430248750_Fig11-02.jpg](img/9781430248750_Fig11-02.jpg)

图 11-2. 从 Xcode 管理器获取设备 ID

然后，你可以将 UDID 复制并粘贴到预置门户中，以注册你的测试设备。

#### 注册应用标识符

注册完测试设备后，你需要注册应用标识符，因为每个预置描述文件仅对一个应用 ID 有效。应用 ID 对应一个包标识符（与你在 Unity 播放器设置中为应用指定的标识符相同），或者当它以星号结尾时匹配许多包标识符（这时它被称为通配符应用 ID）。例如，我注册了一个应用 ID `com.technicat.*`，我可以在一个预置描述文件中将其用于我所有的应用，例如 `com.technicat.fugubowl` 和 `com.technicat.hyperbowl`。

**提示** 创建一个匹配你全部或大部分包标识符的通配符应用 ID，这样你就不必为每个包标识符都创建一个新的应用 ID。

#### 开发预置描述文件

要创建开发预置描述文件，你需要生成一个与该描述文件关联的开发证书。预置门户中有关于如何生成和上传证书签名请求的说明，这将最终生成一个新的开发证书。然后，你可以创建一个与该证书关联的开发预置描述文件。



#### 开发预置描述文件

实际上，一旦你生成了开发证书，预置门户会自动为你生成一个团队预置描述文件，该文件使用星号 `*` 作为应用 ID，因此可以匹配你的任何 Bundle ID，这确实非常方便。但你也可以使用相同的证书和你指定的应用 ID 来生成任意数量的其他开发预置描述文件。

#### 分发预置描述文件

创建分发预置描述文件与创建开发预置描述文件类似。你需要先创建一个分发证书，然后创建一个与该证书关联的分发预置描述文件。

**提示** 重复使用同一个证书请求文件来创建开发证书和分发证书，可以节省一点时间。

### 准备 App Store 用图

您需要为 App Store 准备两种图形：应用图标和一组针对您所支持的所有设备的截图。iTunes Connect 在闲置一段时间后容易自动登出并丢失所有更改，因此不建议在登录状态下折腾创建图标和截图。

#### 创建图标

iTunes Connect 会要求提供一个 1024 × 1024 像素的应用图标（`.png` 或 `.jpeg` 格式），该图标将随 App Store 中的应用描述一起显示。如果您在创建应用图标时早有规划，那直接使用即可。否则，您需要在进入 iTunes Connect 之前创建好它。

理想情况下，您应该请专业图标设计师来创建图标，但如果资金和美术技能都不足，可以像我在紧急情况下所做的那样——从 Unity 编辑器的游戏视图中截取一张大图，并将其缩放到 1024 × 1024。从大图缩小效果更好。将较小的现有图标放大通常效果不佳，Apple 也不鼓励这样做，尽管这种情况可能很常见，因为在不久之前，应用提交时上传的还是 512 × 512 的图标。

**注意** 如果 App Store 图标与设备上显示的任何应用图标不匹配，Apple 可能会拒绝该应用。因此，如果您为 App Store 创建了一个全新的 1024 × 1024 图标，应在 Unity 的 Player Settings 中将其设置为默认图标。

#### 截取屏幕截图

iTunes Connect 还会要求您提供三种尺寸的截图：适配 iPhone 4（640 × 960 或 960 × 640）、iPhone 5（640 × 1136 或 1136 × 640）和 iPad（768 × 1024 或 1024 × 768，新 iPad 则为以上尺寸的两倍）。前提是您在 Unity 的 Player Settings 中选择了 iPhone + iPad 作为支持的硬件。如果只选择了 iPhone，则无需提供 iPad 截图；如果只选择了 iPad，则只需提供 iPad 截图。

截图有三种相当便捷的方法：使用 iOS 模拟器、通过 Xcode Organizer 窗口从设备截取，以及在设备上按按钮截取。

使用 iOS 模拟器截图很简单。使用“编辑”菜单中的“截图”命令，截图将保存到桌面上。如果您的游戏在 iOS 模拟器受限的输入方式下运行不佳，使用模拟器可能不方便。但如果您没有 iPhone 4、iPhone 5 系列以及 iPad，这是获取截图的一种方法。

使用 Xcode 截图几乎同样简单。将设备连接到 Mac 后，在 Xcode Organizer 窗口中选择设备的“截图”面板（所有已连接设备都会显示绿色圆点）。

然后在测试设备上运行游戏时，当看到喜欢的画面，点击右下角的“新建截图”按钮。截图将显示在“截图”面板中（图 11-12），您可以将任意截图从该处（左侧的小预览图或右侧的大图）拖到桌面上，以便轻松访问。

![9781430248750_Fig11-12.jpg](img/9781430248750_Fig11-12.jpg)

图 11-12. 在 Xcode Organizer 中截取设备截图

另外，如果您需要在 Finder 中访问这些截图，它们位于“资源库”![image](img/arrow.jpg)“应用程序支持”![image](img/arrow.jpg)“开发者”![image](img/arrow.jpg)“共享”![image](img/arrow.jpg)“截图”文件夹中。

**提示** OS X 通常隐藏“资源库”文件夹。您可以通过打开“终端”应用程序并输入 `chflags nohidden ~/Library` 来取消隐藏。

我最喜欢的截图方式是直接在设备上同时按下“主屏幕”按钮和“电源”按钮。截图会保存到设备的“照片”中，之后可以同步到 Mac 上的 iPhoto，并可在 Finder 中访问。这种方法可以让我在任何地方（比如沙发上）悠闲地截图，而无需一直连接着 Mac。而且，在关键时刻快速截图，比试图抓住鼠标并迅速点击 Xcode Organizer 中的“新建截图”按钮要容易得多。

### 为拒绝做好准备

如果应用被拒绝，电子邮件将告知你返回 iTunes Connect 中的应用，点击`Resolution Center`按钮查看并选择性地回应拒绝原因，这些原因可能包括技术问题、崩溃 Bug 或违反 Apple 的 App Store 审核指南（在注册成为 iOS 开发者后，你可以在 Apple 开发者网站上阅读该指南）。

如果你选择修改应用，你必须在再次上传前点击`Reject Binary`和`Ready for Upload`。如果你认为拒绝不合理，`Resolution Center`也允许你向 App Review Board 提出申诉。

如果被接受，并且你在 iTunes Connect 中选择了自动发布选项，应用将很快出现在 App Store 上。否则，应用将处于`Pending Developer Release`状态，直到你登录 iTunes Connect，点击应用的`View Details`，然后点击`Release this Version`按钮。

### 更新应用

应用被批准后，你可以随时返回 iTunes Connect 中的应用页面，点击`View Details`来编辑应用描述，修改后的文本将在几小时内显示在 App Store 上。iTunes Connect 页面还会显示一个`Add New Version`按钮。点击此按钮将引导你完成原始提交流程的简化版本，以提交应用更新。除了包 ID（bundle ID）外，你可以更改几乎所有详细信息，但唯一需要更改的是增加包版本（bundle version）。

**提示** 更新应用时，请记住同时增加 iTunes Connect 和 Unity Player Settings 中的包版本。

请注意，包 ID 中以句点分隔的组成部分被视为数字而不是字符串，因此将版本`247`更新为`2.049`是（令人惊讶地）可以的，因为它比较的是数字`47`和`49`（Figure 11-33）。

![9781430248750_Fig11-33.jpg](img/9781430248750_Fig11-33.jpg)

Figure 11-33. 在 iTunes Connect 中成功将应用版本号从`2.47`升级到`2.049`

但是，将版本`1.65`更新为`1.7`会比较数字`65`和`7`，这会导致提交错误消息（Figure 11-34）。

![9781430248750_Fig11-34.jpg](img/9781430248750_Fig11-34.jpg)

Figure 11-34. 在 Xcode Organizer 中未能成功将应用版本号从`1.65`更新为`1.7`

### 跟踪销售情况

### 发放促销代码

对于应用的每个版本，Apple 会提供 50 个促销代码，可用于在 App Store 免费下载应用（即使是免费应用也提供促销代码，尽管在这种情况下使用它们意义不大）。要获取应用的促销代码，请在 iTunes Connect 中点击该版本应用的`View Details`，然后点击右上角的`Promo Codes`按钮（Figure 11-36）。

![9781430248750_Fig11-36.jpg](img/9781430248750_Fig11-36.jpg)

Figure 11-36. 在 iTunes Connect 中点击已上线应用的`View Details`

在同意一项协议后，你将看到一个下载按钮和一个用于指定要下载代码数量的字段（Figure 11-37）。

![9781430248750_Fig11-37.jpg](img/9781430248750_Fig11-37.jpg)

Figure 11-37. 用于输入要下载的促销代码数量的表单

点击`Continue`后，你将看到一份管理代码使用的`App Store Volume Custom Code Agreement`，然后是一个用于下载代码的按钮。点击此按钮将下载一个包含你请求数量的文本文件（Listing 11-1 显示了一些示例促销代码），并且它将通过电子邮件向你发送一份你刚同意的协议副本（如果这听起来很别扭，它类似于 Apple 在确认复选框旁的措辞“同意该协议”）。

**Listing 11-1.** 包含五个促销代码的促销代码文件

```
XY9LJRKMN7JM
TANFXAHR9EM9
XP6TMAFFHFLT
TXJ44ELTHETL
F9W3WEK7JL4Y
```

下载后，代码有效期 28 天，即使在此期间应用进行了更新。

**提示** 由于代码的过期日期基于其下载日期，你可以根据需要从 iTunes Connect 下载代码来节省使用量，而不是一次性下载全部 50 个。
