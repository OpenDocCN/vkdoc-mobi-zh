# 游戏启动/暂停菜单与 Unity iOS 开发

## 启动/暂停菜单脚本

最终完成的启动/暂停菜单脚本比记分板脚本要长得多，因此此处省略了完整代码。本章已展示了所有代码，完整的`FuguPause.js`脚本可在本章的项目文件中获取，项目地址为`http://learnunity4.com/`。

## 探索更深层次

最终，记分板显示让我们的保龄球游戏成为了一款功能完整的保龄球游戏，而添加启动/暂停菜单则提供了玩家所期望的一些打磨效果！

游戏 GUI 的添加不仅完成了保龄球游戏（在商业游戏项目中，这可能会被视为“第一个可玩”里程碑），同时也标志着我们对 Unity 中常用于 3D 游戏的基本功能介绍的结束。

**提示** 虽然我将 GUI 开发留到了这一阶段的末尾（这在游戏开发中很常见，但很不幸），但最好在项目早期就纳入 GUI 设计。即使只是草拟菜单，也有助于明确预期的游戏模式和选项。

## 跨平台特性与 iOS 开发目标

大多数情况下，尽管你一直仅在编辑器中修改和测试游戏，但这些功能都是跨平台的。现在，你可以将此游戏构建为网络播放器、Mac 或 Windows 可执行文件，并且游戏在所有平台上的运行效果都一样（性能差异除外）。但本书的最终目标是让你进入 iOS 开发领域，因此，从下一章开始，本书的剩余部分将全部围绕 Unity iOS 展开。

#### Unity 手册

Unity 手册中“创建游戏玩法”部分的“游戏界面元素”链接指向了“GUI 脚本编写指南”，这是一系列教程风格的页面，详细介绍了 UnityGUI 系统，从创建单个按钮到添加各种其他 UnityGUI 控件（如滑块和单选按钮），再到使用自动布局与固定布局、创建可复用的复合控件，以及通过样式和皮肤自定义 GUI 外观。

“GUI 脚本编写指南”还解释了如何使用 UnityGUI 自定义 Unity 编辑器。事实上，编辑器界面本身就是用 UnityGUI 实现的（这就解释了为什么有时你会在控制台视图中看到与你的代码无关的`OnGUI`错误信息）！

#### 参考手册

我简要提到了可以将除内置 Unity 字体之外的其他字体导入到 Unity 项目中，然后分配给 UnityGUI 样式和皮肤。参考手册在其资源组件部分有一个页面，更详细地描述了字体资源及其导入选项。

#### 脚本参考

自然，脚本参考中的运行时类列表包含了你所使用的 UnityGUI 函数的说明页面，首先是`MonoBehaviour`类中定义的`OnGUI`回调。

在记分板和暂停菜单中使用的大多数 UnityGUI 函数都是用于创建各种 UnityGUI 控件的静态`GUI`和`GUILayout`函数，例如`GUI.Button`和`GUI.Label`。值得逐一查阅每个函数的脚本参考页面，这样你就能了解有哪些 GUI 控件可用，以及自定义变量如`GUI.color`和`GUI.skin`（还有其他的，比如用于自定义背景色）。`GUIStyle`和`GUISkin`也是类，你可以在脚本中随时更改它们的属性。

除了`UnityGUI`类，暂停菜单中还使用了一些其他类。静态变量`Time.timeScale`被用于暂停和恢复游戏，调用了`Application.Quit`来退出游戏，并在选项页面中访问了`AudioListener`、`QualitySettings`和`SystemInfo`类。

#### 资源商店

Necromancer GUI 在暂停菜单中看起来很棒，展示了精心制作的 GUISkin 能让 UnityGUI 变得多么美观，但资源商店上还有很多其他皮肤。它们与 Necromancer GUI 一起，都列在“纹理和材质”下的“GUI 皮肤”分类中。该分类中包含了大量用于 UnityGUI 的皮肤，以及用于第三方 GUI 系统的皮肤，例如来自 Above and Beyond Software（`http://anbsoft.com/`）的流行 EZGUI，和来自 Tasharen Entertainment（`http://tasharen.com`）的 NGUI。

这些第三方 GUI 系统可在“脚本”分类下的“GUI”子分类中找到，该子分类中充满了预先编写好的 GUI，如小地图和菜单。事实上，本章实现的暂停菜单的一个版本可以在资源商店的“完整项目”部分中找到（名称为 FuguPause）。

资源商店在“纹理和材质”的“字体”分类下也有相当多的字体可供选择。但由于 Unity 可以导入 TrueType 和 OpenType 字体，你还可以访问网络上大量的免费字体网站，以及价格适中的字体库（我使用的是 MacXWare 字体库）。
