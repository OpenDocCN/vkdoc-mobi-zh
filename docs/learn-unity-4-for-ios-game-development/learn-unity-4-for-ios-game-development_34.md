# 引言

## 我是如何遇见 Unity 的

严格来说，我第一次编程是在初中图书馆的 TRS-80 上，但实际上我只是每天从杂志上输入同样的 BASIC 代码清单，直到图书管理员提到我可以用磁带保存程序。回想起当我听说图书馆有电脑时的第一反应，我感到很尴尬：“它有什么用？”

一年后，当我拿到 Apple II 时，我豁然开朗。翻开用户手册，学会如何用 BASIC 绘制图形后，我就彻底迷上了。很快我就开始编写黑白棋游戏（一个用 BASIC，一个用 6502 汇编），甚至还有一个 3D 线框显示程序。

在随后的几年里，我游走于 Windows 的荒野，在大小团队中从事计算机图形和游戏开发工作。但快进到六年前，我愉快地重新拥抱了苹果阵营（现在带 Unix 了！），并参加了我的第一次苹果全球开发者大会。Unity Technologies 的联合创始人兼首席技术官约阿希姆·安特为我即兴演示了 Unity 1.5，这正是我一直在寻找的——一款价格低廉的 3D 游戏引擎，能在 Mac 上运行，并能针对包括 Windows、Mac 和网页浏览器在内的多平台发布。

因此，我一回到家就购买了一个 Unity 1.5 Indie 许可（那时 Unity Indie 还不是免费的），后来升级到了 Unity Pro，并在几年后惊喜地发现 Unity 将支持 iOS。当计划实现时，我真是爱死这种感觉了！

与此同时，我曾任职的海波娱乐公司授予我许可，将 HyperBowl（一款我十多年前参与的 3D 街机保龄球游戏）移植到 Unity 以及 Unity 支持的各种平台，这样除了我一直在实验的小型应用外，我有了一个相当有分量的项目可以着手。

我花了六个月时间让 HyperBowl 重制版的首个版本作为 Unity 网页播放器运行，生成了独立的 Mac 和 PC 可执行文件，并在 iPhone 上运行（此外，顺便说一下，还有 Android、Linux 和 Flash 版本）。而且，如果减去我摸索如何从原版游戏中提取美术和音频资源所花的时间，我实际上只用了三个月使用 Unity。

随着每个新版本的 Unity 发布，HyperBowl 和我的其他游戏运行得更快、画面更好，利用了更多 iOS 特性，并在更多 iOS 设备上运行（最终也包括 Android 设备）。我通过第三方插件增加了新功能，添加了一个新的暂停菜单，甚至利用 Unity 资源商店中的资源包创建了一整条全新的 HyperBowl 球道（关卡），而资源商店已方便地集成在 Unity 编辑器中。

这一切都是由一个开发团队（一人）完成的（不算所有投入在原版授权资源上的工作），而且我无需学习任何一行 Objective-C 代码或自己创作美术。从某种意义上说，我感觉自己回归了编程的本源，为自己的项目而工作，出于乐趣，并且额外还能赚钱！希望我能将自己过去六年使用 Unity 的经验（无论是错误还是成功）浓缩到这本书中。

## 关于本书

任何游戏开发书籍都面临一个问题，即试图满足所有人的所有需求；游戏开发、Unity 和 iOS 领域中有大量主题，它们本身就能轻易占据整本书的篇幅。本书是一本介绍如何使用 Unity 为 iOS 开发游戏的入门书，因此我的目标是让每位读者都能熟悉 Unity，并平稳地进入 Unity iOS 开发领域，遵循我多年来走过的相同路径（但时间更短！）
