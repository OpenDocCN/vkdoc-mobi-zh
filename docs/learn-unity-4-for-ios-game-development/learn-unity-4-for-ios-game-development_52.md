# 进一步探索

现在我们有了真正游戏玩法的雏形。虽然它还不是一个完整的保龄球游戏，但它包含了物理（球在地板上滚动）、交互控制（推动球）和摄像机移动（跟随球）。在游戏开发中，每个特性都是丰富的主题，但我们将在下一章中更深入地探讨物理，包括保龄球瓶的添加和基于碰撞的音效。届时，游戏看起来会更像一个保龄球游戏！

#### Unity 手册

在“资源导入与创建”部分，关于“程序化材质”的页面（你在地板上使用过）简要介绍了 Substance 存档格式，并概述了用于创建和分析程序化材质的工具。

在“创建游戏玩法”部分，关于“输入”的页面描述了 `Input` 类及其 `GetAxis` 函数（在本章实现的 `FuguForce` 脚本中使用），并列出了传递给该函数的摇杆和键盘值。该页面的下半部分描述了适用于 iOS 的 `Input` 函数和变量，你可以阅读这部分作为本书关于 iOS 输入章节的预览。

本章主要讨论物理，因此“物理”页面是最值得阅读的。它列出了所有碰撞器类型，包括本章中使用的 SphereCollider、BoxCollider 和 MeshCollider 组件。该页面描述了刚体（球）和静态碰撞器（地板）之间的区别，以及物理材质及其属性如何影响碰撞行为。

#### 参考手册

在“资源组件”部分，关于“程序化材质资源”的页面解释了你在项目视图和检视视图中看到的程序化材质内容。

在“物理组件”部分，详细列出了所有可用的碰撞器，以及可用的关节（包括铰链和弹簧）、恒力和布料组件（Unity iOS 不支持布料）。

本章引入了一个新的设置管理器：“标签管理器”，用于为地板创建新标签。

在“脚本概念”部分，标题为“什么是标签？”的页面详细说明了如何创建标签并将其应用于游戏对象，并提供了一个示例，展示如何在运行时使用标签查找游戏对象。

关于“刚体休眠”的页面解释了刚体在静止时如何“进入休眠”状态。这是一个很好的优化，可以减少不必要的物理计算。

#### 脚本参考

在运行时类列表中，“MonoBehaviour”类的页面列出了本章介绍的碰撞回调函数：`OnCollisionEnter`、`OnCollisionStay` 和 `OnCollisionExit`。

我们查询了 `Collision` 类以获取碰撞信息：相对速度、碰撞到的游戏对象。还有其他数据可用，例如实际接触点。

本章介绍了 `Rigidbody` 组件，这个类的脚本参考页面值得通读。它的变量很大程度上对应检视视图中可用的属性，除了你用来推动球的 `AddForce` 函数外，还有许多相关函数：`AddRelativeForce`、`AddTorque`、`AddRelativeTorque`、`AddExplosionForce`、`AddForceAtPosition`。

`Component` 有几个变量，它们是引用附加到游戏对象上的其他组件的快捷方式。在本书迄今为止创建的两个脚本中，`FuguRotate` 脚本引用了 `transform` 变量，`FuguForce` 脚本引用了 `rigidbody` 变量。Component 的变量与 `GameObject` 中的同类变量相对应。

我们使用的 `Input` 类中的一个函数是 `Input.GetAxis`。如果你计划制作任何桌面或网页游戏，你会想了解用于查询键盘、鼠标和摇杆的各种函数。该页面还列出了适用于移动应用程序的 `Input` 函数。

#### 网络资源

Unity 中的物理实现基于较旧版本（2.x）的 PhysX 引擎。该引擎最初由 NovodeX 开发，被 Ageia 收购后与硬件物理加速器捆绑在一起，现在是 nVidia 的产品。最新版本（3.x）可在 nVidia 开发者专区获取（`https://developer.nvidia.com/physx`）。值得一看 PhysX 软件开发工具包，以便更好地理解 Unity 物理组件和脚本函数所依赖的物理引擎特性。

Unity 网站（`http://unity3d.com/`）上相当新的“学习”部分包含大量物理教程，涵盖了本章讨论的大部分内容以及下一章将要讨论的内容。

## 第 7 章：开始保龄球吧！高级物理

到目前为止，我们已拥有了一个保龄球游戏的雏形，或者至少是 Unity 编辑器中的一个场景，玩家可以在其中让球在地板上滚动。但如果没有至少保龄球瓶，游戏看起来就不像保龄球游戏。本章将弥补这一点，并涉及更多 Unity 物理知识，包括刚体（球和球瓶之间）之间的碰撞以及复合碰撞器（以适应保龄球瓶的形状）。在此基础上，我们还将添加基于碰撞的音效，让游戏听起来也像保龄球游戏。
