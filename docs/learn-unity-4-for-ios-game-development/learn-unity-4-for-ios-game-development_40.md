# 进一步探索

本次 Unity 导览以及 Angry Bots 演示的试玩到此结束。从下一章开始，您将从零开始创建 Unity 项目，以学习游戏引擎的功能。但这不会是您最后一次见到 Angry Bots，我们将在第 10 章和第 11 章中再次回到该项目，重复这一过程来介绍 Unity iOS 开发。在此之前，从下一章开始，您将从头开始创建 Unity 项目，并探索通用的、大多是跨平台的 Unity 游戏引擎功能。

从现在开始，有许多官方 Unity 资源可以扩展我将要涵盖的主题。

#### Unity 手册

如您所见，Unity 的用户界面非常丰富，我们几乎还远未覆盖其全部内容。现在是认真阅读 Unity 手册的好时机，无论是通过 Unity 编辑器内部（欢迎屏幕或帮助菜单），还是通过 Unity 网站（`http://unity3d.com/`）上“文档”部分的“学习”选项卡。当您想查询某些内容或仅阅读关于 Unity 的知识而无需运行 Unity 编辑器时，网页版非常方便。

本章所涵盖的大部分内容都与 Unity 手册中“Unity 基础知识”部分的主题相匹配，特别是“学习界面”、“自定义工作区”、“发布构建”和“Unity 热键”等部分（尽管我认为更好的参考方法是直接查看菜单中列出的键盘快捷键）。

我还提前涉及了 Unity 手册的“高级”部分，并谈到了 Unity 对版本控制的支持。Unity 手册中关于“在 Unity 中使用外部版本控制”的页面对此有更深入的介绍。

#### 教程

除了“文档”部分，Unity 网站的“学习”选项卡还包含一个“教程”部分，其中提供了一系列丰富的“编辑器入门”视频。顾名思义，这些视频介绍了 Unity 编辑器，事实上，这套视频涵盖了本章讨论的大部分内容，包括最重要视图（`Game View`（游戏视图）、`Scene View`（场景视图）、`Hierarchy View`（层级视图）、`Inspector View`（检视视图）和`Project View`（项目视图））的描述，甚至包括发布构建的过程。

#### 版本控制

虽然我仅在解释如何移除 meta 文件的背景下简短地讨论了版本控制，但该主题值得稍加深入讨论，因为 VCS 对软件开发至关重要（当您第一次丢失项目或无法回忆起哪个更改导致游戏出错时，您就会意识到这一点！）。如果您已经有偏好的 VCS，您可能想将其与 Unity 一起使用；如果您尚未使用任何 VCS，那么您可能需要考虑使用它，至少是为了在需要回滚时保留项目的旧版本，并具备检查版本间差异的能力。

在版本控制系统中，`Perforce` 是游戏工作室中流行的商业工具，而 `Subversion` (`svn`) 作为一种开源选项拥有悠久的历史。如今，像 `Git` 和 `Mercurial` 这样的分布式版本控制系统正成为一种趋势。我在 `Bitbucket` (`http://bitbucket.org/`) 上使用 `Mercurial` 管理我的内部项目，并在 `GitHub` 上发布公共项目，包括本书的项目 (`http://learnunity4.com/`)。

说 Unity VCS 支持与具体产品无关，实际上是说 Unity 在 Unity 编辑器中并没有集成任何特定的版本控制系统（尽管在撰写本文时，`Perforce` 和 `Subversion` 的集成正在进行中）。meta 文件以及面向 Unity Pro 用户的 YAML 场景文件，只是提供了与常用于源代码的文本导向型版本控制系统更好的兼容性。您仍然需要在 Unity 外部自行运行 VCS 操作。顺便提一下，您可以在 `http://yaml.org/` 上找到关于 YAML 的更多信息。

同时，我发现使用 GitHub 网站上提供的 Mac GitHub 应用程序以及 Bitbucket 网站上的 Sourcetree 应用程序非常方便。

正如我在解释编辑器首选项中“版本控制模式”选项时提到的，Unity 还提供了一款专门为 Unity 设计的 VCS，称为 Unity Asset Server。它需要购买 Unity 团队许可证。
