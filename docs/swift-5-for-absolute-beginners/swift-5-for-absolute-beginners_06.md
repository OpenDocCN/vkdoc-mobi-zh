# 6. 学习 Swift 和 Xcode

在大多数情况下，所有编程语言都会执行任何计算机所需执行的典型任务——存储信息、比较信息、对信息做出决策，并根据这些决策执行某些操作。Swift 语言使这些任务更容易理解和完成。使用 Swift 的真正技巧（实际上，也是大多数编程语言的技巧）在于理解用于完成这些任务的符号和关键字。本章将继续探讨 Swift 和 Xcode，以便你更加熟悉它们。

## 新手入门

你可能知道，Swift 诞生时间并不长。Swift 语言大约在四年前由 Chris Lattner 开始开发，并于 2014 年 9 月 9 日正式发布了 Swift 1.0。Swift 从 Objective-C 借鉴了许多思想，同时也融合了现代编程语言使用的许多特性。Swift 从设计之初就致力于让普通程序员也能轻松上手。

目前，主要有两种类型的编程语言。编译型语言如 Objective-C 和 C++，以其严格和需要特定语法而闻名。编译型语言的执行速度也明显更快。解释型语言如 Ruby、PHP 和 Python，则以易于学习和编码而闻名，但执行速度较慢。Swift 是一种弥合两者之间差距的语言。Swift 融合了解释型语言广受欢迎的灵活性，以及要求苛刻的应用程序和游戏所需的性能。事实上，Apple 声称 Swift 应用程序的性能将优于用 Objective-C 编写的应用。在 Apple 的一些测试中，Swift 的性能几乎是 Python 的四倍，比 Objective-C 快 40%。

## 理解语言符号

理解符号是任何编程语言的基础。符号是用于在源代码中表达特定含义的标点符号。要使用一门语言，就必须理解其符号。以下是 Swift 中使用的一些符号和语言结构，其中大部分你都已经以某种方式接触过了：

*   `{`：这是**开始**大括号。它用于开始通常所说的***代码块***。代码块用于定义和包围一段代码，并界定其作用域。
*   `}`：这是**结束**大括号。它用于结束一个代码块。只要有开始大括号（`{`），就必须有一个配套的结束大括号（`}`）。
*   `[]`：这是开括号和闭括号。它们用于数组的声明和使用。
*   `func methodName() -> String`：这是 Swift 中定义函数的方式。当然，`methodName` 一词可以代表任何名称。`String` 一词也可以改变。它表示该方法返回的信息类型。在这个例子中，`String` 表示该方法将返回一个字符串，或一组字符（数据类型在第 3 章中介绍过，并将在后续章节中更深入地讲解）。本章稍后将更详细地讨论这一点。

## 在 Swift 中实现对象

Swift 从底层开始构建时就面向对象。它融合了 Objective-C 的最佳部分，同时摆脱了与 C 兼容的限制。它还吸收了脚本语言的一些最佳特性。以下是使 Swift 面向对象的一些概念。如果其中一些术语看起来不熟悉，请不要担心；它们将在后续章节中讨论（第 7 章和第 8 章将介绍基础知识）。

*   几乎所有东西都是一个*对象*。
*   对象包含***实例***变量。
*   对象和实例变量具有定义的*作用域*。
*   类隐藏了对象的*实现*。

### 注意

正如你在第 5 章中看到的，术语***类***通常用于表示对象的定义或类型。***对象***是从类创建出来的。例如，SUV 是车辆的一个*类*。类就像是一种蓝图。工厂生产 SUV。结果是人们驾驶的 SUV 对象。你不能驾驶一个*类*，但你可以驾驶一个从类构建的*对象*。

那么，这些概念如何转化到 Swift 中呢？Swift 在类的实现方面非常灵活。



### 注意

尽管在 Swift 中，单个文件可以包含多个不同的类，但程序员通常会将代码拆分到不同文件中，以便于访问。

### 注意

实例化会将类变成计算机内存中一个真实的对象。类本身在没有实例化之前是无法实际使用的。以 SUV 为例，在工厂生产出一辆（实例化该类）之前，SUV 毫无意义。只有在那之后，SUV 才能被使用。
