## 关于本书

《SwiftUI 和 Combine 异步编程》旨在提供一个实用指南，用于使用苹果提供的最新 API 和工具包构建 UI。它首先介绍`SwiftUI`作为一个现代化且易于使用的框架，用于以声明式方式构建响应式 UI。在涵盖了您在`SwiftUI`中构建高效用户界面需要理解的基本概念之后，本书深入探讨了函数响应式编程，以及如何使用`Combine`解决开发者在构建与异步后端服务交互的、以 UI 为主的应用程序时面临的许多挑战。最后，本书介绍了 Swift 新的结构化并发特性、它们与`Combine`的关系，以及何时应该使用其中之一。

## 谁应该阅读本书

《SwiftUI 和 Combine 异步编程》面向所有使用`SwiftUI`构建应用程序的人。

如果您有一些`SwiftUI`的经验，并希望更好地理解其状态管理是如何工作的，以及如何通过使用视图模型和`Combine`更高效地构建应用程序，那么本书可能适合您。

## 本书的组织结构

本书分为三个主要部分，每部分涵盖在苹果平台上构建响应式 UI 的不同方面。

### 第 1 部分

第 1 部分涵盖`SwiftUI`及其状态管理。您将学习如何使用`SwiftUI`的声明式方法构建用户界面，以及`SwiftUI`的响应式状态管理如何使编写即使在多个屏幕之间也能保持应用程序状态同步的应用变得更加容易。

- **第 1 章** 解释了为什么`SwiftUI`是苹果公司简化应用开发战略中的一个重要构建块。它还对本书后续章节将详细介绍的一些关键概念进行了快速介绍。

- **第 2 章** 引导您从头开始构建一个简单的应用程序。在本章中，您将学习一些关键的`SwiftUI`概念，以及一些保持代码可维护性的有用技巧。

- **第 3 章** 深入探讨了`SwiftUI`的关键概念，如视图、视图修饰符和布局行为。

- **第 4 章** 涵盖了`SwiftUI`的核心内容之一：状态管理。它解释了在`SwiftUI`应用中管理状态的不同方式，并提供了何时使用哪种方法的实际示例。

- **第 5 章** 深入探讨了最常见的 UI 类型之一：列表。`SwiftUI`为构建`List`视图提供了一个非常灵活和强大的 API，本章将为您指导如何使用它。

- **第 6 章** 解释了如何构建另一种非常流行的 UI 类型：（输入）表单。

### 第 2 部分

第 2 部分重点介绍`Combine`，苹果的响应式框架。您将学习`Combine`的工作原理，以及如何将其用于 UI 和与后端交互的代码。

- **第 7 章** 介绍了函数响应式编程和`Combine`的关键概念。您将了解什么是发布者、订阅者和操作符，以及如何使用它们随时间处理事件。

- **第 8 章** 解释了如何使用`Combine`实现复杂的 UI。您将学习如何使用多个`Combine`管道来驱动具有多个输入字段的输入表单的 UI 状态。

- **第 9 章** 展示了如何使用`Combine`访问网络。然后，您将学习如何将其与前面章节构建的管道集成，以构建更复杂的输入验证管道。

- **第 10 章** 涵盖了错误处理这一重要主题，并将为您提供一系列逐步更强大的策略，用于处理错误并以有意义的方式引导用户进行错误缓解。

- **第 11 章** 延续第 10 章的内容，将演示如何构建一个自定义的`Combine`操作符来实现增量退避——一种特别适用于访问网络资源的错误处理策略。

- **第 12 章** 教您如何将现有 API 封装到`Combine`中，以便在`Combine`管道中使用它们。作为一个案例研究，本章将封装一些 Firebase 的异步 API，以演示两种在`Combine`中封装 API 的常见策略。

- **第 13 章** 解释了什么是调度器，以及它们如何帮助您以声明式方式编写多线程代码。

### 第 3 部分

第三部分也是最后一部分，涵盖了 Swift 新的结构化并发（更广为人知的是`async/await`）、它与`Combine`的关系，以及如何使用它来构建感觉流畅且能够与`URLSession`等异步 API 以及其他诸如`Firebase`的异步 API 交互的应用。

- **第 14 章** 介绍了异步编程，涵盖了传统方式（使用闭包）和`async/await`。



*   第 15 章 介绍了如何将 Swift 的新并发 API 与 SwiftUI 结合使用，特别涵盖了为此任务新增的自定义视图修饰符（例如 `task`、`refreshable` 和 `searchable`）。^(³)
*   当 Apple 发布 Swift 新的结构化并发模型时，人们对于是应该继续在应用中使用 Combine，还是转而使用 async/await 感到困惑。第 16 章 重点讨论了这个问题，并就何时使用哪种技术提供了一些指导（剧透：这取决于你的具体用例）。
