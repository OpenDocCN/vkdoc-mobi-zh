# Let

`Let`函数在执行业务处理内嵌计算语句之前，初始化一个或多个变量。该函数接受两个表达式作为参数。*变量声明*参数由一个或多个变量名组成，每个变量名带有一个生成其值的表达式，变量名与表达式之间用等号分隔。变量可以是任何类型的混合：*语句*、*局部*或*全局*（第 12 章，“变量类型”）。每个变量的表达式可以是字面量值、函数调用或生成任意类型结果（将成为对应变量的值）的表达式。*计算*参数可以是任何能产生整个语句结果的表达式。

```
Let ( variable = expression ; calculation )
```

当初始化多个变量时，必须在声明参数周围添加方括号，并在每个*变量-表达式*对之间添加分号。

```
Let ( [
variable = expression1 ;
variable2 = expression2 ;
variable3 = expression3
] ;
calculation
)
```

`Let`函数在构建复杂公式时非常有用。冗长的字段名可以压缩为短变量，以便在公式中多次使用时节省空间。由许多嵌套函数或子句组成的复杂表达式可以分离为一系列较短的表达式，这些表达式逐步生成变量的值，从而更易于阅读和编辑。这还可以消除对大型相关记录集执行相同操作（例如，多次对大型相关记录集使用`Sum`函数）带来的负面性能影响。

举一个简单的例子，对`Contact::Contact Last Name`字段的引用可能在`Case`语句中被多次使用，每次都重复完整的字段引用。通过`Let`语句，该字段的值可以插入到一个名为`lastName`、`name.last`或仅`last`的短变量中，并在原位置使用该变量。以下两个示例说明了这一点，它们构建了一个三级文件夹路径，该路径使用姓氏的首字母、姓氏的前三个字母以及姓氏本身，并在未指定姓氏时使用占位符；例如，“Munro” 将变为 “M:Mun:Munro:”。以下是在公式中重复多次看到对同一字段的四个引用以产生此结果：

```
Case (
Contact::Contact Last Name = "" ; "Missing Name:" ;
Left ( Contact::Contact Last Name ; 1 ) & ":" &
Left ( Contact::Contact Last Name ; 3 ) & ":" &
Contact::Contact Last Name & ":"
)
```

或者，使用`Let`语句，通过一次字段引用将字段的值放入一个短变量中。然后，如果该变量为空，则将其替换为缺失占位符。最后，使用它来构造路径。

```
Let ( [
last = Contact::Contact Last Name ;
last = Case ( last = "" ; "Missing Name:" ; last )
] ;
Left ( last; 1 ) & ":" & Left ( last; 3 ) & ":" & last & ":"
)
```

前面的示例使用了*级联声明*，它使用相同的变量以逐步连续的方式为一个变量建立值，从而将复杂的语句分隔为更易读的格式。在前面的示例中，`last`变量首先被设置为字段名，然后在第二个声明步骤中（如果为空）重置。这允许将复杂的嵌套语句分隔为更易于阅读的步骤。在以下示例中，四个函数嵌套在一行中，可能非常难以阅读，尤其是在更复杂的公式中。从内向外看，此示例首先从`Notes`字段获取内容，`GetValue`函数从中提取第一段，然后使用`TextFormatRemove`函数消除所有格式，最后使用`Substitute`函数将破折号替换为句点。

```
Substitute ( TextFormatRemove ( GetValue ( Contact::Notes ; 1 ) ) ; "-" ; "." )
```
