# 审视产品线

FileMaker 平台，如表 [1-1]() 所示，由多个应用程序组成，每个应用程序都旨在满足开发、使用和共享数据库中的特定功能。

**表 1-1** FileMaker 产品线中各产品摘要

| 产品名称 | 平台 | 开发 | 使用 | 共享 |
| --- | --- | --- | --- | --- |
| FileMaker Pro | Mac, Windows | 是 | 是 | 有限 |
| FileMaker Server | Mac, Windows | 否 | 否 | 是 |
| FileMaker Cloud | 基于 AWS 的 Linux | 否 | 否 | 是 |
| FileMaker Go | iOS 设备 | 否 | 是 | 否 |
| FileMaker WebDirect | Web 浏览器 | 否 | 是 | 否 |



### 使用 FileMaker Pro 创建数据

产品线中的旗舰产品是桌面级应用程序，也是本书的重点，即 `FileMaker Pro`。该应用程序是传统产品的现代化身，它提供了一个多用途界面，可用于在 macOS 和 Windows 电脑上`创建`、`共享`和`访问`数据库文件。它是`唯一`包含定义数据模型、设计界面布局和编写脚本所需的所有结构开发工具的产品。它将前端、后端和中间件整合到一个无缝的低代码应用程序编程接口中。

该软件通过熟悉的桌面文档格式提供对数据的前端访问，该格式可显示界面布局（第 3 章）。图形化的模式编辑器使定义表、字段和关系变得容易（第 7 章 – 9 章）。布局编辑器允许快速开发丰富的界面，以呈现和整合表内容及界面对象（第 17 章 – 22 章）。脚本编辑器巧妙地平衡了对象驱动构建和公式代码，使得创建可重用的自动化事件序列变得简单（第 24 章 – 27 章）。凭据管理使其能够轻松创建安全账户，通过使用各种内部或外部身份验证方式来限制用户在文件内的访问和活动（第 30 章）。

许多高级工具提供了更强大的控制能力。构建一个可在文件中任意位置重用的自定义函数库（第 15 章）。用自定义菜单覆盖整个菜单栏，从而完全掌控用户体验（第 23 章）。脚本调试器有助于识别问题并评估性能（第 26 章）。将整个数据库保存为 XML，轻松生成设计报告，并使用 AES 256 位加密对数据库进行加密，以保护网络数据库（第 31 章）。

数据可以通过多种方式进行共享和集成。用户和脚本可以在数据库和基于文本的数据格式之间轻松地导入和导出数据（第 5 章）。网络共享允许多个工作组成员实时同时访问同一个数据库（第 29 章）。提供了多种集成选项，可通过 AppleScript (macOS)、Claris Connect、动态数据交换 (Windows)、FileMaker API、JDBC、ODBC、SQL 等方式连接到外部系统。

> **提示**  
> 在应用程序偏好设置的`通用`选项卡下启用`使用高级工具`（第 2 章），以访问所有高级功能。

### 使用 FileMaker Server 与 Cloud 进行共享

有两款专用产品用于在网络上托管数据库以供团队共享：`FileMaker Server` 和 `FileMaker Cloud`。

`FileMaker Server` 是一款数据库托管软件包，可为授权用户提供可扩展、安全且可靠的 24/7 全天候数据库访问。托管文件可以被 FileMaker Pro、FileMaker Go 和 FileMaker WebDirect 的多个用户同时访问，或者通过使用托管数据库作为后端数据源的其他系统进行访问。该服务器管理用于运行数据库备份、系统级脚本（shell 脚本或批处理文件）以及在托管数据库中运行 FileMaker 脚本的事件调度。可以配置单个数据库脚本在服务器上运行，而不是在用户计算机上运行，以卸载任务并节省时间。管理员可以访问事件和错误日志以排查问题，或查看性能指标以帮助优化数据库性能。该服务器使用渐进式下载技术来流式传输媒体内容，实现即时显示，无需等待。

`FileMaker Cloud` 是 Claris 管理的 FileMaker Server 云托管版本，具有类似的功能，外加全天候监控与支持，并且增加了远程访问的易用性。

> **注意**  
> 尽管本书侧重于使用和创建数据库，但第 29 章将介绍网络共享。

### 使用 FileMaker Go 与 WebDirect 进行访问

有两款产品允许使用或访问在桌面应用程序中开发的数据库：`FileMaker Go` 和 `FileMaker WebDirect`。

`FileMaker Go` 是一款仅限 iOS 系统的应用程序，可从 Apple App Store 免费获取，它允许 iPhone 和 iPad 用户在离开桌面电脑时访问数据库。用户可以打开存储在 iOS 设备本地的数据库，或者通过 Wi-Fi 或蜂窝网络从 `FileMaker Server` 或 `FileMaker Cloud` 服务器远程访问数据库。可选地，为桌面设计的数据库可以通过自定义脚本进行设计，以检测用户的移动设备并切换到专为较小屏幕和触摸导航设计的布局。计算和脚本可以利用 iOS 技术，包括通过相机进行媒体捕获、通过触摸屏进行签名采集、条形码扫描、触控/面容识别等。数据库可以访问关于设备电量、位置、姿态、气压、加速度、磁航向、步数等信息。

`FileMaker WebDirect` 是网络上共享数据库最简单的方法。只需进行一些设置，任何数据库都可以允许通过网页浏览器进行访问，而`无需`使用像 PHP、HTML5、CSS 或 JavaScript 这样的编码工具。在前端网页中呈现数据库所需的所有网页编程均由 `FileMaker Server` 或 `Cloud` 自动处理。布局会自动在浏览器中渲染，并且除了一些不兼容的脚本步骤和无法在多个窗口中操作外，大多数功能与在桌面或 iOS 设备上相同。
