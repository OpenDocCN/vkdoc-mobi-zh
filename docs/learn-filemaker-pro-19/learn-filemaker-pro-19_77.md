# 设计自定义主题  

自定义主题设计的方法会受到多种因素的影响。两个主要因素是 FileMaker 的具体使用经验和对界面设计原理的普遍了解。熟悉这两者的人可以利用这些经验，在设计界面愿景时专注于创新。经验较少的人则需要从他人作品中汲取灵感，或进行更多的试探性实验。其他因素包括计划构建的解决方案类型、目标部署平台、客户期望以及预算限制。没有一种方法能够适用于每种情况下的每一位开发者，在具体项目之外规划主题可能颇具挑战。虽然不意味着存在一刀切的方法，但在开始阶段还是有一些具体事项值得思考。  

尝试将主题设计规划与任何过去或现在的具体项目分离开来，创建一个抽象的设计沙盒。这能让你专注于创建干净、原创的主题，避免文件被无关负担填满。你仍应参考尽可能多的实际案例来激发灵感，确保你的选择基于真实项目，但又不被任何单一项目所束缚。从众多案例中选取最佳实践，进行比较、对比，然后重新整合为新的设计。  

沙盒文件应包含一两个半真实性的简单表格，以便在过程中使用，例如*联系人*和*公司*，或*发票*和*明细项*。以一个表格作为设计焦点，为其构建列表视图和表单视图布局，另一个表格则作为门户设计的相关表。仅在需要时为对话框、报表、iOS 设备等定义标准项目所需的样式时，才创建额外布局。请记住，目前要将字段和布局数量控制在最低限度，因为你并不是在设计实际解决方案，而只是一个*样式原型*。始终先专注于这一目标。  

开始添加对象并尝试不同的格式选项。重点考虑不同布局或不同功能所需的独特格式。例如，列表视图中的字段可能不可编辑，因此可以设置为透明填充和无边框；而表单视图中可编辑的字段，最好通过不同填充颜色等视觉提示表明其是可接受输入的字段。同样，用于布局标题、字段标签、信息提示、说明、报表页码等文本对象也需要单独的样式。思考你可能使用的各种按钮类型，例如标准按钮、仅图标按钮、带图标的文本按钮等。有些按钮可能有边框和填充，另一些可能只有带透明背景的图标——每种都需要单独定义样式。务必考虑字段的所有不同控件类型；每种类型都需要一个或多个定义的样式，因为不存在通用的*字段*样式类别，只有`编辑框`、`弹出菜单`、`日历`等。  

由于*任何格式差异*都需要单独的样式，字段和标签可能需要根据你构建的解决方案类型进行多种变体。例如，你可能需要为每种对齐方式（左、右、居中）以及每种外观（透明、带边框、填充）分别定义一种样式。花时间考虑不同的对象状态和窗口模式。大多数样式需要适应交互、在不同模式下正确提示信息等。尝试在这些目标之间找到合理的平衡点。一方面，保持样式列表简短以避免过于繁琐；另一方面，要提供足够多的选项，形成实用的设计调色板，避免因样式不足而作弊，对对象进行小的格式调整却不更新或创建新样式。  

在命名样式时，要在清晰描述用途和简洁易读之间取得平衡。名称在整个主题中必须唯一，无论对象类型如何，因此建议将对象类型作为名称前缀，例如，通过将按钮和字段样式分别命名为`按钮 带边框右对齐`和`字段 带边框右对齐`，避免冲突。`默认`样式无法重命名，因此将其作为每种对象类型最常用的样式选择，或者忽略它。  

当样式沙盒中包含足够多的样式，让你觉得能够提供坚实基础时，将主题迁移到实际文件中并开始用于真实开发。当你遇到未预料到的样式需求时，添加或增强主题。如果目的明确且不仓促行事，最终结果将是一个设计良好、实用的主题，可以导入并复用于其他文件。  

虽然这个过程听起来可能像是一项繁琐沉重的工作，但请记住，一个稳固的主题带来的好处最终会让时间投入获得良好回报。
