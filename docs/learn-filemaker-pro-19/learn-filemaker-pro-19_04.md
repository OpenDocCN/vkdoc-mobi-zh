# 适应集成全栈

FileMaker Pro 桌面应用程序采用集成架构，巧妙地将表示前端、流程逻辑中间件和数据存储后端组成的全栈结合在一起。与这些组件在物理上分离并使用不同语言构建的传统方法不同，FileMaker 将所有三层整合到一个熟悉的、基于文档的用户体验中，并提供一个易用的低代码开发环境。熟悉使用命令行语言的多层架构的经验丰富的软件开发人员最初可能会对这种安排感到困惑。他们甚至可能草率且不公正地将其视为仅适合初学者的工具而加以排斥。然而，FileMaker 专业地将易用性与强大的基于标准的技术相结合，为那些深入探究其表象的人提供了一个集两者之长的强大应用程序。尽管初学者也能上手，但它同样是一个非常强大且极其高效的开发工具。

FileMaker 为初学者平缓了学习曲线，同时也为高级用户提供了结构设计选项中惊人的灵活性。可以将数据库构建为自包含的解决方案，其中每个表、布局和脚本的完整资源栈都内置于一个文档文件中。以这种方式构建的解决方案可以像文档文件一样使用，像文字处理模板一样复制和修改，或者像电子表格文件一样传递给同事。它们可以存储在您的文档文件夹中供个人使用，或通过网络与其他用户共享。更复杂的解决方案可以从紧凑的统一设计中受益，但也可以选择突破文档容器的限制，成为由多个文件组成的模块化解决方案。每个模块都可以合并其他模块中的表，从而开辟了大量的设计选择。FileMaker 数据库还可以与其他 ODBC/JDBC 数据库系统交互，和/或充当其他系统的数据源。这意味着存在数量惊人的可能设计组合。

**注意：** 有关使用开放数据库连接 (ODBC) 或 Java 数据库连接 (JDBC) 将外部表添加到 FileMaker 数据库的信息，请参阅第 [7]()

在您开始 FileMaker 之旅或构思第一个数据库时，不要让这些选项的复杂性成为负担。一开始没有必要与它们纠缠。但是，要意识到将解决方案扩展到单个文件之外并超越本书示例的可能性。从一个包含少量表的单个文件开始，可以演变成各种不同的结构设计。一个文件可以与其他文件组合，拆分为数据分离模型（其中界面文件链接到单独的数据文件）等等。总而言之，可以使用以下任意组合来构建数据库解决方案：

-   自定义的单个、自包含、全栈数据库
-   链接在一起的、自定义的全栈数据库组
-   作为后端数据源的自定义数据库，由 FileMaker 数据库、网站或其他平台访问
-   作为前端界面的自定义数据库，合并来自 FileMaker 数据库或其他平台的外部表
-   提供特定数据或界面功能的自定义数据库，例如打印特定报告、存储历史数据、作为移动设备或 Web 浏览器的访问点等。
-   原样使用或自定义的 Claris 入门模板
-   从第三方开发者处购买的模块
-   本地存储、托管在网络服务器或云端的文件
