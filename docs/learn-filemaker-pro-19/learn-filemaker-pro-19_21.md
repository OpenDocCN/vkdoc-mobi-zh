# 规划字段名称

字段命名应遵循与数据表相同的一般准则：可以包含空格，应避免使用保留字和符号，命名应描述内容且不过长。也可以为其指定名称前缀，以便在开发者列表中分组排序。然而对于字段而言，这通常需要逆转自然语言的语序，在某些情况下可能会让用户感到困惑。例如，虽然 `Company Name`（公司名称）和 `Company URL`（公司网址）合乎逻辑，但使用 `Name First`和 `Name Last`代替 `First Name`（名）和 `Last Name`（姓）虽然能让字段在概念上保持列表分组，但可能显得不自然，并可能让用户感到困惑。虽然数据表名称大多对用户保持隐藏，但在执行诸如 `import`（导入）、`export`（导出）、`sort`（排序）等特定功能时，字段名称会更多地暴露出来。在布局上，字段标签可以重命名以避免混淆。然而，熟悉 `First Name`和 `Last Name`字段标签的用户，可能会对对话框里这些字段突然变成 `Name First`和 `Name Last`感到困惑。使用这种命名技巧时，可能需要创建由脚本驱动的流程来处理各种功能，以避免用户看到名称差异。或者，对用户进行名称差异的培训，使其不会感到困惑。
