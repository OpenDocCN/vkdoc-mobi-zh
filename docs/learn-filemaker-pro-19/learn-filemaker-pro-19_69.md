# 编写基本门户筛选公式

门户筛选公式由一个或多个条件语句组成，这些语句计算得出组合结果为 `true` (1) 或 `false` (0)。每个相关记录都使用此公式单独评估，并且仅当结果为 `true` 时才会出现在门户中。由于该公式在相关记录列表内运行，因此它已假定关系本身的记录匹配条件。因此，筛选公式无需重新指定该条件，也不能将结果扩展到该条件之外以包含非相关记录。例如，当门户的数据源匹配分配给当前公司记录的联系人记录时，我们可以按联系地址进行筛选，但无法使公式包含分配给其他公司的联系人。

编写筛选公式时，公式应包含一个条件，用于在未输入任何条件时显示所有记录。因此，请以以下代码开始公式，该代码在筛选字段为空时提供 `true` 结果：

```
Company::公司联系人门户筛选 = ""
```

接下来，在 `or` 运算符之后添加第二个条件，使用 `PatternCount` 函数，指定相关 `联系地址州` 字段与用户在筛选字段中输入的文本之间的部分或完全匹配。以下示例公式允许在未输入筛选值时 `or` 在州字段中找到输入的值时显示记录：

```
Company::公司联系人门户筛选 = "" or
PatternCount (
Company | Contact::联系地址州 ;
Company::公司联系人门户筛选
) > 0
```

保存公式并关闭设置对话框以测试功能。从一个在门户中包含大量联系人的公司记录开始。当在筛选字段中输入州缩写、提交记录并刷新窗口后，门户行应仅包含与输入文本匹配的记录。要使门户在用户输入筛选字段时自动更新，请使用 `ObjectModify` 脚本触发器（第 [27] 章）运行一个脚本，该脚本持续提交、刷新并重新进入筛选字段。
