# 1. 认识 FileMaker

`FileMaker Pro` 是一个用于为现代工作流创建关系型数据库应用的软件平台。该软件由苹果子公司 Claris International Inc. 发布，因其直观、低代码的编程界面而在新手程序员中广受欢迎，同时专业人士也青睐其先进技术、强大的定制选项、丰富的连接性以及可扩展的插件。其独特的集成架构将数据、逻辑和界面层的完整堆栈融合为无缝的编程体验。使用这款旗舰桌面应用，开发者可以协作创建安全、多用户、跨平台的解决方案，并快速部署到移动端、云端和本地工作流中。解决方案的范围可以从简单的电子表格式工作表，到设计精巧、功能丰富、以界面为驱动的方案。FileMaker 的用户包括独立顾问、小型企业员工，以及大中型企业、非营利组织和政府机构的团队成员。尽管它易于学习，但对于那些无法投入时间且没有雇佣开发团队的企业领导者来说，也能轻松找到专业顾问为其定制开发系统。无论技能水平如何，FileMaker 都是构建自定义数据库的绝佳选择。本章将介绍 FileMaker，内容包括：

*   数据库简介
*   FileMaker 发展简史

*   产品线概览

## 数据库简介

`数据库`是一种结构化的信息集合，以通用格式存储，便于各种用途的访问。现代世界充满了数据库，我们通过应用程序和网站不断与之交互。大多数数据库专注于一种特定数据类型，并具有预定义的属性和程序。例如，日历应用允许用户创建和管理具有`日期`、`时间`、`参与者`、`备注`和`提醒`等属性的`事件`。类似地，电子邮件应用允许管理具有`发件人`、`收件人`、`主题`和`正文`等预定义属性的`消息`。通俗地说，这些都可以被视为“事件库”和“消息库”，因为它们在其`基础`上存储并提供对`事件`和`消息`的访问。从更广阔的视角来看，事件和消息都是信息类型，它们都是`数据`的形式。日历应用访问并管理一个`事件数据库`。邮件应用则对`消息数据库`进行同样的操作。与这些以及无数其他为后台数据库提供前端访问的现代应用不同，一个`开放式数据库`应用没有预定义的数据类型；其核心是数据，`任何类型的数据`。像 FileMaker 这样的数据库应用允许开发者创建自定义解决方案，来管理他们定义的任何数据。

像日历或邮件这样的应用提供了一个预定义的数据库，就像隐喻中的文件柜，由供应商锁定结构并预配置以接受特定类型的信息。用户可以在这个预定义的框架中输入信息，但对框架本身几乎没有控制权。相比之下，自定义数据库的隐喻等同于同一个完全空置且解锁的文件抽屉，这样`你`就可以定义它接受的内容，并决定这些信息如何存储、关联、显示、使用和共享。每个项目都从同一张白纸开始，等待你来定义框架并建立可供用户使用的功能。因此，不要错误地认为 FileMaker 只是一个用于管理联系人或项目的系统。就像文字处理文档可以包含各种不同的书面内容一样，使用 FileMaker 构建的数据库可以存储关于开发者定义其管理的`任何`信息：`公司`、`联系人`、`库存`、`发票`、`消息`、`备注`、`人员`、`产品`、`任务`等等。

## FileMaker 发展简史

FileMaker 的早期历史在多个名称、发布商、平台和编号系统之间曲折发展，最终才稳定在现代的轨道上。

### Nashoba Systems

FileMaker 起源于 20 世纪 80 年代初，当时名为 Nutshell，是一款由马萨诸塞州康科德的 Nashoba Systems 为 MS-DOS 开发的计算机程序，并由电子产品营销商 Leading Edge 分销。当 Macintosh 计算机于 1984 年初推出时，Nashoba 看到了机会，将 Nutshell 的数据库引擎与图形用户界面相结合，创建了一个基于表单的数据库产品。由于 Leading Edge 希望保持仅做 DOS 软件的供应商身份，Nashoba 转而寻求新的分销商 Forethought, Inc.。因此，FileMaker 1.0 版本于 1985 年 4 月针对 Macintosh 平台发布。1986 年，为了与 Macintosh Plus 的发布相匹配，它被更名为 `FileMaker Plus`。当微软在 1987 年收购 Forethought 时，他们试图谈判收购 FileMaker，因为其销量超过了他们自己的 Microsoft File 数据库应用。然而，Nashoba 拒绝了，并开始自行发布该程序，此时命名为 FileMaker 4。以今天的标准来看，FileMaker 的这些早期版本非常原始。但在当时，它功能强大，软件通过提供易于使用的界面满足了重要需求，并在“自己动手”的群体中流行起来。

### Claris International

1986 年，苹果公司成立了全资子公司 Claris Corp.，用于开发和发布 Macintosh 软件产品，如 MacWrite 和 MacPaint。1988 年，Claris 收购了 Nashoba Systems 以获得 FileMaker。此时，随着其他 DOS 和 Windows 数据库产品主导市场，Leading Edge 和 Nutshell 已经消失。1988 年，Claris 发布了 FileMaker II，以匹配其其他产品的命名方案。经过几次小更新后，该产品于 1990 年以现代名称和版本格式重新命名，发布了 FileMaker Pro 1.0。Claris 于 1992 年通过支持 Windows 对该产品进行了升级，使 FileMaker Pro 2.0 成为第一个跨平台版本。他们于 1994 年开始发布服务器应用。直到 1995 年 3.0 版本发布，FileMaker 才完全具备关系型数据库功能。到 1997 年，随着 4.0 版本的推出，FileMaker 已成为广受欢迎的产品，销量超过了 Claris 的所有其他产品。苹果公司随后内部吸收了这些产品，并根据其唯一剩余的产品将子公司更名。

### FileMaker

1998 年，该公司更名为 FileMaker Inc.，以反映其新聚焦于数据库产品线。在随后的几十年里，他们将产品从早期的关系型数据库转变为一个功能丰富的开发平台。该产品原生支持了 Mac OS X，后来又支持了现代的 macOS。我们如今习以为常的许多功能都是在此期间添加的。例如，多表文件架构、在单个文件中打开多个窗口、计算变量、按钮、选项卡、门户、网页查看器、条件格式、内置函数和脚本步骤。他们通过添加脚本触发器、集成图表、过滤门户、主题、众多新的布局工具、递归自定义函数和自定义菜单，持续改进产品。近期新增的功能包括对 `JavaScript Object Notation` (JSON)、`client URL` (cURL) 和 `Structured Query Language` (SQL) 的支持。此外还有附加表、用于主-从布局的自查找表、自动目录创建、`FileMaker URL` 和 `FileMaker API` 等功能。这个时代见证了产品线的各种调整，包括桌面版、服务器版和移动版，并且版本编号方案转变为一个基于一致年度升级计划的整数。
