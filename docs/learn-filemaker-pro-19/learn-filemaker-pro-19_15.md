# 正确关闭文件

大多数文件损坏发生在数据库被错误关闭时。这可能导致数据丢失、数据损坏和/或结构缺陷。正确的关闭方法是关闭所有数据库窗口，或通过选择 *文件 ➤ 退出程序* 菜单来退出 FileMaker 应用程序。数据库可能被*错误关闭*的几种方式包括：

*   计算机电源中断，导致打开的数据库意外关闭。
*   系统级崩溃，同时也导致 FileMaker 崩溃。
*   导致 FileMaker 意外退出的故障。
*   当用户计算机进入休眠状态而托管的数据库仍打开时，网络连接断开。
*   在长时间脚本过程中，用户误将旋转的光标当作崩溃，从而仓促强制退出仍有数据库打开的程序。
*   为停止一个关闭了中止功能的、有缺陷的无限循环脚本而需要强制退出。
*   尝试打开或复制由 *FileMaker Server* 主动托管的实时数据库。

**提示**  
托管在 FileMaker Server（第 29 章）上的数据库可免受用户计算机崩溃和强制退出导致的损坏。本地运行的文件则更容易受损。
