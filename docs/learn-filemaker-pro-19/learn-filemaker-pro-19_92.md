### 遍历数据

*循环*也可用于逐条处理存储在变量中的数据。每次循环可对一个以回车符分隔的值列表中的某个段落或 JSON 对象的某个键执行操作，逐一处理直至完成。有几种不同的方式可控制此过程。一种方法是设置几个*控制变量*，一个用于存储值的*计数*，另一个用于跟踪*当前*值的序号，该序号在每次循环时递增。

例如，假设有一个州缩写列表，存放在名为 `$States` 的变量中。列表中的每个州都将用于构建一个查找条件请求，以搜索 `Contact` 记录。如果将所有州输入到单个查找请求中，则不会有任何匹配结果，因为这会指示 FileMaker 查找邮寄地址包含所有这些州的记录。为了获取结果，需要创建多个查找请求，列表中的每个州对应一个请求。以下脚本假设名为 `$State` 的变量包含一个缩写州列表，例如 `OH¶PA¶NY`。它使用 `ValueCount` 函数统计这些州的数量，并将该数值存入名为 `$Count` 的变量中，该变量将控制循环执行的次数。然后，它初始化另一个名为 `$Current` 的变量，以指示从列表中的第一个州开始。接着，执行 `Enter Find Mode` 步骤，并开始*循环*遍历州列表，为变量中的每个州创建一个查找请求。每次循环，`$Current` 变量增加 1，直到超过 `$Count`，循环退出。循环结束后，执行查找操作。

```
Set Variable [ $States ; "OH¶PA¶NY" ]
Set Variable [ $Count ; ValueCount ( $States ) ]
Set Variable [ $Current ; 1 ]
Enter Find Mode [ Pause : Off ]
Loop
Set Field [ Contact::Contact Address State ; GetValue ( $States ; $Current ) ]
Set Variable [ $Current ; $Current + 1 ]
Exit Loop If [ $Current > $Count ]
New Record/Request
End Loop
Perform Find [ ]
```

另一种实现相同目的的方法是，每次循环时使用列表中的第一个值，然后将其从列表中移除，重复此操作直到列表为空。这消除了需要控制变量来存储值数量和当前值的问题。在以下示例中，使用 `GetValue` 将第一个州插入当前请求。接着，`$States` 变量利用 `ValueCount` 和 `RightValues` 函数移除第一个值，保留剩余值供后续迭代使用。当 `$States` 变量为空时，`Exit Loop If` 步骤终止循环，然后执行查找。

```
Set Variable [ $States ; "OH¶PA¶NY" ]
Enter Find Mode [ Pause : Off ]
Loop
Set Field [ Contact::Contact Address State ; GetValue ( $States ; 1 ) ]
Set Variable [ $States ; RightValues ( $States ; ValueCount ( $States ) - 1 ) ]
Exit Loop If [ $States = "" ]
New Record/Request
End Loop
Perform Find [ ]
```

## 管理窗口

有许多脚本步骤可用于操作和管理窗口。`Adjust Window` 步骤可根据从五个选项中选择的一项来更改窗口的可见性或大小。`Resize to Fit` 设置会将窗口尺寸调整到最小可能尺寸，同时继续在当前显示器最大尺寸限制内显示整个内容区域。选择 `Maximize` 将使步骤将窗口大小调整为计算机屏幕的完整尺寸，而 `Minimize` 会将窗口缩小为存储在 Dock (macOS) 或任务栏 (Windows) 中的图标。要将窗口恢复到脚本开始时的尺寸和位置，请选择 `Restore`。最后，`Hide` 选项会将窗口从视图中隐藏，并放入 `Window ➤ Show Window` 菜单中，从那里可以重新打开它。`Arrange all Windows` 步骤可以按四种方式之一排列所有打开的窗口：`Tile Horizontally`、`Tile Vertically`、`Cascade` 或 `Bring All To Front`。其他步骤可以 `Close`、`Refresh` 或 `Scroll Window`。`Select Window` 步骤会将特定窗口置于前台，使其成为当前活动的上下文。通过 `Move/Resize Window`，脚本可以调整指定窗口的 `Height`、`Width`、`Distance from Top` 和 `Distance from Left` 尺寸。使用 `Set Window Title` 步骤可更改窗口的名称。

警告

打开一个没有关闭功能的模态窗口，并且没有按钮或脚本来关闭该窗口，将会困住用户，可能需要强制退出应用程序才能逃脱。

# 创建对话框布局脚本

对话框脚本可以通过多种不同的方式进行配置，具体选择取决于所需的功能。在某些情况下，脚本可以打开对话框，暂停，然后单击按钮时继续。在其他情况下，可以创建一组独立的脚本：一个用于呈现对话框，另一个用于每个按钮选项。下面的示例使用参数将所有操作打包到单个脚本的不同功能部分中。首先，创建一个名为“联系人 - 营销邮件”的脚本，如图 25-15 所示。该脚本以 `设置错误捕获` 步骤开始，然后将脚本参数放入 `$function` 变量中。它接受五个参数之一，以通过不同的条件语句引导工作流：`Start`（未提供参数时的默认值）、`Cancel`、`Introduction`、`Services` 和 `Offers`。

`Start` 参数将启动该过程，当用户从 `脚本` 菜单、自定义菜单、按钮或其他触发方法运行脚本时触发。条件 `If` 语句的第一部分允许使用“Start”参数或不使用参数，这会打开一个显示对话框布局的新窗口，为其分配一个名称，并在屏幕上水平和垂直居中。然后，它进入查找模式并将控制权交给用户，允许他们从弹出式菜单中选择一个州并做出按钮选择。其他四个参数将用于根据单击的四个按钮执行不同的条件步骤。因此，每个按钮都调用相同的脚本，但传入一个不同的参数，该参数会激活条件语句的不同部分。当脚本检测到 `Cancel` 参数时，它将进入浏览模式并关闭对话框，将用户返回到他们之前打开的窗口。三个报告按钮中的每一个都将在第一个 `Else` 步骤之后处理，该步骤包含一个嵌套的条件语句。在那里，`执行查找` 步骤执行搜索，如果找到记录，则遍历生成的找到的记录集。在 `循环` 语句内部，为三个参数中的每一个嵌套了一个条件，但为了简洁起见，省略了具体细节。三个注释指示了将在哪里添加额外的步骤来创建和发送电子邮件，或执行所需的任何操作。

## 介绍卡片窗口

`卡片` 是一个模式窗口，它作为一个覆盖层显示在当前窗口的活动布局之上，如图 25-17 所示。父窗口的内容在卡片打开时变得不活动且不可访问。但是，根据卡片的大小，其背后的内容可能仍在可选地变暗后可见。与对应用程序来说是模态的 `对话框窗口` 不同，卡片仅锁定单个窗口。

卡片在外观和行为上类似于其他环境中找到的模式窗口。例如，macOS 对话框可以作为 `工作表` 打开，附着在窗口顶部。此外，网页通常以类似的 `lightbox` 格式显示各种对象，如广告或图像。在 FileMaker 中，卡片窗口非常适合呈现导航菜单、自定义工具栏、对话框、向导、自定义帮助指南等。由于显示的布局有其自身的上下文，它可以基于与父窗口活动布局不同的表 occurrence。这允许卡片提供在你的解决方案中任何地方都适用的通用界面“服务”，这可以减少技术重复并促进资源共享。

### 使用 cURL 选项

`客户端 URL (cURL)` 是一个开源的命令行工具，用于使用 URL 语法在不同系统之间传输数据。该技术极大地扩展了网站请求的功能，它使用随 URL 一起发送但与之分离的结构化数据。这提供了一种更安全的数据传输方法，并具有更大的结构灵活性。FileMaker 的 `从 URL 插入` 脚本步骤包含一个 `指定 cURL 选项` 按钮，用于当向需要 cURL 格式而非直接将其嵌入 URL 的网站发送请求时。输入 URL 后，单击 `指定` 按钮，并根据目标网站的要求输入选项。例如，如果某个网站需要身份验证，选项的格式可能如下例所示。此示例将一个 URL 放入一个变量，将 cURL 选项放入另一个变量，然后在 `从 URL 插入` 脚本步骤中使用它们，该步骤以 `$Data` 变量作为结果的目标。

```
设置变量 [ $URL ; 值: "https://www.example.com/" ]
设置变量 [ $Options ; 值: "--user name:password" ]
从 URL 插入 [ 对话框: 关闭 ; 目标: $Data ; $URL ; cURL 选项: $Options ]
```

cURL 可用于将数据推送到 Web 表单，如下例所示。将名字和姓氏与字段标签一起构建，作为 `data` 选项的内容包含，然后发送到网站的“联系我们”表单。

```

```

# 26. 调试脚本

当一个简单脚本遇到错误时，排查相对容易。随着数据库变得越来越复杂，故障排查迅速变得更具挑战性。由用户、事件触发器、其他脚本以及外部系统发起的脚本调用形成了复杂交织的层级执行堆栈。脚本可以更改找到的记录集、切换布局和打开新窗口，从而创建了一个由相互关联的界面上下文组成的复杂网络。当出现问题时，通常很难甚至不知道是哪个脚本导致了错误，更不用说知道问题发生的位置、原因以及在何种上下文条件下发生的了。为了减轻这种负担，像 FileMaker 这样的现代开发环境包含了一个调试器。在本章中，我们将介绍脚本调试，并涵盖以下主题：

-   调试简介
-   探索调试界面
-   使用自定义断点

### 调试简介

`脚本调试器` 是一个交互式工具，开发人员可以使用它来可视地逐步执行脚本化过程，并对性能进行故障排查或分析。FileMaker 的调试器使得逐步执行脚本、监控变量、设置断点以及发现错误变得容易，同时还能实时直接观察在数据库界面中执行的相应操作。它包含一个临时禁用所有脚本触发器的选项，以避免出现故障的启动脚本或其他情况。可以通过从 `工具` 菜单或 `脚本工作空间`（第 24 章）的标题中选择 `脚本调试器` 来激活调试器。此外，可以启用 `出错时暂停` 选项，以便当用户处于浏览模式时，如果脚本产生错误，自动打开调试器并暂停操作。

在调试器中运行脚本时，FileMaker 会在堆栈中每个脚本的开头以及遇到错误的任何步骤暂停。您可以在任何步骤定义任意的 `断点`，以强制脚本暂停以便进一步调查。暂停后，可以选择 `继续` 执行脚本到下一个断点，`停止` 以中止脚本，或者使用以下四个选项之一 `单步执行`：

-   `单步跳过` – 继续执行当前脚本，*不*调试地执行任何子脚本
-   `单步进入` – 逐步执行脚本，*同时*调试地执行任何子脚本
-   `单步跳出` – 执行当前子脚本中的剩余步骤，然后将控制权返回给堆栈中的父脚本，并在调用子脚本的那一行之后暂停
-   `设置下一步` – 跳到当前脚本中的选定步骤，*不执行中间步骤*

# 27. 使用脚本触发器

连接到按钮或菜单的脚本需要用户执行手动操作来启动进程。虽然这对于可选任务或需要有意选择的任务来说没有问题，但关键业务功能可能会被遗忘。在某些情况下，要求用户执行额外的点击来启动界面中的强制性操作可能效率低下且不直观。FileMaker 允许您设计动态响应的界面，将用户事件与自定义脚本连接起来。`脚本触发器`是开发者定义的界面事件与脚本进程之间的连接。每当事件发生时，脚本会自动运行。FileMaker 拥有二十多种触发器，用于各种文件、布局和对象事件。一些事件会在用户操作完成之前运行分配的脚本，允许脚本执行自定义步骤并决定是否应将事件执行完毕。另一些则在事件之后运行，允许脚本执行后续任务。本章介绍脚本触发器，涵盖以下主题：

* 定义可用的触发器
* 理解事件优先级
* 在事件完成前访问目标
* 避免触发器异常

> **提示:** 分配了脚本触发器的对象在布局模式下会选择性显示一个徽章图标（第 17 章，“视图菜单”）。

### 定义可用的触发器

脚本触发器分为三类：`文件`、`布局`和`对象`。



### 文件触发器

`文件触发器`（file trigger）可在发生文件事件时运行脚本。这些触发器在`文件选项`对话框（第 6 章）的`脚本触发器`选项卡中配置，如图 27-1 所示。可通过选择`文件 ➤ 文件选项`菜单来访问此对话框。对话框顶部列出了所有可用的`文件事件`。复选框表示该事件处于活动状态，将触发所显示的脚本。下半部分显示所选事件的详细信息。点击`选择`指定与该事件关联的脚本。

![../images/429700_2_En_27_Chapter/429700_2_En_27_Fig1_HTML.jpg](img/429700_2_En_27_Fig1_HTML.jpg)

图 27-1：用于定义文件触发器的选项对话框选项卡

#### OnFirstWindowOpen（首次打开窗口时）

`OnFirstWindowOpen` 事件发生在数据库文件打开后、第一个窗口出现时，这提供了在用户开始工作之前自动执行设置功能的机会。触发的脚本可以初始化全局变量、自动打开其他窗口，或执行默认的查找与排序。此事件仅在用户打开数据库，或从另一个文件执行`打开文件`脚本步骤时发生。如果文件作为隐藏窗口打开在另一个数据库之后，通过关系显示资源，则*不会*触发此事件。

**提示：** 在文件打开时导航至起始布局和隐藏工具栏，可以在`文件选项`对话框的`打开`选项卡中设置，无需触发器。

#### OnLastWindowClose（关闭最后一个窗口时）

`OnLastWindowClose` 文件事件发生在文件开始关闭最后一个界面窗口时。这为脚本提供了执行最后时刻维护任务的机会。如有必要，脚本可以通过返回 false（0）值来停止关闭过程。例如，脚本可以使用`显示自定义对话框`步骤来确认用户是否确实打算关闭文件，并给他们取消的选择。在下面的示例中，对话框配置为第一个（默认）按钮命名为“取消”，第二个按钮命名为“关闭”。它使用`Get ( LastMessageChoice )` 函数检测点击了哪个按钮。如果用户点击第一个按钮取消，则返回零结果，导致触发事件停止，从而保持窗口和文件打开。

```
显示自定义对话框 [ "确定要关闭数据库吗？" ]
If [ Get ( LastMessageChoice ) = 1 ]
退出脚本 [ 文本结果: 0 ]
End If
```

#### OnWindowOpen（打开窗口时）

`OnWindowOpen` 文件事件发生在用户选择`窗口 ➤ 新建窗口`菜单或脚本运行`新建窗口`步骤创建任何新窗口之后。当之前因关系连接而作为隐藏窗口打开的文件首次被前置时，也会触发此事件。这为执行窗口的任何配置或设置程序提供了机会。例如，可用于将窗口精确定位在屏幕上、执行默认查找、排序、设置控制变量，或为使用窗口做其他准备。

#### OnWindowClose（关闭窗口时）

`OnWindowClose` 文件事件发生在*任何*窗口即将关闭*之前*。这为对窗口内查看的一条或多条记录执行最后时刻的验证功能或维护程序提供了机会。与其他发生在动作之前的事件一样，如果脚本返回 false（0）值，过程将被停止，窗口保持打开状态。

**注意：** 当首次打开文件时会触发两个窗口打开事件，关闭最后一个窗口时会触发两个窗口关闭事件。

#### OnFileAVPlayerChange（文件媒体播放器变化时）

`OnFileAVPlayerChange` 文件事件发生在容器字段或 URL 中播放的媒体文件的播放状态改变之后。例如，当暂停、播放、停止，或媒体播放到末尾停止时会触发此事件。此事件仅在 iOS 设备上的 FileMaker Go 中支持。

### 布局触发器

`布局触发器`（layout trigger）可在发生布局事件时运行脚本。这些触发器在`布局设置`对话框（第 18 章）的`脚本触发器`选项卡中配置，如图 27-2 所示。该对话框与`文件选项`对话框类似，只是增加了用于控制事件在哪种窗口模式下启用的复选框。

![../images/429700_2_En_27_Chapter/429700_2_En_27_Fig2_HTML.jpg](img/429700_2_En_27_Fig2_HTML.jpg)

图 27-2：用于配置布局脚本触发器的对话框

#### OnRecordLoad（加载记录时）

`OnRecordLoad` 布局事件发生在记录成为当前记录之后。这为使用自动输入设置中过于复杂的值来设置默认字段值、创建相关记录，或执行其他自定义记录任务提供了机会。任何导致窗口激活记录的活动都会触发此事件。包括以下情况：

* 在找到的记录集中导航到新记录
* 导航到不同的布局
* 创建新记录
* 删除包含多条记录的找到记录集中的一条记录
* 创建显示包含一条或多条记录的表的新窗口
* 显示搜索结果

#### OnRecordCommit（提交记录时）

`OnRecordCommit` 布局事件发生在提交已修改的记录之前。这提供了在必要时停止提交操作的机会。如果脚本结果为 true（1），则原始事件正常进行。如果结果为 false（0），则该事件被取消。任何导致当前记录提交的活动都会触发此事件，包括：

* 用户通过键入 Enter 键或点击活动记录字段外部来显式提交记录
* 导航到不同的记录
* 创建、删除、复制或排除记录
* 脚本执行`提交记录/请求`步骤
* 打开`管理数据库`对话框或其他开发者对话框
* 关闭文件

#### OnRecordRevert（还原记录时）

`OnRecordRevert` 布局事件发生在将已修改的记录还原到之前提交的状态之前。当用户选择`记录 ➤ 还原记录`菜单或脚本运行`还原记录/请求`步骤时会触发此事件。它提供了打开自定义对话框确认该操作、执行其他重置操作，或在必要时停止还原操作的机会。

#### OnLayoutKeystroke（布局按键时）

`OnLayoutKeystroke` 布局事件发生在键盘上键入任何字符时，但操作系统或 FileMaker 的活动菜单集处理的函数键除外。即使没有字段处于活动焦点，也会触发此事件，并且发生在按键事件发送给 FileMaker *之前*。这为脚本提供了机会：当用户在进入字段之前开始输入时，用自定义消息覆盖默认错误对话框。也可以用于响应与菜单无关的组合键来运行脚本，或在向门户过滤器（第 20 章，“过滤门户记录”）输入时刷新视图。以下示例使用两个以触发器为中心的 `Get` 函数，获取按下的按键，并在用户按住 macOS Control 键并键入“1”时运行脚本。脚本执行`执行脚本`步骤（此处未定义），然后使用`退出脚本`且返回结果为 0 来停止按键操作。脚本会忽略用户按下的其他任何按键，并使它们继续进入活动字段。

```
If [ Get ( TriggerModifierKeys ) = 4 and Get ( TriggerKeystroke ) = "1" ]
执行脚本 [ ]
退出脚本 [ 文本结果: 0 ]
End if
```



### 布局事件

#### OnLayoutEnter（进入布局时）

`OnLayoutEnter` 布局事件发生在浏览模式下窗口切换到不同布局之后。这提供了一个机会，可以执行类似于打开窗口或激活记录时触发的设置功能。

#### OnLayoutExit（退出布局时）

`OnLayoutExit` 布局事件发生在导航离开当前布局之前。这提供了一个机会，可以在必要时验证并阻止导航变化。

#### OnLayoutSizeChange（布局大小变更时）

`OnLayoutSizeChange` 布局事件发生在窗口大小改变之后。这发生在打开窗口、调整窗口大小、隐藏/显示工具栏或格式栏，以及在 iOS 设备上切换横竖屏时。这提供了一个执行操作的机会，例如强制设置最小窗口大小或重新定位窗口以使其在屏幕范围内完美居中。以下公式可用于“设置变量”脚本步骤，根据当前窗口尺寸设置 `$Height` 和 `$Width` 变量，或在数值过低时切换至默认最小值。随后可在“移动/调整窗口”脚本步骤中使用这些变量来强制设置最小窗口大小，本例中为 800 x 1000。

```
Let ( [
Height = Get ( WindowHeight ) ;
$Height = Case ( Height < 800 ; 800 ; Height ) ;
Width = Get ( WindowWidth ) ;
$Width = Case ( Width < 1000 ; 1000 ; Width )
] ;
""
)
```

#### OnModeEnter（进入模式时）

`OnModeEnter` 布局事件发生在切换到浏览、查找或预览模式之后。这提供了一个机会，可以为特定活动准备窗口，例如设置默认查找条件、初始化变量或调整打印设置。

#### OnModeExit（退出模式时）

`OnModeExit` 布局事件发生在更改窗口模式之前。这为脚本提供了一个机会，可以在必要时验证并阻止模式更改。

#### OnViewChange（视图变更时）

`OnViewChange` 布局事件发生在表单视图、列表视图和表格视图之间切换之后。

#### OnGestureTap（手势轻点时）

`OnGestureTap` 布局事件发生在布局上执行轻点手势时。这为脚本提供了一个机会，可以使用 `Get ( TriggerGestureInfo )` 函数确定事件的详细信息，并执行自定义操作或在必要时阻止该手势。该事件在以下条件下触发：

- 仅限 FileMaker Go 和 Windows（Windows 7 除外）。
- 仅限浏览和查找模式。
- 轻点操作不在活动的 Web 查看器或活动的编辑框内。
- 单指、双指或三指轻点（iOS）。
- 单指双击（iOS），这将触发该事件两次。
- 双指轻点（Windows）。

#### OnExternalCommandReceived（收到外部命令时）

`OnExternalCommandReceived` 布局事件发生在用户按下按钮以控制 iOS 设备锁定屏幕或外部设备上的播放功能时，例如“播放”、“暂停”、“停止”、“下一首”和“上一首”。触发的脚本在事件发生*之前*运行，并且可以使用 `Get ( TriggerGestureInfo )` 函数获取有关外部事件的信息。

### 对象触发器

*对象触发器* 在针对布局对象或在布局对象内部执行某个事件时运行脚本。这些可以通过图 27-3 所示的对话框进行配置，通过选择一个对象并从“格式”菜单或对象的上下文菜单中选择“设置脚本触发器”功能来访问。此外，“弹出面板设置”对话框中也有一个按钮可以打开此对话框（第 20 章，“探索弹出按钮设置选项”）。

用于配置对象脚本触发器的对话框

#### OnObjectEnter（进入对象时）

`OnObjectEnter` 事件发生在用户点击对象或使用 Tab 键进入对象，当布局对象获得活动焦点之后，或者当脚本执行诸如“转到对象”或“转到下一字段”等步骤时。

#### OnObjectKeystroke（对象按键时）

`OnObjectKeystroke` 事件发生在向具有焦点的对象键入任何字符时，但在事件完成之前。与布局按键触发器类似，此事件会忽略那些由操作系统或 FileMaker 应用程序处理的按键，并且可以通过以假值（0）退出脚本来阻止。

#### OnObjectModify（对象修改时）

`OnObjectModify` 事件发生在对象的值已更改之后。该事件仅响应与界面相关的操作而发生。当窗口更新在外部数据源中修改的值、字段由自动输入公式更新，或使用“设置字段”脚本步骤更新字段值时，该事件*不会*发生。以下操作会触发此事件：

- 在对象中键入按键
- 在值列表格式的字段中选择新值
- 使用“剪切”、“粘贴”或“清除”功能
- 将内容拖放到其中
- 使用与界面相关的脚本步骤（例如“插入文本”）更改对象的值
- 选择新的选项卡控制面板

#### OnObjectValidate（对象验证时）

`OnObjectValidate` 事件发生在修改后字段的值准备进行验证时，即焦点即将移动到另一个字段或正在提交记录之前。这提供了一个机会，可以验证对对象的更改、执行任何预备操作，和/或在必要时阻止移动到另一个对象。

#### OnObjectSave（对象保存时）

`OnObjectSave` 事件发生在修改后的字段已验证但尚未退出之前，提供了另一个阻止退出过程的机会。

#### OnObjectExit（退出对象时）

`OnObjectExit` 事件发生在对象即将失去活动焦点时，无论是否有任何修改。这发生在移动到另一个对象、门户行或字段重复项时。当发生任何导致提交记录的操作时也会发生。脚本在退出对象之前触发，允许其在必要时阻止该过程。

#### OnPanelSwitch（面板切换时）

`OnPanelSwitch` 事件发生在面板控件的当前视图即将更改之前，并且可以通过以假值（0）退出脚本来阻止。内置的 `Get` 函数可以确定对象的布局名称以及当前和目标面板的编号，从而使脚本能够采取与更改相关的特定操作。以下公式可以放入目标脚本的“显示自定义对话框”步骤或“数据查看器”（第 26 章）中，以句子的形式说明这些值。在 `If` 语句中使用它们，可以根据起始和结束面板有条件地执行不同的步骤集。

```
Let ( [
currentPanel = GetValue ( Get ( TriggerCurrentPanel ) ; 1 ) ;
targetPanel = GetValue ( Get ( TriggerTargetPanel ) ; 1 )
] ;
"正在从面板 " & currentPanel & " 切换到面板 " & targetPanel & "。"
)
```

#### OnObjectAVPlayerChange（对象音视频播放器变更时）

`OnObjectAVPlayerChange` 事件发生在对象中的媒体更改状态之后。例如，当用户暂停、播放或停止字段中的视频，或者视频播放到结尾停止时，该事件将触发脚本。此事件仅在 iOS 设备上的 FileMaker Go 中受支持。

## 了解事件优先级

许多用户操作会触发多个事件，FileMaker 将按特定的优先级顺序执行脚本触发器。脚本拦截操作的位置取决于预期的操作类型以及取消事件选项是否重要。



### 打开数据库文件

当 *打开数据库文件* 时，将按顺序触发以下事件：
1. `OnFirstWindowOpen`
2. `OnWindowOpen`
3. `OnLayoutEnter`
4. `OnModeEnter`
5. `OnRecordLoad`

**注意：** 虽然这不是脚本触发器，但文件打开时“*切换布局*”的选项会在所有脚本触发器之前执行（第 6 章，“文件选项：打开”）。

### 提交包含未保存更改的记录

当 *提交包含未保存更改的记录* 时，将按顺序触发以下事件：
1. `OnObjectValidate`
2. `OnObjectSave`
3. `OnObjectExit`
4. `OnRecordCommit`
5. `OnRecordLoad`
6. `OnObjectEnter`

### 打开新窗口

当 *打开新窗口* 时，将按顺序触发以下事件：
1. `OnWindowOpen`
2. `OnLayoutEnter`
3. `OnModeEnter`
4. `OnRecordLoad`
5. `OnLayoutSizeChange`

### 切换布局

当 *切换布局* 时，将按此顺序触发以下事件：
1. `OnLayoutExit`
2. `OnLayoutEnter`
3. `OnRecordLoad`

**注意：** 当在显示包含未保存更改记录的窗口中 *切换布局* 时，将首先执行记录提交序列，然后执行切换布局序列。

## 在事件完成前访问目标

如前所述，许多事件会在事件完成前触发脚本，从而提供在事件效果实施前将其阻止的选项。然而，这意味着操作的结果尚未实现，并且脚本无法通过界面的常规方式识别这些结果。例如，当用户在字段中输入字符时，会触发 `OnObjectKeystroke` 事件。然而，脚本会在字符输入到字段之前运行，因此在脚本步骤中将字段值拉入公式将不会包含最近触发脚本的按键操作。类似地，`OnPanelSwitch` 事件发生在目标面板被激活之前。由于切换之后没有对应的事件发生，因此脚本可能需要确定目标面板才能执行特定的操作路径。

FileMaker 提供了一组内置的 *Get* 函数，专门用于提供有关事件的信息，以便触发的脚本可以访问和使用这些信息。例如，`Get ( TriggerCurrentPanel )` 和 `Get ( TriggerTargetPanel )` 分别以两个由回车符分隔的列表形式返回当前面板和目标面板的编号和名称。`Get ( TriggerKeystroke )` 将返回导致 `OnObjectKeystroke` 事件的、尚未进入字段的最后一个按键，而 `Get ( TriggerModifierKeys )` 返回一个数字，指示用户在键入时按下的任何修饰键。例如，如果用户按住 Shift 键，将返回“1”，而按住 Option 键将返回“8”。当按住组合键时，结果是代表这些键的数字之和，例如，同时按住 Shift 和 Option 将返回“9”。

对于无法访问目标信息的 *事件*，请从事件前触发器切换到事件后触发器。例如，没有可用的函数来确定由 `OnObjectExit` 触发的脚本中的下一个目标对象，因此尝试切换到 `OnObjectEnter`，因为它发生在更改发生之后。

**注意：** 这些目标确定函数仅在目标脚本内有效，*而不会*作为从对象发送到脚本的参数。

## 避免触发器异常

某些对象更改将 *不会* 触发事件。例如，大多数内置的 *拼写* 和 *查找/替换* 功能，尤其是在查找集中执行时，不会触发诸如 `OnObjectModify`、`OnObjectValidate` 和 `OnObjectSave` 等事件。此外，由于像 `导入记录`、`替换字段内容` 和 `重新查询字段内容` 之类的功能会直接修改表数据，它们不会触发界面事件。类似地，像 `设置字段`、`按名称设置字段` 和 `设置 Web 查看器` 这样的脚本步骤也不会触发事件。请务必考虑用户或脚本可能执行的所有操作，并在必要时锁定相关功能或在脚本中使用替代步骤，以确保关键功能被触发。例如，如果布局上的相关字段分配了关键触发器，请使用 `插入文本` 或 `插入计算结果` 脚本步骤代替 `设置字段`。当无法避免非触发步骤时，请添加步骤以编程方式触发相关的目标脚本，或添加步骤以显式执行所需功能。或者，不要过度使用脚本触发器，尤其是在并非绝对必要的情况下，否则当这些异常排除了触发器时，可能会造成不必要的复杂性。

Set Variable [ $installed ; Value : Get ( InstalledFMPlugins ) ]
If [ PatternCount ( $installed ; "Troi File Plug-in;8.0.2" ) = "" ]
If [ Get ( SystemPlatform ) = 1 ]
Install Plug-in File [ Resources::Troi_File.fmplugin ]
Else
Install Plug-in File [ Resources::Troi_File.fmx]
End if
End if
```

该脚本可以作为由` OnFirstWindowOpen `事件（第 29 章）触发的启动脚本的一部分自动运行，以确保每个用户的计算机都正确安装。可以扩展该脚本以检查错误，确认安装过程是否成功，并在检测到问题时警告用户联系数据库管理员。例如，如果未启用`应用程序`偏好设置中的`允许解决方案安装文件`，则脚本将失败。图 28-1 所示的`应用程序`偏好设置对话框中的`插件`选项卡列出了所有已安装的插件。在此处，您可以确认安装、启用、禁用和配置插件。

# 29. 部署和访问数据库

开发完成后，就该将数据库部署到目标工作流中了。部署有三种主要选择：在文件夹目录中、在 iOS 设备上或在主机服务器上。每个选项都有不同的优势，选择会影响谁可以访问文件、如何访问以及从何处访问。FileMaker 系列中的每个产品都有不同的部署和访问能力（参见第 1 章，“审查产品线”）。在本章中，我们将探讨这些选项，介绍与部署和访问解决方案相关的特定主题，包括：

- 使用 FileMaker Server 托管
- 使用 FileMaker Cloud 托管
- 外部访问解决方案

## 部署到文件夹目录

最简单的部署方法是将数据库存储在文件夹目录中。只要有该文件夹的访问权限，`FileMaker Pro` 桌面应用程序就可以打开并使用该数据库。这提供了极大的便利，因为数据库可以在文件夹之间移动，或者像处理文字处理文档或 Excel 文档一样简单地复制以重复使用。虽然这对于单个用户来说是一个简单且有用的选项，但共享的机会有限。可以通过将副本发送给同事来共享数据库，但每个人都将拥有自己独立的文件副本，无法享受协作、同时使用的优势。它可以放在网络文件服务器上的文件夹中；但是，不建议这样做，因为多个桌面应用程序副本同时读取和写入同一文件会导致文件损坏，这可能是灾难性的（第 6 章，“避免文件损坏”）。同样，像 Dropbox 或 Google Drive 这样的服务即使在单用户情况下也可能在同步时损坏文件。`FileMaker Pro` 桌面应用程序可以通过充当具有点对点共享（本章后面描述）的适当主机，安全地共享从本地文件夹打开的文件。但是，这仅限于其他用户最多五个并发连接，并且访问权限仅限于 FileMaker 产品。然而，如果共享不是一个重要的考虑因素，那么从本地文件夹使用数据库是一个合适的选择。只需确保在将文件移动到另一个文件夹之前将其关闭，并将其保存在其他用户无法访问的文件夹中。



# 部署到 iOS 设备

可以将数据库直接部署到 iPhone 或 iPad 上，并通过免费的 `FileMaker Go` iOS 应用程序进行访问。由于文件物理存储在设备上，因此具有便携性的优势。用户可以在任何有设备的地方访问数据库，即使没有连接到蜂窝网络或 Wi-Fi 网络也是如此。部署到 iOS 设备仅限于与数据录入相关的任务，无法对文件进行结构性修改。此外，也无法从设备上共享数据库以供工作组同时访问，因此每个设备都拥有文件的独立副本。然而，对于需要便捷单用户访问为小屏幕设计的数据库来说，这个选项非常理想。

要安装，请先将 `Claris FileMaker Go 19` 应用程序下载到设备上。然后，将设备连接到计算机，并将数据库从某个文件夹复制到设备“文件”中的 `FileMaker Go 19` 文件夹中。在 macOS Catalina (10.15) 上，按照以下步骤操作，如图 29-1 所示：

3. 选择“文件”选项卡。然后将数据库文件拖入 `FileMaker Go 19` 文件夹。

# 在网络上共享数据库

在网络上的共享数据库与在共享服务器文件夹中共享文档文件*不是*一回事。如前所述，数据库文件*可以*放置在文件服务器上的共享目录中，多个人可以看到并访问它。但是，如果多个用户同时打开该文件，当执行数据录入任务时，每个 `FileMaker` 副本将开始持续地读写该文件。*这将导致文件损坏和数据丢失！* 如果发生这种情况，该文件必须通过恢复过程进行处理，并且可能损坏到无法修复（第 6 章，“避免文件损坏”）。

在网络上共享数据库的正确方法是*通过*一个启用了共享功能的 `FileMaker` 软件版本来充当主机服务。这可以通过使用 `FileMaker Pro` 桌面应用程序进行*点对点共享*，或使用运行 `FileMaker Server` 或 `FileMaker Cloud` 的专用服务器来实现。数据库存储在主机软件安全隐藏的文件夹中，该软件管理连接并协调无冲突的读/写进程，如图 29-3 所示。

![../images/429700_2_En_29_Chapter/429700_2_En_29_Fig3_HTML.jpg](img/429700_2_En_29_Fig3_HTML.jpg)

**图 29-3** 数据库及由主机软件管理的客户端访问

一旦主机开始提供数据库服务，用户就可以通过其客户端软件访问它。在 `FileMaker Pro` 桌面应用程序中，选择 `文件 ➤ 主机 ➤ 显示主机` 菜单，或者在“打开文件”对话框中点击“主机”，即可看到列出收藏主机和本地主机的对话框。iOS 版 `FileMaker Go` 应用程序中的“主机”选项卡也会列出相同的内容。当数据库配置为 `WebDirect` 时，用户还可以通过网络浏览器中的 URL（例如 `http://127.0.0.1/fmi/webd`）从主机访问数据库。当使用 `FileMaker Server` 或 `FileMaker Cloud` 托管时，还可以使用 `ODBC`/`JDBC` 和通过 `FileMaker API` 进行的*表述性状态传递* (REST) 来启用传入连接。有关这些连接方法的更多信息，请访问 Claris 的文档网站。

## 了解协作限制

当多个用户在同一个托管文件中工作时，会自动强制执行一些限制以避免冲突。各部分每次*仅限一名用户*。如果用户各自修改的是*不同*的记录、布局或脚本，那么前三个操作可以同时进行。其余的开发操作在各自的开发者对话框中每次仅限一名用户，即，一名用户可以编辑值列表，同时另一名用户可以编辑模式，因为这些操作在不同对话框中进行。

-   编辑*相同*记录
-   修改*相同*布局
-   修改*相同*脚本
-   修改*任何*数据库模式：表、字段和关系
-   修改*任何*值列表
-   修改*任何*数据源
-   修改*任何*安全访问权限

### 启用点对点共享

*点对点共享* 选项允许用户通过 `FileMaker Pro` 桌面应用程序打开最多 125 个数据库，并充当最多五个并发远程客户端连接的主机。由于点对点主机运行的是客户端版本的软件，因此主机和其他已连接的用户都可以在文件中执行数据录入和开发任务，具体取决于其访问权限。`FileMaker 网络设置` 对话框的上半部分用于为客户端本地应用程序启用点对点共享。此处会显示一条警告，指出点对点共享不是安全的网络协议，因为只有 `FileMaker Server` 和 `FileMaker Cloud` 具备使用*安全套接层* (SSL) 认证的选项以启用网络上的安全传输。虽然此方法不适用于大型工作组或对安全性有重要要求的场景，但对于与一小群同事共享数据库来说，它是可行的。

## 使用 FileMaker Server 托管

`FileMaker Server`是一个快速、可靠且可扩展的服务器软件包，可在网络上安全地共享数据库文件。该服务器具有强大的功能和优势，包括：

- 为运行在 macOS、Windows 或 iOS 上的任何`FileMaker`产品线的用户提供可靠的全时数据库服务。
- 多种灵活的许可选项，可以混合搭配以满足团队成员的个人需求。
- 使用`FileMaker WebDirect`（文件管理器网页直连）无需网络编程即可在 Web 上共享数据库，或使用`Custom Web Publishing`（自定义网络发布）通过 PHP 或 XML 创建数据驱动的网站。
- 使用`Open Database Connectivity`（开放数据库连接，ODBC）和`Java Database Connectivity`（Java 数据库连接，JDBC）与远程系统共享托管数据库。
- 使用`Admin Console`（管理控制台）网络界面或`fmsadmin`命令行本地或远程监控和管理服务器。
- 通过自动数据库备份和计划脚本实现管理任务的自动化。
- 可以使用`Perform Script On Server`（在服务器上执行脚本）步骤将脚本从客户端软件卸载到服务器上运行。

**注意**：有关超出本节所述托管基础知识的更多信息，请参阅位于`https://www.claris.com/resources/documentation/`的`FileMaker Server`产品支持部分。

### 准备主机计算机

在选择用作服务器主机的计算机时，有许多重要的考虑因素。计算机应具有快速处理器、充足的内存和固态硬盘。使用有线的、高速的、专用的网络连接将其连接到网络，并理想地为其分配一个静态 IP 地址。为了允许优先使用处理器、硬盘和网络容量，*专门*将该计算机用于托管数据库。为避免损坏，不应直接备份包含托管数据库的文件夹。相反，请使用`FileMaker Server`内置的计划备份功能，然后将这些备份的额外副本复制到其他驱动器。此外，禁用屏幕保护程序和节能休眠功能，改为使用显示器变暗。关闭诸如 macOS `Spotlight`（聚光灯）功能之类的索引服务，并且不要使用防病毒软件扫描。

**提示**：始终查看 Claris 网站以获取服务器的最低硬件要求和其他服务器建议。

#### 定义已安装资源

`FileMaker Server`不是一个传统应用程序，不会出现在计算机的`Applications`（应用程序）文件夹中。相反，将安装以下资源：

- `Database Server`（数据库服务器） – 托管数据库，与`FileMaker Pro`和`FileMaker Go`（文件管理器 Go）客户端共享。
- `Web Publishing Engine`（网络发布引擎） – 处理`WebDirect`服务，管理`Web Server`（网络服务器）和`Database Server`之间的流量。
- `Web Server`（网络服务器） – macOS 使用其自己的 Apache 网络服务器副本。在 Windows 上，安装期间会启用`Internet Information Services`（互联网信息服务）并将其用作网络服务器。
- `PHP Engine`（PHP 引擎） – 用于将`FileMaker API`的调用路由到`Web Publishing Engine`。
- `Admin Console`（管理控制台） – 通过服务器或客户端计算机上的网络浏览器访问，用于配置和管理服务器。
- `Command-Line Interface Executable`（命令行界面可执行文件） – 使用`fmsadmin`命令通过命令行界面管理服务器。
- `User Account`（用户账户） – 指定服务器运行所用的账户。macOS 上的默认选择是安装程序创建的新`fmserver`用户，Windows 上则是`Local System`（本地系统）。如果需要，可以指定自定义账户。
- `User Group`（用户组） – 会创建一个名为`fmsadmin`的新用户组，并且必须将其分配给托管数据库文件。

### 访问管理控制台

`Admin Console`（管理控制台）用于管理服务器，可通过网络浏览器访问。安装服务器后，服务器计算机桌面上会有一个指向本地主机的书签。也可以通过打开网络浏览器并输入以下地址来访问控制台：

```
http://localhost:16001/admin-console
```

要使用另一台计算机访问它，请将`localhost`替换为以下服务器地址：

```
https://10.0.1.20:16000/admin-console
https://10.0.1.20/admin-console
```

当控制台页面打开并请求身份验证时，输入您在安装期间定义的管理员名称和密码。服务器管理控件分布在六个选项卡页面上：

- `Dashboard`（仪表盘） – 汇总服务器信息和状态
- `Databases`（数据库） – 列出所有已安装的数据库和活跃连接的客户端，并带有管理数据库以及与客户端通信或断开连接的控件
- `Backups`（备份） – 用于计划并查看备份
- `Configuration`（配置） – 包含常规设置、客户端、文件夹、计划脚本、通知、证书和日志记录
- `Connectors`（连接器） – 包含`Web Publishing`（网络发布）、`FileMaker Data API`（文件管理器数据 API）、插件以及 ODBC/JDBC 连接的设置
- `Administration`（管理） – 包含许可证、管理员密码和外部身份验证设置

### 上传文件到 FileMaker Server

可以通过手动将数据库文件移动到`Databases`（数据库）文件夹中，或通过`FileMaker Pro`桌面应用从客户端计算机上传它们来将数据库文件添加到服务器。



### 将文件移入 Databases 文件夹

服务器将托管的文件存储在 `FileMaker Server` 文件夹的 `Databases` 子文件夹中。如果安装到默认位置，该文件夹将根据操作系统位于两个位置之一：macOS 上的 `Library` 文件夹或 Windows 上的 `Program Files` 文件夹，两者均如下所示。要手动安装文件，只需将其拖入 `Databases` 文件夹即可。但是，以这种方式添加文件时，必须手动将它们分配给 `fmserver` 用户和 `fmsadmin` 组。

```
/Library/FileMaker Server/Data/Databases
[drive]:\Program Files\FileMaker\FileMaker Server\Data\Databases\
```

**注意**  
如果服务器正在主动托管数据库，则移除、替换或使用 `FileMaker Pro` 桌面应用程序直接打开这些数据库将导致严重损坏。在执行任何这些操作之前，请确认文件未被主动托管。

## 管理数据库文件

要管理托管的数据库，请登录 `Admin Console` 并单击 `Databases` 选项卡。列表中每个数据库旁边的下拉菜单包含以下管理命令：

- `Open` – 打开数据库以供网络访问。如果文件已加密，则需要输入密码。
- `Close` – 关闭并停止托管数据库。
- `Download` – 将已关闭的数据库副本下载到本地硬盘驱动器。在 macOS 上，如果浏览器偏好设置阻止了弹窗，此功能将无法使用！
- `Remove` – 将已关闭的数据库从列表中移除，并将其放入 `Databases` 文件夹中一个不可用于托管的特殊文件夹中。
- `Verify` – 关闭、验证并重新打开数据库。
- `Clear the Encryption Passwords` – 移除所选数据库文件的加密密码。

**提示**  
数据库列表顶部的一个类似菜单提供了这些选项的镜像，但其命令会影响所有数据库。

## 重新启动服务器计算机

当服务器正在主动托管文件时，它可能正在与客户端通信、在缓存中保留最近的更改，或正在将这些更改写入磁盘。如果强制退出服务器进程或硬件突然强制关闭，文件可能会损坏。为避免损坏，务必规避某些风险并正确关闭数据库。始终使用足够的不间断电源（UPS）设备，并在关闭计算机之前关闭 `Admin Console` 中的所有数据库。除非因重大崩溃或故障而绝对必要，否则切勿在不先关闭文件的情况下强制退出应用程序或执行硬重启计算机。

## 使用命令行界面

`fmsadmin` 命令行工具允许使用 `Terminal` 应用（macOS）或命令提示符（Windows）来管理服务器。这些命令随服务器自动安装到以下特定于平台的文件夹中：

```
macOS = /Library/FileMaker Server/Database Server/bin/fmsadmin
macOS（符号链接） = /usr/bin/fmsadmin
Windows = [drive]:\Program Files\FileMaker\FileMaker Server\Database Server\fmsadmin.exe
```

### 格式化 fmsadmin 命令

命令的公式为：`fmsadmin <command> <options>`。例如，要获取所有托管数据库的列表，请在 Terminal 中键入以下命令并按 Enter 键。

```
fmsadmin list files
```

某些功能需要身份验证，并会提示您输入服务器管理员用户名和密码。或者，您可以在命令中包含凭据信息，以避免出现二次提示。只需按照以下模式和示例添加它们：

```
fmsadmin  -u  -p
fmsadmin list files –u Admin –p J56TF3
```

### 可用的 fmsadmin 命令

`fmsadmin` 工具包含以下命令，每个命令都有各种可用选项：

- `autorestart` – 获取或设置服务器的自动重启功能
- `backup` – 备份一个数据库或文件夹中的每个数据库
- `clearkey` – 移除已保存的数据库加密密码
- `close` – 关闭一个或多个数据库
- `certificate` – 管理 SSL 证书
- `disable` – 禁用计划或统计记录
- `disconnect` – 断开一个或多个客户端
- `enable` – 启用计划或统计记录
- `help` – 获取可用命令的帮助信息
- `list` – 列出客户端、数据库、插件或计划
- `open` – 开始托管数据库
- `pause` – 临时停止数据库服务器
- `remove` – 将文件移出 Databases 文件夹
- `resetpw` – 重置管理员用户密码
- `restart` – 重启服务器、adminserver、FMSE、WPE 或 XDBC 进程
- `resume` – 结束数据库服务器的临时暂停
- `run` – 运行一个计划
- `send` – 向已连接的用户发送消息
- `standby` – 管理备用服务器连接、角色和更新
- `start` – 启动服务器、adminserver、FMSE、WPE 或 XDBC 进程
- `status` – 获取客户端或数据库的状态
- `stop` – 停止服务器、adminserver、FMSE、WPE 或 XDBC 进程
- `verify` – 检查数据库的一致性

# 使用链接、URL 和 AppleScript 控制 FileMaker

有多种方法可以从 FileMaker 外部打开数据库并在其中执行操作：`Snapshot Links`（快照链接）、`FileMaker URL` 和 `AppleScript`。

## 使用快照链接共享书签

`快照链接`是一个 XML 文件，存储关于一个记录或一组已找到记录的信息，可用于重新创建先前存在的已找到记录集。它们的作用类似于书签，但引用的是数据库中特定的一组已找到记录。打开一个快照文件会自动打开数据库，导航到相应布局，并恢复已找到的记录集，从而重新创建用户在保存快照时所查看的相同上下文。这些文件可以存储在文件夹中并发送给同事，允许用户与其他用户交换记录列表。例如，公司文件服务器上的每个客户文件夹都可以包含一个快照链接，点击即可立即打开该公司的联系人列表视图。

要生成快照链接，请选择 `File ➤ Save/Send Records As ➤ Snapshot Link` 菜单。这将打开一个 `Save Records` 对话框，其中包含快照配置 `options`，如图 29-9 所示。为链接输入名称并选择保存位置。选择 `Current record` 或 `Records being browsed` 的 `Save` 选项来控制保存哪些记录。可选的复选框可用于创建带有快照附件的电子邮件。`Send Records as Snapshot Link` 脚本步骤使用相同的对话框来配置自动生成快照链接的脚本。

## 使用 FileMaker URL

`FileMaker URL` 是一种已注册到 FileMaker Pro 客户端应用程序安装中的互联网协议，可以使用 URL 打开数据库并运行脚本。如果用户帐户的权限集被明确授予对 `fmurlscript` 扩展权限的访问权（第 30 章），则以下 URL 选项变为可能：

- **在 Web 浏览器中键入** — 在 Web 浏览器中键入或粘贴 URL，命令将被路由到 FileMaker。
- **嵌入网页中** — 使用 `href` 标签在浏览器中的网页上或数据库中某个布局上的 Web 查看器中创建链接。或者，新的 `FileMaker.PerformScript` JavaScript 函数是用于 Web 查看器的更好选项（第 20 章，“使用 JavaScript 调用 FileMaker 脚本”）。
- **在脚本中使用** — 使用 `Open URL` 脚本步骤来触发其他脚本。但是，`Perform Script` 步骤提供了通过计算名称运行脚本的更好选项。
- **在外部应用程序和脚本中使用** — 编程语言（如 AppleScript）可以指示 FileMaker 应用程序运行该 URL。

### 格式化基本 FMP URL

地址必须包含一个 `前缀`、`地址`和`文件名`。扩展名的包含是可选的。从版本 18 开始，前缀可以包含版本号来指定多个已安装版本中的其中一个。如果未指定版本，URL 应路由到最后安装的版本。打开数据库需要地址。要引用一个已打开的数据库，请使用美元符号作为地址。例如，要引用一个已打开的 `Contacts` 数据库，地址应格式化如下例所示：

```
fmp://$/
fmp://$/Contacts.fmp12
fmp18://$/Contacts.fmp12
fmp19://$/Contacts.fmp12
```

如果数据库存储在用户的 `Documents` 文件夹中，请使用波浪号作为地址，如下例所示，该地址将自动打开数据库：

```
fmp19://~/Contacts.fmp12
```

所有 URL 都必须进行 `百分比编码`，但在 FileMaker 内使用 `Open URL` 脚本步骤时除外。例如，一个名为“Learn FileMaker”的数据库需要将空格改为“%20”，如下例所示：

```
fmp://$/Learn%20FileMaker.fmp12
```

> **提示**  
> 在 FileMaker 计算中构建 URL 时，使用 `GetAsURLEncoded` 函数自动处理编码。

### 寻址托管数据库

要访问网络服务器上托管的数据库，请包含主机计算机的地址。例如，如果 `Contacts` 数据库已托管，FMP URL 可以使用 IPv4 地址、IPv6 地址或 DNS 名称进行格式化，如下三个示例所示：

```
FMP://10.1.0.10/Contacts.fmp12
FMP://[2001:0db8:0a0b:12f0:0000:0000:0000:0001]/Contacts.fmp12
FMP://filemaker-server.local/Contacts.fmp12
```

### 包含访问凭据

为避免弹出请求帐户凭据的对话框并实现无缝操作，可以在 `URL` 中指定帐户和密码。这些可以以冒号分隔的值形式添加在地址之前，并用“`@`”符号分隔。例如，一个帐户名为“admin”、密码为“58Jt234”的数据库可以使用以下示例中的地址打开，而无需密码对话框：

```
FMP://admin:58Jt234@10.1.0.10/Contacts.fmp12
```



## 包含脚本名称

URL 还可以指示 FileMaker 在数据库中运行脚本。在数据库名称后添加问号，后跟 `script=` 和一个经过 URL 编码的脚本名称。例如，以下示例将确认指定主机上的 `Contacts` 数据库已打开，然后运行名为 `Find` 的脚本：

```
FMP://10.1.0.10/Contacts.fmp12?script=Find
```

为简洁起见，本示例及其他示例中均省略了帐户凭据。但可以包含这些凭据，以确保数据库打开时不会出现要求输入信息的对话框。运行脚本时，将使用登录数据库的用户帐户的权限和限制来执行。

### 添加脚本参数

当目标脚本接受参数时，URL 可以通过在脚本名称后附加一个与号（ampersand）和 `param` 值来包含该参数。例如，无需总是搜索相同的条件，可以修改前面示例中的 `Find` 脚本，使其接受一个参数并使用该值来确定执行哪种类型的搜索。在 `If` 脚本步骤中使用的参数，可以让脚本根据 URL 提供的值来搜索 `Status` 字段。现在，要查找状态为 `Active` 或 `Hold` 的记录，请在 URL 中添加一个与号和 `param` 组件，如下例所示：

```
FMP://10.1.0.10/Contacts.fmp12?script=Find&param=Active
FMP://10.1.0.10/Contacts.fmp12?script=Find&param=Hold
```

### 添加脚本变量

URL 还可以更进一步，通过在脚本中初始化局部变量，添加与号、变量名和变量值，遵循如下所示的模式：

```
FMP:///?&param=&=
```

此示例初始化了一个变量，为查找脚本指定了州名：

```
FMP://10.1.0.10/Contacts.fmp12?script=Find&param=Active&$State=Pennsylvania
```

## 使用 AppleScript 自动化 FileMaker

`AppleScript` 是一种符合 `开放脚本架构`（`OSA`）的命令语言，可以与 macOS 应用程序通信并进行控制（[`https://goo.gl/i70lnx`](https://goo.gl/i70lnx)）。早在 1994 年该语言首次作为 Macintosh System 7.5 的一部分引入时，FileMaker 就是首批支持脚本化的应用程序之一。从那时起，对该语言的支持稳步发展，并继续为从外部脚本自动化 FileMaker 功能以及将其与其他应用程序的数据集成提供了绝佳选择。如果用户帐户的权限集被明确授予 `fmextscriptaccess` 扩展权限（第 30 章），那么 AppleScript 将能够控制应用程序和数据库的操作。

tell application "FileMaker Pro"

### 编写基本任务脚本

作为使用脚本自动化 FileMaker 的简要介绍，本节提供了一些简单示例，包括打开数据库、激活已打开的数据库、更改布局、执行查找和统计记录数。这些可以通过与其他功能结合来创建复杂的脚本化工作流，从而执行更复杂的操作。它们还可以与发送到其他应用程序的脚本命令交织在一起，创建强大的应用程序间数据传输解决方案。

**注意**

FileMaker 还包括一个名为 `Perform AppleScript` 的脚本步骤，该步骤允许将 AppleScript 命令嵌入到针对其他应用程序命令的数据库脚本中。

#### 打开数据库

AppleScript 可以使用 `open file` 命令从任何文件夹打开数据库。此示例将三个变量的值设置为数据库的路径、帐户名和密码。这些变量用于指示 FileMaker 打开该文件：

```
set pathToDatabase to "Macintosh HD:Users:shannonmiller:Learn FileMaker.fmp12"
set textAccount to "Admin"
set textPassword to "58Jt234"
tell application "FileMaker Pro"
open file pathToDatabase with passwords textPassword for Accounts textAccount
end tell
```

要打开托管的数据库，`getURL` 命令将接受本章前面描述的 FileMaker URL。

```
tell application "FileMaker Pro"
getURL "fmp19://Admin:58Jt-234@10.0.1.20/Learn%20FileMaker.fmp12"
end tell
```

#### 激活已打开的数据库

要使已打开的数据库显示在文档窗口堆栈的最前面，此示例检查该文档是否存在，然后使用 `show` 命令激活该数据库：

```
set nameDatabase to "Learn FileMaker"
tell application "FileMaker Pro"
if (document nameDatabase exists) = true then
show document nameDatabase
end if
end tell
```

#### 更改布局

此示例使用 `show` 命令激活名为 `Contact` 的布局：

```
tell application "FileMaker Pro"
tell document "Learn FileMaker"
show layout "Contact"
end tell
end tell
```

#### 查找所有记录

要执行相当于 `查找全部` 命令的操作，请使用带有 `every record` 引用的 `show` 命令。

```
tell application "FileMaker Pro"
tell database "Learn FileMaker"
tell table "Contact"
show every record
end tell
end tell
end tell
```

#### 基于字段值查找记录

此示例在 `show` 命令后附加一个 `whose` 子句，以根据某个字段中的特定值进行搜索。

```
set nameTable to "Contact"
set nameField to "Contact Address State"
set textToFind to "NY"
tell application "FileMaker Pro"
tell database "Learn FileMaker"
tell table nameTable
show every record whose cell nameField of table nameTable contains textToFind
end tell
end tell
end tell
```

`whose` 子句可以使用多种运算符来匹配字段中的数据，包括 `contains`、`does not contain`、`is equal to` 和 `is not equal to`。此外，子句可以与其他子句组合以形成复合搜索条件，如下例所示：

```
show every record
whose cell nameField1 of table nameTable contains value1 or
whose cell nameField1 of table nameTable is equal to value2 and
whose cell nameField2 of table nameTable is not equal to value3
```

##### 统计找到集中的记录数

要统计当前窗口找到集中的记录数，需要一条 `tell window` 语句来引用所显示的上下文，无论布局或表如何。

```
tell application "FileMaker Pro"
tell window "Learn FileMaker"
return the number of every record
end tell
end tell
```

##### 统计表中的所有记录数

要统计表中所有记录的总数，无论当前窗口的布局如何，请使用 `tell database` 语句。

```
tell application "FileMaker Pro"
tell database "Learn FileMaker"
return the number of every record of table "Contact"
end tell
end tell
```

# 30. 定义账户与权限

在这个全球网络互联的现代时代，保护数据库安全的需求显而易见。关于数据泄露、信息泄漏以及黑客恶意利用的新闻报道，为我们提供了关于未受保护数据危险的警钟。虽然阻止外部人员的未授权访问非常重要，但安全权限是一个更为复杂的主题。即使是授权用户，也需对其在数据库内部的活动进行限制。限制他们可以创建、查看、修改和使用的*内容*值得考虑。同样，限制对数据库*结构设计*的访问，有助于阻止对模式、界面和脚本函数的粗心或恶意更改。即使是对公式或脚本的微小意外更改，也可能对公司的业务流程造成灾难性影响，并可能导致严重的经济后果。只有具备足够技术技能和业务逻辑知识的用户才应被授权担任开发者角色。本章探讨可用的凭据认证功能，涵盖以下主题：

-   定义默认安全
-   定义用户账户
-   探索权限集
-   在公式中使用凭据
-   利用自定义扩展权限
-   控制文件访问

## 定义默认安全

*账户凭据*定义了谁可以打开文件以及他们被授予了哪些权限。每个 FileMaker 数据库都从默认凭据设置开始，这些设置被配置为不引人注目，并允许任何人立即访问文件的全部内容和结构。任何能看到存储在目录中或由其他客户端共享的文件的人都可以打开它，并拥有*对所有内容的完全访问权限*。该文件将显示为已解锁，并且无需登录提示即可打开。对于在本地计算机上将数据库作为文档文件使用的个人情况，或者在无需考虑安全性的小团队中，此配置可能是可以接受的。然而，在大多数情况下，建议使用凭据认证来保护敏感内容和结构编程。当使用 FileMaker Server 或 FileMaker Cloud 托管数据库时，这是*必须的*。

FileMaker 要求每个数据库至少有两个默认账户设置：一个允许完全开发访问的活动账户，另一个为访客提供只读访问权限的账户。每个新创建的数据库都包含这两个账户。一个名为 `[Guest]` 的账户允许用户*无需*输入用户名或密码即可打开文件，但权限受到严格限制。此账户无法删除，但默认情况下处于禁用状态。启用后，登录对话框中会添加一个*以访客身份登录*按钮。第二个名为 `Admin` 的账户会自动启用，但未设置密码。由于*文件选项*对话框（第 6 章，“数据库文件选项”）中的*登录方式*设置会自动配置为在打开文件时使用此账户登录，因此造成了新文件没有访问设置的假象。实际上，新文件只是绕过了登录屏幕，并以无密码的默认完全访问账户身份登录。

**注意：** 当文件配置为自动登录*完全访问*账户或未分配密码时，FileMaker Server 和 FileMaker Cloud 将拒绝托管该文件。

## 定义用户账户

账户在*管理安全性*对话框中定义，如图 30-1 所示。通过选择*文件*菜单下的*管理 ➤ 安全性*来打开此对话框。它列出了文件中定义的账户，并用于创建、配置、编辑、复制和删除账户。它还有一个按钮，用于访问另一个*高级设置*对话框（本章稍后讨论）。

![../images/429700_2_En_30_Chapter/429700_2_En_30_Fig1_HTML.jpg](img/429700_2_En_30_Fig1_HTML.jpg)

**图 30-1** 用于管理文件访问凭据的对话框

虽然两个默认账户完全存储在数据库文件内部，但也可以定义使用外部授权系统对用户进行身份验证的账户。*通过以下方式验证*弹出菜单可根据选定的身份验证方法切换账户列表。

默认的 *FileMaker 文件或外部服务器* 身份验证选项将两种账户类型合并到一个列表中。*FileMaker 文件*账户使用完全存储在数据库文件内部的账户名和密码进行定义。*外部服务器*账户在文件中仅使用一个名称进行定义，该名称引用了在主机计算机上或集中管理的身份验证服务器（例如 *Apple Open Directory* 或 *Windows Active Directory*）上外部创建的*用户账户组*。这样，各个用户的凭据可以由 IT 部门在计算机级别进行存储和管理，因此人员变动无需在数据库内部进行修改。一个或多个外部用户账户可以由操作系统或身份验证服务器附加到一个组。数据库账户将权限授予分配给该组的任何用户，允许他们使用其外部凭据登录。当数据库解决方案由许多独立的文件组成，并且跨所有文件管理用户更改过于费力时，这尤其有用。外部身份验证仅在数据库由 FileMaker Server 托管时有效。

当开发者的本地应用程序已登录到 *FileMaker Cloud* 账户时，*Claris ID* 选项会出现在身份验证菜单的顶部。通过选择*启动中心*窗口的*我的应用*部分登录账户。然后，返回安全性对话框，定义哪些云用户在该文件托管于该 *FileMaker Cloud* 上时可以访问它。

FileMaker 还包括对*开放授权* (OAuth) 的支持，这是一种开放标准的授权框架，允许第三方系统代表另一个系统对用户进行身份验证，而无需直接共享凭据。托管在 *FileMaker Server* 上的数据库可以配置为使用来自用户的 *Amazon*、*Google* 和 *Microsoft Azure AD* 账户的凭据进行身份验证。

### 默认权限集

每个数据库都有三个默认权限集。它们的名称周围带有方括号，并且除了可以启用和禁用`扩展权限`之外，这些权限集是不可编辑的。

* `完全访问权限` – 允许对`每个可用功能`和所有内容进行不受限制的访问。此权限只应保留给开发者。FileMaker 要求至少有一个允许完全访问的账户。
* `仅限数据输入` – 仅提供对数据输入工作的有限访问权限。分配此权限集的用户可以在每个表中创建、编辑和删除记录。他们可以使用但无法结构性修改布局、脚本和值列表。他们可以打印和导出。空闲时会自动与服务器断开连接，并且可以更改自己的账户密码。
* `只读访问权限` – 与前一权限类似，但更严格，不允许以任何方式修改内容或结构。

# 分配扩展权限

之前在图 30-6 中看到的*编辑权限集*对话框中的*扩展权限*部分，包含一个可以启用的附加权限列表。*扩展权限*是一种基于关键字的权限设置，可以分配给一个或多个权限集。这些权限用于访问特定类型的功能。FileMaker 包含几个固定用于特定功能的扩展权限，主要是各种类型的入站网络和脚本访问。

一些扩展权限允许用户通过特定方法登录。例如，`fmwebdirect` 允许从 Web 浏览器访问服务器托管的数据库，而 `fmapp` 允许从 *FileMaker Pro* 和 *FileMaker Go* 应用程序访问文件。

其他权限则允许外部系统通过特定协议访问服务器托管的数据库：`fmxdbc` 允许通过 ODBC/JDBC 进行入站访问，`fmxml` 允许 XML 访问，`fmphp` 允许 PHP 访问，`fmrest` 允许通过 *FileMaker Data API* 从*表述性状态转移*（RESTful）Web 服务进行访问。

另外两个权限提供脚本访问：`fmextscriptaccess` 允许通过 *AppleScript*（macOS）和 *ActiveX*（Windows）进行访问，而 `fmurlscript` 允许通过 URL 触发数据库脚本（第 29 章，“使用 FileMaker URL”）。

虽然所有默认的扩展权限都是硬编码的，只能简单地打开或关闭 FileMaker 内置功能，但可以定义新的扩展权限来控制数据库中的自定义功能（本章稍后讨论）。

# 在公式中使用凭据

使用*账户*和*权限集*进行凭据验证，在限制对文件、内置功能和自定义结构元素的访问方面表现出色。尽管凭据接口对自定义资源具有相当出色的控制能力，但其覆盖范围仍有一定限制。一个权限集可以控制布局的可见性，但无法限制对单个对象的访问。它可以使脚本可执行，但无法控制脚本中的单个步骤。它无法控制自定义菜单或自定义函数的可访问性或行为，也无法深入公式内部。这些限制仅仅是自定义解决方案的本质所致。在某个节点之后，内置的凭据接口就无法*向外延伸*足够远，来控制每个对象、脚本步骤和自定义配置的权限。超过这个节点，自定义资源必须使用公式来*向内查看*用户的凭据，并确定应施加哪些限制以及执行哪些变体。为此，有三个 *Get* 函数可用于公式中，以施加*基于凭据的条件*。使用这些函数，公式可以识别用户的*账户*名称、*权限集*名称，以及在使用外部身份验证方法时的*组*名称。以下示例展示了每个函数的假设结果。

```
Get ( AccountName )                    // 结果 = 张三
Get ( AccountPrivilegeSetName )        // 结果 = [完全访问权限]
Get ( AccountGroupName )               // 结果 = dbmarketing
```

脚本可以使用 *If* 步骤来限制为特定用户执行一组单个步骤。例如，一个发票报表脚本可能由许多用户运行，但其中有一部分步骤是生成敏感的财务报告，该部分应仅在脚本由首席财务官运行时才执行。账户名可用于确定这些步骤是否应该执行。

```
If [ Get ( AccountName ) = "李华" ]
转至布局 [ "发票 – 特殊报表" ; 动画: 无 ]
排序记录 [ 恢复 ; 对话框: 关闭 ]
打印 [ 对话框: 打开 ]
转至布局 [ 原始布局 ; 动画: 无 ]
结束条件
```

另一个脚本可能会根据用户的权限集导航到特定部门的菜单布局。此示例脚本使用一系列 *If* 和 *Else If* 脚本步骤，每个步骤针对一个单独的布局。

```
设置变量 [ $set ; 值: Get ( AccountPrivilegeSet ) ]
If [ $set = "人力资源" ]
转至布局 [ "主页 HR" ; 动画: 无 ]
Else If [ [ $set = "销售" ]
转至布局 [ "主页 销售" ; 动画: 无 ]
Else If [ [ $set = "生产" ]
转至布局 [ "主页 生产" ; 动画: 无 ]
结束条件
```

这些函数可用于在任何可以使用公式的地方创建特定于凭据的行为。隐藏布局对象或自定义菜单、更改按钮或标签名称、个性化工具提示、在计算字段中产生不同结果等等。*然而，使用此技术应持谨慎态度。*

## 理解嵌入凭据的风险

在公式中嵌入凭据以创建条件性功能应谨慎使用。公司员工的每一次变动都可能需要更改在数十、数百或数千个不同公式中使用的员工姓名。即使像同事结婚更改姓氏这样简单的事情，在复杂的数据库中也可能需要大量工作。虽然可以为新员工更改现有账户的名称，但公式中所有使用旧名称的地方不会自动更新，因为它们是文字文本，而不是动态引用。在人员流动率高的组织中，定位和更改这些名称可能会成为一项全职工作。

这个问题可以通过使用“职务”作为账户名而不是员工的实际“姓名”来部分缓解。例如，张三可以以“账户主管”的身份使用唯一密码登录数据库。然后，如果新人接替该职位，他们沿用相同的账户名和新密码。由于实际账户名未更改，任何使用它的公式也无需更改。然而，这种方法有问题，因为它使用户身份失去了个性，并且在查看历史记录列表时，看到所有记录都由“账户主管”修改，但跨越了三个不同的人担任此职位的时期，这会令人困惑。理想情况下，用户应使用个人标识符登录数据库。

当公式使用*权限集*而不是*账户名*时，条件性功能可以稍微更能适应团队变化。可以根据需要删除和创建单个用户账户，而分配的权限集继续控制公式中的界面和行为条件。虽然这是一个更好的方法，但仍然存在风险因素。除非权限集经过精心设计，几乎无所不能，否则随着公司工作流中角色和职责的变化，某些功能几乎不可避免地需要进行重大更改。此外，由于每个用户只能分配一个权限集，功能可能需要大量的复合子句来授予一组不同权限集的权限。

在公式中使用凭据确实将安全设置的能力扩展到了*管理安全性*对话框之外，但这种方式可能非常耗费人力。一开始可能只是为某个用户或组设置一个简单的临时例外，但很快就会成为一种坏习惯，使开发者陷入困境，因为他们必须不断记住在拥挤界面设计的黑暗角落中丢失和遗忘的数千个凭据例外。幸运的是，FileMaker 有一个极好的替代方案，通过允许将自定义功能例外轻松直接连接到安全接口中来解决这个问题：*自定义扩展权限*。

# 利用自定义扩展权限

*自定义扩展权限*是一种由开发者定义的、基于关键字的权限，可以安全地在公式中使用以创建条件功能。这些权限可以轻松地链接到一个或多个权限集，从而适应公司人员未来的变动。条件公式不是直接连接到不断变化的*账户*和*权限集*，而是通过关键字连接到一个不变的*自定义扩展权限*。随着单个用户账户的创建和删除，以及权限集的修改和重新分配，由扩展权限授予的条件访问和行为可以直接在*管理安全*对话框中轻松启用或禁用。扩展权限允许您为数据库的自定义功能命名，并将其作为新设置插入到凭据接口中，而不是将凭据命名插入到公式中。

要创建自定义扩展权限，首先要确定一组条件功能以及所涉及的对象/公式。为了便于说明，我们使用一个简单的示例：通过点击布局上的按钮来批准*发票*记录的能力。该按钮需要被条件隐藏，以便未经授权的用户无法看到或点击它。首先，为新权限选择一个名称。名称应是一个由两个或多个单词组成的短关键字，该关键字应唯一且能清晰地指示其将控制的功能。对于此示例，像 `InvoiceApproval` 这样的名称将描述我们在示例中定义的能力。

> **注意**
>
> 在多个对象公式中使用扩展权限名称之前，请仔细选择，以避免日后需要多次更改，而这正是我们试图避免的问题！

打开*管理安全*对话框，点击*高级设置*按钮，然后选择*扩展权限*选项卡。这里列出了文件中定义的所有内置和自定义扩展权限。点击*新建*按钮，在*编辑扩展权限*对话框中打开一个新的扩展权限，如图 30-10 所示。

-   获取有关组件数量和使用情况的统计信息。
-   保存数据库在特定时间点的结构快照，该快照可以与之后的结构进行比较，以识别变更。

#### 从文件中删除管理员访问权限

#### 启用 Kiosk 模式

`为非管理员帐户启用 Kiosk 模式`选项会保存文件的副本，该副本对所有非管理员访问强制使用全屏模式且无菜单栏。这是创建触摸屏 Kiosk 的一个很好的选项，在这种模式下，您不希望用户访问除您在布局上提供的界面控件之外的任何内容。数据库必须至少定义一个非管理员用户帐户，并且必须使用该帐户以 Kiosk 模式打开。一旦进入 Kiosk 模式，用户在文件中导航或关闭文件的能力将通过您在布局上提供的控件来实现。将没有菜单栏，甚至访问操作系统功能（如强制退出或切换应用程序）也将被完全禁用。当添加一个用于退出旨在作为公共 Kiosk 的数据库的按钮时，请确保包含某种身份验证过程，以防止普通用户退出方案并访问底层操作系统。

> **警告**
>
> 如果您没有在布局上提供退出应用程序或关闭数据库的按钮，那么唯一的选项就是强制重启计算机，这可能会导致文件损坏。

#### 创建错误日志

`为任何处理错误创建错误日志`选项将在`项目文件夹`中生成一个日志文件，其中包含处理其他实用功能时发生的任何错误。此选项可以与其他任何实用功能一起选择，但在启用加密时是必需的，并且对于加密选项将默认选中。一旦选中，下方的一个额外选项允许通过单击`指定`按钮来自定义日志文件的名称和位置。默认为一个名为“Logfile.txt”的文件，它将保存在`项目文件夹`中。



#### 探索加密功能

FileMaker 包含多种可用于保护数据的加密功能。使用受管外部存储的容器字段（第 11 章）可以配置为自动加密存储的文件。各种内置函数提供了*Base64*和*Crypt*选项，用于对存储在字段中或与其他系统交换的数据进行编码/解码和加密/解密。已安装的*SSL 加密*证书可保护客户端与服务器之间的传输数据；FileMaker Server 推荐使用此类证书，而 FileMaker Cloud 则要求必须使用。此外，通过`Developer Utilities`对话框提供的两个*解决方案选项*可为整个数据库文件提供*静态加密*保护，这在标准凭据访问之外增加了额外保护层（第 30 章）。加密后的数据库在结构上进行了修改，以防止其在存储于磁盘时被未经授权的访问、篡改或分析，例如，如果有人获取了数据库的备份或存档副本。在桌面上打开文件或在 FileMaker Server 上托管文件时，*必须*输入您在加密过程中定义的密码。FileMaker Cloud 要求这样做，并将自动加密上传的文件。一旦托管，用户访问仅需标准凭据。

**注意：**请务必将加密密钥安全存储，因为密钥一旦丢失将无法恢复。

`共享 ID`文本字段包含一个最多 32 个字符的区分大小写的字符串，用于链接多文件解决方案中的所有数据库文件。当使用加密密码打开一个文件时，该文件随后可以访问任何其他已使用*相同共享 ID*和*加密密码*保存的文件，而无需重复提示用户输入加密密码。

##### 移除数据库加密

`移除数据库加密`选项会以未加密状态保存文件副本，恢复正常的操作行为和访问方法。在`指定解决方案选项`对话框中选择此选项，并输入图 31-5 所示所需的设置。输入文件的*加密密码*以证明您有权修改文件。然后，点击`指定`按钮输入具有完全访问权限的*FileMaker 账户*。这两项对于*所有*选定的文件应该是一致的。关闭`指定解决方案选项`对话框以返回`开发者实用工具`对话框。然后，点击`创建`以运行该实用工具。解密后的文件将出现在指定的`项目文件夹`中。
