# 处理字段对象

在 FileMaker 中，术语*字段*的含义会根据讨论语境而有很大差异。请花点时间区分通常被混为一谈的四种事物，如图 20-2 所示：

![../images/429700_2_En_20_Chapter/429700_2_En_20_Fig2_HTML.jpg](img/429700_2_En_20_Fig2_HTML.jpg)

图 20-2 – 术语“字段”各种用途的图示

- 表模式中的*字段定义*，即其底层数据结构
- 数据记录的*字段单元格*实例
- 正在设计的布局上的*字段对象*
- 浏览模式下布局中呈现的*字段显示*

将这四种都称为“字段”本身并无问题，也无需强制使用所示的标准术语。事实上，本书有时也会统一用该术语指代它们。然而，请花点时间理解其中的区别，以避免混淆。

表的*字段定义*是存储*潜力*的独立单元，它确立了数据库模式中的数据类型和设置。这些定义会被复制为*字段单元格*，对应表中数据内容创建的每条记录，成为包含信息的*实际*存储单元。这类似于电子表格的隐喻：字段定义如同*列*，记录如同*行*，而记录中字段的每个实例则是一个*单元格*。在 FileMaker 中，由于用户是*通过*布局与数据结构交互，因此隐喻性的电子表格被隐藏起来，这种区别对用户来说不太重要。

在设计布局时，*字段对象*是一个图形元素，它定义了字段单元格显示的位置、行为和外貌。在布局模式下，*字段对象*被赋予一个*字段引用*，它是指向*字段定义*的指针。在浏览模式下查看时，该布局对象会渲染成*字段显示*，展示当前查看记录的*字段单元格*中存储的信息；其格式和行为遵循*表的字段定义*和*布局的字段对象*两者的设置。

## 向布局添加字段

可以通过多种方法将字段添加到布局中。有些方法会先放置字段对象，然后要求指定字段引用。其他方法则要求在创建对象之前指定引用。还有一种方法可以根据情况双向操作。下面简要介绍所有向布局添加字段的方法。

### 添加后指定字段引用

以下所有方法都会先在布局上添加一个*未定义字段对象*，然后立即弹出*指定字段*对话框（下一节描述），要求选择字段引用：

- 将工具栏中的`字段`工具拖放到布局上。
- 选择工具栏菜单中的`字段/控件`，将光标置于*字段绘制*模式，然后点击并拖动以创建字段对象。单击一次可自动选择默认的`编辑框`控件类型。按住鼠标点击并保持，可从菜单中选择一种控件类型。
- 选择`插入 ➤ 字段`菜单。
- 使用`编辑 ➤ 复制`菜单复制选定的字段。

### 添加时指定字段引用

以下所有方法在添加字段时，引用已作为过程的一部分指定好：

- 从`对象`面板（第 [19]章(#429700_2_En_19_Chapter.xhtml)）的`字段`标签页中，拖放一个或多个字段。
- 复制并粘贴现有字段。
- 创建新布局时，在对话框中添加字段（第 [18]章(#429700_2_En_18_Chapter.xhtml)，“创建新布局”）。
- 使用`编辑 ➤ 复制`菜单复制*多个字段*，它们当前分配的字段将保持不变。
- 按住`Option`（macOS）或`Windows`（Windows）键，拖放现有字段以创建该字段对象的新副本。此操作可在单个布局内或两个窗口的布局之间进行。

当在两个不同文件的窗口之间拖放复制字段时，字段分配会有所不同。如果接收文件在相同表名中定义了与被拖字段同名的字段，则该字段将引用该字段进行放置。如果接收文件没有匹配的字段，但有匹配的表名，则字段将以`<字段缺失>`引用进行放置。如果接收文件既没有匹配的字段也没有匹配的表名，则该字段将作为空白字段对象放置。

## 探索“指定字段”对话框

使用任何需要选择引用的方法添加字段时，都会先临时放置一个空白字段对象，并打开*指定字段*对话框，如图 20-3 所示。在顶部，从菜单中选择一个表 occurrence。当前布局的表 occurrence 始终位于顶部，文件中所有其他 occurrence 则分列于两个列表中：相关和不相关。可以添加任何字段，但只有来自当前表或相关表的字段才能按预期工作。找到并选择字段后，勾选可选地自动创建标签的复选框，然后点击`确定`将字段添加到布局。

### 编辑现有字段的引用

要重新分配现有字段对象的引用，请双击该字段打开*指定字段*对话框，或从其上下文菜单中选择`指定字段`，或点击`检查器`面板中`数据`标签页下`字段`组里“显示数据来自”设置旁边的铅笔图标。

## 处理字段标签

*字段标签*是放置在字段旁边的文本对象，用于帮助用户识别其内容。标签可以手动插入文本对象创建，也可以使用`指定字段`对话框中的`创建标签`选项自动创建。创建后，可以像其他文本对象一样对其进行编辑、移动、格式化和操作。只要没有手动编辑过，自动创建的标签将保持与字段名称*动态链接*。在这种情况下，如果在模式中修改了字段的已定义名称，标签会自动更新以反映新名称。稍后可以通过双击现有字段重新打开`指定字段`对话框，选中`创建标签`框并点击`确定`，为现有字段添加标签。该标签将作为动态链接值重新出现在现有字段对象旁边。

*控件样式*是一种布局设置，用于修改字段的外观和输入选项，以简化数据输入。可以为字段分配七种类型之一，这些类型位于`检查器`面板`数据`标签页下`字段`组的`控件样式`菜单中，如图 20-4 所示。

### 为双字段值列表使用弹出菜单

弹出菜单控件样式具有一项独特功能，可在与双字段值列表配合使用时减少对额外资源的需求。通过配置已分配的值列表使其仅显示*第二个字段*（第 11 章，“使用字段中的值”），并将字段的控件样式设置为启用了*使用值列表覆盖数据格式*选项的弹出菜单，用户将仅看到第二个字段，即使第一个字段会被输入到字段中。

为说明这一点，向 `Contact` 表中的 `Contact Company ID` 字段添加一个用于 *Company* 记录的值列表，该列表将插入公司 `Record ID` 并在联系人与公司之间建立关系。首先，设置一个名为 `Company List` 的值列表，如图 20-18 所示，该列表生成所有公司记录的 `Record ID` 列表，并将 `Company Name` 作为第二个字段。

![../images/429700_2_En_20_Chapter/429700_2_En_20_Fig18_HTML.jpg](img/429700_2_En_20_Fig18_HTML.jpg)

**图 20-18** 按第二个字段排序的双字段值列表示例

接下来，将该值列表作为下拉列表应用于 `Contact Company ID` 字段，如图 20-19 所示。

![../images/429700_2_En_20_Chapter/429700_2_En_20_Fig19_HTML.jpg](img/429700_2_En_20_Fig19_HTML.jpg)

**图 20-19** 分配给字段的值列表（左），配置为下拉列表（右）

通过在值列表定义中包含第二个字段，用户可以按*名称*查看并选择公司。然而，一旦选择并提交后，菜单消失，仅保留*id*可见。传统上，此问题可通过在 `Contact Company ID` 字段与 `Company` 表的 `Record ID` 之间添加关系，然后通过该关系提取 `Company Name` 字段并显示在布局上附近位置来解决，如图 20-20 所示。

![../images/429700_2_En_20_Chapter/429700_2_En_20_Fig20_HTML.jpg](img/429700_2_En_20_Fig20_HTML.jpg)

**图 20-20** 公司名称字段（右）从相关表匹配项（右上）中提取

虽然这能解决问题，但存在一些缺点。它会使布局变得杂乱，同时显示 id 和名称。而且，当您点击 id 字段、按名称选择，id 进入字段，而名称却显示在侧边时，操作会显得有些不自然。由于用户通常无需查看 id，一种创造性的做法是重叠两个字段，使用户只看到顶部的公司名称字段，而隐藏其下方的 id 字段。通过将顶部的名称字段配置为不允许输入，点击操作会穿透它并激活其后方的 id 字段，从而打开菜单。做出选择后，id 字段将被名称字段隐藏。使值列表仅显示第二个字段，会导致 id 几乎完全不可见（菜单打开时除外）。虽然此方法可行，但专用关系与笨拙的字段叠加组合使其不太理想。

相比之下，可以消除所有这些混乱，直接使用可覆盖数据格式的弹出菜单控件样式。首先，扩展 `Contact Company ID` 字段的宽度。然后，确认值列表设置为仅显示第二个字段（如前所述）。最后，将字段配置为显示为弹出菜单，并选择*使用值列表覆盖数据格式*选项，如图 20-21 所示。现在，列表将仅显示公司名称，而该字段虽然仍*包含* id，但会通过布局格式将其覆盖，显示从值列表中选择的值，即公司名称。此方法*无需*重叠字段或要求额外的表匹配项和关系。

**图 20-21** 有焦点（左上）与无焦点（左下）的字段，配置为带覆盖功能的弹出菜单（右）

### 绕过值列表输入限制

使用值列表的四种控件样式均隐含地鼓励遵守某些数据输入限制，即输入*单个值*或将输入限制为*仅限值列表项目*，如表 20-1 所示。值得指出的是，*这些并非严格执行的规则，狡猾的用户可以绕过它们*。就其机制本质而言，弹出菜单和单选按钮都大致排除了用户输入多个项目或不在分配列表中的值的可能性。同样，复选框集（未设置允许其他值的选项）似乎也排除了选择不在列表中的值。然而，所有这些“限制”都是可以覆盖的。在必须限制用户只能使用控件样式提供的选项的情况下，可能需要采取明确措施，使用户无法绕过值列表。

**表 20-1** 值列表驱动控件样式的隐含输入限制

| 控件样式 | 单个项目 | 仅限值列表项目 |
| --- | --- | --- |
| 弹出菜单 | 是 | 是 |
| 下拉列表 | 是 | – |
| 复选框集 | – | 是 |
| 单选按钮 | 是 | 是 |



#### 阻止输入自定义值

有几种方法可以输入字段指定值列表中不存在的值。例如，当字段获得焦点时，可以粘贴剪贴板中的文本。此外，*插入*菜单中的任何文本输入功能都可以用来插入预定义值。用户可以从另一个字段甚至另一个应用程序中拖动一段文本并将其放入该字段。导入记录以及许多脚本步骤都可以绕过该界面。有几种方法可以加强数据输入，使其无法绕过指定的值列表。

##### 使用字段验证

阻止用户绕过指定值列表的一种方法是在字段定义级别使用验证（第 8 章）。这将适用于任何布局上该字段的每次使用（*包括*未使用值列表控件样式的布局），并会向用户发出错误警告。对于需要全面覆盖的情况，这是一个不错的选择。

##### 使用计算过滤错误值

或者，在字段定义级别使用自动输入计算可以自动预处理数据输入，以确认任何输入都是合适的，并过滤掉不合适的输入。这提供了更大的灵活性，因为自定义公式可以根据各种条件动态改变行为，例如，使字段在不同的布局上或根据不同的因素以不同的方式运行。由于该技术会在没有错误对话框的情况下静默地删除错误输入，因此非常适合进行无中断的修正，并且不会减慢用户的数据输入速度。然而，在某些情况下，它可能会让人困惑，对于没有经验的用户来说，这看起来像一个错误，他们不知道犯了什么错误，也不知道他们的输入已被修改。

以下公式假设输入的是单个值，并将自动过滤掉任何不在列表中的输入值。它使用 `Let` 语句和 `ValueListItems` 函数，将名为“公司 - 行业”的值列表中的所有项目拉取到一个名为 `itemsAllowed` 的变量中。然后，`FilterValues` 函数会删除所有不在允许列表中的输入项。这个公式可以通过 `While` 语句进行扩展，以循环处理并拒绝包含多个值的字段中的一个或多个条目，就像使用复选框控件集时一样。

```
Let (
itemsAllowed = ValueListItems ( Get ( FileName ) ; "Company - Industry" )
;
FilterValues ( itemsAllowed ; Self )
)
```

##### 使用脚本触发器

最后，可以使用 `OnObjectValidate` 脚本触发器（第 27 章）来运行一个脚本，验证用户的输入并采取纠正措施，自动清理和/或通知用户其错误。它甚至可以中止验证过程，强制用户在继续之前纠正错误。对于需要特定布局的干预，这是一个不错的选择，可以通过自定义通知自动纠正，或者通知并强制用户更改。

#### 阻止输入多个值

复选框集是唯一一种直观地允许从值列表中选择多个项目的值列表驱动控件样式。其他控件样式在选择新值时总是替换现有值，并且*看起来*限制为只能单选。然而，可以使用以下任何方法从弹出菜单、下拉菜单甚至单选按钮中选择多个项目：

*   点击其他选项时按住 Shift 键。
*   使用 `粘贴` 命令输入一个以回车符分隔的列表。
*   从另一个字段或应用程序拖拽一个以回车符分隔的列表。
*   当字段启用了“允许输入其他值”选项时，用户可以手动输入多个值。
*   使用诸如 `设置字段`、`插入文本`、`替换字段内容`、`导入记录` 等脚本步骤。

##### 限制绕过机制输入多值

要禁止在字段中输入多个值，请使用自动输入计算或脚本触发器来阻止用户输入。

###### 使用自动输入将字段限制为单个值

自动输入计算可以自动删除除一个值之外的所有输入值。在下面的示例中，只会保留最后输入的值。下面的第一个示例使用 `GetValue` 和 `ValueCount` 函数将输入限制为最后输入的值。第二个示例将始终返回第一个输入的值。当此公式执行时，所有其他值都将消失。


## 使用文本

文本对象可以以四种类型之一添加到布局中：*静态文本*、*动态符号*、*合并字段*和*合并变量*。

### 创建静态文本

在布局上，*静态文本*是一个包含不变字符串的对象，用于向用户提供标签或传达信息。静态文本对象可以包含丰富格式和多段落。在布局模式下，通过选择工具栏中的`文本`工具，然后在布局设计区域点击并拖拽以定义对象边界，即可创建静态文本对象。释放光标后，即可开始输入。创建后，可以通过选择工具并点击对象，或双击对象，来编辑和格式化文本。

### 创建动态占位符符号

*动态占位符符号*是文本对象中的一个特殊字符串，它会自动替换为内容随时间变化的预定值类型（例如日期或时间）。有数十种可用符号，每种对应一个`Get`函数（第 13 章，“Get 函数介绍”）。这些符号可以手动输入，也可以通过从`插入`菜单中选择一项来添加到布局中。该菜单会将符号插入到当前具有编辑焦点的文本对象中，或作为新对象添加。符号由双花括号括起的关键词构成。例如，要在布局上显示*当前日期*，请创建一个包含如下值的文本对象：

```
{{CurrentDate}}
```

占位符符号可以与其他符号和静态文本组合，以创建复合字符串。以下示例可以放在打印布局的页脚中，并自动更新以显示报告打印时的日期、时间和用户账户：

```
报告打印于 {{CurrentDate}} {{CurrentTime}}，由 {{AccountName}} 打印
```

### 创建合并字段

与动态函数占位符类似，*合并字段*是一种符号标记格式，允许字段内容显示在布局上的不可编辑文本对象中。使用`插入 ➤ 合并字段`菜单，可以将其添加到当前具有编辑焦点的文本对象中，或作为新对象添加。合并字段由字段名构成，字段名两侧各有两个左尖号和两个右尖号，也可以手动输入。

```
<<字段名>>
```

引用当前布局表的本表字段时，只需字段名称即可。引用相关字段时，必须包含表出现名称以提供上下文参考。以下示例假定放置在`联系人`布局上，将显示联系人的全名及分配给他们的相关`公司`名称：

```
<<联系人>> 在 <<公司>>
// 结果 = 联系人 王小明 在 山韵肩垫有限公司.
```

**警告：** 在复合合并字段字符串中使用的空字段，会使其在周围静态文本构成的句子中显示为空白，导致语句不通！

### 创建合并变量

*合并变量*是一种符号标记格式，允许全局变量（第 12 章，“变量”）在布局上以文本对象的形式呈现。与合并字段和动态符号类似，合并变量可以与其他合并变量、文本、符号及合并字段任意组合使用。使用`插入 ➤ 合并变量`菜单，可以将其添加到当前具有编辑焦点的文本对象中，或作为新对象添加。合并变量由变量名构成，变量名两侧各有两个左尖号和两个右尖号，遵循如下模式：

```
<<$变量名>>
```

*按钮*是一种布局对象，它将操作控件嵌入到对象中。按钮在浏览和查找模式下工作，根据类型和配置的不同，可以运行脚本、执行单个脚本步骤或打开弹出界面元素。按钮允许开发人员将活动控件嵌入界面，用于导航、创建、删除或执行任何其他功能。这些控件为用户提供了快捷方式，减轻了手动重复复杂任务的负担。FileMaker 提供了四种按钮选项：三种原生按钮类型——*按钮*、*弹出按钮*和*按钮栏*——以及将*任意对象*格式化为按钮的功能。

要创建弹出按钮，请选择*插入 ➤ 弹出按钮*菜单，或按住*按钮*工具栏图标，然后从菜单中选择*弹出按钮*，如图 20-34 所示。然后在内容区域中单击并拖动以绘制对象的边界。

#### 定义基本弹出按钮界面

创建后，新的弹出按钮会显示为一个带有打开弹出界面的按钮，如图 20-35 所示。

![../images/429700_2_En_20_Chapter/429700_2_En_20_Fig35_HTML.jpg](img/429700_2_En_20_Fig35_HTML.jpg)

**图 20-35** – 准备好进行编辑的新弹出按钮

弹出按钮界面由以下元素组成：

1.  **按钮** – 当弹出窗口关闭时，按钮保留在布局上。与标准按钮一样，在您指定标签和/或图标之前，此按钮为空。
2.  **界面/设计区域** – 当用户点击按钮时，此区域会弹出。在布局模式下，这是一个可调整大小的设计区域，您可以在其中根据需要添加布局元素。
3.  **关闭按钮** – 此按钮仅在布局模式下编辑弹出按钮时可见，它将关闭弹出界面。
4.  **标题** – 弹出按钮的可选标题。

> **提示：** 点击弹出设计区域可显示渐变控制条和调整大小手柄。
