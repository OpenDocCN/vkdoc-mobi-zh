# 管理字段

在“管理数据库”窗口的“字段”标签页中，可以创建、复制、重命名、修改和删除字段。

## 创建新字段

在“管理数据库”对话框的“字段”标签页中，通过在“字段名称”文本区域输入名称来手动创建新字段。然后在“类型”菜单中为字段选择数据类型，并单击“创建”按钮。新字段应出现在列表中。创建新的“汇总”或“计算”字段时，FileMaker 会根据所选类型立即打开一个选项对话框。对于现有字段，以后可以通过单击“选项”按钮（本章后续部分介绍）打开这些选项。

## 复制现有字段

当新字段与现有字段相似时，可考虑复制以节省时间。选择一个或多个字段，然后使用“复制”和“粘贴”按钮，或单击“复制”按钮。所复制的字段将在列表中高亮显示，每个字段与原字段同名，但根据您使用的复制方法，会附加一个唯一的数字后缀或 `副本`后缀。复制字段的所有设置将与原字段相同。但是，现有记录中的字段内容不会从原字段复制过来，因此复制字段的内容为空。字段可以粘贴到同一数据表、同一文件中的不同数据表，或另一个数据库文件的数据表中。

## 修改字段名称或注释

可以通过选择现有字段，在相应的文本区域输入新值，然后单击“更改”按钮来修改其名称和注释。

## 修改字段类型

字段一旦创建并输入数据后，很少需要更改其类型。但是，如果出于错误或需要更改数据库结构的需要，请选择该字段，从“类型”菜单中选择一个新值，然后单击“更改”按钮。根据原始类型和新类型的不同，可能会出现一个对话框警告：所有现有记录中该字段的现有内容将转换为新类型。将 `文本`更改为 `日期`或 `时间戳`会将之前作为文本输入的日期转换为正式日期，并根据需要自动将年份转换为四位数（参见本章前面的“两位数字日期转换”）。将 `容器`更改为其他数据类型将删除字段中的任何非文本数据。例如，实际嵌入的文件或图像数据将被删除，只保留文件的名称或路径。更改为`计算`或`汇总`将导致任何先前手动输入的内容永久丢失。
