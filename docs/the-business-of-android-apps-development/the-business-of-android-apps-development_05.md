# 第 3 章：法律问题：安全总比遗憾好  

罗伊曾涉及多起法律诉讼，他可以告诉你，这绝不是什么好事。了解法律如何影响你作为应用开发者的角色，可以让你远离麻烦。我们并非律师，因此无法在法律上给你任何具体的法律建议。话虽如此，或许我们可以推荐一些你可能觉得有用的法律资源，以避免不必要的麻烦。  

## 聘请律师——控制成本  

你可能会因为费用问题而犹豫是否聘请律师。虽然律师通常不便宜，但他们往往比你可能面临的后果要便宜得多。如果你对应用的合法性有任何疑虑，或者对法律事务有任何不确定，应立即联系律师。大牌律所业务出色，但收费也非常昂贵。如果你资金有限，应考虑找一家较小的律所，或者找一位独立执业的律师。通常，他们的收费比大律所更合理。虽然他们可能没有大律所那么深厚的专业积累，但一般也能很好地提供基本的法律服务。理想情况下，你可以找到一位曾在大律所工作、如今已独立执业的律师。他们既有大律所的工作经验，收费却更为合理。  

请记住，与律师协商费用是有可能的。如果你与多家律所接触，你就有更多的议价筹码。对于常规工作（例如公司注册或撰写通用雇佣合同），通常可以要求固定费用。然后你可以在不同律所之间货比三家，进行直接的价格比较。  

年轻律师，即所谓的“初级律师”，通常刚从法学院毕业没几年，往往需要在职培训。要小心别花钱去培训一位律师。你可以要求律所不要在你的案件上使用初级律师。通常，你希望由律所的合伙人或资深律师来处理你的工作。只与那些已经做过你所需类型工作的初级律师合作。典型的安排是初级律师完成大部分工作，然后由合伙人复核。自然，如果初级律师做得不对，你就在为合伙人纠正错误而买单。你应当注意不要为律师之间的内部沟通付费。一般来说，涉及的律师越少，流程效率就越高。  




如果你开发应用时以从投资者处筹集资金为目标，有时可以说服律师提供无需预付费用的咨询，这种安排被称为“递延补偿”。这通常意味着将一定数量的计费小时数推迟到你与律师商定的融资门槛达成后再收取。律师还可能协商获得你创业公司的部分股权。一旦你达到融资门槛（通常是首轮融资），你就需要支付未结清的款项。这种交易通常只有专门帮助初创公司的律所才会提供。该领域在业内被称为“风险投资法律”。通常，风投律所会在你的公司未能达成融资目标且被迫解散时放弃收费。具体条款通常取决于你向律师推销自己和创意的能力。请记住，这种支付结构的前提是，你计划筹集足够资金来覆盖法律费用。一家风投律所可能会提供 1 万到 2.5 万美元甚至更多的递延咨询服务。切记，递延不代表免费；你要警惕服务收费过高。不道德的律师可能会对同等服务收取更高费用，因为他们知道你在此种收费结构下不会太关注成本。

从外部投资者处筹集资金的过程超出本书范围，但请记住，若选择此路，你将稀释公司所有权，并耗费大量时间用于融资而非实现创意。如果你认为市场规模足够大，这或许是个好选择，但需权衡利弊。

请谨记，在聘请律师时，务必注意你所接受的条款。如果访问《美国新闻与世界报道》旗下的 Best Law Firms 网站（`http://bestlawfirms.usnews.com/`），你会看到一个搜索引擎，可帮助你查找当地专精“风险投资法律”或“公司法律”等业务领域的律所。

## 成立公司

如果你以个人名义发布 Android 应用，法律上该业务被视为独资企业。你当然可以随心所欲地命名业务，但作为独资企业，你需对任何债务、不当行为和/或疏忽承担个人责任。例如，若你订购物资后因收入低于预期而无法支付，你需个人承担债务。若你的地图应用导致某人驾车坠崖，原则上他们可以直接起诉你个人索赔。另一方面，当应用由公司（包括有限责任公司，即 LLC）拥有时，责任通常归属于该独立法律实体。更详细地说，成立法律实体可将企业的资产与负债与所有者(们)的资产和负债分离。因此，通常宜成立公司实体，以保护自己免受与你编写应用相关的诉讼和债务影响。成立公司还可能带来税务优势：你可以抵扣某些商业开支，长期来看能节省资金。

请记住，必须谨慎维护资产与负债的分离，否则法院可能不会保护你免于个人责任。例如，若你将公司资金用于个人事务，这明显违背了个人资产与公司资产分离的原则，也属非法行为，可能导致债权人或股东以挪用公款罪名起诉。

你可以选择不同类型的公司实体。C 股份有限公司、S 股份有限公司和有限责任公司（LLC）各有独特优势和税务影响。法律专家已就此主题撰写了许多书籍，由于我们并非律师，建议你参考这些书籍了解更多细节。

## 最终用户许可协议与隐私政策

EULA 即“最终用户许可协议”。通常，EULA 会向用户展示复杂的法律条款，用户必须同意才能访问网站或安装软件包。

本质上，EULA 是你与用户之间的合同。它规定了最终用户使用你应用的条件和条款。EULA 提供了一种保护方式，防止因软件损坏用户计算机或数据，或因用户以你从未预期的方式使用应用而引发的诉讼。例如，若你的应用用于报税，你可以通过 EULA 限制用户因错误使用软件而被国税局处罚时的损害赔偿。顺便一提，如果这是你的应用创意，我们强烈建议你在继续前进前先咨询律师。

关于 EULA 的法律可执行性存在许多复杂的法律问题，因此如果你认为应用需要 EULA，我们再次建议你联系有起草此类合同经验的律师。许多 Android 应用并未附带 EULA，通常是大型应用公司（有能力聘请律师）才会包含。但每种情况都不同，你需要深入思考是否愿意放弃 EULA 所能提供的法律保护。

网上通常能找到 EULA 范例。EULA 种类繁多，我们无法推荐一种适合你的特定需求。一如既往，若对此问题有疑问，我们建议你咨询律师。

## 知识产权

你计划通过分发实际上只是一堆存放在内存中的程序和数据的某个东西来赚钱。幸运的是，即使软件不是实物，你仍然可以像保护有形资产一样保护它免遭盗窃。另一方面，你也可能被指控窃取知识产权。我们想你会同意，每个开发者都应该对知识产权法律有一些基本了解。让我们聚焦于知识产权的四种不同类型：版权、商标权、商业秘密和专利权。

### 版权

版权赋予原创作品的创作者（例如书籍、照片或软件）在有限时间内的专有权利。具体来说，版权持有人有权决定谁可以复制、分发、公开表演、修改或基于其原创作品创作衍生作品。这意味着作为应用的创作者，在版权有效期内，你独享复制、销售或传输该应用的权利。当然，你也可以选择将该权利转让给他人。在大多数司法管辖区，版权在作者去世后至少持续 50 年，通常会更长。

出于这些原因，你可能对获得版权感兴趣。好消息是获得版权非常容易。从法律上讲，从你落笔到纸上或向电脑输入字符的那一刻起，版权保护即告成立。版权可以保护诸如源代码、图形、音效以及你在应用程序中投入的其他原创创意作品，但它不保护那些未以创意或艺术方式使用的任何事实或思想。要保护一项发明，你应该申请专利，正如本章后续“专利权”部分所述。

获得版权可以为你提供基础层面的保护，这在起诉明显模仿你作品的人时会派上用场。上一句中强调的重点是*明显*，因为版权保护可以扩展到竞争对手窃取你的图形或应用程序本身的情况，但当模仿者的应用程序恰好与你的基本想法相同（巧合）时，则不适用于此。

为了享受版权保护，并非必须注册版权。然而，版权注册是有效版权的*初步证据*。如果在作品发表后 3 个月内或在侵权行为发生前完成注册，注册将使版权持有人能够寻求法定赔偿和律师费。否则，权利人只能就实际损失和利润提起诉讼。

注册可以在线完成，要求你上传源代码的前 25 页和后 25 页（最好采用 PDF 格式）。由你自行决定应用程序的首页和末页是什么。应用程序的每个新版本都必须重新注册。你也可以为你的应用程序截图进行注册。详细说明请见此处：

```
http://www.copyright.gov/circs/circ61.pdf
```

虽然并非享受版权保护所必需，但包含版权声明（无论是文字形式还是使用版权符号）有助于克服“善意侵权”的推定。换句话说，如果版权侵权者看到了你的作品受版权保护，那么复制行为就不可能是善意的。

目前的申请费是 65 美元，但如果通过`http://www.copyright.gov/eco/`在线提交，则仅需 35 美元。你可以在`http://www.copyright.gov/eco/eco-tutorial.pdf`的教程中找到许多关于电子申报问题的答案。

### 商标权

现在我们来谈谈商标权。商标旨在防止他人将你公司的产品和服务与其他公司的产品和服务混淆。商标不保护概念或思想，但它能保护你免受他人使用你的名称、标识或其相似物进行经营。在当今市场，保护你的品牌极其重要。想想像麦当劳、微软这样的公司，以及为了讨论需要，谷歌自己的安卓系统。毫无疑问，它们标识的形象会浮现在脑海中：分别是金色拱门、Windows 标志和那个绿色机器人。商标法阻止竞争对手以可能混淆消费者的方式使用这些标识。

与版权不同，商标权并非自动授予。但是，根据一项名为“假冒”的普通法原则，你或许可以对未经许可的商标提起法律诉讼。当你使用某个名称或符号作为安卓应用的标识时，你可能就已经获得了该项法律下的保护。然而，法律要求你必须证明该交易商（即你）拥有“商誉”。除非你拥有大量通过你的商标认识你的客户，否则这很难做到。在美国境内，你可能希望通过向美国专利商标局提交商标注册来获得更好的保护，这是在联邦法院提起商标诉讼所必需的。一旦你的商标获得批准，你就得到了保护，无需证明你的商标是众所周知的。在美国以外，你会发现大多数国家按照世界贸易组织《与贸易有关的知识产权协议》的规定提供类似的商标保护。在全球范围内寻求商标保护可能成本高昂，因此你应该思考在哪些地区你最需要这些保护，然后将申请限定在这些区域。

在其他问题中，如果其他人已经在使用你计划使用的商标，你不仅可能得不到任何保护，甚至还可能侵犯了他们的商标。美国专利商标局维护着一个注册商标数据库。你可以使用商标电子检索系统免费搜索该数据库，但你应该聘请商标律师来帮助你解读结果并注册商标。你可以通过以下链接访问商标流程和搜索详情：

```
http://www.uspto.gov/trademarks/
```

### 保护你的商业秘密

在开发你的知识产权时，有些事情你可能不会向普通人透露。作为一名专业博主，Mark 不会在他的个人博文中列出他的商业联系人，并且他有一些策略不会与所有人分享。这类事物受商业秘密法保护。

特别是在美国法律下，“根据美国法典第 18 编第 1839(3)(A)、(B)条(1996 年)的定义，商业秘密包含三个部分：(1)信息；(2)为保护该信息而采取的合理措施；(3)该信息因不公开而具有独立的经济价值。”商业秘密不像版权、商标或专利那样有固定的存续期限。但是，独立发现你秘密的第三方并不违反其规定。



正如您所见，如果您期望获得商业秘密法的保护，就必须采取合理措施来保护相关信息。例如，您需要为关键数据设置密码保护，并粉碎任何包含敏感信息的文件。如果有人为您工作或与您一同开发 Android 应用，他们必须了解将所有应用相关信息保存在私有驱动器和服务器上的重要性，并且要求他们签署保密合同（如保密协议（`NDA`））至关重要。

网上有许多`NDA`样本可供参考。您可以利用它们入门，但如同所有法律事务一样，您应该始终咨询律师，以解决针对您个人情况的任何法律问题。

以下是一些供您查阅的样本：

```
http://www.hbs.edu/entrepreneurship/pdf/Sample_NDA.pdf
http://www.bitlaw.com/forms/nda.html
```

如果您的商业秘密被盗，并且您能向法庭证明您已做出合理努力来保密，那么您就有权获得不同的法律救济，包括禁令和损害赔偿金。

## 专利

在讨论了版权法、专利法和商标法之后，现在我们进入专利部分。专利是发明的一份书面公开声明。作为公开披露（理论上丰富了世界的知识）的回报，发明人有权在一段时间内禁止他人制造、使用、销售、许诺销售或进口该发明主题。大多数 Android 开发者不太可能从申请自己的专利中受益，因为获取专利的成本极高。起诉其他公司侵权则更加昂贵；一场专利诉讼轻松就能达到七位数。一项普通专利在其有效期内很容易花费 10,000 美元甚至更多。此外，很少有 Android 应用能够符合专利申请条件。

如果您决定申请专利，请注意并非任何想法都能获得专利。任何“新颖且有用的流程、机器、制造物或物质组成，或对其任何新颖且有用的改进”都可能获得专利，但发明主题还必须具备新颖性和非显而易见性。专利审查员会将您的发明与现有发明（即现有技术）进行对比，以确定您的发明是否同时满足新颖性（新）和非显而易见性的要求。显然，如果您的想法已被发明，您就不应期望获得专利。此外，任何单独或组合起来会使您的发明变得显而易见的类似发明，也会阻碍专利性。一位优秀的专利律师（或一位糟糕的专利审查员！）可以帮助您的专利更容易通过审查。

在继续开发应用之前，您应该搜索以确认您的应用想法是否已存在（是否有现有技术？）。如果存在但未获得专利，那么您将无法获得专利。如果存在且*已经*获得专利，那么您尝试自己制作该应用就会侵犯专利权人的权利。一如既往，我们建议您联系律师；在专利方面，则需要联系专利律师。不过，简单的谷歌搜索是您自己查找现有技术的好方法。此外，谷歌可以通过以下链接专门搜索专利：

```
http://www.google.com/?tbm=pts
```

假设您确信自己的应用符合新颖性和非显而易见性的要求。您也确信它带来的收益将远超过申请专利所需的 10,000 多美元。一种对冲风险的方法是提交临时专利申请。这是一种法律程序，允许您向专利局发送一份您发明的基本描述，并利用该描述为您发明确立一个专利申请日。该流程对于小型实体仅需 130 美元，但费用每年会变动。您有一个不可延长的 1 年期限，可以按正常费用提交正式专利申请。临时专利的优势在于，它允许您以低成本声明您的发明，然后您有一年时间来决定是否值得投入“真金白银”来确保您的专利权。临时专利还授予您使用“专利未决”（`patent pending`）这一术语的权利，您在许多产品广告中都能看到它。如果您想知道它是什么意思，它仅仅是向竞争对手表明，针对您的产品和/或服务，有一份专利申请正在处理中。您可以通过以下网站在线提交临时专利申请：`https://efs.uspto.gov/efile/portal/efs-unregistered`。

## 许可

在编写应用时，利用开源软件是很有诱惑力的。有大量免费软件库能实现各种神奇的功能。然而，其中大多数都遵循各种开源许可证进行授权。未能遵守这些许可证的条款可能会使您面临法律诉讼。例如，`Creative Commons`许可明确禁止任何商业用途。`GPL`（通用公共许可证）在将您的非`GPL`软件与`GPL`库链接方面引发了许多法律歧义。有些人认为，虽然静态链接`GPL`库违反了许可证，但动态链接是可以的。相比之下，`LPGL`（宽松通用公共许可证）明确允许与非`LGPL`软件进行链接。以上仅仅是许可条款的几个例子。简而言之，您在使用某款免费软件之前，应该研究其许可证。

许多专利发明也可以进行许可。在软件领域，需要许可的技术包括`MPEG`、`MP3`和`Dolby Digital`。在许多情况下，手机（或平板电脑）制造商已经协商了这类许可，但如果您发现自己在使用一个 Android SDK 中未包含的标准，您可能需要仔细核实以避免麻烦。您可以通过以下链接找到常见许可标准的列表：

```
http://www.musemagic.com/papers/licensinginfo.html
```

此外，如果您发现有人已经为您打算实现为应用的确切想法申请了专利，那么在放弃之前联系他们可能是值得的。发明者通常有兴趣许可他们的想法，并希望有人能出于这个目的联系他们。
