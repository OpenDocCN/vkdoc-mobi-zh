# PhotoScroller 项目与分页技术

`PhotoScroller`项目还演示了一种名为**分块**（tiling）的高级技术。在第 5 章的开头，我解释了 iOS 设备没有足够的内存或 CPU 能力来为表格创建数千个单独的行对象。相反，它只绘制用户可见的表格部分，当用户滚动列表时动态更新。

任何异常大的内容视图都可能属于同一类别。`PhotoScroller`项目演示了如何动态地只准备那些当前通过滚动视图的“窗口”可见的视图对象。表格视图（正如你所记得的，它基于`UIScrollView`）已经做到了这一点，只为其在表格中可见的行准备视图对象。

滚动视图的一个不太常见的用途是查看“分页”内容。这可以通过将`pagingEnabled`属性设置为`true`来实现。当你这样做时，滚动视图会强制内容视图（具体来说是它的`contentOffset`属性）以离散的步长移动，这些步长精确是其框架尺寸的整数倍。从概念上讲，它将你的内容视图划分为一个网格（大小恰好等于窗口的大小），任何滚动最终都会停留在其中的一个片段上。有一个名为`PageControl`的示例项目演示了这个功能。

**注意**：`PhotoScroller`项目允许你在图像之间滑动，但它并没有使用`UIScrollView`的分页功能。相反，它使用了`UIPageViewController`。你将在第 12 章中使用`UIPageViewController`创建一个类似的界面。

滚动视图的高级用法不适合胆小者。这可能非常复杂，但正是这些复杂的设计造就了真正酷炫的应用。如今已成为 iOS 应用主流的“下拉刷新”手势，完全是通过滚动视图和滚动视图代理实现的。如果你需要在表格视图中使用此功能，大部分工作已经为你完成：创建一个`UIRefreshControl`对象，并将其连接到表格视图控制器的`refreshControl`属性。现在，用户可以通过下拉来刷新表格。要深入探索滚动视图的强大功能，请从 *Scroll View Programming Guide for iOS* 开始。
