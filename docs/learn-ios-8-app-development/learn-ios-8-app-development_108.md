# 地理编码与路线规划

地理编码通过 `CLGeocoder` 对象执行。`CLGeocoder` 会将描述性信息字典或地图坐标转换为 `CLPlacemark` 对象。地标对象是地图坐标与坐标处描述的集合。这些信息包括坐标所在的国家、地区和城市、街道地址、邮政编码（如适用），甚至能识别该位置是否为水域。

### 获取路线规划

你的应用可调用的另一项资源是地图应用——没错，就是 iOS 自带的**标准地图应用**。有一些方法能让你的应用启动地图应用来协助用户。这是一种简单的方式，无需在应用中添加任何地图、定位、路线和导航功能，即可为用户提供这些服务。

你通过 `MKMapItem` 对象使用地图应用。你可以从当前位置（`mapItemForCurrentLocation()`）或地理编码后的地标对象（`MKMapItem(placemark:)`）创建一个或多个 `MKMapItem` 对象。

创建后，调用地图项对象的 `openInMapsWithLaunchOptions(_:)` 函数（适用于单个地图项），或将地图项数组传递给 `openMapItems(_:,launchOptions:)`。启动选项是一个字典，可告知地图应用要显示哪个地球区域、高亮地图上的哪个区域、是否提供驾车或步行路线、使用何种显示模式（地图、卫星、混合）、是否显示交通信息等。

使用 `MKMapItem` 的代码示例见《定位服务与地图编程指南》中的“提供路线规划”章节。

如果你想在应用中提供路线规划，请使用 `MKDirectionsRequest` 类。首先为行程起点和终点创建两个 `MKMapItem` 对象，然后启动 `MKDirectionsRequest`。该对象会联系地图服务器，返回一条或多条可能路线。这些路线由 `MKRoute` 对象描述。若要在地图中显示某条路线，请获取 `MKRoute` 对象的 `polyline` 属性；它是一个 `MKPolyline` 覆盖对象，可直接添加到地图，操作方式与你之前在 Pigeon 应用中做的完全一致。更多详情及示例代码，请参考《定位服务与地图编程指南》中的“提供路线规划”章节。

App Store 上的 Homer 应用同样执行地理编码并获取路线规划。你可以在 `https://github.com/JamesBucanek/Pigeon` 下载 Homer 的源代码。
