# 排版后的内容

但如果你需要在多个不同的地方，对同一类视图对象进行完全相同的调整呢？你是否需要在每个视图控制器中都编写代码来修改每个对象？假设你正在开发一款多人游戏，玩家要么属于 **Rose 队**，要么属于 **Periwinkle 队**。你希望所有按钮和导航栏的背景色，在 Rose 队玩家看来是浅红色，在 Periwinkle 队玩家看来是浅蓝色，并且这些视图要出现在每个视图控制器的每个位置。

实际上，有一种方法可以通过外观代理（*appearance proxy*）来实现这一点。*外观代理* 是一个对象，它可以一次性调整其所属类中所有视图对象的一个或多个属性。其工作原理如下。

你向特定的视图对象类请求其外观代理：

```
let navBarAppearanceProxy = UINavigationBar.appearance()
```

如果你设置了代理对象的显示属性，它会将该属性应用于应用中每一个 `UINavigationBar` 对象，即使是那些尚未加载的也不例外：

```
navBarAppearanceProxy.barTintColor = roseTeamColor
```

**注意：** 你还可以请求一个仅适用于特定特征环境或包含在特定视图层次结构中的对象的外观代理。

代理并不能影响所有属性。只有那些在框架头文件中标记有 `UI_APPEARANCE_SELECTOR` 的属性，才是它可以更改的。我从未找到过官方的列表，但大多数视觉属性（如背景、色调、字体等），如果需要统一应用于多个视图对象，都可以通过这种方式进行自定义。

你可以在 `UIView` 的自定义子类中实现自己的外观属性，并且你的类的外观代理也会调整这些属性。有关详细信息，请参阅 `UIAppearance` 的文档。
