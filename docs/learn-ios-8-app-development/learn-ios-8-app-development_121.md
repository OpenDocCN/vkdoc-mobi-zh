## Swift 的（简短）历史

大多数计算机语言的历史可以追溯到几十年前。`Swift` 是一个例外，它像智慧女神雅典娜从宙斯的额头中全副武装地诞生一样——或者，在这个例子中，是来自苹果公司。`Swift` 的概念只有几年历史，直到苹果公司在 2014 年全球开发者大会上发布它之前，几乎没有任何苹果公司以外的人听说过它。

`Swift` 融合了许多现代语言特性，但并未（至少目前还没有）受到遗留特性和语法的拖累。它是一种简洁、快速、高效、精炼、安全、现代且灵活的语言。令人惊讶的是，它在实现所有这些目标的同时，还能与现有的 `C`、`Objective-C` 和 `Objective-C++` 代码保持二进制兼容¹。在深入了解 `Swift` 的具体使用之前，让我简要解释一下这些概念。

`Swift` 是一种简洁的语言。这里的简洁，我指的是它是正交且规整的——这些是描述一致性的技术术语。在许多较老的语言中，新特性被叠加在其他特性之上，而这些特性又是叠加在原始语言之上的。最终你会得到一个语法和规则拼凑起来的补丁，而这些规则并不总是一致的。例如，在 `Objective-C` 中，你使用一种语法来表示 `Objective-C` 字符串字面量，使用另一种语法来表示 `C` 字符串字面量，再用第三种语法来表示字符字面量；而且这三种类型彼此不兼容。`Swift` 消除了这种不一致性——只要在逻辑上是可能的。

`Swift` 是快速的。在过去的几十年里，编译器开发者们学到了很多关于如何将你编写的代码转换为高效机器指令的知识。其中一条经验是：编译器越了解程序员想要做什么，它就能越高效地生成代码。`Swift` 的语法旨在巧妙地将这些信息传达给编译器。`Swift` 语法倾向于描述你*想要*发生什么，而不是你*希望*它如何发生。这让编译器能够为你找到最佳解决方案。

`Swift` 也是一种完全现代化的语言，借鉴了诸如 `Rust`、`Haskell`、`Ruby`、`Python` 等语言的创新特性。然而，这并不是什么新鲜事。软件工程的世界里充斥着各种计算机语言的“遗骸”，这些语言引入了各种强大的新编程范式，其中大部分由计算机科学家开发。这些语言中的大多数最终被淘汰，因为它们也未能满足行业对性能和兼容性的要求。相比之下，`Swift` 最初是由一位“编译器专家”构思和设计的。`Chris Lattner`，苹果公司首席编译器工程师，也是 `LLVM` 和 `clang` 编译器的主要开发者之一，构想了 `Swift` 作为一种不会牺牲速度或兼容性的现代语言。换句话说，你终于可以“鱼与熊掌兼得”了。

最重要的是，`Swift` 是一种安全的语言。它的语法和规则旨在禁止许多常见以及一些不常见的编程错误，这些错误是全世界软件开发人员的祸根。2014 年，科技界因保护互联网数据的核心加密算法之一中发现的漏洞而震动。这个漏洞是由一行 `C` 代码中一个误放的分号（`;`）引起的，这是自该语言诞生以来 `C` 程序员就一直犯的经典错误。`Swift` 的语法经过精心设计，旨在帮助你避免犯这类简单但有时是灾难性的错误。（这并不意味着你不能写出有缺陷的软件；只是更不太可能归咎于 `Swift` 本身。）

最后，`Swift` 是精炼的，但不晦涩。所谓精炼，是指用 `Swift` 编写的同一个程序，很可能比用 `C`、`C++`、`Java`、`C#` 或 `Objective-C` 编写的相同代码要短。这意味着你的代码量更少，更易于阅读，也更易于编写。这间接意味着你需要写的东西更少，而且你写的东西也更容易理解。以下 `Objective-C` 语句将一个 `Person` 对象数组按出生日期升序排序：

```
[people sortUsingComparator:^NSComparisonResult(id l, id r) {
        Person *p1 = (Person*)l;
        Person *p2 = (Person*)r;
        return [p1.birthdate compareDate:p2.birthdate];
    }];
```

将其与 `Swift` 中的相同代码进行比较：

```
people.sort { $0.birthdate < $1.birthdate }
```

相比之下，一门晦涩的语言（例如 `Perl`）可以用少得惊人的代码完成任务，但牺牲了可读性。例如，下面是用 `Perl` 编写的一个完整程序：

```
print("".(($line=join("",<>))=~s/.*\n/$h{$&}++?"":$&/ge,$line));
```

如果你读了那段语句并说道“哦，是的，那是读取一个文件并使用哈希表消除所有重复记录”，那么你是一位天生的计算机语言大神，可以跳过本章的其余部分了。

现在大神们已经离开了，我应该指出，`Swift` 在很大程度上仍是一项进行中的工作。我在 2014 年 10 月撰写的 `Swift` 版本，在很多方面，与几个月前在苹果公司 `WWDC` 上发布的版本有着显著的不同。我预计 `Swift` 会继续变化，有时甚至是剧烈变化，随着它的发展。因此，请对你在此读到的东西持保留态度，并阅读有关 `Swift` 最新改进的资料。一个好的起点是 `Swift` 博客：`http://developer.apple.com/swift/blog/`。是的，`Swift` 有一个博客。

## 类

到目前为止，你已经看到过这个上百次了，我在第 6 章中也解释过，但在这里再次呈现它的完整面貌。在 `Swift` 中，类通过一个 `class` 指令声明，如下所示：

```
class ClassName: SuperClassName, ProtocolName {
    private var instanceVariable: Int = -1
    func instanceFunction(firstParam: Int, secondParam: Int) -> Bool {
        return firstParam >= secondParam
    }
}
```

以下是重点：

*   `class` 声明通知编译器你在声明一个新类，它继承自哪个类，采用哪些协议，拥有哪些属性，以及定义了哪些方法（实例函数）。
    *   父类是可选的。如果你的类不继承自现有类，可以省略父类名称。
    *   协议是可选的。如果你的类不采用任何协议，可以省略协议名称列表。
*   类的主体由它的属性和函数组成。在 `class` 语句内部声明的任何内容都属于这个类。
*   你可以在属性或函数声明之前，选择性地插入访问控制指令（`public`、`private` 或 `internal`）。

**提示** 按照惯例，类型名称以大写字母开头（`MyClass`、`ViewController`、`Weekday` 等）。变量名称以小写字母开头（`dataSource`、`frame`、`userInteractionEnabled` 等）。
