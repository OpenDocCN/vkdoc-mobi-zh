# 归档版本控制

在实现 `NSCoding` 时，你可能需要考虑类发生变化时会发生什么。将对象归档到持久化存储中的一个后果是，数据是——嗯——持久的。用户会期望你的应用能够打开多年前创建的文档。我一直在努力改进我的软件，我相信你也是如此。我经常向类中添加新的属性，或者更改属性的类型和范围。这通常意味着创建新类，并定期废弃旧类。所有这些更改都会改变类的编码方式，并在解档由你软件的旧版本（有时甚至是新版本）创建的数据时带来挑战。

有许多处理归档兼容性的技术。你的新代码可以使用不同的键来编码其值。在解码时，你的软件可以测试该键是否存在，以确定归档数据是由新版本还是旧版本软件创建的。你可以在归档数据中编码一个“版本”值，并在解码时测试该版本。较新的软件可以用新格式和旧格式编码同一个值，这样旧版软件（对新格式一无所知）仍然可以解释文档数据。

甚至还有在解档过程中用一个类替换另一个类的技术。（你在第 14 章中实际上已经用到了，将归档文件中的 `SKScene` 类替换为你的 `GameScene` 类。）这可以解决编码类已不存在的问题。关于这些问题的详细讨论以及一些解决方案，请参阅 *Archives and Serializations Programming Guide* 中的“Forward and Backward Compatibility for Keyed Archives”章节。
