# 第 12 章 物理引擎

物理引擎是驱动热门 iOS 游戏（如《愤怒的小鸟》、《粘粘高尔夫》、《果冻车》和《台阶摔跤》）的核心技术。它们允许你创建一个感觉动态且逼真的世界。

`Cocos2d`内置了两个物理引擎：`Box2D`和`Chipmunk`。两者都设计为仅在二维空间工作，因此与`cocos2d`完美契合。

在本章中，你将学习这两个物理引擎的基础知识，在此过程中，你可能会逐渐对其中一个更有好感。我将简要解释它们之间的区别，但很大程度上，这是一个基于个人偏好的选择。

如果你从未接触过物理引擎，请不要担心——我也会快速介绍它们的基本概念和关键元素。

## 物理引擎的基本概念

你可以将物理引擎视为游戏对象的动画系统。当然，这取决于游戏开发者来连接和同步游戏对象（如精灵）与物理对象（称为*刚体*）。之所以这样称呼，是因为物理引擎将它们的动画视为僵硬、不可变形的物体。这种简化使得物理引擎能够计算大量的物体。

通常有两种类型的物体：*动态*（移动）物体和*静态*（不可移动）物体。这种区分很重要，因为静态物体从不移动——并且永远不应该被移动——物理引擎可以基于静态物体永不相互碰撞这一事实进行某些优化。

另一方面，动态物体彼此碰撞，也与静态物体碰撞。除了位置和旋转之外，它们至少还有三个定义参数。一个是*密度*或*质量*——换句话说，衡量物体有多重的指标。其次是*摩擦*——动态物体相对于表面移动时的阻力或光滑程度。最后是*恢复系数*，它决定了物体的弹性。尽管在现实世界中不可能，但物理引擎可以创建在弹跳过程中永不损失动量，甚至每次从其他物体弹起时速度都会增加的动态物体。

动态和静态物体都有一个或多个*形状*，这些形状决定了物体所占据的区域。形状最常见的是圆形或矩形，但也可以是多边形、形成任何复杂形状的若干顶点，或者仅仅是直线。物体的形状决定了其他物体及其形状将在何处发生碰撞。反过来，每次碰撞都会产生*接触点*——两个物体的形状相交的点。你可以利用这些接触点来播放粒子效果，或在物体碰撞的确切位置添加划痕。

物理引擎通过施加力、冲量和扭矩来对动态物体进行动画，而不是直接设置它们的位置和旋转。不推荐直接修改位置和旋转，因为物理引擎会做出某些预测，如果你手动重新定位物体，这些预测将不再成立。

最后，你可以使用一系列*关节*将物体连接在一起，这些关节以各种方式限制连接物体的运动。有些关节可能带有*马达*，例如，可以充当汽车的驱动轮，或者为关节提供摩擦，使关节试图弹回其原始位置。

## 物理引擎的局限性

物理引擎有其局限性。它们必须采取捷径，因为现实世界过于复杂而无法模拟。刚体的使用就是一个例子。在某些极端情况下，物理引擎可能无法捕捉到每一次碰撞——例如，当物体移动得非常快时，它们可能会相互穿过。尽管这在量子物理学中已被证明会发生，但我们肉眼可见的现实世界物体尚未显示出这种效应。

刚体有时会相互穿透并卡住，特别是当它们受到关节约束时，这些关节将两个或多个物体连接在一起并限制了它们的运动范围。这可能导致物体出现不希望的颤抖运动，因为它们努力分开的同时又要保持关节约束满足。即使在现代游戏中你也可能见过这种情况；例如，第一人称射击游戏中死亡角色的布娃娃可能会呈现出非常不自然的身体姿势，其四肢根本无法静止，导致整个布娃娃颤抖或更糟。这类事件仍然让玩家感到惊奇和好笑。在你最喜爱的视频网站上搜索“物理故障”，你就会明白我的意思。
