# 确保博客中不要显得傲慢自大，但也不要妄自菲薄。你或许是个初学者，但你在学习，所以专注于你学到的东西。你学到的那些事可能看起来其他数百万开发者已经知道了，于是你可能会想，自己是否真的该写博客谈论它们。我对这种想法很熟悉，坦白说，你总会怀疑自己写的东西是否能让别人感兴趣。相信我，会的。即便不会，也没什么大不了的。事实是，虽然有数百万开发者可能已经了解你发布的内容，但还有数百万的人不知道，他们能够向你学习。

同时记得把最擅长的技能摆在前面，避免在博客中谈论自己的弱点。你或许并非无所不知，但你可以学会。如果你真的想知道如何通过预测客户端移动来增强多人游戏编程，就去学习它。网络是绝佳的资源——收集你找到的内容，并写下你的发现和所学。别人会因此尊重你。写博客需要多年的实践，所以最好现在就开始，因为它最终会带来回报——可能以你目前无法想象的方式。

另一个推销自己的重要方式是 Twitter。一旦你有话想对世界说，你会很高兴能同时触达几十、几百乃至成千上万的关注者。Twitter 是一个非常有效的营销工具。有效使用它的方法遵循几个简单的步骤。首先，不要保护你的推文——这会严重限制关注你的人数（反正我肯定不会关注）。然后，提供一份有趣的个人简介，应包括你的兴趣、职业以及任何能让你看起来值得关注的特点。仅仅用一个笑话、诗歌或语录作为简介是不可取的。而且别忘了链接到你的博客！最重要的事情是定期发推，并发布别人可能感兴趣的内容。只发布关于你自己和产品的内容（或者只是转发别人的帖子）可不会说服多少人关注你。

### 公共关系与新闻稿

如果你独自工作且不想与发行商合作，雇佣一家公关公司或代理来让你和你的游戏在市场上有更好的机会，可能是个明智的决定。但也可能完全是愚蠢且毫无意义的。后一种情况通常不是公关人员的错；而是关乎理解公关的益处——需要什么才能让公关有效运作——并评估是否值得花费数千美元。

作为独立开发者，你的预算很可能非常有限。即便公关代理的入门级价格也会让你惊掉下巴。如果情况如此，那就转身离开，回去继续开发你的游戏吧。你的直觉是正确的。

现在，如果你已经有一款拿得出手的游戏，并且不仅赚了钱，还在周围人中积累了一些影响力，情况可能就有所不同了。也许你有一些闲钱，从第一款游戏中学到了很多，第二款游戏甚至更好，而且已经有玩家在等待它发布。你能借助专业公关来推波助澜吗？这种情况下可能性更大。

如果你决定寻求公关代理的帮助，一定要找一位有游戏行业工作背景的——最好是与独立游戏开发者以及移动游戏领域合作过的人。这样的人不容易找到，所以多问问。用一位不涉足游戏开发的公关代理是浪费时间和金钱——你需要的是有能力触达游戏目标受众的公关人员。但最重要的是，决定因素应该是你的游戏本身。

如果你知道自己有一款特别的作品，测试者也这么告诉你，而且游戏看起来非常棒，那么投入几千美元在公关上可能是个好选择。公关代理会让游戏评测者撰写关于你游戏的文章，游戏开发网站也会注意到你的特殊能力。如果这些特质不明显，专业公关也无法带来巨大改变。当你能够提供精美的截图以及引人入胜、有趣且吸引人的预告片时，公关效果最佳。

你怎么知道自己的游戏是否特别？去问问那些会毫不犹豫告诉你他们不喜欢什么的人。家人和朋友通常过于友善，无法提供改进游戏所需的尖锐批评。你应该在论坛上招募私人测试者，并提供一些“开胃菜”——比如截图或游戏特色描述——让他们对你的游戏更感兴趣。你如何处理批评和功能请求，以及总体上如何与测试者互动，这本身也是公关问题。处理用户反馈，特别是批评，是一项你应尽早磨练的关键技能。

但谁说不能先自己试试呢？有专门为独立开发者服务的新闻稿发布服务，费用仅为专业公关代理收费的一小部分。而且，遵循规则写一份新闻稿并不难。以下新闻稿发布服务正好能对接游戏玩家和开发者的正确渠道。最热门的选择是 Indie Press Release Service（网址：`www.gamerelease.net`）和 Mitorah Games 的 Game Press Release Submission Service（网址：`www.mitorahgames.com/Submit-Game-Press-Release.html`）。

毫不意外，这两个服务都由独立开发者自己运营。具体来说，Juuso Hietalahti 创建了 Indie Press Release Service，还运营着非常富有洞见的博客`www.gameproducer.net`。如果你想了解更多关于制作和营销方面的内容，这个博客尤其值得关注。

你还应该考虑 Games Press 网站，全球游戏记者经常访问该网站（详见`www.gamespress.com/about_howtosubmit.asp`）。

## 吸引玩家以增加收入

每个平台都有一些外围技术，在某些情况下对某些开发者来说即使不是必需，也很有帮助。在 iOS 平台上，除了苹果的 Game Center 之外，还包括越来越多的社交网络平台可供选择。此外，还有服务器开发工具包，你可能需要它们来开发需要连接到你自己的服务器的持久化游戏——无论是为了查找和运行超过四名玩家的比赛，还是仅仅为了在线保存角色、进度、设置和世界。

另外，尽管有时被认为有争议性，但在游戏中提供广告可以创造额外的收入来源，尤其是对于免费游戏和精简版。通常与广告结合，你还可以研究一下为游戏添加分析和度量是否有帮助，以便找出玩家最容易失败的地方、他们点击最多的按钮以及他们玩不同游戏模式的频率。这可以帮助你调整游戏，使其对更多玩家来说更有趣。

### 吸引玩家

游戏行业的热门流行词显然是*社交*。无论是热潮还是泡沫，无论投资者、评测者和玩家是关注还是厌倦，社交游戏元素正变得越来越强，并已成为一种期望。社交游戏涵盖从被动地向 Twitter 和 Facebook 等网络推送更新，到引导多人游戏中的玩家互动等内容。

此外，推送通知是一个强大的工具，可以提醒你的用户应用程序中发生的重要事件。无论是更新、新内容，还是你发布的下一款游戏，它都为你提供了一个额外的渠道，让用户持续参与你的应用和你的品牌。

#### 社交网络



### 游戏社交网络平台

除了苹果的 Game Center 技术外，还有多种不同的社交网络平台。它们或多或少都有相似之处：允许玩家互联、发布高分、赢取成就，以及执行许多其他操作（包括将游戏事件发布到 Twitter 和 Facebook）。而且，无论对玩家还是开发者，这些平台全部免费！

由于它们的特性集在不断演变，且社交网络市场蓬勃发展，最终决定权在您手中。我在此列出几家主要平台，并提及它们的一些突出特性。

**注意：** 大多数社交网络 SDK 以及 ShareKit (`http://getsharekit.com`) 都已包含将您的用户连接到 Twitter 和 Facebook 的支持，因此您无需学习并实现各自独立的 Facebook 和 Twitter API。如果您只需要接入 Twitter，可以看看 Matt Gemmell 开发的优秀 `MGTwitterEngine` API，地址为 `http://mattgemmell.com/2008/02/22/mgtwitterengine-twitter-from-cocoa`。如果您需要将 Facebook 集成到应用中，官方的 Facebook iOS SDK 位于 GitHub 上：`http://github.com/facebook/facebook-ios-sdk`。

- **OpenFeint** 被视为 iOS 社交网络 SDK 的领导者。它凭借 Game Center 兼容性和回合制多人游戏功能脱颖而出。但最重要的是，它非常受欢迎，拥有数百万玩家群体。请访问 OpenFeint 的开发者门户：`www.openfeint.com/developers`。
- **Scoreloop** 通过提供额外收入渠道（如可下载内容、虚拟物品共享和游戏内货币）在竞争中独树一帜。它还包含分析功能，可了解玩家的行为。请访问 Scoreloop 主页了解功能简介：`www.scoreloop.com`。
- **Plus +** 是由 ngmoco（一家大型 iOS 游戏发行商）创建的社交网络平台。目前仅对 ngmoco 的开发合作伙伴开放。Ngmoco 正在寻找精选开发者进行合作；如果您认为自己具备相应能力，可申请访问其开发者网站：`http://plusplus.com/developers`。
- Chillingo 的 **Crystal** 是另一家大型 iOS 发行商的社交网络平台。与 Plus+ 类似，其访问权限仅限于与 Chillingo 及其新东家 Electronic Arts 合作的开发者。您可以在 `http://devsupport.crystalsdk.com` 上获取 Crystal SDK 的相关信息。

如果您发现此列表中缺少 Agon-Online，那么您可能尚未得知其已于 2011 年 6 月 30 日关停的消息。至于 Geocade，它目前在 `www.geocade.com` 上仍可使用，但在竞争对手中已无关紧要。

### 多人游戏的 Socket 服务器技术

如果您正在构建一款需要比社交网络平台所能提供的更复杂的服务器端游戏逻辑和存储功能的多人游戏，那么您需要编写自己的 Socket 服务器。幸运的是，编写具有 Socket 连接和其他网络魔法的服务器/客户端架构中最困难的部分已经有人为您代劳了。

在 Socket 服务器上托管在线游戏具有多项优势。对于某些游戏而言，防止玩家作弊至关重要，因此将最关键的逻辑隔离在服务器上运行，同时能够验证来自客户端的数据，可以有效阻止许多常见的作弊机制。基于服务器的方法也是编写一款能同时支持大量玩家游戏的唯一途径。如果采用点对点技术，您很快就会遇到可扩展性问题，因为每台设备的带宽相当有限，并且点对点对局中的每个玩家都会增加该对局中所有连接设备的计算开销。正因如此，苹果的 Game Kit 将设备间的点对点连接限制为最多四名玩家。服务器硬件最终比任何连接到它的设备都更强大，带宽也更高，这使得您能在同一对局中容纳更多玩家。此外，服务器还托管着数据库，并通常运行验证程序，以确保玩家没有篡改其应用或传入的数据流，从而抵御作弊和漏洞利用。

- **Electrotanks 的 Electroserver** 是一款功能丰富的服务器技术，被全球专业人士广泛使用。它按此定价，但确实提供一个免费版本，允许最多 25 个并发用户连接到游戏服务器。如果这对您来说足够，可以尝试一下：`www.electrotank.com`。
- 由 gotoAndPlay() 开发的 **SmartFoxServer** 紧随 Electroserver 之后。它的功能没有那么丰富，但在独立游戏开发者中非常受欢迎，这主要是因为其有一个完全免费的 Lite 版本，而 Basic 和 Pro 版本也仍然允许最多 20 个用户免费使用。总体而言，其价格对独立开发者来说更为实惠。请在产品页面比较 SmartFoxServer 的各种版本：`www.smartfoxserver.com/products`。
- **Exit Games** 采取双管齐下的策略，提供了名为 Photon 的网络组件和名为 Neutron 的托管服务，用于开发持久在线游戏。Neutron 原生支持回合制多人游戏的开发，并且可以挂起和恢复游戏会话。Photon 有一个免费版本，每个应用或服务器支持最多 50 个并发用户，而 Neutron 则提供免费试用版。请访问 Exit Games 网站比较功能和价格：`www.exitgames.com`。

### 推送通知提供商

推送通知是那些无论您是否在使用 iPhone，都会弹出在屏幕上的小提醒消息。一些应用使用它们来通知用户股市行情、实时体育赛事，或者仅仅是让您知道有新的漫画连载可用。其用途无穷无尽。推送通知最大的力量在于，它能激励您的用户持续使用您的应用；如果他们让应用提醒自己新闻和事件，就不容易忘记它。

您可能听说过苹果的推送通知服务，如果尚未了解，可以在苹果的《推送通知编程指南》中学习更多内容：`http://developer.apple.com/library/ios/#documentation/NetworkingInternet/Conceptual/RemoteNotificationsPG/Introduction/Introduction.html`。

推送通知允许应用向单个设备广播消息。如果您实施了某个社交网络平台，您将免费获得此功能，无需关心推送通知编程的复杂性。然而，在某些情况下，您可能需要实现自己的解决方案，为此您需要一个推送通知提供商。这是一个能够与苹果服务器和 iOS 设备通信的服务器。您可以使用免费开源的 `apns-sharp` 库（见 `http://code.google.com/p/apns-sharp`）编写自己的服务器，或者注册 Mono Push 服务（`www.monopush.com`）或 Urban Airship（`http://urbanairship.com`）。
