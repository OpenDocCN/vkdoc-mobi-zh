# 第 17 章：非同寻常

最后一章不包含源代码。我甚至不会过多讨论 cocos2d 引擎。相反，我想重点谈谈读完本书后，如果你想提问并学习更多知识，可以去哪里寻找答案。毫无疑问，读完本书后最应该去的地方是 `www.learn-cocos2d.com`。我每隔一周会发表一篇深度文章，内容涉及示例代码、指导、参考资料、当前状况，以及偶尔的评论文章。你还会找到 Cocos2D 播客和我的 LearnCocosTV 频道的链接。

你还应该研究哪些技术可能对实现你的游戏有用，例如广告、分析、众多社交网络库之一，甚至是用于持久世界游戏的服务器技术。

你想知道的一切，可能都在互联网的某个角落。它可能只是藏起来了。如果你想了解去哪里寻找美术、音频和自由职业者，那算你走运——我在本章中提供了良好的入门点。

在本章的最后一节，我还会带你一窥市场营销和公共关系。这些是我经常被问及的话题，它们充满了神秘感和误解。我将讨论与发行商合作、如何从这种关系中获益，以及如何推广你的游戏和你自己。

对于独立开发者而言，让社区认可你是一个富有创造力、热情洋溢的游戏开发者，并与社区建立联系，这一点至关重要。你所有的社交网络努力都将帮助你推广游戏，只需通过接触更多志同道合的人。如果你能建立一个关注者网络，游戏的成功也会随之而来。很多人把这件事搞反了，以为情况正好相反。其实并非如此。



你还会了解到通过 `cocos2d` 提供的参考游戏和应用。它们会让你对 `cocos2d` 游戏引擎的能力以及你作为一名独立开发者能达到的成就，留下深刻的印象。最令人兴奋的学习工具之一就是他人的源代码，因此我也列出了一些在售的商业 `cocos2d` 源代码项目，正是基于这个原因。

最重要的是，“与众不同”应该成为你所有行动的指导原则。创造一些与众不同的东西，并且不要害怕与众不同。

## 学习与工作的额外资源

本节旨在帮助你找到问题的答案，获得特定问题的支持，获取更多可供学习并作为自己游戏基础的源代码，当然还有，向你介绍所有的 `cocos2d` 工具以及一些最佳的 `cocos2d` 参考游戏。

这也是我的博客 `www.learn-cocos2d.com/blog` 的目的。你可以在我的博客上找到最新的更新以及 `cocos2d` 游戏开发的技巧和诀窍，我每隔一个周四就会在那里发布一篇新的深度文章。

### 在哪里寻求帮助

无论你是遇到了自己无法解决的技术问题，还是需要更多人参与你的游戏开发，你都可以获得帮助。此外，如果你正在寻找美术、音频或工具，我也知道在哪里能找到你想要的，或者至少知道从哪里开始搜索。

**提示** 如果你卡住了，不知道该尝试其他什么方法，仅仅是把问题描述清楚、你想达到的目标以及到目前为止你做了什么都写下来，就会有所帮助。大多数时候，这能解放你的思维，让你想到尚未尝试过的方法，而且往往能带来解决方案。如果还是不行，至少你现在有了一份摘要，可以发布到论坛或问答网站，这有助于你更快地获得一个好的答案。提问的艺术在于让别人更容易回答你。

### Cocos2D 主页

这也许显而易见，但如果你有与 `cocos2d` 相关的问题，你应该前往 `cocos2d` 论坛并加入社区：`http://cocos2d-iphone.org/forum`。

在 `cocos2d` 论坛上，你可以询问任何与 `cocos2d` 相关的问题。它有专门讨论音频编程、物理引擎、社交网络、`cocos3d` 和广告等热门话题的子论坛，以及一个用于提问 Objective-C 和 iPhone SDK 相关问题的通用论坛。大多数情况下，`cocos2d` 社区人很友好且乐于助人，论坛上分享了许多优秀的示例代码和开发故事。

在提问之前，请务必搜索论坛以及官方的 `cocos2d` 文档维基：`http://cocos2d-iphone.org/wiki/doku.php`。

此外，你可以在 `cocos2d` 游戏论坛上发布你新上架的游戏。别忘了也把它添加到用 `cocos2d` 制作的游戏列表中。你可以在 `cocos2d` 游戏页面完成这个操作：`www.cocos2d-iphone.org/games`。

### Cocos2D Central

我创办了一个名为 Cocos2D Central 的社区中心，专门收集与 `cocos2d` 相关的一切内容。最重要的是，这里有关于 `cocos2d`、本书、Kobold2D 以及我的游戏套件的论坛版块。Cocos2D Central 也托管了 `cocos2d-javascript`（`cocos2d` 的网页浏览器移植版）的官方论坛。

在“资源”部分，有若干小教程可供使用，“下载”部分则托管了我目前提供下载的所有文件，包括 `cocos2d` 安装程序和本书的源代码。

### Stack Exchange 网络

论坛是社区彼此互动的好方式。但正因如此，它们往往会变得有点闲聊化，搜索特定答案也可能很麻烦，因为论坛的内容并不严格局限于解决问题。

而这正是像 Stack Exchange 这样的问答网站大显身手的地方。你进去提出一个具体问题，然后得到答案。因为焦点在问答上，所以更容易找到已有的答案。如果你特别喜欢某个问题或答案，可以点赞，这样它会在搜索结果中排在更靠前的位置。

我时常对 Stack Exchange 网络贡献者展现出的专业知识感到惊叹。这在一定程度上要归功于内置的徽章和积分系统，这使得提出有趣的问题和撰写深思熟虑、深入浅出的答案都变得非常有回报。Stack Exchange 网络由多个免费的问答网站组成，其中最受欢迎也是我个人最爱的是 `http://stackoverflow.com`，它主要涉及编程问题。

你在 Stack Overflow 上找到的关于 `cocos2d` 的问题数量可能不如在官方 `cocos2d` 论坛上多，但 Stack Overflow 上的问题质量很高，而且几乎所有问题都能得到很好的答案。关于在该网站上使用搜索标签存在一些混淆，`cocos2d` 和 `cocos2d-iphone` 都被用来标记与 iPhone 版 `cocos2d` 相关的问题。这可以归因于 `cocos2d` for iPhone 的成功，以至于它已经成为 `cocos2d` 这个名称本身的代名词。使用以下两个链接可以找到 Stack Overflow 上所有与 `cocos2d` for iPhone 相关的问题：

*   `http://stackoverflow.com/questions/tagged/cocos2d`
*   `http://stackoverflow.com/questions/tagged/cocos2d-iphone`

Stack Exchange 作为一个问答网站网络正在不断扩张。最新的成员之一是“游戏开发”问答网站。在这个网站上，你可以提出通用的游戏编程问题，以及任何与游戏开发相关的问题，包括设计、营销和销售。请访问 Game Development Stack Exchange 网站：`http://gamedev.stackexchange.com`。

### 教程和常见问题解答

网上有很多关于 `cocos2d` 的教程，但有一位教程作者明显脱颖而出：Ray Wenderlich。他撰写了十几篇 `cocos2d` 教程，并发布在他的网站上：`www.raywenderlich.com/tag/cocos2d`。除了 `cocos2d` 教程，你还可以在这个网站上找到其他非常有趣的 iPhone SDK 相关技巧和诀窍。至少有一个特定案例对 `cocos2d` 开发者也有帮助，Ray 在其中解释了如何保存和加载你的应用数据。这也可以应用于保存和加载游戏（参见 `www.raywenderlich.com/1914/how-to-save-your-app-data-with-nscoding-and-nsfilemanager`）。

我的 Learn & Master `cocos2d` 网站（`www.learn-cocos2d.com/` category/idevblogaday）上的 iDevBlogADay 文章主要是教程和指南。你将学会如何在 `cocos2d` 项目中启用 ARC，深入了解 iOS 销售统计数据，如何用 `cocos2d` 编写一个网络摄像头查看器，如何将现有的 `cocos2d` 项目升级到更新版本——以及更多内容。

你绝对应该加入推 terverse，并创建一个关于 `cocos2d` 和 `kobold2d` 的列表或保存的搜索。有非常多非常好的教程，但它们分布得很广。另一个追踪 iOS 开发优秀文章的好方法是访问 Cocoa Literature 网站

就我个人而言，浏览他人的代码是我最好的学习方式。当我拿到 `cocos2d` 并发现文档不足时，我做的第一件事就是投资了 Sapus Tongue 源代码项目，看看 `cocos2d` 在游戏中是如何实际使用的。这对于快速入门非常有帮助。

**注意** 由于 Zynga 收购了 Sapus Media 的部分资产，Sapus Tongue 源代码和 Level SVG 产品已不再提供。不过，我在以下部分介绍的源代码产品是优秀的替代品。

与开源项目相比，商业出售源代码的优势在于，开源项目很少能得到支持，而且它们几乎从不更新，以至于你会发现大多数开源项目使用的都是 v1.0 之前的 `cocos2d` 版本——只有一个例外，那就是官方的 `cocos2d-iphone-extensions` 项目：`www.cocos2d-iphone.org/forum/topic/17546`。
