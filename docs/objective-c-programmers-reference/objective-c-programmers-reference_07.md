# 1. Objective-C 中的 C 语言  

## 摘要  

本章将介绍 Objective-C 与 C 语言共有的主要概念。Objective-C 基于最初的 C 语言。由于这种特殊关系，任何有效的 C 代码在 Objective-C 中同样有效。因此，熟悉 C 语言的基础知识，以及它们如何与面向对象编程的各个领域（如类、消息和接口）相关联，就显得至关重要。  

数组、结构、指针、变量声明、变量类型和函数等主题，是这两种语言共享的通用词汇的一部分。熟练掌握这些基本 Objective-C 技术的程序员，在理解和运用面向对象概念时会更加得心应手。  

为了让你快速掌握 Objective-C 的高级能力，我们将从概述继承自 C 语言的最常用特性开始本书。在接下来的章节中，你将看到如何使用这些基础技术创建简单的程序，这些程序将成为你未来 Objective-C 应用程序的构建模块。  

> **注意**  
> 在阅读接下来的几节内容时，请记住一个隐含的假设：这里讨论的每个 Objective-C 特性，在纯 C 语言中同样可用。除非另有明确说明，本章内容均遵循此规则。  

## 简单语句  

Objective-C 程序由语法元素组成，这些元素可进一步归类为函数、类和变量声明。函数包含语句或变量声明。语言中的每条语句都以分号结尾，或者用花括号括起来（在复合语句的情况下），这使得无论是人还是编译器都能轻松识别。与某些使用缩进来定义声明可能结束位置的语言不同，Objective-C 使用分号以明确无误的方式标记其结束。例如，以下是一些有效的语句和变量声明（其含义将在后续章节解释）：  

```
int numberOfEmployees;
double totalPayment;
totalPayment = numberOfEmployees * 1000;
NSLog(@"test log");
if (1 > 2) totalPayment = totalPayment + 10;
```

这些行中的每一行要么定义一条可执行指令，要么声明一个程序稍后将使用的实体。我们将详细检查每种情况下使用的语法，但目前重要的是认清 Objective-C 语句的本质，以及它们如何组合起来构成有效的程序。  

## 变量  

变量是内存中一个可以保存和修改值的位置。Objective-C 中的变量必须属于特定的类型，该类型在变量声明时被明确定义。以下是一些例子：  

```
int anIntegerVariable;
float aFloatingNumber;
```

请注意，变量类型是声明中的第一个元素，后面跟着变量名，最后以分号结尾，分号表示语句结束字符。这两个变量声明创建了一个整型变量和一个浮点型变量。整型变量必须只包含整数，编译器会对此进行检查。即使程序员强行将其他类型的值赋给整型变量，在涉及该变量的所有表达式中，其内容仍会被转换为整数。  

可以在一条语句中声明多个变量。这是通过在同一语句中添加额外的变量名来实现的，变量名之间用逗号分隔。以下是另一个在同一表达式中声明多个变量的例子：  

```
int anIntegerVariable, aSecondIntegerVariable;
bool booleanVariable1, booleanVariable2;
```

命名变量时需要遵守一些规则。第一条规则是，变量只能以字母字符或下划线字符开头。变量名中的其他字符同样适用此规则；但从第二个位置开始，也允许使用数字字符。以下是一些有效的变量声明：  

```
int var1;
int _single_2_Position;
int long_variable_name;
```

在 Objective-C 中，下划线字符经常用作长变量名的分隔符。其目的是在不借助大小写字母混写的情况下，提高这些长名称的可读性。然而，一些程序员认为在同一个变量中混用大小写没有问题。因此，下面的例子也非常常见：  

```
int longVariableName;
```

> **注意**  
> 变量命名是一种风格问题。不同的程序员对哪种形式更好看有不同看法。不过，我们建议你在使用变量命名风格时保持一致。不要在一个地方使用大小写混合的变量，而在另一个地方使用下划线分隔的变量。基本思想是避免混淆，并减少每次在程序中引入新名称时你需要做的决策数量。  

变量名以及 Objective-C 中所有其他标识符都是区分大小写的。这意味着以下两个是对不同变量的有效声明：  

```
int myVariable1;
int myvariable1;
```

变量名区分大小写这一事实，使得使用统一的命名风格更加重要。如果因为不加区分地使用两个仅由一个拼写错误的字符区分的变量而导致程序失败，那将是非常不幸的。  

> **注意**  
> 在 C 语言中，以单个或双下划线开头后跟大写字母的名称是保留字。只有编译器才能使用它们，用于语言实现的目的。请避免使用此类名称，因为它们可能与未来语言版本中使用的内部名称冲突——即使它们在你当前的编译器实现中能够正常工作。  

### 变量初始化  

变量声明也可用于定义变量的初始值。这通过赋值操作符实现。以下是一个简单的例子：  

```
int daysInTheWeek = 7;
int monthsInTheYear = 12;
float PI = 3.14;
```

初始化还可以使用算术表达式甚至函数调用。因此，可以编写如下代码：  

```
int secondsPerHour = 60 * 60;
```

> **注意**  
> 变量应始终被初始化为合适的值，因为缺乏适当的初始化是编程错误最常见的来源之一。Objective-C 将变量初始化与定义相结合，正是为了减少这类错误的发生。



## 变量类型

变量在使用前需要声明为特定类型。Objective-C 中有多种类型可供使用，通常可分为原生类型和用户自定义类型。

原生类型由语言和编译器实现预先定义，无需使用任何库或头文件即可使用，因此可在 Objective-C 定义的任何函数或类中使用。而用户自定义类型由程序员指定，它们可以是库的一部分，也可以是用户代码的一部分。我将首先介绍可用的原生类型；结构和类将在后面讨论。

原生类型数量不多，大多数是数值类型，但也可用于存储字母数字数据以及逻辑值。最后，原生类型还可用于存储内存地址，即所谓的指针。

在数值类型中，最常见的是整数和浮点数。例如，整数值由类型 `int` 表示。浮点数是数学中实数的计算近似值，可由 `float` 或 `double` 等原生类型表示。

### 整数类型

整数类型用于存储整数，包括 `char`、`int`、`short`、`long` 和 `long long`。不同数值类型的出现源于它们各自拥有不同的存储容量，以及在内存中表示此类数字所需的内存大小。例如，在 32 位机器上，定义为 `int` 的整数通常占用 4 个字节，这意味着它可以存储大约 40 亿以内的数字。另一方面，`short` 类型的值可能小至 2 个字节，足以存储大约 200 万以内的数字。以下是一些示例：

```
char val0 = 12;
```

```
int val1 = 123400;
```

```
short val2 = 123;
```

```
long val3 = 12345;
```

```
long long val4 = 12345678;
```

`long` 和 `long long` 变量的大小取决于编译器和使用的机器，但它们能存储的值通常大于 `int` 允许的范围。

整数值的类型还可以声明为有符号或无符号。有符号数可以是正数或负数，因为符号与数值表示一起存储。无符号数不能为负，取值范围从零到变量大小允许的最大值。

当表示已知为非负的数量时，无符号数非常有用。例如，内存大小通常由无符号数表示。然而，通常的做法是使用更为通用的有符号类型——除非确定某个数字不可能为负。默认情况下，声明为 `int`、`short`、`long`、或 `long long` 的整数都是有符号的。

## 浮点数

浮点数是实数的计算机表示。由于计算机存储数据量有限，实数无法被精确存储。对于某些数字，例如 0.3，由于使用二进制表示会引入误差，计算机中只能存储其近似值。对于其他实数，例如圆周率 Pi，众所周知它没有有限表示形式。

Objective-C 程序使用浮点表示法，其中实数分为两部分存储：尾数和指数。你无需了解这种表示法的复杂性，但重要的是要意识到这些数字与整数之间的区别。同样重要的是要知道，浮点数的运算可能导致难以理解和追踪的舍入误差。

浮点数可以声明为两种形式：`float` 或 `double` 值（尽管 `double` 可以接受 `long` 修饰符）。`float` 和 `double` 的区别在于每种类型存储所用的数据量。具体字节数取决于所使用的机器，但 `double` 数的精度和指数范围都是 `float` 的两倍。以下是一些示例：

```
int degrees = 360;
```

```
float e = 2.7182;
```

```
double elarge = 2.718281828459;
```

## 字符

Objective-C 支持的一种常见原生类型是字符（`char`）。字符可以存储任何标准 ASCII 值，其范围通常为 0 到 255。尽管用于存储字符串元素，但字符类型只是前述整数数字类型之一，因此其行为与 `int` 或 `short` 等类型类似。`char` 类型也可以声明为有符号（允许负数）或无符号（当已知不会存储负数时）。

## 逻辑值

变量还可以存储单个逻辑值：真或假。虽然 C 语言没有专门的布尔值，但在 Objective-C 中，你可以使用 `BOOL` 类型来表示此类变量。`BOOL` 类型的变量可以存储值 `YES` 或 `NO`，也可以拼写为 `TRUE` 或 `FALSE`。然而，该语言允许将任何整数类型转换为布尔值。在这种情况下，零值被转换为 `FALSE`，而任何非零值都被视为 `TRUE`。对于不注意这些类型之间转换规则的程序员来说，这可能是错误的来源。

## 指针

Objective-C 支持存储内存地址的变量，这是引用程序所存储数据最灵活的方法之一（此类变量的使用将在本章后面介绍）。指针使用星号符号声明，以下是声明常见类型指针的示例：

```
int *pointer_to_int;
```

```
double *double_pointer;
```

第一个示例是一个可以存储整数地址的变量。第二个是一个指针，可用于存储包含 `double` 类型值的内存地址。在 Objective-C 中创建对象时，指针符号使用频繁，你将在下一章中看到这一点。

## void 类型

关键字 `void` 是 Objective-C 中使用的特殊类型名称。与其他类型名称不同，`void` 不代表真实的数据类型，而是表示数据的缺失，或者类型未知这一事实。`void` 类型仅能在少数情况下出现。一种情况是当函数没有返回值时，这通过使用 `void` 关键字替代返回类型来表示。

使用 `void` 的另一种情况是标记指向未知数据类型的指针。当类型未知且使用 `void` 时，生成的指针将不包含关于该内存位置存储的实际数据类型的任何信息。此类变量可以在 Objective-C 中用作泛型指针，可应用于任何数据类型。你将在后续学习指针的章节中看到这种用法的示例。

**注意**

由于 `void` 只是一个特殊关键字，因此无法声明 `void` 类型的变量。例如，尝试声明像 `void var;` 这样的变量将导致编译错误，因为编译器无法推断该变量的真实类型。



## 注释

大规模项目中使用的软件并不总是易于理解的。尽管应尽一切努力使代码更易于理解，但复杂的算法和其他假设可能会妨碍对程序中特定区域的理解。这就是为什么始终需要在程序本身中添加文档的原因。

Objective-C 使用两种不同风格的代码注释。第一种风格，也是最简单的用法，采用序列 `//` 作为新注释的开始。以这种方式开始的任何注释将一直有效，直到行尾。

```
int integerVariable; // 该变量保存一个整数
float floatVariable; // 一个浮点数变量
```

第二种注释风格用于跨越多行的注释。在这种情况下，注释以字符 `/*` 开始。从该位置起的所有文本都被视为注释的一部分，直到遇到字符对 `*/` 为止。

```
/* 这是一个多行注释的示例。以下
表达式分别用于定义整数和浮点
类型的变量 */
int integerVariable;
float floatVariable;
```

多行注释不能嵌套。也就是说，如果你已经有了一条注释，就不能在其周围再使用第二对注释字符。这样做是行不通的，因为第一次看到字符 `*/` 时，它们会关闭之前开始的任何多行注释，而不仅仅是最近的那一条。

```
/* 一条新注释
  /* 原始注释，未被嵌套 */
这不是注释，因为第一条多行注释
已经被关闭了。 */
```

**注意**

如上所述，一些程序员避免使用多行注释的原因之一是它们可能导致混淆。为避免这种情况，请仅使用单行注释，并将多行注释保留用于特定情况，例如 Objective-C 文件顶部的通用注释。

## 算术表达式

使用变量的最简单方法之一是将其作为算术表达式的一部分。这些表达式用于使用常见的运算符（如加法、减法、乘法和除法）计算数值量。

Objective-C 拥有丰富的运算符集，可用于数值表达式。使用这些运算符最直观的方式是表示四种常见的算术运算。然而，一个不太明显的运算符是取余运算符。它返回第一个数除以第二个数的整数余数。

```
int remainder = 10 % 4;
```

此操作的结果应等于二。这种表达式的一个常见用途是判断一个数字是否为偶数。

```
if (number % 2 == 0)
{
        /* 这是一个偶数 */
}
```

表达式可以使用括号进行组合，以便程序员控制编译器分组运算的顺序。

```
int value = 25;
int result = (10 + value) * (2 / value);
```

## 函数基础

Objective-C 程序由函数、声明和类的混合体组成。函数是 C 程序使用的基本概念，而类是 Objective-C 提供的基本扩展机制。因此，在本节中我们将只讨论函数；我们将在下一章介绍类。

每个程序都从一个名为 `main` 的函数开始。`main` 函数并非从程序的任何部分调用，而是在应用程序启动时由编程环境调用。程序在 `main` 函数返回时精确结束。以下是一个简单的 `main` 函数示例：

```
int main()
{
   NSLog(@"hello to objective-C");
   return 0;
}
```

该程序的目的是写入一条日志字符串。程序通过使用 `NSLog` 函数实现这一点，该函数是 Objective-C 提供的 Foundation 框架的一部分，因此在标准 C 程序中不可用。`NSLog` 的参数是一个以 `@` 字符开头的字符串字面量；此类字符串字面量也是 Objective-C 独有的。程序中的其他所有内容都只是为了符合语言的要求。

函数名为 `main`，函数名后的括号对用于确定所需的任何参数。在这种特定情况下，`main` 函数被定义为一组空参数，这意味着没有参数。然而，某些应用程序期望接收参数，这些参数可以从命令行提供。如果是这种情况，`main` 函数也可以以稍微不同的方式声明。

```
int main(int argc, char *argv[])
{
   NSLog(@"hello from objective-C");
   return 0;
}
```

在这个示例中，程序接收两个参数：第一个是整数值 `argc`，也称为参数计数器，它指定了传递给程序的参数数量。第二个参数 `argv` 是一个指向字符的指针数组，可用于存储传递给应用程序的每个参数。在后续示例中，你将学习如何在程序中使用这些参数。

### 定义新函数

函数是将一组指令打包的简单方法，这样程序员就不必重复自己。例如，这是一个计算两个数平均值的简单函数：

```
float avg(float a, float b)
{
   return (a+b)/2;
}
```

尽可能使用函数有以下几个优点。

*   **简化程序**：每当需要重复执行一个常用操作时，你可以创建一个封装该操作的函数。在上面的例子中，通过调用函数 `avg`，你无需在代码的其他部分重复用于计算平均值的加法和除法运算。这在这里可能看起来没有必要，因为函数非常简单。然而，当算法更复杂时，简化其使用就变得更为重要。例如，你可以在大型工程包中拥有一个用于数学积分的例程。或者你可以在图形应用程序中拥有一个窗口绘制例程。在每种情况下，使用函数都会简化最终的程序。
*   **改善维护**：事情在变化，程序员必须每天适应变化。如果客户决定修改某个计算的方式，而你没有使用函数，那么所需的更改就必须传播到当前进行该计算的所有地方。另一方面，如果你足够聪明地使用了函数，那么只需要在一个地方应用更改。代码中调用该函数的每个地方都会自动使用更新后的版本。这样使用函数可以减轻维护负担。在某些情况下，可以避免修改成百上千行代码——并且将修改限制在仅仅一个地方。
*   **改善文档**：创建函数也是一种记录过程的方式。每当你创建一个函数时，都需要给它一个名字来描述它做什么。如果做得好，调用者至少会对该函数的功能有一个高层次的了解。这种技术可以被看作是一种简单但有效的自我文档形式。当编程项目中的每个人都定期采用这种做法时，可以生成更易于维护且更具前瞻性的代码。

由于这些优点，函数在 Objective-C 中被广泛使用，并且实际上也是 C 语言中代码组织的基本原则。你将在下一章中看到，类可以被视为函数的泛化，这样你就能通过使用这些原则实现更多。



#### 函数语法

函数的语法由其输入参数和返回类型，以及花括号及其包含的一组语句决定。参数列表是函数名后括号内显示的一组参数。即使函数不需要任何参数，其函数名后也会有一对空括号。函数名前缀有作为结果返回的值的类型。例如，

```
int add_values(int a, int b)
{
   return a + b;
}
```

将始终返回一个整数值，该值取决于两个参数 `a` 和 `b`。

本章接下来的示例将使用函数的语法，以便你了解函数如何在编程项目中使用。

## 条件运算

条件运算用于编码逻辑语句。在像 Objective-C 这样的编程语言中，任何产生非零值的表达式都被视为真，而零（或 `NULL`）值则被视为假。逻辑运算用于操作此类真或假的表达式，并构成了使用 `if` 语句进行条件执行的基础。例如，考虑以下代码片段：

```
void test_function(int x)
{
   if (x > 10)
      NSLog(@"Number is greater than 10");
}
```

此函数演示了将表达式 (`x > 10`) 作为 `if` 语句的一部分来使用。逻辑表达式可以使用以下运算符之一：`>`、`<`、`>=`、`<=`、`==`、`&&`、`||` 或 `!`。前五个运算符提供了一种表达大于、小于、大于等于、小于等于或等于的逻辑关系的方式。它们可以应用于数字或字符，以及布尔值。

这五个运算符可用于比较整数、字符、布尔值或浮点数的表达式中。此类表达式的结果将是一个逻辑值，例如 `YES` 或 `NO`。

你还可以使用逻辑运算符 `&&`、`||` 和 `!` 来组合这些布尔值。第一个运算符组合两个逻辑值，如果两者都为真，则结果为 `YES`。第二个运算符作用于两个逻辑值，如果其中至少一个为真，则结果为 `YES`。最后，第三个运算符执行逻辑非运算。当值为假时结果为 `YES`，当值为真时结果为 `NO`。这些运算符也可以应用于数值或指针值。当这种情况发生时，如果数字或指针对应于零或 `NULL`，则被视为假；否则，该值被视为真。

复杂的逻辑表达式还可以使用括号分组，以便更容易理解。以下是使用这些逻辑运算符的一个示例：

```
void logical_func(int x, int y)
{
   if ((x > 2 && x <= 10) ||
       (y > 15 && y <= 20)
      )
      NSLog(@"the numbers are acceptable");
}
```

在这段代码中，变量 `x` 与 `2` 和 `10` 进行比较，使得 `x` 的值位于 `3` 到 `10` 之间。变量 `y` 也与 `15` 和 `20` 进行比较，因此可接受的数字仅在 `16` 到 `20` 之间。如果 `x` 或 `y` 满足这些要求，该函数将打印一条消息说明情况。

### `if` 语句

`if` 语句，如上例所示，可用于基于逻辑表达式执行操作。`if` 语句的最简单形式类似于以下内容：

```
if (<logical expression>)
  <statement>;
```

这种形式的 `if` 语句很有用，但当需要执行多条语句时则不适用。如果需要执行多条语句，你应该将表达式分组到一个单个块中（也称为复合语句）。块是 Objective-C 中分组表达式的常用方式，类似于函数定义的分组。以下是一个示例：

```
void logical_func2(int x, int y)
{
   if ((x > 2 && x <= 10) ||
       (y > 15 && y <= 20)
      )
   {
      NSLog(@"the numbers are acceptable");
      NSLog(@"the total value is %d", x + y);
   }
}
```

每当逻辑表达式为真时，这段代码执行两个操作：打印日志字符串并显示 `x + y` 的值。为此，你需要一个表达式块，如上所示，定义在花括号之间。

还有第三种更有用的 `if` 语句形式。假设你有一些操作需要在逻辑表达式为真时执行，而另一些操作需要在表达式为假时执行。这可以通过在 `if` 语句中添加一个 `else` 分支来实现。`else` 分支可以是单个语句或块。以下是另一个示例：

```
void logical_func3(int x, int y)
{
   if ((x > 2 && x <= 10) ||
       (y > 15 && y <= 20))
   {
      NSLog(@"the numbers are acceptable");
      NSLog(@"the total value is %d", x + y);
   }
   else
   {
      NSLog(@"the numbers are not acceptable");
   }
}
```

## 循环

编程可以被描述为定义重复任务的艺术。但是，如果没有可控重复的方法，执行编程任务将非常困难。这就是为什么每种语言都提供一组允许重复表达式的语句。循环是主要用来重复一组指令的语句的通用名称，其重复取决于逻辑条件的结果。

Objective-C 中可用的循环表达式有 `for`、`while` 和 `do while` 循环。这些循环形式中的每一种都提供了执行可控重复所需的语法。你将在接下来的章节中看到它们各自的工作原理。

### `while` 循环

`while` 循环是三种循环形式中最简单的一种。`while` 循环的基本格式为

```
while (<logical expression>)
   <statement>
```

其中 `<statement>` 可以是单个语句或复合语句。例如，

```
void printNums(int min, int max)
{
   while (min < max)
   {
      NSLog(@"Value %d", min);
      min = min + 1;
   }
}
```

`while` 循环的执行过程是：计算给定的逻辑表达式，并且仅当结果为真时才进入循环。然后执行花括号内块中的所有表达式。接着再次计算逻辑表达式，并重复该过程。当逻辑表达式为假时，整个块被跳过，程序在块后的第一行继续执行。

就像 `if` 语句一样，`while` 语句也允许使用单个表达式而不是块。因此，可以编写以下代码：

```
bool isOdd(int num)
{
   while (num > 0)
      num = num - 2;
   return num == -1;
}
```



### `do { } while( )` 循环

迭代语句的另一种变体是 `do`/`while` 循环。它与之前 `while` 形式的区别在于，其执行条件的判断在循环体末尾（即 `while` 关键字所在位置）进行。这使得 `do`/`while` 循环在需要至少执行一次循环体的情况下非常有效。

可以将 `do`/`while` 循环视为一种避免初始条件判断的方式，这与标准 `while` 循环的工作方式相反。借助这种额外的重复特性，可以编码大量需要在逻辑判断之前先执行一次的算法。

例如，考虑一个处理从文件读取输入的算法。假设文件中包含一组数字，最后一个数字是 -1。在这种情况下，算法包括读取文件中的数字，可能使用该数字进行一些处理，并在数字为 -1 时停止。如果假设存在一个名为 `nextNumber()` 的函数，它返回文件中的下一个数字，那么该算法可以表示为：

```
void process_numbers()
{
   int number;
   do
   {
      number = nextNumber();
      // 执行其他任务
   }
   while (number != -1);
}
```

请注意，此函数将始终至少读取一次下一个数字并执行相关任务。这正是 `do`/`while` 循环可能成为最佳工具的典型场景。只要遇到的数字不等于 -1，末尾的条件判断就会评估为真。当该表达式为假时，重复停止，函数结束其当前执行。

### `for` 循环

在 Objective-C 的三种循环语句中，`for` 循环是最通用的。原因在于它可以在每次迭代的开始（或结束）时执行不同的操作。结合其他更专门的迭代语句的强大功能，`for` 循环很可能是你目前所见过的控制结构中使用最广泛的一种。

`for` 循环的语法由用括号括起来的前置部分和循环体组成——这与 `while` 循环类似。然而，与 `while` 循环不同的是，括号内包含三个由分号分隔的表达式。第一部分是初始化表达式。第二部分是逻辑表达式，类似于你在 `while` 循环中编写的内容。这个逻辑判断决定 `for` 循环体是否会被执行，并且在每次即将开始新一轮迭代时都会重复判断。

最后，第三部分在每次循环体执行结束时运行。因此，就像循环的第二部分一样，这最后一部分在每次运行循环体时执行一次。这里有一个简单的例子：

```
int for_test(int max)
{
   int i;
   for (i=0; i<max; i=i+1)
   {
      NSLog(@"value is %d", i);
   }
}
```

这是一个展示此类循环在实践中如何工作的最小示例。在 `for` 循环顶部表达式的第一部分中，你看到了初始化。这里的目标是将计数器变量 `i` 的初始值赋为 0。中间部分将检查该数字是否小于最大值。如果该判断为真，则执行循环体，在循环体中将数字打印到应用程序日志中。

`for` 多部分表达式的最后一部分仅在每次迭代结束时执行。在这里，`i` 的值增加 1。这使得计数器在每次迭代结束时增加其值，直到达到所提供的最大值。当达到该值时，逻辑判断失败，循环结束。

### `switch` 语句

Objective-C 提供了另一种选择两个或多个选项来执行的方法。除了 `if` 语句，当存在两个或多个可以根据单个值选择的选项时，你可以使用 `switch`/`case` 复合语句。

例如，假设你有一个字符变量，它只能包含三个值：`'B'` 代表蓝色，`'R'` 代表红色，`'G'` 代表绿色。在这种情况下，你可以使用 `switch`/`case` 根据此变量的内容执行三种不同的选项。示例代码如下：

```
void switch_test(char value)
{
   switch (value)
   {
   case 'R':
      NSLog(@"red");
      break;
   case 'G':
      NSLog(@"green");
      break;
   case 'B':
      NSLog(@"blue");
      break;
   }
}
```

请注意，`switch` 控制结构接收控制切换的变量作为参数。然后，对于每个有效的情况，你都有一个 `case` 标签。`case` 定义了执行开始的位置。在这个例子中，它打印与字符关联的颜色。

在每个有效的代码路径之后都有一个 `break` 语句。之所以需要这样做，是因为当遇到新的 `case` 时，`switch` 控制结构不会停止执行。实际上，如果没有 `break` 指令，剩余的代码将继续执行，直到 `switch` 块结束。为了防止这种情况发生，`break` 语句会将程序移动到 `switch` 体结束后的下一条指令。

可以在 `switch`/`case` 语句中使用的另一个元素是 `default` 标签。此类标签可用于检查无效输入。以下是上面所示函数的改进版本：

```
void switch_test(char value)
{
   switch (value)
   {
   case 'R':
      NSLog(@"red");
      break;
   case 'G':
      NSLog(@"green");
      break;
   case 'B':
      NSLog(@"blue");
      break;
   default:
      NSLog(@"error: invalid color code");
   }
}
```

也可以让多个 `case` 指向同一段代码。这可以通过不向会“穿透”到下一个 `case` 的 `case` 添加 `break` 来实现。请考虑以下示例：

```
void vowel_test(char x)
{
   switch(x) {
      case 'A':
      case 'E':
      case 'I':
      case 'O':
      case 'U':
         NSLog(@"vowel\n");
         break;
      default:
         NSLog(@"consonant\n");
   }
}
```



### 赋值运算符

Objective-C 提供了一种通用的表达式来更新变量的值，这种表达式使用了赋值运算符。这类赋值表达式被频繁使用，因为它们能以更简洁的方式完成某些任务，而这些任务在其他语言中可能需要两个或更多步骤才能完成。

基本赋值运算符使用 `=` 符号，并修改左侧的变量，使其包含右侧表达式的结果。这是更新变量的常见方式，你在目前看到的示例中已经遇到过。然而，除了这种基本用法，赋值运算符还可以作为子表达式出现在其他上下文中。其结果是你可以将赋值嵌入到其他表达式中，使其成为它们的一部分。

```
bool check_total(int a, int b)
{
   int total;
   if ((total = a + b) > 10)
   {
      NSLog(@"total value of %d is too high", total);
      return false;
   }
   return true;
}
```

在此示例中，变量 `total` 用于存储 `a` 和 `b` 的和。`if` 语句检查这两个参数的和是否大于 `10`。然而，该测试同时将 `a + b` 的结果赋值给了变量 `total`，以便稍后在调用 `NSLog` 时使用。表达式 `(total = a + b)` 不仅执行了赋值操作，还产生了所赋的值，这样其他表达式就可以访问这个值。

**注意：** 在此示例中，很明显 `total` 本可以在其他地方更新，但在更复杂的代码中，有时将赋值作为子表达式是更自然的做法。在编写这类代码时，请尝试思考是否有其他解决问题的方式能产生更好、更易于维护的方案。

赋值运算符这一特性的另一个用途是链式赋值。其基本思想是：同一个值可以赋值给多个变量；只需让两个或更多变量共享同一个赋值表达式即可实现。

```
int height, width;
void make_square(int dimension)
{
   height = width = dimension;
}
```

函数 `make_square` 用于将 `height` 和 `width` 变量更新为相同的值。这是通过在这两个变量之间链式赋值实现的。实际上，正确的解读方式是从右向左阅读这个赋值语句。

```
void make_square(int dimension)
{
   height = (width = dimension);
}
```

显然，同样的思路可以扩展到任意数量的变量。

### 自增与自减运算符

赋值的一种简单形式是自增运算符。该运算符被创建用于对包含整数值的变量执行加一操作。类似地，也存在自减运算符，它的应用是将赋值给变量的值减一。自增运算符常用于循环中，例如更新临时计数器的值。

```
int print_forwards_and_backwards(int n)
{
   int i;
   for (i=0; i<n; i++)
   {
      NSLog(@"value is %d", i+1);
   }
   for (i=n; i>0; i--)
   {
      NSLog(@"value is %d", i);
   }
}
```

在此示例中，你有机会同时使用自增和自减运算符来移动计数器变量 `i`。自增和自减运算符也有一个版本可以在另一个表达式内部使用，其方式类似于赋值运算符的使用方式。前置自增运算符将变量加一，并且确保自增操作在包含它的表达式的其他部分执行之前发生。类似地，也存在具有相同特性的前置自减运算符。

```
int print_forwards_and_backwards(int n)
{
   int i;
   for (i=0; i<n; )
   {
      NSLog(@"value is %d", ++i);
   }
   for (i=n; i>0; )
   {
      NSLog(@"value is %d", i--);
   }
}
```

这个示例展示了使用前置自增和后置自增运算符的区别。在两个循环中，我们都将自增和自减运算符从循环表达式中移除了。它们已被挪到了 `NSLog` 调用中，并成为了其子表达式。然而，在第一个例子中，我们希望先递增变量再将其打印出来，因此我们使用了前置自增运算符。在第二个例子中，目标是先按原样打印变量，然后执行自增操作，这通过后置自增运算符实现。

**注意：** 自增和自减运算符不能在同一个表达式中对同一变量使用多次。一旦发生这种情况，编译器将无法得知变量的预期值，结果很可能是错误的。作为编程规范，应避免在大型表达式中使用自增运算符，以免意外发生此类情况。

### 复合赋值

有一类赋值运算符是对自增和自减运算符的推广。例如，要向一个变量加上整数值，你可以使用 `+=` 运算符。因此，

`width = width + 10;`

等价于

`width += 10;`

例如，下面的代码增加了矩形的尺寸：

```
int width, height;
void increase_rectangle_dimensions(int increase)
{
   width += increase;
   height += increase;
}
```

减法操作类似：

```
void reduce_rectangle_dimensions(int reduction)
{
   width -= reduction;
   height -= reduction;
}
```

对于每一种算术运算符，都有对应的复合赋值运算符版本：`*=`、`/=`、`%=`、`<<=`、`>>=`。它们很容易理解，你将在本书的其他章节中看到它们的使用示例。

## 结构体

结构体是一种用户定义的类型，它包含其他数据元素，这些元素可以通过标签来访问。结构体使用关键字 `struct` 声明。

```
struct address
{
   int number;
   NSString *street;
   NSString *city;
   NSString *state;
   int postal_code;
};
```

这个结构体定义了一个新的数据类型，它包含五条数据，这些数据可以作为 `address` 结构体的一部分来访问。要使用这样的结构体，你只需创建一个这种新定义类型的变量即可。

```
void create_address()
{
   struct address famous_address;
   famous_address.number = 1600;
   famous_address.street = "Pennsylvania Av";
   famous_address.city = "Washington";
   famous_address.state = "DC";
}
```

结构体的巨大优势在于它们允许将信息打包到逻辑单元中，这样你就不需要处理多个不相关的变量。相反，你只需处理一个单一的数据表示形式。因此，例如，使用上述定义的地址结构体，你可以创建以地址为参数的函数，而不必处理一组不相关的项目。

```
void print_address(struct address an_address)
{
   NSLog(@"street: %@", an_address.street);
   // ... 打印地址中的其他字段
}
```

你也可以将结构体视为一种仅包含数据的原始版本的对象。在下一章中，你将看到 Objective-C 如何提供用户定义的数据类型，这些类型不仅包含关联的数据，还能响应消息。



## 指针

指针是内存中某个地址的标识。Objective-C 通过使用指向特定类型的指针，提供了一种便捷的方式来访问内存地址。指针类型的主要优势在于，它能让你将任何内存地址解读为变量的潜在位置。因此，你不再局限于使用预先声明的变量：你可以将任何内存位置转换为可由算法操控的数据类型。

声明指针时，你需要从一个已有的数据类型开始，并在该数据类型的右侧加上星号字符。例如，你可以按照以下方式为整数和双精度浮点数定义指针：

`int *pointer_to_int;`

`double *my_double_pointer;`

**注意：** 指针声明仅仅定义了一个指向某数据类型的地址，但并不会创建与该数据类型关联的任何内存。例如，上面展示的变量 `pointer_to_int` 并没有与之关联的整数变量。要能够使用指针，首先需要让它指向有效的内存。

一旦拥有了指针，你就可以用它来操控已有的变量。这可通过使用两个运算符来实现：星号运算符用于检索指针所指向内存中存储的数据。例如：

```
void print_value(int *pint)
{
  NSLog(@"the value is %d", *pint);
}
```

表达式 `*pint` 表示你正在解引用 `pint` 所指向内存中存储的值。简而言之，星号运算符允许你使用指针来替代给定类型的变量。

与指针配合使用的第二个运算符是取地址运算符（与符号）`&`。该运算符用于获取现有变量的内存地址。结合使用时，星号运算符和取地址运算符允许你从变量获取地址，并访问和修改存储在这些内存地址中的数据。它们为程序员提供了在应用程序中管理内存所必需的基本工具。这里有一个简单的例子：

```
void modify_value(int *pint, int new_value)
{
   *pint = new_value;
}

int width, height;
void init_rectangle_dimensions()
{
   modify_value(&width, 100);
   modify_value(&height, 50);
   NSLog(@"Now width = %d and height = %d", width, height);
}
```

这段代码使用指针来修改一个整数变量的内容。这种间接操作很有用，因为现在同一段代码可以被用来修改任何整数变量，无论它在程序中的实际名称是什么。

### 参数传递

现在是解释参数传递约定概念的好时机。每个函数都由其名称、参数和返回类型定义。在 Objective-C 中，传递给函数的参数总是会被拷贝。这意味着，例如，当将一个整数传递给函数时，函数内部使用的值仅仅是原始整数的一个副本。特别是，如果你修改了参数的值，你并不会改变被传入的原始变量的值。例如：

```
void try_to_change(int value)
{
   value = 10; // 一个新值
   NSLog(@"value is now %d", value);
}

void use_change()
{
   int orig_value = 20;
   try_to_change(orig_value);
   NSLog(@"after change %d", orig_value);
}
```

如果你运行这个函数，你会看到在执行了 `try_to_change` 函数之后，存储的值并没有改变。我们说变量 `orig_value` 是按值传递给 `try_to_change` 的。如果你真的想要改变一个变量的值，你需要的不仅仅是它的副本，而是它的内存地址。这可以使用指针来实现，如上一节所示。因此，在这个例子中，正确的做法是：

```
void really_change(int *value)
{
   *value = 10; // 一个新值
   NSLog(@"value is now %d", *value);
}

void use_change()
{
   int orig_value = 20;
   really_change(&orig_value);
   NSLog(@"after change %d", orig_value);
}
```

注意，现在你传递的不再是一个值，而是一个内存地址，这样你就可以使用星号运算符来访问那块内存。在调用函数 `really_change` 之后，`orig_value` 的值从 `20` 变成了 `10`，这与之前那次不同。这意味着函数 `really_change` 是在原始变量的内容上执行修改，而不仅仅是在其副本上。

**注意：** 理解语言如何使用指针参数很重要，因为这也是操作对象的方式。Objective-C 中的所有对象都是作为指针来创建和使用的。

## 数组

数组是一种按顺序存储多个相同类型值的方式。例如，假设你想存储学校里所有学生的年龄。一种方法是创建大量变量来存储这些数据，例如：

`int age1, age2, age3; // ... 这里还有更多`

这种安排很容易被视为过于僵化而难以维护。显然，这将需要将数据存储到不同变量中的代码。

`age1 = 11;`
`age2 = 9;`
`// ... 这里还有更多`

这不仅不方便，而且容易出错。数组提供了一种解决这个问题的方法。如果你只需要存储学生的年龄，你可以创建一个包含 1000 个整数的数组，这些整数可以通过从 `0` 到 `999` 的值进行索引。

`int ages[1000];`

现在，你可以编写代码来根据需要访问和修改数组。例如，为了存储所有年龄，假设你有一个返回学生人数的函数，以及另一个返回编号为 `i` 的学生年龄的函数。然后，你可以编写以下代码来计算平均年龄：

```
void average_student_age()
{
   int total_age = 0;
   int max = number_of_students();
   int i;
   double denominator = max;
   for (i=0; i<max; ++i)
   {
      ages[i] = age_of_student(i);
   }
   for (i=0; i<max; ++i)
   {
      total += age[i];
   }
   return total / denominator;
}
```

这个函数首先将数据读入 `ages` 数组，然后计算平均值（总年龄除以学生人数）。

数组的声明语法类似于单个变量的声明方式。但是，要创建一个特定大小的数组，你需要在方括号内输入元素的数量。在上面的例子中，你有一个包含 1000 个整数的数组。为了声明数组，你需要一个固定的大小，例如 `1000`。对于可变大小的数组，稍后你将使用 Objective-C 的容器类，例如 `NSArray`，它为数组的创建和调整大小提供了更灵活的机制。

要访问数组的元素，你只需要使用括号来定位所需的元素。例如，要给数组的第三个元素赋值，你需要使用以下代码：

`ages[2] = value;`

这段代码片段会将 `value` 的内容存储到数组 `ages` 的第三个位置。

**注意：** Objective-C 数组是从零开始索引的，因此第一个元素位于位置零。对于学习该语言的人来说，这可能会造成混淆，但你只需要记住，索引表示你想从数组起始位置偏移多远。第一个元素距数组起始位置的距离为零，因此索引为零。类似地，第二个元素的距离为一。



### 数组与指针

Objective-C 中数组最重要的特性之一是与指针紧密相关。这两个概念之间的关系源于 Objective-C 应用程序中数组内存的创建方式。数组仅作为内存块存储，其中的元素通过距数组起始位置的距离来访问。

例如，考虑前面示例中提到的 `ages` 数组。当编译器读取数组声明时，它会预留一块足够大的内存区域，以容纳所请求数量的元素。

要访问特定元素，编译器会获取数组起始地址并加上特定的偏移量。因此，以下代码是可行的：

```
int ages[1000];

// 初始化数组的代码 ...

void print_item(int pos)

{

   int *pointer = ages;

   NSLog(@"item value: %d is the same as %d", *(pointer + pos), ages[pos]);

}
```

这段代码说明了一个事实：数组可以转换为同类型的指针。此外，对指针加值相当于访问数组的特定元素。因此，上述代码等同于以下代码，后者以不同方式访问数组：

```
int ages[1000];

void print_item(int pos)

{

   int *pointer = ages;

   NSLog(@"item value: %d is the same as %d", pointer[pos], (ages+pos));

}
```

鉴于 Objective-C 编译器存储数组的方式，使用指针访问数组元素不仅是可行的，而且也是专业程序员经常采用的做法。

## 总结

本章我们简要概述了 C 语言和 Objective-C 语言共有的主要概念。虽然我们没有足够的篇幅来描述 C 语言语法和语义的每个细节，但这里讨论的基本元素足以用来创建复杂且实用的应用程序。

C 语言的编程方法论——结构化编程——要求将程序划分为相关的函数。而另一方面，在 Objective-C 中，主要概念是将程序分解为类，这些类封装了代码以及其方法所使用的数据。此外，Objective-C 通过消息传递的概念提供了一种灵活的运行时代码执行机制。这是下一章的主要主题，届时你将开始探索面向对象编程的概念。
