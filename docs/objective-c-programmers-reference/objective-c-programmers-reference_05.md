# Objective-C 进阶指南

`Objective-C`提供了另一种通过其块（`block`）语法来减少编程工作量的技术。`block`是一段代码，可以传递给应用程序的其他部分。`block`不仅保留代码信息，还保留了创建`block`时作用域内的变量信息。这样，`block`就类似于其他语言中闭包的概念。通过定义`block`，你可以使某些算法的实现更具扩展性。你还可以使用`block`来实现将在其他线程中运行的代码，正如一些`Objective-C`库所实现的那样。

### 第 7 章：动态绑定

`Objective-C`最大的优势之一是它允许程序员根据需求选择速度和灵活性的最佳组合。`Objective-C`的运行时系统是程序员可以用来在动态和静态编程之间切换的主要机制。在本章中，你将学习如何探索运行时系统，以在你的软件中实现静态检查和动态灵活性的最佳组合。

### 第 8 章：内存管理

使用具有动态特性的面向对象语言的优势之一是，内存管理可以简化，由系统库完成大部分繁重的工作。`Objective-C`使用基于引用的内存管理模型，其规则使得在任何应用程序中分配和释放内存几乎变得直接简单。在本章中，我将回顾你需要掌握的一些简单规则，以编写正确的`Objective-C`代码。我还将介绍`Apple`编译器提供的最新技巧，这些技巧实际上消除了在内存管理机制中手动干预的需要。

### 第 9 章：键值编程

在`Objective-C`中与对象交互的一种非常常见的方式是为特定的属性名称（也称为键）设置值。键值编程非常重要，因为它被用于`Objective-C`的众多库中。它允许通过一个简单的接口使用对象，避免了在对象之间唯一区别是它们包含的值集时创建多个子类的需要。

### 第 10 章：文件系统

`Objective-C`库提供了一种简化的机制来访问系统资源。其中一个例子是如何使用`Foundation`框架来访问文件和目录。在本章中，我将展示如何在这个标准`Objective-C`框架中使用类来创建文件、读写现有文件中的数据以及操作目录的内容。

### 第 11 章：Foundation 框架

`Objective-C`中的类被组织成框架，以便于分发和使用。`Foundation`框架提供了编写与环境交互的程序所需的类。基本的实用类，如字符串、字典和数组，也包含在`Foundation`框架中。在本章中，我提供了对`Foundation`框架类的参考，并在必要时附有示例。

### 第 12 章：编译器

为了有效地使用`Objective-C`，了解语言的基础设施很重要。编译器在软件开发周期中扮演着特殊角色，因为它决定了源代码如何被翻译成可执行程序。在`Apple`环境中，主要使用两种编译器：`gcc`，即`Mac OS X`使用的传统编译器；以及基于`LLVM`（`Lower Level Virtual Machine`）的新编译器，这是`Apple`在过去几年中开发的。在本章中，我将介绍`Objective-C`编译器提供的几个选项，以及如何使用它们编写快速可靠的代码。

### 第 13 章：预处理器

`Objective-C`自带一个强大的预处理器，可以简化将重复代码输入程序的过程。然而，如果你不谨慎使用，预处理器可能会成为许多编程陷阱的源头。在本章中，我将介绍预处理器的主要特性以及如何避免它的一些粗糙边缘。

### 第 14 章：单元测试

测试驱动开发是一种软件方法论，要求程序的每个特性都要通过单元测试代码进行彻底测试。测试驱动开发是利用面向对象语言（如`Objective-C`）模块化特性的绝佳方式。在本章中，你将学习如何在`Objective-C`语言中创建单元测试。你还将学习如何使用`Xcode IDE`管理和运行单元测试。

### 第 15 章：调试

任何程序员都知道编写无缺陷软件有多困难。这就是为什么每个平台都有许多集成的调试辅助工具。`Xcode`为编写和调试程序提供了完整的解决方案，这使得在新建或现有应用程序中查找错误变得更加容易。在本章中，我将解释如何使用这些调试功能。通过教程风格，我将描述如何使用断点、监视窗口、条件表达式和日志终端。

### 第 16 章：在 Cocoa 中构建 OS X 应用程序

`Objective-C`最重要的用途是实现使用`Cocoa`框架的应用程序。作为一名`Objective-C`程序员，你可能会参与创建`iOS`应用程序或完整的`Mac OS X`应用程序。在本章中，我将介绍`Cocoa`，这是用于在`Mac OS X`中开发桌面应用程序的框架。

### 第 17 章：在 Cocoa Touch 上构建 iOS 应用程序

`Cocoa Touch`是用于为`iOS`（驱动`iPhone`、`iPad`和`iPod touch`的操作系统）创建丰富应用程序的框架。虽然使用与`Cocoa`类似的技术，但与功能全面的`Cocoa`框架相比，`iOS`提供了简化的架构。`Cocoa Touch`还提供了对移动资源的访问，如摄像头、加速度计和`iOS`管理的其他硬件。



## 编译代码示例

本书中给出的示例均在 OS X 系统上使用免费的 Xcode IDE 编译。要使用 Xcode，你需要拥有一台 Mac OS X 系统的机器。如果你的电脑尚未安装 Xcode，可以从 Apple 开发者网站 [`http://developer.apple.com`](http://developer.apple.com) 下载。

另一种安装 Xcode 的方式是使用 Mac 上的 App Store 应用程序。搜索 Xcode 并点击安装即可。

下次启动 Xcode 时，通过点击“文件 ➤ 新建 ➤ 项目”来创建一个新项目。这将弹出一个向导窗口，你可以在其中为新项目设定一些基本选项，例如位置、名称以及目标操作系统（请将 Mac OS X 设为目标系统）。

在新项目中，你可以创建文件并输入书中展示的示例代码。要下载本书包含的所有示例的完整集合，请访问本书的网页 [`http://coliveira.net`](http://coliveira.net) 或 [`www.apress.com`](http://www.apress.com)。
