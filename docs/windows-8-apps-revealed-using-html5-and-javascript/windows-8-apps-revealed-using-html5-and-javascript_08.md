# `default.js` 文件与 Windows 商店应用

`default.js` 文件是 Windows 商店应用开始脱离标准 Web 应用环境的地方，你会在其中看到一些 Windows 专属 API 的初步呈现。与 Web 应用相比，Windows 商店应用拥有更为复杂的生命周期，而 Visual Studio 添加到 `default.js` 中的代码则提供了处理不同应用状态的基本支持。它并没有看起来那么复杂，我将在第 5 章解释你需要了解的 Windows 8 应用生命周期，以及如何应对它。

### 在 Windows 8 中使用你喜爱的 JavaScript 库

至此，你会意识到，你在 Web 应用开发中的许多经验，可以直接迁移到 Windows 8 开发的世界中。你可以通过使用你喜爱的 JavaScript 库，为你的 Windows 8 项目抢占先机。例如，我就是 jQuery 的忠实粉丝，任何读过我的《Pro jQuery》一书的人都知道。在大多数情况下，只要避免使用那些 Windows 8 与主流浏览器采用不同模型的功能区域，你使用编写良好的库时就不会遇到任何问题。举例来说，jQuery 在 Windows 商店应用中运行良好，但使用 `ready` 事件时要小心。在 Windows 8 应用中，你需要直接响应生命周期事件。

另一个需要避免的领域是异步脚本加载；Windows 8 应用的 JavaScript 文件是本地安装的，加载速度非常快。无需管理脚本加载过程，而且由于我在 Windows 8 上遇到过这方面的问题，我发现直接使用常规的 `script` 元素加载代码更为简单。

然而，有些库使用起来意义不大。例如，像 jQuery UI 和 jQuery Mobile 这样的用户界面工具包。你可以让它们在 Windows 8 中工作，但最终得到的应用不会遵循 Windows 8 独特的风格，并且可能无法像其他 Windows 商店应用那样以完全相同的方式响应触摸事件。

作为一般性指导，我建议你在开始使用你喜爱的 JavaScript 包之前，先熟悉一下 `WinJS` API 的功能。微软提供了一套相当扎实的基础功能集，包括界面控件、数据绑定，甚至还有 jQuery 的精简版本。这些功能有时会存在缺陷，本书中你会看到其中一些，但我建议你在添加其他 JavaScript 库之前，先了解 `WinJS` 提供了什么。

### 启动和调试 JavaScript Windows 8 应用

测试和调试 Windows 8 应用的最佳方法是使用*模拟器*，它是 Visual Studio 安装的一部分。在 Visual Studio 窗口中，你会看到 *Local Machine* 文字旁边有一个右箭头。点击文字右侧的小向下箭头，然后从菜单中选择 Simulator，如图 1-4 所示。

***图 1-4.** 为 Windows 8 应用选择模拟器*

要启动 Windows 8 应用，请点击现在显示为 Simulator 的按钮。将会出现一个新窗口，显示 Windows 8 应用，如图 1-5 所示。

***图 1-5.** Windows 8 模拟器*

这是测试 Windows 商店应用的最佳机制，因为它允许你模拟开发机器本身不具备的设备功能。如果你探索模拟器窗口右侧边缘的按钮，你会看到更改屏幕分辨率、更改设备方向以及模拟触摸交互和位置数据的选项。

**注意：** 还记得在本章前面为 `default.css` 文件添加样式时，我将网格布局的宽度设置为可用空间的 120%。你可以在图中看到这个效果。右下角容器的文字被裁剪了，部分布局没有立刻显示出来。你可以通过触摸或使用鼠标滑动视图。

### 重新加载 Windows 8 应用

使用 JavaScript 开发 Windows 商店应用的一个好处是，你无需停止并重新启动模拟器中的应用来反映你所做的更改。为了演示这一点，我做了几个简单的更改。首先，我更改了 `default.html` 中一个 `div` 元素包含的文本，如列表 1-6 所示。

***列表 1-6.** 进行 HTML 更改*

```html
. . .
<div id="leftContainer" class="gridLeft">
<h1 class="win-type-xx-large">左侧完整容器</h1>
</div>
. . .
```

我还对 `default.css` 进行了更改，如列表 1-7 所示，为另一个元素分配了不同的背景颜色。

***列表 1-7.** 进行 CSS 更改*

```css
. . .
#topRightContainer {
-ms-grid-column: 2;
-ms-grid-row: 1;
background-color: #808080;
}
. . .
```

**提示：** 当我在展示此类部分代码列表时，只有加粗部分发生了更改。文件的其余部分与先前的列表保持一致。我知道有些读者喜欢看到完整的代码列表，但我需要在一本薄书中包含大量信息，这是提高内容密度的一个绝佳方法。别忘了，你可以免费从 Apress.com 获取完整的源代码。

图 1-6 展示了控制 Windows 8 应用在模拟器中执行的 Visual Studio 控件。前三个是暂停、停止和重新启动执行。这些是自 Visual Studio 诞生以来我一直记得的传统调试器控件。

***图 1-6.** 用于重启和重新加载 Windows 8 应用的控件*

新增的是重新加载按钮，我已将其突出显示，它位于其他控件的右侧。点击此按钮会快速重新加载 Windows 8 应用的内容，并立即反映任何更改。你也可以按 F4 键或选择“调试” ➤ “刷新 Windows 应用”来重新加载项目。你可以在图 1-7 中看到我的 HTML 和 CSS 更改的效果。

***图 1-7.** Windows 8 模拟器现在显示 HTML5 和 CSS 更改*

不仅仅是 HTML 和 CSS 被重新加载。应用的 JavaScript 也会被刷新。这是为 Windows 商店应用建立迭代式“编写-测试”开发循环的一个极好且快速的方法。

**提示：** 如果添加了新文件，则无法重新加载应用，但 Visual Studio 会在发生这种情况时发出警告，并提供重新加载应用作为替代方案。

### 调试 Windows 商店应用

Visual Studio 拥有出色的调试器，可以非常轻松地用于追踪 JavaScript Windows 商店应用中的问题。在我自己的应用代码中，我发现最简单的方法是在代码中添加 `debugger` 关键字。当运行时遇到该关键字时，调试器会中断，然后我可以单步执行代码、检查变量，并使用 JavaScript 控制台执行少量代码片段。

但是，`debugger` 关键字只能用于你可以修改的代码，这意味着当我想要在 Microsoft 的 `base.js` 和 `ui.js` JavaScript 文件中的某个语句上中断调试器时，我必须采取不同的方法。为此，我需要选中感兴趣的语句，右键单击，然后从弹出菜单中选择“断点” ➤ “插入断点”。效果相同，但不会对文件进行任何修改。

我建议你探索一下 Visual Studio 调试器；它比浏览器中可用的开发者工具更复杂，花上几个小时来掌握它的工作方式是非常值得的。
