# 第 3 章：应用程序控件

## 本章小结

在本章中，我介绍了 Windows 8 应用的三个重要结构特性：`AppBars`（应用栏）、`Flyouts`（浮出控件）以及导航模型。这些机制开始弥合通用 Web 应用与将应用程序分解为可管理模块的工具之间的差距。

你不必非要使用`AppBars`和`Flyouts`，但如果不用，你的应用将无法融入 Windows 8 模型。Windows 8 的魅力之一在于能够运用你的 HTML5 和 JavaScript 技能来创建 Windows 应用程序。如果创建的应用不遵循 Windows 8 的惯例，那就错过了 Windows 8 为 Web 程序员提供的机遇。

同样，你也可以选择使用单个 HTML 文档来构建你的 Windows 8 应用。但话说回来，这同样会错失良机。驱动 Web 应用走向内容整合的那些限制条件，在 Windows 8 应用中并不存在，这意味着因分解内容和代码而带来的开发、测试和维护上的便利是值得探索的。`HTMLControl`和页面功能是实现这种开发风格的关键要素。
