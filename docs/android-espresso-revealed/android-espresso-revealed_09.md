# 第 12 章：使用 Espresso 和 Kotlin 测试机器人模式

在本章中，你将学习如何应用一种测试机器人模式，该模式将测试实现与业务逻辑分离到 Espresso UI 测试中。



## 第 13 章：使用 Espresso 和 UI Automator 进行有监督的 Monkey 测试

本章将展示如何利用 Espresso 和 UI Automator 实现有监督的伪 Monkey 测试，这些测试既适用于可访问源代码的应用程序，也适用于第三方应用程序。

## 第 14 章：AndroidX Test 库

本章演示如何将测试代码从 Android 支持库迁移到 AndroidX Test 库。你将了解 AndroidX Test 库中引入的新 API，并学习如何将它们应用于 UI 测试。

## 第 15 章：提高生产力与测试非常规组件

本章包含其他章节未涉及的代码示例，以及能够提升日常测试编写效率的 Espresso 测试技巧。内容包括创建参数化测试、将测试聚合到测试套件、在 UI 测试中使用 AndroidStudio 实时模板、在 Espresso UI 测试中设置 `SeekBar` 进度，以及 Espresso `Drawable` 匹配器等相关主题。

## 本书未涵盖的内容

本书旨在创建一份指南，指导如何为 Android 应用程序编写端到端的 UI 自动化测试，而无需模拟或存根应用程序依赖项。在我看来，这是最接近重现终端用户行为的方式。本书不涉及如何模拟应用程序数据和网络连接请求，或绕过应用程序工作流中的某些状态。

## 工具要求

要使用本书，你需要至少具备 Android Studio IDE、Gradle、GitHub 和 shell/bash 等工具和平台的基本操作知识。在大多数情况下，我会说明如何配置你的 IDE，并指出运行特定脚本时应使用哪些命令。
