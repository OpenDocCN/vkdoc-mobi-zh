## 引言

尽管目前大多数 iOS 开发都集中在使用苹果软件开发工具包的原生应用程序上，但随着不同移动设备和操作系统版本数量的增长，这变得越来越不适用。整合资源来制作可以跨平台运行的 Web 应用程序，远比雇佣一个团队来构建特定版本的应用程序更具成本效益。

从这个意义上说，Web 应用程序可能才刚刚开始发力。

事实上，Web 浏览器的性能和特性使得其在用户体验和提供的服务方面的可能性越来越大，并且 Web 浏览器为 Web 开发者创造了一个激动人心的竞技场。随着新标准的出现和实施，原生应用程序和 Web 应用程序之间的界限从未如此模糊；例如，地理定位和多点触控控制已不再是梦想。另一方面，HTML5 不仅仅是标记语言的一个新版本；新的规范带来了一系列全新的 API，这些 API 将使你忘记大量使用演示性图片，并让你能够丰富地集成多媒体元素。

由于这些新特性大多源于公认的规范，它们不仅在 Mobile Safari 布局引擎中实现，而且在其他移动设备和桌面浏览器中也广泛可用。

Web 浏览器无处不在；它们是有史以来最广泛的发布平台，可在所有平台上使用，并越来越多地在所有设备上使用。这使得 Web 开发成为开发者能够用最少的资源接触到最广泛受众的领域。在未来的几年里，这很可能将构建 Web 应用程序变成新的“黄金国”。

### 本书读者对象

本书特别适合在前端 Web 开发方面经验丰富，并希望转向移动开发，主要是针对基于 WebKit 的浏览器进行 Web 应用程序开发的开发者。尽管本书中的示例和解释旨在适用于 iPhone、iPad 和 iPod touch，但你应该会发现它们对于开发 webOS、Android 或最新 BlackBerry 设备的浏览器，以及任何现代桌面浏览器同样适用。

本书对于原生应用程序开发者来说也很有用，因为经常需要借助 iOS SDK 中的 `UIWebView` 来更好地控制应用程序的界面，并且牢牢掌握本书所介绍技术的最新进展会大有裨益。

### 你需要了解什么

本书假定你已经对 Web 及其底层技术有了扎实的了解，包括熟练掌握 JavaScript 开发。JavaScript 不仅对增强现代 Web 应用程序中的用户体验至关重要，而且为了使用 HTML5 以及其他万维网联盟（W3C）或 Web 超文本应用技术工作组（WHATWG）规范带来的最新 API，它也是必需的。为了构建高效的应用程序，你还需要对面向对象编程和 JavaScript 特有的相关机制有很好的理解。

### 本书内容

本书分为三个主要部分。它们将依次为你提供构建 Web 应用程序和高效开发移动设备所需的背景知识；带你了解 CSS3、HTML5 和 Mobile Safari 的许多令人兴奋的新特性；并使你能够进一步使用 Web 标准。以下是每部分的简要描述：

**第一部分，Web 应用开发入门：** 本书的第一部分介绍了用于开发 Web 应用程序的有用工具。当然，你很可能有自己的习惯，拥有一个功能完备且随时可用的开发环境。然而，深入研究 WebKit 浏览器的特定开发工具将帮助你更快地构建更好的 Web 应用程序。

**第二部分，使用 HTML5 和 CSS3 的 Web 应用设计：** 第二部分将让你更详细地了解 Web 应用程序的样子以及构建成功 Web 应用程序时应遵循的规则。遵循用户界面和用户体验的具体指南——这应该是你的两个首要关注点——我们将介绍最新的技术，这些技术将使你能够有效地实现风格、内容和交互方面的目标。

**第三部分，进一步使用 JavaScript 和 Web 标准：** 最后一部分将带你更深入地了解客户端开发，主题包括 Ajax 和 HTML 模板、苹果多点触控 API 的高级处理以及基于位置感知的 Web 应用程序。本部分的最后一章将为你介绍使页面能够离线可用和功能完整的方法，为构建表现如原生应用程序般的 Web 应用程序画上点睛之笔。

### 你准备好了吗？

得益于 iPhone、iPad 和 iPod touch 上令人难以置信的 iOS 平台，以及 Mobile Safari 对最新 Web 标准的实现，你不仅会发现你的 Web 应用程序几乎没有限制，还会注意到你的开发流程和可用选项将更加流畅并充满创造力。

本书中的一些例子可能看起来比其他例子更技术性和更高级。但是，不要害怕，因为一切都会逐步解释。同时，不要忘记自己测试代码并进行实验是学习和理解的最佳方式。所以，不要犹豫，修改本书中的示例并根据你的需求进行调整。

阅读本书后，你将掌握构建高级 Web 应用程序的主要工具，这些应用程序具备吸引用户的主要优势。那么，来吧，翻过这一页，跳入这个激动人心的创作领域吧！
