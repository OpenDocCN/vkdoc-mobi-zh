# 排版后的文本

`maximumAge`选项告诉设备在尝试确定用户实际位置之前先查找此类数据，并允许你为此数据设置一个时效限制。如我们之前所见，此解决方案的另一个优点是它减少了电池消耗。

`maximumAge`的默认值为`0`，意味着设备将尝试为每个请求返回一个更新的值；如果设置为`Infinity`，用户代理将必然返回一个缓存值（如果有的话）。如果不存在缓存值，它将获取一个新位置。

`timeout`属性为位置获取操作完成设置了一个以毫秒为单位的时间限制。默认值为`Infinity`，这可以让你想象`timeout`选项在开发应用程序时的用处。如果未指定，应用程序将一直运行直到终端返回所请求的信息，即使此信息由于某些原因不可用。指定一个`timeout`值后，当达到时间限制时，一个`TIMEOUT`错误将被发送到错误回调函数。

在前面的示例中，如果没有缓存任何位置值，函数的失败回调将被立即调用。请注意，用户授予访问其位置信息所花费的任何时间都不被计入。

组合这些选项可以使你的应用程序更具响应性。然而，当你确实需要这种额外的精度时，第三个选项`enableHighAccuracy`告诉 API 获取尽可能接近用户实际位置的结果。此选项默认设置为`false`。

请记住，用户通常关心超本地信息，因此你可能希望尝试提供精准定位。当你轮询位置时，精度会提高。这可以通过使用 API 的跟踪功能来实现。作为最后的手段，如果位置不可用，你可以依赖一个文本字段，以便用户可以指定他们的城市或邮政编码，这通常是最后一个可接受的精度级别。
