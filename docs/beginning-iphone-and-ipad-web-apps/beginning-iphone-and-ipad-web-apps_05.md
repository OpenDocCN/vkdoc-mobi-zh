### 客户端数据存储

WebKit 中一项令人兴奋的新功能是，不仅可以通过 Cookie，还可以通过直接在浏览器中设置的数据库，在用户端存储数据。我们将在第 15 章详细讨论此功能；现在，我们先看看 `Storage` 部分（旧版本中为 `Database` 部分）如何允许你检查多种客户端存储类型（见图 3-31）。

**图 3–31.** Storage 检查器

此时，Storage 检查器的布局应该看起来很熟悉，它包含常见的侧边栏和主视图。左侧的侧边栏中列出了存储类型的窗格，每个窗格下会显示当前正在使用的各种存储组或 Cookie（如果有）。

#### 数据库存储

第一个组用于数据库。每个数据库都可以通过数据库图标前的箭头进行折叠。要查看某个存储块的数据表，只需点击它。选中的表会显示在窗口右侧，以数据网格的形式呈现所有列和行。

与 `Scripts` 部分一样，如果你想进行更深入的检查，可以点击存储块来调用交互式 shell。此处的 shell 专门用于 SQL 命令，允许你借助自动补全功能查询数据。请注意，在当前版本中，在此处运行命令是与表交互的唯一方式，不能像其他存储功能那样直接编辑表。

不过，查询结果会立即显示在屏幕上，因此过程相当快速简单。

#### Cookie

Cookie 是一种早已为人熟知的客户端存储方法。从左侧选择 `Cookies` 窗格可以极大地简化你对 Cookie 的管理。当前页面的所有可用 Cookie 都会以表格形式呈现，包括每个 Cookie 的名称、值、域（是来自你页面的 Cookie 还是来自第三方站点，如广告）、路径、过期日期、大小、HTTP 和仅 HTTPS 标志。你可以删除任何 Cookie；但所有 Cookie 删除都是不可逆的，因此请谨慎使用——你肯定不想因为误操作而丢失所有常用网站的偏好设置。

#### 其他存储功能

Cookie 的大小限制为 4KB。除了数据库存储，WebKit 还通过本地存储和会话存储为你提供了突破此限制的解决方案。这两个功能在左侧边栏中都有专门的窗格。与 Cookie 一样，它们以键值对的形式存储信息。为了说明这一点，你可以在控制台中输入以下内容：

```
localStorage.myKey = "Some value";
sessionStorage.anotherKey = "Some other value";
```

这些行将分别在本地存储和会话存储区域中创建新的存储部分。与 Cookie 一样，你可以通过点击表格下方的叉号图标来删除条目，但不同于 Cookie 和数据库存储，你可以直接在表格中修改值，并通过双击空白区域来添加新条目。

### 审核你的页面

`Audits` 部分可以通过提供与页面加载相关的指导和最佳实践，帮助你优化网站。这些指南主要基于 W3C WebApps 工作组（WWAWG）关于构建更好的 Web 应用程序的建议。在撰写本文时，你无法扩展审核功能，但这是未来版本的计划，届时应该允许你专门针对特定问题类别，例如安全性或特定设备。

同样，左侧边栏中列出了可用的审核项，窗口右侧则包含用于运行新审核的数据和选项（图 3-32）。你可以选择“网络利用率”、“页面性能”或“全部”。你还可以选择在页面加载时进行审核，或者审核页面的当前状态。一切准备就绪后，只需点击窗口底部的“运行”按钮，审核结果便会很快出现在检查器中。

**图 3–32.** 审核结果

结果根据关键程度分类——红色表示重要问题，绿色表示可以考虑的次要改进。在建议中，你可能会找到未使用的 CSS 选择器列表、浏览器缓存技巧或 Cookie 处理技巧。例如，你可能会被建议将静态内容放在单独的服务器上，以防止每次新请求（图像、样式表等）时都携带无用的 Cookie；显然，这会不必要地增加流量负载，因为 Cookie 在服务器上被图像等静态内容处理的可能性很小。

如果我们还没足够强调，你正在面向的设备连接质量不可靠，且整体性能低于台式计算机或笔记本电脑。每节省一个千字节，就意味着节省了时间，也意味着离用户离开你的页面（可能再也不回来）更远一步。

### 仍然不确定？

尽管我们认为 WebKit Web Inspector 是前端调试的绝佳工具，但在开发过程中你自然可能更喜欢其他浏览器或工具。如果你已经是前端开发人员，你可能习惯于 Firefox 上的 Firebug 或 Opera 上的 Dragonfly。即使是 Internet Explorer，其最新版本也附带了开发者工具。无论你情况如何，请注意，当今大多数浏览器都内置了调试和开发工具，或者可以通过扩展程序获得。Web Inspector 实际上并非从零开始构建：例如，`Console API` 最初是在大获成功的 Firebug 中使用的。这最终是一个习惯和个人选择的问题。

如果你喜欢 Firebug，或者想要更多直接在目标设备上调试的功能，我们建议你关注 Firebug Lite 的更新。该工具是 Firebug 的轻量级版本，可作为书签小程序在所有浏览器中使用。目前，它在 iPhone 上功能还不算非常完善，但随着它的发展，它可能成为 Web 应用程序开发者的一个严肃选择。

## 总结

现在，你应该已经牢牢掌握了可用的工具，以加快你的开发进程。你也会感到欣慰，因为在你日常使用的浏览器中，就有这么多有用的工具来让开发者的生活更轻松。你并非孤军奋战，因为我们介绍的工具是用 Web 语言构建的，来自社区的演进也证明会大有裨益：谁能比 Web 开发者更了解他们需要什么呢？在接下来的章节中，我们会在必要时提供关于其中一些工具的更多细节。我们打赌你感觉到事情正变得更加具体。在下一章中，我们将剖析通用的 Web 应用程序元素，然后再深入到 Mobile Safari 开发的核心。
