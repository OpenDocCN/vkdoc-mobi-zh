# 第一部分：Web 应用开发入门

## 第 1 章：开发工具

与原生 iPhone 应用程序开发不同，Web 应用程序开发不需要特定的集成开发环境（IDE）。只要你有一个可以保存纯文本的编辑器，就没问题。同样，前端 Web 开发者的主要工具——Web 浏览器——有各种形式，众所周知且易于获取。

免费并不意味着可以粗心大意，在你的开发过程中，你会希望获得所有能得到的软件效用。第一章旨在帮助你建立一个高效的工作环境，同时确保我们从 iOS 冒险之旅的一开始就有共同的基础。

### 源代码编辑器

市面上有很多出色的编辑器——开源的、专有的，免费的、付费的，像点唱机一样漂亮的，或者直接运行在终端里的。从我们的角度来看，一个真正有用的编辑器至少应该支持语法高亮、自动补全以及某种代码片段工具。

#### 科莫多巨蜥

如果你已经涉足 Web 开发，你很可能已经找到了你完美的代码伙伴。如果没有，我们建议你下载并安装 Komodo Edit，这是一个跨平台、免费且功能丰富的开源编辑器，基于 Mozilla 代码库构建，我们将在所有示例中使用它。

**注意：** 你可以从 ActiveState 网站 `www.activestate.com/komodo-edit` 下载适用于 Mac OS X、Windows 和 Linux 的 Komodo Edit。

与 Mozilla 一样，Komodo Edit 是一项开放 Web 和开放标准倡议。开放 Web 旨在鼓励社区驱动的、开放且非专有的 Web 技术发展，以确保 Web 的生存力和开放演进。

从 6.0 版本开始，Komodo Edit 显着地立即支持 HTML5 和 CSS3 语法，包括对 `-webkit-` 专有前缀的支持。这些在本书中都将非常有用。

#### 让自己舒适

Komodo Edit 功能如此丰富，以至于它本身几乎就像一个轻量级的 IDE。

当你启动 Komodo Edit 时，它会向你提供恢复上次会话的选项；回答“否”将带你进入起始页（始终作为特殊标签页访问），如图 1-1 所示。在那里，你会找到打开文件和执行各种操作的快速链接，并且 ActiveState 甚至为每个选项添加了示例来帮助你入门。你可以通过选择“视图 ➤ 标签页和侧边栏”菜单，然后选择“工具箱”来访问这些内置的“工具箱”面板。你还会注意到 Komodo Edit 的一个有趣优点：项目管理，它可以让你有序地组织项目中的所有元素。在第 4 章中，我们将创建一个将在本书中使用的 Web 应用程序模板，而使用 Komodo Edit 是创建、存储和重用此模板的一种简单方法。

另一个可以显著提高生产力的方面是 Komodo Edit 通过处理代码片段、模板甚至宏来增强代码和文件重用的能力。你可以直接通过右键单击一段代码，在上下文菜单中选择“添加为工具箱中的代码片段”来创建代码片段和模板，所以，放手去做吧——节省一些时间。

此外，如前所述，Komodo Edit 基于 Mozilla，这意味着它可以通过附加组件进行扩展。可以从“工具 ➤ 附加组件”菜单访问附加组件管理窗口。

你也可以查看社区网站和扩展部分，网址为 `http://community.activestate.com`。很可能你缺少的东西就在那里。我们不打算详细介绍 Komodo Edit，但它绝对值得一试。如果你尚未确定源代码编辑器，或者你希望偶尔改变一下工作习惯，请务必研究一下。更多文档可在其网站上找到。

#### Dashcode 怎么样？

你可能会想，“好吧，当我安装 Snow Leopard 时，它附带安装了 Xcode 和 Dashcode，那么为什么不直接用那个呢？” 你是对的，因为苹果专门为此目的开发了这个软件（图 1-2）。

Dashcode 是一个很好的工具，可以生成小巧而漂亮的 Web 应用程序，并在视图之间实现漂亮的过渡。更重要的是，Dashcode 大量使用模型-视图-控制器（MVC）范式来合理化和简化应用程序层之间的交互。尽管如此，它并不符合我们的需求。

首先，它只能在运行 Mac OS X 的计算机上使用，将使用范围限制在拥有 Mac 的开发者；如今这些开发者仍然只占编码人员的一小部分。

其次，它是可视化编程软件。你完全可以用 Dashcode 构建应用程序，而无需触及任何一行代码。这可能很方便，但这不是我们的重点。作为本书的读者和专业 Web 开发者，你需要完全控制你的代码，并且你想了解开发过程每个阶段正在发生的事情，以确保 Web 应用程序的质量。

最后，就像 Dreamweaver 和其他图形工具一样，它倾向于产生许多特定于框架的属性，使得整个代码的修改和维护变得繁琐，并且会生成太多不必要的标记和 JavaScript。在为 Web 开发时，特别是针对诸如移动浏览器这类内存有限的设备时，轻量级代码应该是首要考虑因素。

### 使用正确的浏览器

为 Web 开发主要是为 Web 浏览器开发。尽管我们将专门针对 Mobile Safari（用于 iPhone、iPod touch 和 iPad 的浏览器），但清楚地了解该领域的现状会有所帮助。此外，如果你已经是 Web 开发者，你可能习惯于主要使用特定浏览器来完成开发过程，而在桌面浏览器上开发通常比在移动设备上开发更高效。

#### WebKit

为苹果设备开发时，你会一直把 Mobile Safari 放在心上。当然，你可以努力使你的应用程序跨浏览器兼容，但这可能意味着更多的代码甚至更少的功能。而且，正如你将要看到的，Mobile Safari 并不是一个像最初看起来那样受限的目标。

Mobile Safari 和 Desktop Safari 都基于 WebKit 布局引擎。该引擎从设计之初就追求轻量化和标准兼容。WebKit 不仅实现了最新的 W3C 候选推荐标准，还实现了带有 `-webkit-` 前缀的草案阶段特性。

WebKit 引擎还附带了一整套开发者工具。我们将在第 3 章详细介绍 WebKit 的 Web 检查器，你很快就会看到它在你的 Web 应用程序开发过程中会有多大帮助。

**注意：** 开发 iPhone（或 iPod touch 或 iPad）Web 应用程序的最佳选择是在苹果平台上工作。但是，你可以轻松地在 Windows（无论是 XP、Vista 还是 7）上安装 Safari。作为 Safari 的替代方案，我们推荐基于 Chromium 项目的浏览器，例如 Google Chrome、Chromium 或 Iron。这三个浏览器都适用于 Mac OS、Windows 和 Linux。

WebKit 被许多平台上的浏览器使用，包括移动设备。其中，你可能会对 SymbianOS、谷歌的 Android、ChromeOS、BlackBerry 以及 Palm 最近的 webOS 感兴趣。你为苹果浏览器构建的大多数 Web 应用程序应该可以在其他移动设备上运行。

#### Gecko 和 Firefox

浏览器市场的另一个重要参与者是 Firefox，一个基于 Gecko 引擎的开源项目。它对标准的遵守非常好，你可能会欣赏 Firefox 最强的优势之一：它的社区。它可以通过丰富的附加组件无限扩展，例如 Firebug，这是一个全面且可用的开发和调试工具。这使得 Firefox 成为为 Web 开发时的绝佳伴侣，其功能与 WebKit 类似。

#### Opera

Opera 也适用于所有操作系统。Opera 团队一直将速度和遵守标准放在首位，尽管其浏览器没有占据主要市场份额，但它是前面介绍的浏览器的绝佳替代品，也是开发者的不错选择。事实上，Opera 自带了一套开发者工具——Dragonfly。Opera 也适用于移动设备（Opera Mini），但与 Mobile Safari 不同，它是浏览器的一个大幅精简版本，因此无法让你受益于桌面浏览器支持的所有标准。

#### 迷航的 Internet Explorer

如果你是一个正在考虑 Internet Explorer 的 Windows 用户，恐怕我们不能推荐这个浏览器用于 Web 开发目的。尽管从版本 3 到 5（第一个主要实现 CSS 支持的浏览器，以及第一个提供当时未命名的 Ajax 功能的浏览器），它绝对是一个先进 Web 浏览器，但它已经走上了一条偏离标准的道路，并且在第 8 版中仍然缺乏对许多特性的支持，例如 CSS 选择器和属性（即 CSS2 选择器和属性），在 Acid3 测试中仅获得 20/100 的分数。Internet Explorer 9 似乎在这方面将有重大改进，但就目前而言，基于 Trident 的浏览器并不是一个选择。

### 从 Web 角度进行开发

Web 有标准，其中最重要的是万维网联盟（W3C）的规范。这些规范为你——开发者——提供了构建出色的基于 Web 的应用程序的工具。这就是为什么在开发 Web 应用程序时，你对 Web 浏览器的首要关注点是其对 W3C 的兼容性。你选择作为首要目标的浏览器越符合标准，你的页面在其他平台上的功能性就越强，你需要做的额外工作也就越少。

#### Acid...Acid...Acid

Web 标准项目是一个致力于推广和鼓励尊重和采用 Web 规范的组织。由于并非所有浏览器在这方面都做得一样好，它提供了一种工具来轻松检查布局引擎的跟进情况。该测试的最新版本称为 Acid3。

**提示：** 要了解浏览器在该测试中的得分，只需访问 `http://acid3.acidtests.org`，几秒钟后你就会得到结果。

Mobile Safari 以 100% 的兼容性通过了测试，这很好，因为这意味着可以使用许多方法来增强你的应用程序。如果你正在使用另一个浏览器进行开发，你应该记住，你可能会在无意中忽略 HTML、CSS 和 JavaScript 中一些非常有趣的特性。

#### HTML5 一致性

尽管 Acid3 测试是评估浏览器 CSS 标准兼容性的好方法，但它不能精确指示支持了哪些内容。不过，你可以使用测试页面 `http://html5test.com` 检查浏览器对 HTML5 特性的支持，并获得支持和不支持标准的详细表格。

该测试页面将列出所有新的 API、扩展和 HTML5 特性，并指示所使用浏览器对这些特性的支持程度。Mobile Safari 在 300 分中获得了 125 分（以及 7 个奖励分）。虽然这看起来可能很低，但它实际上表明了对新标准的出色支持，尤其是对于移动浏览器而言。

要了解每个已实现特性的更多细节，列表中的每个元素都是一个链接，点击将直接带你到该规范的相关页面。

#### 浏览器中的卫星

现代前端开发（尤其是 Web 应用程序开发）的另一个核心部分是 JavaScript。我们将在大多数章节中大量使用 JavaScript，并在第 10 章介绍面向对象的 JavaScript 编程。

浏览器脚本引擎对 ECMA-262 规范（截至第 3 版）的符合性可以使用 Google 构建的 Sputnik 测试进行评估，该测试可从 `http://sputnik.googlelabs.com` 获得。它是一个包含超过 5,000 个测试点的开源测试。Safari 获得了非常好的分数，仅次于 Opera，排名第二。

### 针对 iOS 进行开发

尽管我们已经介绍了你可能需要的所有工具的跨平台选项，但你要针对 iOS 进行开发，以下软件虽然很有用，但只能在 Macintosh 系统上使用。如果你没有 Mac，请仍然继续阅读；尽管你无法使用下面部分描述的应用，但我们仍将介绍一些对你在开发冒险中有用的资源。

#### 使用 iPhone 和 iPad 模拟器

在开发 Web 应用程序时，有许多理由让你不想使用 iPhone 设备。好消息是，一旦你注册成为苹果开发者，苹果就会提供一个免费的工具来实现你的需求。只需在 `http://developer.apple.com/ios` 注册。然后，下载并安装免费的 iOS 软件开发工具包（SDK），特别是其模拟器，如图 1-3 所示。

**注意：** 下载前，先喝杯茶。你需要下载近 3GB 的数据。安装完成后，iPhone 和 iPad 模拟器应用程序将隐藏在 `/Developer/Platforms/iPhoneSimulator.platform/Developer/Applications/` 文件夹中。如果你不想每次工作前都疯狂地寻找它，请将其快捷方式拖到 Dock 或桌面！

模拟器的主要好处是，你不需要实际拥有一个合适的设备来进行测试，也不必通过反复加载页面来耗尽电池电量。更重要的是，它不仅支持 iPhone 模拟，现在还使用真实版本的 Mobile Safari 模拟 iPad。它还允许你在不同版本的 OS 上运行测试，而无需经过漫长且烦人的设备重置过程。

这是你能在桌面上获得的最接近 iPhone 的体验。至于非 Mac 用户，我们只能建议你在真正的 iPhone、iPod touch 或 iPad 上双重测试所有代码，以正确了解实际设备的表现。

#### 无论如何使用真实设备

使用模拟器，你会获得较少的功能，只够满足开发的最低要求。请记住，模拟器使用你的台式计算机资源，性能可能比在设备上更好。因此，过渡和高级视觉效果或多媒体内容（在第 8 章和第 9 章中介绍）应始终在目标产品上进行系统测试。

此外，由于屏幕尺寸确实有影响，你应该始终检查所有文本和元素（例如按钮）是否可读和可用，以免损害 Web 应用程序的用户体验。

#### ADC 是你的朋友

无论你是否拥有 Mac 来安装模拟器，注册始终是个好主意。苹果开发者连接（ADC）是一个资源丰富的地方，可以浏览并找到帮助。成为注册开发者将使你获得技巧、屏幕录像、访谈等更多资源。如果你从与你无关的信息（即关于原生 iOS 应用程序开发的信息）中找出路，你会发现苹果自己的信息可能非常有用。最好的起点是 Safari 开发页面 (`http://developer.apple.com/safari`)，该页面提供了关于支持特性的有趣信息。

## 第 2 章：开发环境

现在你有用于创建和测试的工具了。在第 1 章中，我们建议，为了正确评估你的应用程序，你应该始终在合适的设备上进行彻底测试，无论你的目标设备是什么。同样，了解你的页面在用户端表现如何的最可靠方法，就是尽可能在接近真实条件的开发过程中使用它们。

一旦你的应用程序准备就绪并大放异彩，你将需要将其托管在服务器上，以便它可以通过万维网提供服务。然而，理所当然地，你可能不想在仍在开发阶段时将网站托管在生产平台上。由于某些限制和缺点仅当网站托管时才会出现——同时也有一些受欢迎的好处——我们将指导你完成一个并非你想象的那么难的过程：在你的工作机上安装一个本地服务器。

你的新服务器将提供开发 Web 应用程序和按照本书示例进行操作所需的一切。基本上将其视为另一个工作工具。只需按照本章的步骤操作，你就不需要深入高级功能或花哨的配置。你将拥有一个随时可用的服务器，无论你的平台如何，它都能托管你的页面，并使其可供共享同一无线网络的外部设备访问。

### 提供 Web 应用程序服务

我们提到，直接在本地文件上开发 Web 应用程序时会受到一些限制。在你着手安装本地 Web 服务器之前，这里有一个例子：你构建了一个带有建议功能的搜索应用程序；当用户开始输入一个单词时——砰！——他正要输入的单词就出现在他面前！为了实现这一点，你从某个远程站点的列表中检查单词。你已经在本地文件上彻底测试了你的应用程序，它运行得非常完美。然而，当你将其上线时，魔法消失了——不再出现建议。这是因为 Ajax，实现此类功能的基础技术，仅当请求发生在同一来源时才有效。这个“跨域”限制在你测试时没有出现，因为本地文件没有域，并且由于使用了 `file://` URL 方案，执行上下文不同。这就是为什么你从未注意到这个问题——直到为时已晚。

如果你的开发过程中使用了 iPhone Simulator 或合适的设备——这是你应该做的——另一个缺点是它们都无法访问本地文件系统上的文件。托管你的页面是使用模拟器以及你的目标设备——iPhone、iPod touch 或 iPad——的必要步骤。

### Mac OS，化繁为简

最新版本的 Mac OS X 附带了设置工作服务器所需的所有组件。尽管站点托管对开发者（等等）很有用，但这通常不是他们的主要知识领域。幸运的是，配置过程已经变得非常简单。只需按照下面几节的步骤操作即可。

#### 在你的 Mac 中嵌入 Apache

从菜单中打开“系统偏好设置”，在出现的窗口中，打开“互联网与无线”部分。点击“共享”图标，然后选中“Web 共享”复选框（见图 2-1）。搞定了吗？嗯，好消息——你的服务器已经设置好了！

点击个人网站链接，Safari 或你默认的 Web 浏览器将使用你新个人站点中的默认文件打开一个窗口，如图 2-2 所示。你可以在 `/Users/username/Sites/` 文件夹中修改和添加此站点的文件，其中 `username` 是你的会话用户名。

第二个链接将带你到计算机的网站。访问此 URL 将显示在 Apache 默认安装中找到的常见“It works!”消息。此网站的文件位于 `/Library/Webserver/Documents/` 文件夹中。

如果你记不住像 `192.168.1.100` 这样的 URL 或者懒得输入，你也可以使用计算机名称访问你的网站。URL 看起来类似于 `http://computername.local/~username/`。

这些简单的步骤允许你使用你的主要语言——HTML、CSS 和 JavaScript——构建一个网站。这是你开始制作托管 Web 应用程序所需的最低配置，但要构建一个现代网站，你可能希望使用某种服务器端脚本语言来增加网站与用户之间的交互。我们将在本书的一些示例中使用 PHP，因此我们建议你遵循以下步骤。它不像安装的第一部分那样“一键式”，因为你需要以管理员权限修改配置文件。但别担心：它并不像看起来那么困难。

#### 脚本引擎

打开一个终端窗口，并输入以下命令：

```
sudo vi /private/etc/apache2/httpd.conf
```

系统会要求你输入管理员密码。你很可能就是自己电脑的管理员，在这种情况下密码就是你的会话密码。但如果你使用的是公司机器，你可能需要请别人为你输入密码。`apache2` 配置文件应该在终端窗口内的 Vi 中打开。

**注意：** 在本书中，我们不常使用命令行工具来编辑文件。尽管如此，当我们这样做时，我们将使用著名的 Vi。如果你不熟悉命令行文本编辑器，你可能会觉得它们有点吓人。它们通常学习曲线陡峭。然而，由于其中一些编辑器很受欢迎且功能确实丰富，我们认为可以在这里介绍一个。

相反，如果你不喜欢 Vi 并且已经安装了 Komodo Edit，请查看在 Komodo ➤ 偏好设置菜单项中启用 Vi 行为模式的选项；选择“编辑器 ➤ 键绑定”，然后选择“启用 Vi 模拟”，这是从上往下第二个选项。

文件打开后，将光标向下移动，直到到达看起来像这样的行：

```
...
#LoadModule php5_module libexec/apache2/libphp5.so
#LoadModule fastcgi_module libexec/apache2/mod_fastcgi.so
...
```

你想通过删除 `#` 符号取消这些行的注释，以启用这些模块并允许 PHP 脚本正确执行。为此，输入 `a`（用于追加），像在任何编辑器中一样取消注释这些行，通过输入 `esc` 退出编辑模式来保存更改，然后输入 `:wq`（用于写入并退出）。

为了使配置更改生效，请通过取消选中并重新选中“系统偏好设置”中的“Web 共享”选项来重新启动服务器，或者直接在终端窗口中输入以下命令：

```
sudo apachectl restart
```

你可能想知道你刚刚做了什么。第一行告诉服务器加载 PHP5 模块，允许你在服务器端运行 PHP 脚本。PHP 是一种流行的、开源且非常强大的通用脚本语言。在其最新版本中，它具有面向对象编程和许多使开发者生活更轻松的函数。

第二行允许分离脚本执行和其他服务器任务。简单来说，服务器和 PHP 模块（或任何实现 FastCGI 协议的其他模块）作为独立的应用程序运行，拥有各自的资源和进程。这相应地意味着，如果某个脚本导致 PHP 崩溃，则无需重启服务器。

所以，甚至没有注意到，你已经成功了：你已经在你的工作机上设置了一个带有 PHP5 的服务器，使开发过程既简单又专业。如果你是一个好奇的人，我们打赌你甚至已经修改了你新网站的一些东西（当然，如果你是 Mac 用户的话）。接下来是如何在 Windows 机器上实现类似结果的操作指南。

### 基于 Windows 的系统

其内容与 Unix 和 BSD 系统上的内容非常相似。你应该应用与上一节相同的更改。你的更改将自动生效。

### 配置多个网站

现在，作为额外福利，我们将讲解最后一件事。此步骤并非必需；我们只是认为部分读者可能对此感兴趣。具体来说，我们将解释如何在服务器上创建多个站点。这样，在同时处理多个项目时，你可以让浏览器理解正确的地址，而不是使用子文件夹。

除了在不同文件夹中使用不同站点名称带来的组织优势外，你还能受益于“根目录”的概念，无论是使用 PHP、HTML 还是 CSS。

实际操作中，在 PHP 中，你可以使用`$_SERVER['DOCUMENT_ROOT']`环境变量访问此信息。在编写 HTML 和 CSS 时，如果 URL 以斜杠（`/`）开头，浏览器会理解你调用的路径将从目录结构的顶层开始。在站点`http://localhost/example/`上，`/images/hey.png`表示`http://localhost/images/hey.png`。

### Apache 2：深入探索

以下步骤适用于 Apache 服务器，也就是你在 Mac OS X 和 Linux 平台上安装的服务器类型。作为幸运的 Unix 用户，你还需要稍作调整，因为如前所述，不同平台安装 Apache 的方式并不相同。你需要找到 Apache 的主配置文件，并对其进行修改。还有其他管理虚拟主机的方法，但这种方法最接近跨平台。

#### 检查配置

如同安装部分，你将再次在终端中使用`Vi`编辑文件。打开一个终端窗口，根据你的系统输入以下两条命令之一：

```
sudo vi /private/etc/apache2/httpd.conf # 适用于 MacOSX
sudo vi /etc/apache2/apache2.conf       # 适用于 Debian
```

一个文件将会打开，其内容因系统而异。你要做的第一件事是检查以下两行：

```
#### 确保 Apache 监听 80 端口
Listen 80

#### 在所有 IP 地址上监听虚拟主机请求
NameVirtualHost *:80
```

如果它们存在，很好，你可以进入下一步。如果不存在，你应该将它们添加到文件末尾。要在`Vi`中执行此操作，请将光标移动到文件末尾，再次输入`a`（Append，追加）以进入编辑模式。这些规则确保你的 Apache 服务器将监听 80 端口上的 HTTP 请求，无论 IP 地址如何（这就是使用`*`的原因）。

#### 配置你的第一个虚拟主机

接下来是创建实际的虚拟主机。你至少需要为你的站点指定一个名称（URL）和文档根目录的路径（你的文件将存放的位置）。在你的配置文件末尾，使用你想要的名称和路径添加以下代码：

```
<VirtualHost *:80>
    DocumentRoot /var/www/example
    ServerName www.example.local
</VirtualHost>
```

**警告：** 使用此指令添加虚拟主机将覆盖最初安装的主机。为防止这种情况，你应该在添加新虚拟主机之前，使用全局服务器名称及其文档根目录创建一个虚拟主机。你可以在正在编辑的主 Apache 配置文件中找到所需的信息。

完成所有必要的更改后，你可以保存文件并退出`Vi`，方法是按下`Esc`键，然后输入`:wq`（写入并退出）。然后，重启你的服务器。如果你是 Mac 用户并且遵循了 Apache 安装过程，你已经知道如何操作。对于 Debian 用户，请运行以下命令：

```
sudo /etc/init.d/apache2 restart
```

Apache 现在知道你的站点存在了，但系统的其余部分还不知道，除非你已将其添加到`hosts`文件中。要解决此问题，你应该按照前面解释的步骤修改此文件。

现在你可以在浏览器中检查你的新站点，输入你选择的服务器名称。由于你的文档根目录中没有文件，应该会显示一个相当空白的页面。不过，它能够正常工作，所以我们会在接下来的章节中与你见面，用令人兴奋的自制 Web 应用程序来填充它。
