# 第 5 章：用户体验与界面指南

目前，你只是改变了页眉的颜色，使其与 iPad 的配色方案相匹配。不过，完全没必要无意义地重载你的 iPhone 样式表，因为当你的网页应用在 iPad 或桌面浏览器上使用时，你也可以加载一个不同的、特定的文件。这可以通过在`<link>`标签的`media`属性中指定设备宽度来实现：

```html
<link rel="stylesheet" href="styles/big.css"
media="only screen and (min-device-width:640px)">
```

这样一来，如果用户使用的是 iPhone，就不会加载任何内容，但你可以为更大的设备添加合适的样式。将上述代码添加到你的模板`styles`目录下的一个名为`big.css`的文件中。

现在你有了针对 iPad 的特定样式表，让我们在其中添加一些代码，以实现之前提到的分屏视图。你将按照原生 iPad 已安装应用程序的实现方式来应用它。请注意，在横屏模式下，两列的高度必须相同，这一点很重要。因此，你对主视图和辅助视图使用了`table`和`table-cell`的显示属性值，这避免了使用 JavaScript 或利用背景图片的视觉假象，那些方式对渲染引擎负担更重、加载时间更长，而且整体上不够优雅且难以维护。

```css
.split-view {
  display: table;
  width: 100%;
  height: 100%;
}

.split-view .view {
  display: table-cell !important;
}

.split-view .view:first-child {
  border-right: solid 1px black;
  width: 320px;
}
```

你的标记代码将会像这样简单：

```html
<div class="split-view">
  <div class="view">
    第 5 章：用户体验与界面指南
    ...
  </div>
  <div class="view">
    ...
  </div>
</div>
```

现在，只需别忘了在你的主样式表中添加以下代码，以向 iPhone 用户隐藏辅助内容：

```css
.split-view .view:first-child {
  display: none;
}
```

这不会在 iPad 上生效，因为你之前已经在`display:table-cell`规则上指定了`!important`。`!important`声明会覆盖样式表或内联样式中的所有其他规则。

### 勇于创新

你现在已经具备了高效构建 iPhone 和 iPad 应用的知识和工具。一个摆脱默认设置的简单起点是：保持整体布局不变（你已经理解了所呈现的布局选择如何对网页应用的可用性产生积极影响），然后调整应用的颜色方案。iOS 的默认颜色虽然独特，但有时会显得有点冷冰冰。其相关性也取决于你的目标用户群体。例如，灰蓝配色方案可能会让青少年用户感到乏味。

一个简单又有趣的配色方案尝试方法是访问网站[`kuler.adobe.com/`](http://kuler.adobe.com/)。在那里，你可以从众多用户提交的主题中选择，或者创建自己的主题。你的想法是轻松生成符合苹果设计初期精神的配色组合。你也可以去那里了解当前流行的配色方案，或者寻找灵感。

当然，为你的网页应用寻找原创设计方法的最佳途径，就是尽可能多地实际使用你的目标设备。不仅是作为开发者，还要作为用户去深入了解它们，这将极大地帮助你理解什么“管用”，什么不管用。

模仿目标操作系统的用户界面是一个良好的开端，可以让你的应用更容易被用户接受。即使是懒惰的用户也应该能轻松适应他们熟悉的环境。因此，你的应用界面应该在现有基础上进行精心设计和直观构建。

尽管如此，你还应该经常能从日常生活中找到好的灵感。你喜欢整理文件来让自己井井有条吗？你需要一幅能让自己在良好状态下工作的宁静风景画吗？找到将这些体验移植到你的应用中的方法。很可能你不是唯一有这种需求的人，这种创新就能区分“好”与“优秀”。为了提供不同的体验，在简洁清晰的 iOS 界面基础上进行构建，创造出美丽而新颖的环境。

## 第 5 章：用户体验与界面指南 — 总结

为移动设备进行开发并不容易。你必须摒弃许多在传统 Web 上占主导地位的原则和习惯，才能打造出成功的网页应用。所有基于 iOS 的设备都为你的应用（从而也为用户）提供了出色的标准支持、简洁的界面和精巧的功能。

然而，这同时也伴随着一系列约束。你必须极为小心，不要让用户失望，始终为他们带来尽可能最佳的体验，并考虑到他们已习惯的标准。

要从根本上思考你的设计流程，但不要忘记你从以往开发中学到的东西。你拥有大量可用的工具，因此要充分利用它们，创造相关且独特的体验。在接下来的章节中，我们将更详细地探讨利用 WebKit 和 iOS 技术可以实现哪些功能。

## 第 6 章：用于网页应用用户界面的有趣 CSS 特性

HTML5 在文档语义方面比该语言的先前版本走得更远。然而，如果你从事前端开发已有一段时间，你一定意识到，吸引人的设计往往意味着艰难的选择和多余的标记。在本章中，我们将介绍大量 CSS 特性，它们将使你的网页应用开发过程更快、更丰富、更清晰。有些特性使得期待已久的增强功能变得轻松且易于维护，而另一些则更为复杂，但将为你的网页应用在 Mobile Safari 上开启全新的布局和设计可能性。

请注意，本章讲解的许多特性要么相当新颖且被认为不稳定，要么是 WebKit 特有的，因此都带有 WebKit 专有的`-webkit-`前缀。这对于属性值可能是必需的，并且对于我们即将解释的大多数属性都是必需的。作为一种良好实践，你应该始终先声明 WebKit 特定的规则，然后声明标准的、不带前缀的规则，以实现最大的向前兼容性。

请牢记这一切，然后深入探索 Mobile Safari 为你提供的这些新奇迹。

### 用 CSS 提升用户体验

在介绍 CSS3 规范带来的一些 CSS“甜点”以及其他一些常因浏览器兼容性而被牺牲的特性之前，让我们先了解一些 Mobile Safari 特有的 CSS 特性。虽然本章将要展示的 CSS 优秀特性将帮助你增强和细化对页面布局和设计的控制，但你的目标是一个特定的操作系统和浏览器，它们两者都具有强大的用户界面，并为用户创造了习惯和期望。我们将首先介绍的 Mobile Safari 功能，将让你构建的应用能更好地集成到用户设备中，从而提升他们的用户体验。

#### 用户反馈

在用户与界面交互如此紧密的设备上，用户的操作能在你的网页应用中触发视觉反馈至关重要，正如我们在第 5 章中解释的那样。



Mobile Safari 初始实现的交互状态基于 `-webkit-tap-highlight-color` 属性，该属性定义了一个覆盖框的颜色，用于在用户操作时高亮视口中的可点击区域。例如，在任何被点击的链接上都能看到这种效果。以下规则会在用户触摸可点击元素时使这些区域变暗（图 6-1）：

```
a { -webkit-tap-highlight-color: rgba(0, 0, 0, 0.5); }
```

图 6-1. 使用自定义高亮颜色的被触摸元素

你可能一直以为 Mobile Safari 中没有其他“已点击”状态。这并非完全正确。不过，需要一点小技巧才能通过 CSS 的 `:active` 和 `:hover` 伪类来访问这个属性。

许多人推荐使用多点触控中与 `mouseover` 和 `mouseout` 对应的 `touchstart` 和 `touchend`，通过事件处理器添加和移除特定类来实现。这种方法确实有效，但使用起来有些笨重且繁琐，尤其是有一种更简单的方式可以实现同样效果。

你可以通过在 `window` 上添加事件处理器来激活这些对应状态，即使不为此处理器添加任何功能，它也能生效并触发所需行为。因此，将以下代码添加到我们 Web 应用模板的 `main.js` 文件中，就能为 `main.css` 文件添加后续规则，如图 6-2 所示：

```
/* main.js */
document.addEventListener("touchstart", new Function(), false);
```

```
/* main.css */
.group-wrapper ul li a {
  ...
  -webkit-tap-highlight-color: transparent;
}

.group-wrapper ul li a:active {
  color: #fff;
  background: #015de6 -webkit-gradient(linear, left top, left bottom, from(rgba(255, 255, 255, .25)), to(transparent));
}

.group-wrapper ul li a:active * {
  color: #fff;
}
```

图 6-2. 犹如原生 iPhone 菜单

同样，我们设计这些样式时使其易于调整。这里只需改变背景颜色，就能调整按钮外观，而无需修改渐变效果。

#### 禁用复制/粘贴功能

用户可以轻松地对页面元素执行长按操作，这会触发复制/粘贴功能，并显示一个影响体验的丑陋框体，如图 6-3 所示。

图 6-3. 长按可能触发复制/粘贴功能

当不需要此行为，或希望为此操作添加自定义功能时，可以禁用它。仅需使用 CSS 的 `-webkit-user-select` 属性即可实现。该属性可取值 `auto`（允许选择）、`text`（仅允许用户选择文本），或 `none`（完全禁用选择）。

不过，务必注意不要让可用性下降，例如禁止选择文本字段。以下代码使用通用选择器为你处理了这一问题：

```
*:not(input):not(textarea):not(select),
input[type=image],
input[type=file],
input[type=submit],
input[type=button],
input[type=reset] {
  -webkit-user-select: none;
}
```

这将阻止除文本输入框（涵盖所有新输入类型）之外的所有元素被选中，使你的 Web 应用体验更接近原生 Mobile Safari 应用。

#### 控制弹出菜单

在链接上触摸并长按会触发如图 6-4 所示的菜单。如果你的 Web 应用已设计得类似原生应用，你可能不希望、不需要或用户预期中不存在此行为。

图 6-4. 触摸并长按链接或图片等元素可能触发选项弹出菜单

幸运的是，同样只需简单的 CSS，通过设置 `-webkit-touch-callout` 属性即可禁用它。



将`callout`属性设置为`none`。与上一个属性一样，请考虑用户可能期望该功能的情况，以避免造成困扰。

```
body {
  ...
  -webkit-touch-callout: none;
}
```

为文档的`body`元素添加此规则，足以全局禁用所有元素的该行为。

### 选择器

在深入了解 CSS3 布局和样式特性的神奇之处之前，我们认为有必要花些时间来介绍现代浏览器中编写 CSS 最令人兴奋的方面之一：选择器。的确，如果你过去只习惯使用 ID、类和后代选择器来定位元素，那么你错过了大部分乐趣。WebKit 浏览器实现了所有 CSS1 和 CSS2 选择器，以及 CSS3 引入的大多数选择器。

> **注意**：此处讨论的所有选择器都可以在 WebKit 的 Web 检查器（位于“开发”菜单下）的搜索字段中使用，这在你的开发过程中会非常有用。

这让你几乎完全掌控想要定位的元素，而无需为太多标签添加额外的标识符。CSS 选择器允许你在文档结构中精确移动，仅选择具有特定特征的元素。

#### 可用 CSS 选择器概述

随着 CSS3 的推出，可用选择器的数量变得相当可观，但我们将只深入探讨那些看起来更复杂的选择器。你可以在规范页面找到完整列表及每个选择器的描述，网址是：[www.w3.org/TR/css3-selectors/#selectors](http://www.w3.org/TR/css3-selectors/#selectors)。

你会看到，那里列出的大多数选择器要么是已知的，要么是显而易见的。你可能会注意到一些有趣的新成员，它们允许进行否定选择或仅匹配属性的部分内容。在第 9 章中，我们还将使用功能非常强大的`:target`，它可以省去不少 JavaScript 代码。

然而，其他一些选择器虽然有用，但可能不那么直观。以下是对这些选择器的一些说明和解释。

首先，伪元素现在通过使用双冒号标记（`::`）而不是单冒号来与伪类区分。这看起来很琐碎，但 CSS2 并没有做出这种区分。这对选择器的本质有意义：伪元素不属于文档树。通常它们属于文档文本内容，无法通过文档语言正常访问；而伪类旨在根据元素在 DOM 中的位置进一步识别它们。

其次，你应该注意空格，因为一个“空选择器”可以匹配任何元素。

```css
/* 2 个等效定义...（带空格） */
p ::first-letter { ... }
p *::first-letter { ... }

/* ...但不同于 */
p::first-letter { ... }
```

接下来，请注意你可以链式使用选择器。例如，你可能想在所有文章的末尾添加一个视觉标记。假设`<div>`的最后一个子元素是一个段落元素，可以按如下方式实现：

```css
div p:last-child::after {
  content: "The End";
  font-weight: bold;
}
```

这可能会相当复杂，因为实际上，你可以精确到几乎无极限，正如以下示例所示：

```css
a.class1.class2:not(#id1):first-child:hover {
  color: red;
}
```

这里，我们定位任何同时具有`class1`和`class2`类、但不具有`id1`ID、且是父元素的第一个子元素的锚点，以对其悬停状态应用特定颜色。

#### 位置相关选择器：结构伪类

你可能已经习惯了使用`:first-child`和`:last-child`选择器。如果是这样，你大概知道它们可能会令人沮丧，原因有二。

第一个限制是，显然，它们只允许你分别定位第一个和最后一个子元素。然而，常常需要为元素的倒数第二个子元素、每隔一个元素，或某个类型的第一个子元素应用不同的样式。



第二个原因是，这两个选择器都具有模糊性：`p:first-child`并非直接选中第一个`<p>`元素，而是选中父元素中的第一个子元素，如果该子元素恰好是`<p>`类型。因此，在以下代码中，没有任何段落会被应用不同的样式：

```html
<style>
div p:first-child {
  font-weight: bolder;
}
</style>

<div>
  <h1>Some Title</h1>
  <p>First paragraph, but not first element: will not be bold text.</p>
  <p>Second paragraph.</p>
</div>
```
