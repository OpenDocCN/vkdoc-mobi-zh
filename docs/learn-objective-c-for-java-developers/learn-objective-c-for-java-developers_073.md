# 对象实例化

尽管我将 NIB 对象描述为通过标准反归档过程创建的，但 NIB 加载器对某些对象做了一些特殊处理。以下是 NIB 中对象的创建方式：

-   自定义视图对象（`NSView` 的子类）通过 `-initWithFrame:` 进行初始化。这是为了与编程方式创建自定义视图对象保持一致。
-   非视图自定义对象（`NSObject` 的子类）通过发送 `-init` 消息进行初始化。
-   Interface Builder 提供的所有其他对象（即库面板中出现的那些对象）均通过常规的 `-initWithCoder:` 消息进行反归档。这也适用于您通过 Interface Builder 插件提供的任何自定义对象。

通过标准的初始化消息 `-init` 和 `-initWithFrame:` 初始化自定义对象，意味着自定义对象无需遵循 `NSCoding` 协议即可归档到 NIB 文档中。这也意味着您可以实例化应用程序中几乎任何可用的对象，并将其连接到 NIB 中的其他对象。使用 `-initWithCoder:` 解码对象的好处在于，NIB 可以包含预设的属性值。但由于 Interface Builder 不允许您编辑自定义对象的属性，这一点其实无关紧要。
