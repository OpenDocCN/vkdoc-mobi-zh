# 线程状态

如果线程已启动且尚未结束，则返回 `YES`。

`-isFinished`

如果线程已执行并退出，则返回 `YES`。

`-isCancelled`

如果线程曾收到过 `-cancel` 消息，则返回 `YES`。

##### 线程特定值

有时你需要一个全局变量，但仅限于特定线程的上下文中。每个 `NSThread` 对象都有一个 `threadDictionary` 属性专门用于此目的。该属性是一个可变字典，你的应用程序可以用它来存储所需的任何值。清单 15-3 中的示例为单个线程创建了一个 `NSNotificationCenter` 对象。

**清单 15-3.** 存储线程特定值

```
NSMutableDictionary *threadLocal = [[NSThread currentThread] threadDictionary];
[threadLocal setObject:[NSNotificationCenter new]
               forKey:@"NotificationCenter"];
```

其目的是创建一个通知中心，用于在线程内发送和观察通知。任何对象都可以通过以下语句获取其线程特定的通知中心：

```
[[[NSThread currentThread] threadDictionary] objectForKey:@"NotificationCenter"]
```

##### 优先级

可以使用 `+[NSThread setThreadPriority:]` 消息来更改当前运行线程的优先级。优先级是一个介于 `0.0` 和 `1.0` 之间的浮点值，`1.0` 为最高优先级。你可以通过 `+[NSThread threadPriority]` 获取线程的当前优先级。典型的默认优先级是 `0.5`。实际优先级范围（如果内核认可的话）取决于操作系统和内核。

内核除了倾向于将更多 CPU 时间分配给优先级较高的线程外，几乎不保证线程的优先级。请明智地使用线程优先级。

##### 栈大小

另一个很少使用的属性是线程的栈大小，该属性必须在线程启动前设置。这要求你先创建一个 `NSThread` 对象，设置其栈大小，然后使用 `-start` 消息启动线程。栈大小必须以 4K（4096）字节的偶数倍设置，当前默认为 512K。

以下两种情况你可能需要更改栈大小：

- 线程发生栈溢出。在这种情况下，请在启动线程前增加其栈大小。
- 你创建了大量不占用太多栈空间的线程，并且分配给每个线程的栈空间影响了地址空间的分配。请减少默认栈大小，以便每个线程分配更少的内存。

##### 名称

每个线程还有一个可变的 `-name` 属性，它是一个任意的 `NSString` 值。操作系统在很大程度上会忽略线程名称，因此你可以将其用于任何目的。

### 终止线程

线程在以下情况下停止运行：

- 线程的 `-main` 方法或创建线程时指定的方法返回
- 线程代码抛出一个未捕获的异常
- 进程终止

在前两种情况下，线程会被销毁，其资源会被回收。第 14 章中描述的未捕获异常处理程序会处理未捕获的异常。`NSThread` 对象无法重新启动。如果你想启动另一个线程，请创建一个新的 `NSThread` 对象。

当进程终止时，所有线程都会被放弃，无论其状态如何。这与 Java 不同，Java 的虚拟机会一直运行到所有非守护线程完成。如果你创建了后台线程，则无需安排它们完成即可终止应用程序。相反，如果你的后台线程需要在应用程序终止前完成，则必须通过信号量或其他方式进行协调。

### 运行循环

运行循环是 Cocoa 框架不可分割的一部分。它们是应用程序、用户界面事件、分布式对象、定时器、延迟对象消息和分布式通知的主要事件分发机制。

运行循环是一种事件处理服务，它同步处理来自一个或多个异步源的事件。事件源被称为输入源，有几种预定义的类型：

- 用户界面事件（鼠标事件、键盘事件等）
- 延迟方法调用（`-performSelector:withObject:afterDelay:` 等）
- 分布式对象消息
- 定时器事件

还有其他输入源，并且你也可以开发自己的输入源，但这属于特殊情况。

每个线程都关联一个（可选的）运行循环对象。代码 `+[NSRunLoop currentRunLoop]` 会返回与当前线程关联的单个运行循环对象，而 `+[NSRunLoop mainRunLoop]` 会返回主线程的运行循环。运行循环是延迟创建的；如果线程没有运行循环对象，则在发送消息时会创建一个。与主线程关联的运行循环称为主运行循环。

> **警告：** `NSRunLoop` 不是线程安全的。对运行循环对象的操作应在其所属的线程内进行。

运行循环非常高效。它们会阻塞线程，直到其中一个输入源上出现事件。然后运行循环会立即处理该事件，采取适当的操作，然后“循环”等待下一个事件。运行循环还会处理周期性的清理工作，例如管理内存和栈对象垃圾回收。你可以将线程的控制权完全交给运行循环，或者编写自己的循环来保持运行循环工作。

#### 启动运行循环

通常你不需要启动运行循环。如果你在编写标准应用程序，`NSApplication` 对象会自动启动主运行循环。主运行循环会处理你应用程序中的所有重要事件。模态对话框和 `NSConnections` 也会启动运行循环。

在以下情况下，你需要启动自己的运行循环：

- 你创建了一个需要触发定时器或向对象发送延迟消息的新线程。
- 你在自己的线程中提供分布式对象，或者通过引用传递了可能从远程进程接收异步消息的对象。
- 你正在编写一个不使用应用程序框架的命令行工具或守护进程，并且需要在主线程上处理延迟方法调用、定时器事件或分布式对象。（第三种情况只是前两种情况的特例，因为命令行工具的主线程不会自动启动运行循环。）

运行循环需要输入源，并且只有在有输入源时才会运行。要启动运行循环，你必须首先为其提供一个或多个输入源。否则，运行循环将不执行任何操作并立即结束。

- 创建并调度一个定时器（本章稍后介绍）将为当前运行循环创建一个定时器输入源。
- 将延迟方法调用加入队列会向运行循环添加一个消息分发源。
- 一个 `NSConnection` 会将其通信端口添加到当前运行循环。
- 主线程拥有 Objective-C 框架已创建的几个输入源。在主线程上启动的运行循环将始终启动并永远运行。

通过启动运行循环，你基本上将线程的控制权交给了运行循环。线程会阻塞，直到从输入源接收到事件。要启动运行循环并让它永远运行（或者至少在有输入源时运行），请执行以下操作：

```
[[NSRunLoop currentRunLoop] run];
```

消息 `-runUntilDate:` 是一个轻微变体，它可以在指定的时间段内运行运行循环。你可以用此方法执行周期性操作，但使用定时器更可取。
