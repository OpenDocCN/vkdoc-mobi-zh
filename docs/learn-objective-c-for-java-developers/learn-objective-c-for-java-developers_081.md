# 鼠标跟踪矩形与响应者链

第二种方法是定义一个鼠标跟踪矩形。鼠标跟踪矩形定义了视图中需要跟踪鼠标的离散区域。当鼠标移入该区域、在区域内移动以及移出该区域时，它会通知你的对象。这比处理每一次鼠标移动要高效得多，但设置起来稍微复杂一些。

## 第 20 章 ■ 模型-视图-控制器模式

1. 在一个对象中实现 `-mouseEntered:`、`-mouseExited:`、`-mouseMoved:` 和 `-cursorUpdate:` 方法。该对象可以是你的视图对象，也可以是其他对象。
2. 使用要跟踪区域的矩形（以视图坐标表示）以及在步骤 1 中创建的所有者对象，创建一个 `NSTrackingArea` 对象。你还可以指定额外的跟踪选项，或提供一个上下文信息字典。
3. 通过向 `NSView` 对象发送 `-addTrackingArea:` 消息，将 `NSTrackingArea` 对象附加到该视图上。

一旦附加完成，所有者对象将接收到鼠标进入、移动、离开以及光标更新事件。

#### 响应者链

响应者链是从第一响应者开始，沿对象层级向上的一系列对象。响应者链是确定应用程序如何响应命令和其他操作的主要机制。

完整的响应者链如图 20-15 所示。

***图 20-15.** 响应者链*

响应者链通过检查链中的每个对象是否响应（即实现）给定的消息，来处理操作消息（形式为 `-(IBAction)action:(id)sender` 的 Objective-C 消息）。如果对象响应，则接收该消息。否则，该过程会继续检查链中的下一个对象，直到找到能响应消息的对象，或者没有更多对象为止。

图 20-15 中的两个第一响应者需要稍加解释：第一响应者是界面中作为“焦点”的单个对象。当窗口失去关键窗口状态时，它会记住之前作为其第一响应者的对象。这将成为窗口的初始第一响应者。当窗口成为关键窗口时——假设在此过程中没有其他视图成为第一响应者——初始第一响应者会自动成为第一响应者。当关键窗口和主窗口不同时，响应者链从第一响应者开始，经过关键窗口，然后到达主窗口的初始第一响应者。

图 20-15 中的许多响应者链部分是可选的，实际中很少会遇到遍历整个链的情况。如果关键窗口没有第一响应者对象，响应者链从关键窗口本身开始。如果关键窗口和主窗口相同，则响应者的父视图层级只会被遍历一次。没有控制器的窗口不会在链中包含窗口控制器。非文档窗口没有控制器或文档对象，会直接跳转到应用程序。窗口和应用程序对象可能没有委托。非基于文档的应用程序没有文档控制器对象，因此这些应用程序中的响应者链以应用程序或其委托结束。

如果你正在开发 iPhone 应用程序，响应者链的原理是相同的。响应者链本身往往相当简单。iPhone 应用程序一次只显示一个 `UIWindow`，因此关键窗口和主窗口始终是同一个。传递给 `UIView` 的操作会先传递给它的 `UIViewController`（如果有的话），然后再传递给其父 `UIView`。

### 操作消息

操作、菜单状态查询、服务和错误解释都通过响应者链进行过滤。解释响应者链如何与应用程序集成的最简单方法是使用一个例子。以图 20-14 中描绘的应用程序为例，考虑四个菜单命令：复制、关闭、保存和新建。每个命令都会向第一响应者发送一个操作消息（分别是 `-copy:`、`-performClose:`、`-saveDocument:` 和 `-newDocument:`）。

- `-copy:` 操作消息被发送给第一响应者。图 20-14 中的第一响应者是一个实现了剪贴板功能的可编辑文本字段。它通过将当前选中的内容传输到应用程序的粘贴板（剪贴板）来响应 `-copy:` 消息。
- 第一响应者没有实现 `-performClose:` 方法，它的任何父视图也没有实现。关键窗口实现了该方法，并通过关闭自身来响应 `-performClose:` 消息。
- 第一响应者没有实现 `-saveDocument:` 消息。它的任何父视图、关键窗口、关键窗口的委托、主窗口中的视图、主窗口、主窗口的委托或窗口控制器也都未实现。`NSDocument` 对象是负责文档管理的对象，也是实现“保存”和“另存为…”命令的对象。它通过将活动文档的内容编码并写入文件来响应 `-saveDocument:` 消息。
- `-newDocument:` 消息会经过响应者链中的每一个对象，直到到达为基于文档的应用程序创建的单个文档控制器对象。文档控制器会创建一个新文档，并打开一个无标题的窗口。

应用程序对命令的响应会随着响应者链状态的变化而改变：

- 如果另一个视图对象是第一响应者，应用程序可能不响应“复制”命令。
- 应用程序中的其他窗口会忽略“关闭”操作，因为它们不在响应者链中。
- 如果主窗口不是文档窗口，其响应者链中将没有 `NSDocument` 对象，因此该窗口不会响应“保存”操作。
- 应用程序始终响应“新建”命令，因为链中的顶层对象会一致地响应它。

#### 发送操作消息

发送给特定对象的操作消息称为定向操作。发送给响应者链的操作称为非定向操作。一个操作由一个 Objective-C 消息标识符（`SEL`）和一个对象标识符（`id`）组成。如果目标标识符指向一个对象，则为定向操作。如果目标标识符为 `nil`，则为非定向操作。

操作通过向 `NSApplication` 对象发送 `-sendAction:to:from:` 消息来执行。目标（`to:`）参数要么是一个对象（定向），要么是 `nil`（非定向）。应用程序会判断目标是否响应该操作，或者尝试在响应者链中找到一个响应该操作的目标。如果成功，目标对象会收到该操作消息，并且 `-sendAction:to:from:` 返回 `YES`。你可以通过编程方式发送操作，但 `NSControl` 视图、菜单项和键盘等效键通常会为你发送它们。

在 Interface Builder 中，操作被定义为消息/对象对。如果操作直接连接到 NIB 文档中的另一个对象，则定义了一个定向操作。将操作连接到“第一响应者”占位符 NIB 对象会创建一个非定向操作，该操作将在运行时传递给当前的第一响应者。

#### 菜单操作

Cocoa 应用程序中的菜单项是响应者链的另一个重要应用。当你下拉一个应用程序菜单时，只有那些适用于当前界面状态的菜单项才会被启用。不合适的菜单项则会被禁用。
