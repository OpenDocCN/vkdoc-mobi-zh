# 第 2 章

## Java 与 C 的关键差异

本章重点介绍 Java 和 C 之间的重要区别。我们将特别关注那些看起来熟悉但在 C 语言中含义截然不同的语法和关键字——正如法国人所说的“假朋友”（faux amis）——以及 Java 中没有的结构。在 `Objective-C` 以明确的方式扩展 C 语言的领域中，Java 和 C 之间的相似性反而可能成为混淆的根源。因此，你通向 `Objective-C` 之旅的第一步，就是熟悉 C 语言。

Java 和 `C++` 都是 C 语言的衍生品。两者都创建了全新、复杂、面向对象的语言，同时修正了 C 语言中一些公认的不足。它们各自引入并强制执行了诸如构造函数、方法访问、封装等概念。

`Objective-C` 则采取了一种极其极简主义的方法。`Objective-C` 通过引入一层薄薄的语法和少量关键字——重点就在于这个“薄”字——将类和对象的概念添加到了传统的 C 语言中。`Objective-C` 并不会以任何显著的方式改变或重新定义底层的 C 语言。在 Java 中，你用 Java 编程；在 `C++` 中，你用 `C++` 编程；而在 `Objective-C` 中，你是在工业标准的 C 语言之上叠加一些面向对象的扩展来进行编程。

**注意** 我或许应该提一下，还有一门 `Objective-C++` 语言；它将 `C++` 与 `Objective-C` 的对象和特性叠加在一起。这主要对那些必须将遗留的 `C++` 代码集成到 `Objective-C` 运行时环境中的程序员有用。本书不讨论 `Objective-C++`，但关于 C 和 `Objective-C` 的大部分内容都是适用的。

如果你之前用 C 语言编程过，那么可以跳过本章。由于 `Objective-C` 就是 C，你不会发现任何意外之处。如果时间有点久远了，你可以快速浏览各个章节，以提醒自己它们之间的差异。

如果你从未用过 C 语言编程，那么你需要了解 Java 和 C 之间的差异。Java 的语法模仿自 C 语言，因此你已经熟悉了方法、代码块、条件语句、赋值语句、运算符等的结构。以下是关于 Java 和 C 主要差异的速成课程。

### 基本类型

基本类型是由语言定义的原子类型，是所有计算的基础。它们大多是具有不同大小、数值范围和格式的标量变量类型。总的来说，`Objective-C` 中的整数和浮点类型与 Java 中的相似。表 2-1 列出了两者的基本数值变量类型。

|------|-------------|-----------|------|-------|
| `boolean` | `BOOL` | | 1 位 | `false` … `true` |
| `byte` | `char` | `int8_t` | 8 位 | -128 … 127 |
| `unsigned char` | `uint8_t` | | 8 位 | 0 … 255 |
| `char` | `int16_t` | | 16 位 | -32768 … 32767 |
| `unichar` | `uint16_t` | | 16 位 | 0 … 65535 |
| `short` | `short int` | `int16_t` | 16 位 | -32768 … 32767 |
| | `unsigned short int` | `uint16_t` | 16 位 | 0 … 65535 |
| `int` | `int` | `int32_t` | 32 位 | -2147483648 … 2147483647 |

| `long` | `long long int` | `int64_t` | 64 位 | -9223372036854775808 … 9223372036854775807 |

C 语言没有严格的布尔类型。在 C 语言中，布尔运算符（例如 `==` 和 `&&`）返回整数结果。C 将零值解释为 `false`，将任何非零值解释为 `true`。这使得你可以分别将语句 `if (i!=0)` 和 `if (p!=nil)` 写成 `if (i)` 和 `if (p)`。这些语句是等价的，但我认为使用更冗长的形式——即使语言本身并不要求——能使意图更清晰。为了方便和可读性，`Objective-C` 定义了一个 `BOOL` 类型来声明布尔值。实际的类型是一个无符号 8 位整数，但你应该将其视为只能表示 `YES` (`true`) 或 `NO` (`false`)。C 语言也定义了一个 `bool` 类型，可与 `BOOL` 互换，但 `Objective-C` 程序员几乎只使用 `BOOL`。

**注意** 没有布尔类型的一个副作用是，C 语言的控制语句接受整数表达式。这使得你很容易在你本想写比较的地方使用了赋值语句。小心不要在你本想写 `if (i==1)` 的时候写了 `if (i=1)`。前者意味着“将数值 1 赋值给变量 `i`，然后测试该值是否不为零”。

C 语言的 `long` 和 `short` 关键字在技术上是对 `int` 类型的修饰符，如 `short int` 或 `long int`。Java 的 `long`、`int` 和 `short` 是三种不同的整数类型，不会组合使用。在 C 语言中，每当出现 `long` 或 `short` 关键字时，都隐含了 `int` 类型，因此 `short int` 和 `short` 是等价的。你使用哪一种只是风格问题。

Java 对编程的重大贡献之一就是标准化了整数类型的大小。传统的 C 语言没有规定像 `char` 和 `int` 这样的类型的大小；它们会因编译器或系统的不同而变化。现代 C 语言在很大程度上已将大小标准化为表 2-1 中列出的值。为了避免任何可能的歧义，C 标准现在包含了 `typedefs`（本章后面会介绍），这能够保证变量大小的一致性。一个类型为 `int32_t` 的变量将始终是一个有符号的 32 位整数，无论编译器对 `int` 或 `short int` 的首选字长是什么。

从历史上看，C 语言的 `int` 是 16 位，`long int` 是 32 位。随着 32 位和 64 位 CPU 取代了更小的微处理器，`int` 的标准大小变成了 32 位。为了保持向后兼容性，`long int` 保持为 32 位，并引入了 `long long int` 类型来定义 64 位整数。
