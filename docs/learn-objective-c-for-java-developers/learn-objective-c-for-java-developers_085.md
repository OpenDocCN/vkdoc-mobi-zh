# TicTacToe 应用与撤销管理

`TicTacToe` 应用使用了`Undo`管理器。为你的应用添加基本的撤销功能极为简单：

1. 当文档发生更改时，获取该文档的撤销管理器，并“推入”以下三元组：一个目标对象、一个 Objective-C 消息标识符，以及一个包含先前值的对象。如果你正在实现一个绘图程序，调整形状大小可能会推入一个`-setShapeSize:`消息以及该形状的先前尺寸。
2. 礼貌的做法是，为推入的操作命名。再次以绘图程序为例，将操作命名为“调整形状大小”，会在“编辑”菜单中向用户显示“撤销调整形状大小”。这有助于用户理解将要撤销的操作内容。
3. 当用户选择`撤销`命令时，撤销管理器会获取推入栈中的最后一个操作，并将带有值的消息发送给该对象。在此示例中，绘图对象将收到一个包含其旧尺寸的`-setShapeSize:`消息。
4. 撤销操作处理程序还应将（当前的）先前尺寸作为“重做”操作推入。推入操作到撤销栈的消息完全相同——当撤销一个操作时，撤销管理器会切换到一个临时模式，将操作推入到一个“重做”栈。这使得你的单个`-setShapeSize:`方法能够同时作为“执行”、“撤销”和“重做”方法。

`TicTacToe`的数据模型极为简单，因此它仅在每次移动后将文档（游戏和消息）的整个状态推入撤销栈。更复杂的数据模型会推入特定的更改，但原理是相同的。

为了简化撤销和打开/保存命令，`TTTDocument`实现了一个`contents`属性，该属性将整个文档状态作为一个不可变对象返回。

实现撤销有一个隐藏的优势：撤销管理器还会为你维护文档的`isDocumentEdited`属性。这能告诉文档控制器何时文档包含未保存的更改。如果不使用撤销管理器，则必须自行管理`isDocumentEdited`属性。

### 资源

资源是复制到应用程序包中的数据文件。框架定义了多种资源文件，例如 NIB 文档、托管对象模型文件以及可执行文件。你可以随意添加任何所需的其他文件。通常这些是图像、声音和帮助文件，但并无限制。程序可以包含框架，甚至另一个应用程序，作为其资源的一部分。

你可以使用`NSBundle`类获取资源。像`NSImage`这样的类通常提供便捷方法，例如`+[NSImage imageNamed:]`，可以定位资源文件并将其加载到对象中。

#### 本地化资源

应用程序包中的几乎任何资源都可以按语言组织。这被称为本地化。文件的本地化版本存储在一个变体子文件夹中。`NSBundle`会首先在变体文件夹中查找用户偏好的语言。如果找不到合适的文件（或变体），它将使用不在任何变体中的文件。这允许你为任何资源创建特定于语言的替代版本。这包括 NIB 文档、图像、声音、向用户显示的字符串列表等。

在 Xcode 中，资源的本地化版本显示为主文档的子文档。要创建本地化版本，请打开文件的“信息”窗口，然后选择“添加本地化”。每个本地化都是一个完全独立的文件，需要你自行维护。

### 总结

本章涵盖了大量内容。模型-视图-控制器设计模式贯穿于 Cocoa 框架之中。它影响着应用程序的许多不同方面，并将用户界面、业务逻辑和数据组织结合在一起。它涉及大量类，并使用了迄今为止讨论过的几乎所有通信技术。

理解并拥抱 MVC 是理解 Cocoa 应用程序的关键一步。你还了解了关键的开发工具，例如 Interface Builder，它利用许多 Objective-C 技术，几乎无需代码即可创建复杂的用户界面。
