# 注意事项

在 64 位架构中，整数的大小再次发生了变化。如果你为 64 位 CPU 编译代码，`long int` 会变为 64 位整数——而 `short int`、`int` 和 `long long int` 则保持原有大小。如果你需要一种在 32 位架构上为 32 位、在 64 位架构上为 64 位的整数值，请将其声明为 `NSInteger` 或 `NSUInteger`。它们是 C 语言中的 `typedef`，会根据架构改变自身大小。请留意 API 以确定正确的类型。如果某个方法返回 `NSInteger`，请将接收该值的变量也声明为 `NSInteger`。这样，无论编译为 32 位还是 64 位可执行文件，其行为都将保持一致。

Java 通过消除无符号整数类型简化了传统的 C 语言。在 C 语言中，整数分为有符号和无符号两种类型。与 Java 的 `char` 类型最接近的是 C 语言的 `unichar`，但请小心——`char` 是有符号的，而 `unichar` 是无符号的。无符号整数是滋生细微编程错误的温床，这大概就是 Java 将其移除的原因。以下是最常见的几种情况。

避免编写无意义的不等式语句，例如清单 2-1 中所示。

**清单 2-1.** 无意义的比较语句

```
unsigned int i = 99;
while (i>=0) { … } // 无意义/始终为真；i 不可能小于 0
```

请注意，符号位可能会通过赋值被重新解释，如清单 2-2 所示。

**清单 2-2.** 符号位重新解释

```
int i = -3;
unsigned int j = i; // j = 4294967293
```

清单 2-3 演示了在比较运算中混用有符号和无符号类型的情况。请避免这种做法，或者对其中一个值进行强制类型转换，使类型兼容。

**清单 2-3.** 混合类型比较

```
int i = -3;
unsigned int j = 3;
if (i<j) { … } // 不可预测的条件
```

大多数大小、索引和数组计数都是无符号的。在声明将用作集合索引的变量时，请声明为无符号整数。在其他几乎所有场合，都可以使用有符号整数。

### 常量

常量是赋予给不变、已知值的名称。比起使用原始数字值（即使该值显而易见），采用符号化编程更为简单且信息量更大。

现代的代码库和框架定义了数千个常量，但表 2-2 列出的是你经常使用的一些常量。

**表 2-2.** 常见常量

| Java | Objective-C | 用途 |
|------|-------------|------|
| `true` | `YES` | 逻辑真 |
| `false` | `NO` | 逻辑假 |
| `null` | `nil` | 空或无效的对象引用 |
| | `NULL` | 空或无效的指针 |

从技术上讲，`nil` 和 `NULL` 是可以互换的。为了提高可读性，在设置和测试 Objective-C 对象引用时应使用 `nil`，就像你在 Java 中使用 `null` 一样。`NULL` 是 C 语言的空指针常量，用于设置和测试指针。采用这种约定可以强调代码中内存指针与对象引用之间的区别。

整数类型前的 `const` 关键字含义与 Java 中的 `final` 大致相同，但它很少用于定义简单的整数常量。C 语言中的大多数常量都是通过预处理器（本章后续部分也会介绍）或枚举语句来定义的。

清单 2-4 中的枚举（`enum`）定义了一个数值变量 `v`，它可以保存整数常量 `Zero`、`One` 或 `Two` 中的一个。与 Java 不同，`v` 不是一个对象，它只是一个整数。这三个常量（`Zero`、`One` 和 `Two`）是公共符号，其值分别为 0、1 和 2。它们可以在任何能接受整数常量的上下文中使用。变量 `v` 通常会被省略，`enum` 语句仅用于定义这些常量。

**清单 2-4.** 枚举

```
enum {
    Zero = 0, // = 0 是可选的，如果省略，枚举从 0 开始
    One,
    Two
} v;
```

### Typedefs

`typedef` 是 Java 中所没有的概念，但在 C 语言中却被广泛使用。在 Java 中，新的数据结构和功能是通过创建新类来实现的。而在 C 语言中，你可以定义自己的原始数据类型。清单 2-5 中的程序使用一个整数值来标识库存中的零件。



### Typedefs

`typedef`关键字放在变量声明前会将其转换为类型定义。与定义一个整型变量`PartCode`不同，`Listing 2-5`中的`typedef`语句创建了一个与`int`同义的新变量类型（`PartCode`）。一旦定义完成，新的`PartCode`类型就可以替代`int`类型使用。`aCode`和`bCode`的声明是相同的。

```
typedef int PartCode;
PartCode aCode;
int bCode;
```

`typedef`有助于使代码更具可读性。函数原型`add( PartCode c )`比`add( int c )`的描述性更强。`typedef`也提升了可维护性。`PartCode`可以在后续重新定义为`unsigned int`或`long long int`，而无需修改每个部件代码变量的声明。

`typedef`通常用于复杂声明，例如结构体和枚举。`Listing 2-6`中的`typedef`语句同时创建了一个新的枚举类型（`Flavor`）和三个整型常量（`Strawberry`、`Rhubarb`和`Apricot`）。现在你可以编写`Flavor favorite = Apricot;`，这会将值`2`赋给变量`favorite`。

```
typedef enum {
    Strawberry,
    Rhubarb,
    Apricot
} Flavor;
```

### 指针

Java 通过特意省略任何允许程序员直接访问任意内存位置的编程结构，来“保护”你免受物理内存的影响。在 Java 中，你只能访问命名的基本类型和对象的引用。

C 语言没有在代码和物理内存之间提供这种隔离。这给了你极大的自由来访问和操作数据。同时，它也将不越界（访问内存）的责任落在了你——程序员——身上。

可以将 C 指针想象成 Java 对象引用，但有两个主要区别。C 指针可以指向任何类型的值或结构——而不仅仅是对象。你可以声明一个指向整数、浮点数、复杂结构、函数、甚至另一个指针的指针。其次，C 指针始终是进程地址空间中的一个真实内存地址，而 Java 的对象引用是一个抽象且不透明的值。

除此之外，大多数关于 Java 引用的编程规则也适用于指针。例如，将指针作为参数传递会复制值的引用，而不是值本身。

由于 C 语言同时提供值和指向值的指针，因此必须使用编程语法来区分两者。一元指针运算符（`*`）可以出现在类型定义和表达式中。在声明中，指针运算符将变量类型从简单变量更改为指向该类型变量的指针，如`Listing 2-7`所示。

```
int i = 1;
int *iptr;
iptr = &i;
*iptr = 2;
```

在`Listing 2-7`中，变量`iptr`是一个指向整型变量的指针。`iptr`的值是一个存储了整数值的内存地址，而不是存储在那里的整数值本身。换句话说，它是对进程地址空间中其他位置的一个基本整数值的引用。

`&`符号是一元“取地址”运算符。它返回变量的地址，而不是变量的值。`iptr = &i`语句将`iptr`的值设置为变量`i`的内存地址，而不是其值。

表达式中的一元指针运算符（`*`）将引用重定向到指针变量所指向的值。在某种意义上，它是一元取地址运算符的补充。最后一条语句使用指针运算符将赋值重定向到`iptr`所指向的值，而不是`iptr`本身的值。用英语来说，该语句的意思是“将值`2`存储在地址包含在`iptr`中的整型变量里”。最后一条语句不会改变`iptr`的值；它改变了`iptr`所指向的整型变量的值。在这四条语句执行完毕后，变量`i`的值将是`2`。

### 结构体

C 结构体定义了内存块的布局。结构体（`struct`）由其他变量类型构建而成。一旦定义，这些变量集合就可以作为一个单一单元对待。`struct`语句非常类似于 Java 的类定义，但不包含方法。简单的结构体声明（如枚举）定义了一个具有给定结构的单一变量。更典型的是，结构体会出现在`typedef`语句中，以便可以通过名称重用该结构定义（参见`Listing 2-8`）。

```
typedef struct {
    int key;
    int count;
} KeyCount;

...

KeyCount counter;
counter.key = 1;
counter.count = 0;
```

`Listing 2-8`中展示的`KeyCount`结构体定义了一个包含两个成员变量的内存单元，这两个成员变量位于连续的地址中，分别名为`key`和`count`。与 Java 类似，C 语言使用成员运算符（`.`）来访问结构体中的成员变量。更常见的是，通过指向结构体的指针来间接引用结构体，如`Listing 2-9`所示。

```
KeyCount *nextCounter = &counter;
nextCounter->count += 1;
```

变量`nextCounter`是一个指向`KeyCount`结构体的指针。当变量是指向结构体的指针时，使用间接成员运算符（`->`）来指定成员变量。这在技术上是`(*nextCounter).count`的简写，但更易于读写。

将一个结构体赋值给另一个结构体会复制整个结构体。语句`KeyCount save = counter`会将`counter`结构体中的两个整数复制到`save`结构体中对应的整数。方法参数是按值传递的，因此在参数中传递结构体会复制整个结构体。这与 Java 不同，后者不允许按值复制对象。

因此，大多数方法声明的参数是指向结构体的指针（引用），而不是结构体本身。但如果你需要按值传递结构体，你也有这种选择。

### 对象引用

如果你认为 C 结构体看起来很像类，而指向结构体的指针作用很像对象引用，那么你非常敏锐。Objective-C 使用 C 结构体来定义对象，并使用结构体指针作为对象引用。

一个 Objective-C 的类定义本质上定义了一个 C 结构体。你首先使用 Objective-C 语法来定义类。一旦定义完成，你就可以使用 C 语言中关于结构体和指针的语法来与对象交互。两者都在`Listing 2-10`中展示。

```
@interface KeyCounter {
@public
    int key;
    int count;
}
@end

KeyCounter *countObject = ...;
countObject->count += 1;
```

在幕后，为类创建的结构体包含了额外的成员变量，这些变量用于定义对象的类、提供对其方法的访问、管理内存等等。

Objective-C 甚至提供了一个运算符，可以将类类型转换为 C 结构体类型，允许你无限制地访问对象的内部结构，或者将对象视为一个简单的 C 结构体。这很少使用，但它凸显了 Objective-C 的透明性。

如果所有这些让你感到不知所措，别担心。你可以在很大程度上忽略`typedef`、结构体和指针的细节。要使用 Objective-C 编程，你需要知道类是结构体，指针运算符（`*`）用于声明指向对象的指针（引用），以及间接成员运算符（`->`）用于通过对象指针（引用）直接访问成员变量。
