# 懒加载模式

懒加载模式体现了数据交付的即时加载理念。它将值或数据结构的构建延迟到真正需要该数据的时候。懒加载是 Java 和 Objective-C 中一种流行的设计模式。Objective-C 通过实现其自身的类结构懒加载增加了另一个层次，你可以将其集成到自己的类中。

### 实现该模式

以下是实现懒加载模式的方法：

1. 创建一个私有实例变量或私有静态变量，该变量最终将包含数据，但初始化为一个占位值（通常是`nil`）。
2. 将该变量包装成一个提供公共 getter 方法的属性。
3. 在 getter 方法中，测试该变量是否为占位值。如果尚未初始化，则构建它并保存结果。将新构建的或先前构建的数据返回给调用者。

懒加载通常适用于以下情况：
- 创建或计算数据所需的内存或资源过于庞大，可以推迟。
- 有很大可能永远不需要这些数据。
- 实例化对象时，数据的前置条件不存在或不可用。

延迟计算某个值，或组装一个大型或复杂的数据结构，可以节省内存和 CPU 资源，直到实际需要数据时再使用。在某些情况下，数据可能永远不需要，从而显著提高性能。通过将操作推迟到稍后时间，可以提升感知性能，使应用程序能够立即更新显示或响应用户事件，而更复杂、耗时的工作则在之后完成。典型的例子是照片浏览应用程序，它立即向用户显示占位图或缩略图，而实际图像则在后台加载。用户会感觉应用程序运行得更快，因为它的响应性更好，即使加载和显示最终图像的实际时间可能更慢。

一个能够快速创建对象且内存需求极小的对象构造器称为轻量级构造器。即使对象本身是重量级的，其构造器也可以是轻量级的。使用上面的例子，图像视图对象可以几乎瞬间初始化，只需图像的尺寸和可以获取图像数据的 URL。构造器以及更新窗口或布局多页文档所需的属性是轻量级的，并且立即可用——即使实际图像需要数兆字节的内存和几秒钟才能获取。

### 全局变量的懒加载

懒加载通常应用于全局变量。Java 相对于 Objective-C 的一个优势是它能够为静态变量声明对象初始化器。Java 运行时在类加载时评估初始化语句。

```
public final static String[] ColorNames = new String[] { "White", "Black" };
```

Objective-C 中真正的全局变量是 C 静态变量。C 静态变量只能使用常量或静态地址进行初始化，并且此初始化发生在 Objective-C 运行时启动之前。要达到相同的效果，需要在 Objective-C 运行时启动后以某种形式进行自动初始化。
