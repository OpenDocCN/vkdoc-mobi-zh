# 第一章：Windows Embedded Compact 设备驱动开发基础

### 宏内核架构与 Windows CE

`Windows Embedded CE 6.0`（下文简称为 `Windows CE 6.0`）及更高版本基于宏内核架构。图 1-2 展示了 `Windows CE 6.0` 和 `Windows Embedded Compact 7` 的架构。该图中需要特别注意的是，设备驱动宿主（即 `GWES.dll` 和 `DevMgr.dll`）本身是动态链接库。设计者仍可根据需求添加或删除系统服务，实现组件化。

#### 宏内核架构中的用户模式与内核模式驱动

在内核模式下运行意味着故障设备驱动可能危及整个系统。为了增强稳健性，可安装的设备驱动可以运行在由 `Udevice.exe` 进程托管的用户模式下，从而保护内核免受恶意设备驱动的侵害。其主要缺点是性能较慢。根据经验，宏内核中的内核模式设备驱动似乎比微内核架构中的用户模式设备驱动提供更好的性能，而宏内核中的用户模式设备驱动性能则会大幅下降。更多信息请参阅本章后续的“Windows CE 内核模式与用户模式设备驱动”一节。

### Windows CE 系统架构与 I/O 处理

与之前的版本 `Windows CE 6.0` 相同，`Windows Embedded Compact 7` 是一款基于宏内核的商业嵌入式实时操作系统（RTOS）。该操作系统采用的特定设备驱动模型简单易用，便于实现设备驱动。

#### 与设备驱动相关的系统组件

与任何现代嵌入式操作系统一样，`Windows Embedded Compact` 包含多种系统组件，每种组件都有特定用途，共同构成一个功能广泛的平台。内核提供核心操作系统服务；其他服务（如 `GWES` 和 `设备管理器`）则提供附加功能和接口。图 1-3 突出显示了那些对设备驱动开发者尤为重要的系统组件。

*图 1-3. 与设备驱动相关的 Windows CE 系统组件*

`Windows CE` 依赖以下系统组件来实现灵活的设备驱动模型：

- `GWES.DLL` – 即图形窗口和事件子系统，它负责加载本机设备驱动，并在加载期间对其及接口进行管理。本章后续的“Windows Embedded Compact 设备驱动模型”一节将更详细地介绍 `GWES`。

- `DEVMGR.DLL` – 即设备管理器，它负责加载内核模式流接口设备驱动。此外，`设备管理器` 会跟踪驱动所公布的接口，并支持基于全局唯一标识符（GUID）搜索驱动。详情请参阅本章后续的“Windows Embedded Compact 设备驱动模型”一节。

- **I/O 资源管理器** – `设备管理器` 包含一个 I/O 资源管理器，用于从注册表中枚举可用资源列表。

- `FILESYS.DLL` 和 `FSDMGR.DLL` – 分别是 `Windows Embedded Compact` 文件系统管理器和文件系统驱动管理器，负责将分区驱动和文件系统驱动加载到 `Windows CE 存储管理器` 中。你将在第六章“理解设备驱动类型”中了解更多关于这些驱动类型的信息。

- **网络 DLL** – 这些 DLL 实现了网络功能，例如 `TCP/IP`。第十章“网络驱动接口规范”将提供 `Windows CE` 中网络通信的更多细节。
