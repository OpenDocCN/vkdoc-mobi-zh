### 引言

Windows Embedded Compact 7 实际上是 Windows CE 的最新版本。专门介绍此操作系统的书籍相对较少。当 Windows CE 3.0 发布时，Windows CE 才真正成为一个合适的 RTOS。当时出版了几本书，其中最著名的是 Douglas Boling 和 James Wilson 的著作。第一本书侧重于应用程序编程，并随着新版本系统的出现而更新。第二本书是关于创建基于 Windows CE 系统的全面教程。最近，Samuel Phung 等人的书籍延续了这一趋势，向嵌入式系统开发者介绍 Windows CE。然而，撰写一本专门针对系统某个方面的书籍尚未有人尝试。设备驱动程序是嵌入式系统开发的核心。

每个计算机系统，无论是通用计算机还是嵌入式计算机系统，无论是否时间关键，都必须处理三个阶段：数据输入、数据处理和数据输出。嵌入式系统通常无需人工干预即可完成这三个阶段。数据输入和数据输出必须通过外围 I/O 设备来处理。

本书旨在引导读者了解 Windows Embedded Compact 7 及之前版本的 Windows CE 中使用的流接口设备驱动程序模型的开发过程。流接口设备驱动程序模型是 Windows Embedded Compact（Windows CE）操作系统中最常用的模型。存在用于用户界面输入的原生设备驱动程序和显示设备驱动程序，但大多数设备驱动程序开发人员使用流接口驱动程序模型来满足其开发需求。
