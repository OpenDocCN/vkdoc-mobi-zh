# 第 7 章 ■ 流式设备驱动程序的本质

`Device Manager` 暴露了三个新函数，可供内核模式设备驱动程序用于支持异步 I/O 请求：

- `CreateAsyncIoHandle` — 设备驱动程序使用此函数来更改 `XXX_Read`、`XXX_Write` 和 `XXX_IOControl` 入口点的实现，以支持异步 I/O
- `CompleteAsyncIo` — 此函数用于标记异步 I/O 的完成，并更新异步 I/O 完成的最终字节数
- `SetIoProgress` — 异步设备驱动程序使用此函数来更新异步 I/O 请求的状态和进度

以下代码示例展示了如何在设备驱动程序中实现异步 I/O 操作，以及此类异步 I/O 处理的一种可能流程。在该示例中，设备驱动程序在 `XXX_IOControl` 入口点内实现了一个长写入操作（在本示例中并非实际那么长）。为了执行这个长写入操作，设备驱动程序创建了一个专用线程，该线程实际将输入缓冲区复制到设备驱动程序的本地缓冲区。这种做法并不十分巧妙，但确实能有力地阐明要点。区分此设备驱动程序与任何其他内核模式流式设备驱动程序的唯一代码，正是这一特定行为。以下**清单 7-1** 是一段演示 `XXX_IOControl` 入口点实现的代码片段。红色标记的代码表示 `Device Manager` 为设备驱动程序开发人员提供的特殊补充，用于支持异步 I/O 处理。首先要注意的是传递给 `XXX_IOControl` 入口点的新参数——`hAsyncRef` 句柄，它指向一个 I/O 数据包对象。`Device Manager` 提供的下一个函数是 `CreateAsyncIoHandle`。此函数将 I/O 数据包连接到一个异步缓冲区，以分配一个复制缓冲区，在此例中仅分配了输入缓冲区。

在**清单 7-2** 中，代码设置了将传递给处理 I/O 操作的线程的参数，创建了该线程并运行它。其余工作由线程负责。这里的代码较为简单，并将线程优先级设定为 251。在设计驱动程序时，如果 I/O 操作规模较大，您可能需要更改此线程的优先级。但是，您应记住这将对系统产生的影响，并考虑用户模式调用线程的优先级。

线程本身执行 I/O 操作，当操作完成时，代码调用由 `Device Manager` 提供的另一个函数 `CompleteAsyncIo`，以通知调用进程 I/O 操作已完成。结果，由调用进程创建的事件会被触发。

该进程本身是一个非常简单的控制台应用程序，它打开设备驱动程序，创建一个 `OVERLAPPED` 结构和一个事件，并调用 `DeviceIoControl`，如**清单 7-3** 所示。为清晰展示代码，列表中的大部分调试消息已被省略。以粗体红色字体显示的代码强调了异步 I/O 请求支持特有的部分。

> **对应用程序开发人员的警告：** 在设备驱动程序通知 I/O 操作完成之前，应用程序不得修改用于异步 I/O 请求的内存缓冲区，因为这可能导致不可预测的结果。
