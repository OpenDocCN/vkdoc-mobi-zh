# 排版后的内容

然而，这里的问题不在于弗兰克，而在于主持人。弗兰克感觉不到进展，所以他离开了。主持人忘记了……*如果听众没有从耳中冒出“蒸汽”（意指全神贯注），他们可能就会停止倾听。* 主持人的责任在于不断扫视整个房间，判断谁在参与、谁没有。这很困难。

**主持人**。明确的议程。七个人。在会议的任何一个时间点，有三个人在就某个话题进行言语交锋。主持人不仅要确保这三名活跃参与者不相互“厮杀”，还需要实时判断其他四个人是否在精神上投入其中，如果没有，又该如何应对。这非常困难。

这之所以非常困难，是因为主持这类会议高度依赖于具体情境。你有七个人，每个人都有各自的个性和议程。你还要考虑他们在那个精确时刻的心情。此外，还要顾及召开这场会议所讨论的话题本身。面对所有这些模糊变量，我能给出什么切实可行的建议来让每个人都保持参与呢？这里有几个小贴士：

*   **把他们拉回来。** 如果他们看起来没有参与感，就把话题引向他们，问一个与当前话题状态相关的问题：“斯坦，没有代码审查？真的吗？”
*   **用沉默重置会议。** 如果好几个人都走神了，我最喜欢的一个招数就是主持人保持沉默。当所有人的目光都集中在你身上时，从 10 开始倒数，看看会发生什么——弗兰克会从《植物大战僵尸》中抬起头来，心里纳闷：“怎么这么安静？我错过什么了？”
*   **改变场景。** 你们现在坐着吗？好，那就站起来。你一直在白板上写东西吗？没有？那试试看。对场景做些微调可能改变不了什么，但也可能让某个人从精神恍惚中被推一把。

会议的进展是通过流程来衡量的，主持人的工作就是让会议以良好的节奏向前推进，这就是为什么主持人有时需要……

## 掌控全局

作为主持人和参会者，你会遇到各种各样的与会者。我想谈谈的是那些认为仅仅因为被邀请，就负有道德义务必须在会议上发言的人。是的，说话是积极参与的表现。是的，你永远不知道哪个随口提出的奇怪问题会神奇地改善会议。是的，这个人总是在说话……每次会议……永远没完没了。

某个时刻，主持人需要变成“独裁者”，掌控会议。他掌控一切。他积极展示对会议的控制，当你就是那个被“掌控”的人时，会有点刺痛，但这场会议不是关于你的。而是关于房间里的每一个人，他们都想离开这场会议，去做真正有成果的工作。

对于主持人来说，在会议中决定介入并制止某个人并非轻率之举。一个好的主持人知道，滥用“独裁者”角色最终会导致所有人都闭口不言，这和那个永远说个不停的人一样低效。召唤“独裁者”是最后一搏，旨在立即解决问题，并且以一种让问题不再重现的方式来解决。这是一种凭直觉做出的主持人判断，在你熟练掌握之前总会搞砸；然而，成功主持一场会议的一个重要且不可估量的部分还涉及到……

## 即兴发挥

无论难题是什么，解决方案都将通过以下两种方式之一出现：随机的灵光乍现，或艰难的埋头苦干。通往任何一种方式的路径都需要一个称职的主持人做到我刚才描述的一切，同时还要知道何时应该忽略这些规则。

一个好的主持人知道：

*   什么时候会议完全偏离了既定议程，但房间里每个人的非言语信号都显示正取得进展——那么，管它呢，顺其自然，看看会走向何方。
*   什么时候这个看似在漫无边际地闲聊、浪费大家时间的人，可能正在接近某个随机灵感的火花——那就让他说下去。
*   一场注定失败的会议所发出的明显危险信号，无论是准备不足、关键人物缺席，还是团队正完全被另一个问题困扰。
*   在预定会议开始五分钟后就有勇气叫停，因为实在看不到任何取得进展的途径。

会议管理，如同人员管理，通常是关于管理“当下瞬间”的艺术。这意味着唯一适用的规则完全取决于当下那个如同雪花般独一无二的语境。

## 会议文化

在成长型公司发展的某个阶段，会议会占据主导地位。在当时，这似乎是个好主意，因为产品路线图散落一地，关键人物正在离职，或者走廊里充斥着争吵声。无论灾难是什么，一场与正确的人一起举行、领导得力且高效的会议，能为难题提供解决方案。看到这种情况的人会注意到并想：“好吧，我们现在有了一种解决问题的新工具——它叫会议。”

带着这种全新的认可，会议如雨后春笋般涌现。它们成了解决问题的时尚方案——取得进展的方案。更多的人被邀请参加这些会议，因为每个人都相信*如果你被邀请参加会议，你在某种程度上就更能体现专业相关性。* 人们开始在大楼里变得难找，查看别人的空闲/忙碌日程成为文化的一部分，突然间*我们更关心的是如何照料和喂养这些会议，而不是真正地把事情搞定。*

会议必须存在，但会议不能被看作是取得进展的唯一解决方案。如果必须开会，那么会议开始时就要牢记：一场成功的会议的定义是，当会议结束时，它再也不需要召开了。
