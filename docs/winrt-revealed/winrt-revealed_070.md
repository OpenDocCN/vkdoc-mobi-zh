# 第二部分

## 过程即产品

人会犯错。我们每个人都会。有些错误无人知晓，只有我们自己知道；而有些则是公开的——那些令人难堪的灾难，当你抬头时，团队所有人的目光都聚焦在你身上。哎呀。

为了防止这些错误，团队中更有条理的成员创建了流程。他们的目标是为我们的工作提供结构，并消除猜测。这些人用心良苦，但他们仍然让那些明白以下两点的人感到厌烦：第一，无论我们制定多少流程，我们总是会犯错；第二，犯错往往比不犯错能揭示更多有用的信息。

流程在衡量者与创造者之间创造了一种令人愉悦且健康的张力。在这种张力中，会出现无用的会议、走廊里的争吵和充满火药味的邮件。我倾向于认为接下来的章节会帮助你避免这些冲突，但你需要它们。就像一次有益的犯错一样，一场健康的争论提供了不同的视角，如果每个人都认真倾听，这将有助于流程的演变。无论你爱它还是恨它，流程就是你构建产品的方式。
