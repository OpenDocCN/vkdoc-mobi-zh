# 管理信息流与团队沟通

其他人都在猜测自己是否还有工作，而在缺乏信息的情况下，他们就开始编造各种离谱的谣言。

这就是谣言工厂，也就是小道消息传播的途径，它的存在与你作为管理者的沟通能力直接相关。

信息的创造，是在没有背景和依据时，人为构建出某种语境和基础的行为。你可以称之为谣言或八卦，但它本质上是对沟通失败的一种反应。当我听到一个匪夷所思的八卦时，我会关注两件事。第一，它到底在说什么；第二，这个荒诞的编造填补了人们认知中的什么信息空白。

回到那位以为自己要被解雇的"摇滚明星"员工。既然我知道她绝不可能被解雇，那我听到了什么？首先，我听到的是"我不知道自己在组织中的位置"。她并非真的认为自己会被解雇——而是她不明白自己的价值。其次，我听到的是"既然我不知道自己的价值，那我就编造一个疯狂的后果，虽然这不太可能发生，但至少能引起别人的注意。"

八卦、谣言，无论以何种形式出现，都意味着你组织中的某个人正在寻求帮助。

## 防止信息饥荒

你需要对这位"摇滚明星"员工进行危机处理。你需要和她坐下来，提醒她五次她是如何在你模糊的指示下创造出优秀产品的。你需要多次跟进，确保她知道自己的价值。但更重要的是，你需要弄清楚她最初是如何陷入信息饥荒的。

好吧，这一切始于你没有传达信息，这一点我们很清楚，但问题是：为什么？以下是一些常见的失败原因。

### *"别担心，我会记得转达的。"*

关键信息丢失的最大原因，可能就是管理者把大脑当成待办事项清单。这是新手管理者常犯的错误，他们还没学会如何做好信息传递的"管道"。他们在会议上听到一个需要转达给某位工程师的任务，心想："记住了，得提醒鲍勃，除非 X 发生，否则菲尔会搞砸构建。我能记住。"

如果这是清单上唯一的事项，这位管理者还能应付，但问题是他还有四个会议和十件其他待办事项要记住，直到他在走廊里遇到鲍勃时才想起来："菲尔……什么来着。"

把它写下来。随身带个笔记本，任何听起来有点意思的东西都记下来。这就引出了下一点……

### *"我觉得没人需要知道那个。"*

比尔是我团队里的一位新任管理者，他刚刚能参加员工会议，感到很高兴。我的员工会议分为两部分。第一部分是传达我从上级会议中获得的所有信息，并加入我自己的想法和意见。第二部分是团队成员的快速状态更新和相关内容。

第一部分中有很多冗余信息。产品表现如何，谁得到了晋升，最近有哪些客户成功案例。如果你每周都听，肯定会觉得无聊，但我每周都会重复同样的内容。

比尔很快就觉得无聊了，在第一部分时停止了做笔记。当我问他时，他说："我觉得我的团队不会关心那些他们不负责的产品的客户成功案例。"

"真的吗？你为什么不问问他们？"

接下来的一周，比尔疯狂地做着笔记。他的评论是："没错，他们想知道所有信息。"

尤其是在较大的组织中，你需要特别注意保持组织信息的持续流动。你可能觉得自己在传递无用的信息，但经验法则是：你永远不知道你的团队会关心什么。我有一位工程师，忠实地记录着每一个新客户的信息，几个月后，他对客户群的了解比大多数销售人员还要深入。

我明白，转达你接触到的每一条信息可能是一项全职工作，而作为管理者，你的职责之一就是判断哪些信息需要传递。我的经验法则是：如果我犹豫超过几秒钟是否要传递某条信息，那我可能就没有资格做这个决定，所以还是传递出去，看看会发生什么。

### *"我的员工能读懂我的心思。"*

也许你在信息传递方面做得不错。你召开了内容丰富的员工会议，并且持续转发邮件。这很棒。我很高兴你传递了所有信息，但问题是："你的员工听到了什么？"

创业公司的上一位副总裁在转发每封邮件方面做得"出色"，他收件箱里的每一封邮件都转发。每一封邮件。能参与副总裁的所有沟通当然很好，但这几乎成了垃圾邮件。更糟糕的是，他在每封邮件开头都加上了一些简短而晦涩的评论。

"有意思。"
"我们应该做。"
"哈！早告诉过你！"

大多数时候，我都能根据最近的对话推测出他那些只言片语的含义，但有时我完全不知道该怎么处理某封邮件。

仅仅因为一封邮件或一个想法在你脑海中是清晰的、有趣的，并不意味着其他人也能一眼看出其中的深意。让团队紧密协作、共享思维是我的一个目标，但花点时间为每条你传递的信息添加一些个人背景，绝无坏处。这需要时间，但能保证信息的质量，同时防止信息在缓慢传递中变得混乱。

好消息是，如果你想知道你的团队在接收到信息后听到了什么或读懂了什么，你可以问我最喜欢的一个跟进问题："你刚才听到了什么？"

## 积极的沉默

有组织地传播信息是让团队与组织其他部分保持联系的第一步，但你仍然会犯错。无论是前面提到的失败原因，还是完全不同的其他问题，你的职责是不断评估团队的需求。而我有一个非常好的消息。

你的团队会告诉你他们需要什么。无论是通过八卦、谣言、凝视、踱步还是喊叫，你的团队总是在告诉你他们需要知道什么。这意味着你的职责不仅仅是充当信息管道，还要采用一种"积极的沉默"策略。

在我的员工会议上，我会偶尔插入一段长时间的停顿。也许我刚说了一些有争议的话，却没有收到任何反对意见。也许我知道在座的某个人正对我们最近的讨论感到恼火。所以我等待。我用沉默填满整个房间。

在这种令人不安的安静中，如果他们正想说点什么，他们就会说出来。试试看：闭上嘴，看看当你不说话时，你的团队会说什么。

## 微妙、诡计与沉默

管理者、准管理者，以及想要理解管理者的人，只需要读一读罗伯特·格林和朱斯特·艾尔弗斯合著的《权力的 48 条法则》。

我故意没有对这本书做任何背景调查，因为我最初读到这份清单时的反应非常深刻，我想保留那种感觉。书中记载了一些相当邪恶的手段，也揭示了一些管理者日常行为的基本真相。起初我分不清作者写下"让他人处于持续的恐惧中：培养一种不可预测的氛围"之类的内容时是认真的吗，但读了几遍之后，是的，我认为他们是认真的。

我对这份清单以及它与管理者的关系感到的问题是，其中许多"法则"涉及对你所领导的人的心理折磨，这在我看来是加剧人们对管理者的强烈负面反应的好方法："那个家伙是个权欲熏心的混蛋。"

尽管如此。
