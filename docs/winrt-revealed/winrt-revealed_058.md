# 章节：信息饥渴

有个人站在你办公室外面，一言不发。

这是一种抓狂的表现，但此刻他只是站在那儿。或者也许不是站着。他可能在使用“徘徊法”——每隔几分钟就来回走一趟。也可能是“长凝视法”，他就站在外面死死地盯着。我最喜欢的是“雪崩法”，当你从屏幕前抬起头，只有几秒钟时间做好准备，然后一堆沮丧的情绪就会涌进你的办公室。

你给这些抓狂行为起什么名字并不重要。关键是要弄清楚，作为经理，你本该如何避免它们。这很可能与信息匮乏有关。

## 信息管道

作为经理，你的众多职责之一就是充当信息管道，规则看似简单：对于你看到的每条信息，你必须正确判断团队中谁需要这条信息来完成工作。

很简单，对吧？一封邮件来了，你读完，觉得得转发给你的某个开发人员。于是你转发了。问题来了：信息比你想象的要多得多；需要信息的人也比你预期的多；你对谁需要信息的判断常常会出错；而且你还有一大堆其他重要的事要做。

首先，我们来关注信息管理不善的后果，因为这样才能引起你的注意。

一个不那么普通的工作日。走廊里流传着裁员的谣言，而且谣言是真的。现在，你办公室外出现了一个“长凝视者”，当然，你做的第一件事就是请她进来，以便解决“长凝视者”的问题。

“为什么长凝视？”

“什么？”

[否认。]

“你在我办公室外站了两分钟，盯着我看。”

“我有吗？”

[哇，完全否认。]

“那么，怎么了？”

“呃，我会被解雇吗？”

现在，这可是你最好的开发人员。她是你的明星员工。她是那个你抛出一个最模糊的想法，她就能把那些手势比划变成功能、变成产品的人。她已经做到了*五次*，而现在她却在你的办公室里担心自己是否要被解雇。

发生了什么？你最重要的一位工程师怎么会认为自己如此无足轻重，以至于可能被解雇？

她对信息感到饥渴，而在缺乏信息的情况下，人们会自己编造信息。

### 自然厌恶真空

回想你上次经历裁员。发生了什么？首先，你听到“裁员要来了”的谣言，结果证明这不是谣言，并且那是你在接下来三周里听到的最后一个事实。

你看，管理团队花了这三周时间试图弄清楚接下来会发生什么，以及谁会被裁。但他们当时实际上还不知道，所以什么也没说，而这恰恰就是最错误的做法。
