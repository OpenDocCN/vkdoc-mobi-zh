# 附录 A：设计与创建图形

在图 A-7 中，我们看到 iPhone 版本应用有四个可以拖放图片的位置；图 A-8 显示了 iPad 版本。请注意，对于 iPad，可以指定横屏和竖屏版本，而 iPhone 只允许竖屏图片。在图 A-7 和图 A-8 中，图片被标记为“未指定图片”。顶部两个位置允许您指定应用程序的图标。您可以为 Retina 显示屏指定不同的图片，从而提供高分辨率版本。要指定图片，只需将图片拖入这些位置之一。该图片可以是项目已有的，也可以是其他图片。Xcode 会检查您拖入这些框中的图片是否适合该角色。如果不适合，系统会提示一个错误对话框，如图 A-9 所示。

**图 A–9.** *指定为图标的图片不合适*

图 A-9 显示了一个错误对话框，当尺寸不正确的图片被拖入 Xcode 中的应用程序图标位置时触发。尽管提供了简单的拖放界面，但要正确格式化所有图片实际上可能有点棘手。之所以棘手，是因为包括苹果网站在内的大多数网站上的图片尺寸信息都不准确。我希望等到你读到这些内容时，苹果已经在其网站上解决了这个问题。表 A-2 列出了每种支持图片的正确尺寸。

**表 A–2.** *支持图片尺寸*

| **设备** | **图标** | **启动（竖屏）** | **启动（横屏）** |
|------------|----------|-----------------------|------------------------|
| iPhone 3GS, iPod Touch 3 | 57x57 | 320x480 | 无 |
| iPhone 4, iPod Touch 4 | 114x114 | 640x960 | 无 |
| iPad 1 + 2 | 72x72 | 768x1024 | 1024x768 |

一旦你创建了正确尺寸的文件，就可以将它们拖入 Xcode 的相应位置。Xcode 会自动将文件复制到项目目录（如果需要），并将文件添加到项目中。较旧版本的 iOS 依赖于命名约定来指定哪个文件对应哪个功能。这相当容易出错且难以维护。

较新版本的 iOS 和 Xcode 允许你随意命名文件。

清单 A-2 显示了这些信息如何体现在我们的 `info.plist` 文件中。
