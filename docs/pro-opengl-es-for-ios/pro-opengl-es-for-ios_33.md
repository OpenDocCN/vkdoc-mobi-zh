# 第 7 章：精心渲染的杂项

## 斑点阴影

斑点阴影是一种完全的取巧手段。它直接假设不存在真实的直接光源，因此物体的阴影仅仅是其下方的一个斑点，如图 7-10 所示。如你所见，如果我们的遮挡物（投射阴影的物体）是一个巨大的食人玉米煎饼，这种方法的效果就不太理想。

## 投影阴影

投影阴影是动态阴影算法中“最容易”实现的，但当然这也意味着它们附带了许多限制，即投影阴影在向大型平坦表面投射阴影时效果最佳，如图 7-11 所示。此外，阴影无法投射到任意物体上。与其他方法一样，其基本流程是从光源视角和相机视角分别获取某种快照。光源视图被压扁到平面上，涂上合适的阴影颜色（即深色），然后遮挡物被渲染在最上层。

图 7-11：将阴影投射到平坦平面上，然后“重新投影”到观察者眼中的效果。阴影区域通过计算从光源出发、经过每个顶点的向量与平面的交点来确定。平面上的每个点构成一个“新”的物体，可以像平面上其他物体一样进行变换。清单 7-15 展示了如何编写这段代码。

让我们再次从基本的弹跳立方体演示开始（尽管其中大部分内容将针对阴影代码进行修改，但它仍可作为工作模板），但我们将替换新的视图控制器代码。清单 7-11 涵盖了一些初始化参数。

清单 7-11：需要添加到视图控制器的初始化内容

```
- (void)viewDidLoad
{
  [super viewDidLoad];

  // 为清晰起见，删除了常规向导生成的代码。
  m_WorldRotationX=35.0; //1
  m_WorldRotationY=0.0;
  m_SpinX=0.0;
  m_SpinY=0.0;
  m_SpinZ=0.0;
  m_WorldZ=-6.0; //2
  m_WorldY=-1.0;
  m_LightRadius=2.5; //3
  [(EAGLView *)self.view setContext:context];
}
```

在第 1 行附近，设置了世界旋转角度；90°时你将俯视场景，而 0°时你将正视场景侧面。

场景的世界位置在第 2 行附近确定。减小`z`值会使场景远离；增大`z`值则使其靠近。减小`y`值会使场景在屏幕上向下移动，增大则向上移动。

第 3 行中的`m_LightRadius`表示光源在场景上方移动的圆的半径。

`drawInRect()`方法在清单 7-12 中介绍。你最先注意到的变化是几何体和颜色数组不见了。实际上，我现在已将它们移到了文件顶部，作为全局定义的数据。这使得将渲染内容拆分成更小的部分变得更容易。有趣的是，很多 OpenGL 几何描述都保存在`.h`文件中——非常、非常、非常大的`.h`文件。苹果公司实际上在某些任务中推荐这种方法，并在其绘制著名的茶壶的 OpenGL 示例中使用了这种方法。该文件长达 3000 多行。

清单 7-12：用于投影阴影的`drawInRect()`方法

```
- (void)glkView:(GLKView *)view drawInRect:(CGRect)rect
{
  int lightsOnFlagTrigger=300; //1
  bool lightsOnFlag=true;
  static int frameNumber=0;
  GLfloat minY=2.0;
  static GLfloat transY=0.0;
  m_TransY=(GLfloat)(sinf(transY)/2.0)+minY; //2
  glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
  glClearColor(0.0, 0.0, 0.0, 1.0);
  glEnable(GL_DEPTH_TEST);
  [self updateLightPosition]; //3
}
```
