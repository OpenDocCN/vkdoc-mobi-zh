# 检查器概览

检查器可让您选择要记录的跟踪类型。每选择一个项目，就会在轨道窗口中新增一条轨道。颜色用于区分不同跟踪数据，而符号则仅当您在底部附近的“轨道显示”部分选择了折线图轨道样式时才会使用。

`配置`按钮会将页面翻转至更大的统计列表。这些都是详列在详细信息窗格中的项目。由于并非所有项目都适合以图表形式呈现，因此您有更多选项可供选择。

截至目前，您可以采集以下类型的跟踪数据，除非特别说明，这些数据均非累积值，而是每帧的单独值（在大多数情况下，数值越小越好）：

-   `#批处理调用次数`：各种批处理例程（如 `glDrawArrays()` 或 `glDrawElements()`）的调用次数。
-   `#禁用调用次数`：`glDisable()` 的调用次数。
-   `#启用调用次数`：`glEnable()` 的调用次数。
-   `#刷新调用次数`：`glFlush()` 被调用的次数。
-   `#GL 与平台调用次数`：每帧中对 OpenGL 和 iOS API 的总瞬时调用次数。
-   `#GL 调用次数`：与前一项相同，但仅限 OpenGL 调用。
-   `#渲染线条数`：渲染的图元线条数。
-   `#平台调用次数`：每帧中对 iOS API 的总瞬时调用次数。
-   `#渲染点数`：渲染的图元点数。在我的测试案例中，在我那台被亲切地称为“牛顿”的 iPad 1 上，它显示为 880,000。（想想看。）您也可以将这些点等同于已处理的顶点数。
-   `#冗余状态变更`：如前所述，每帧内不必要的状态变更总数。我目前只显示了 19 次。您的目标是将其降至 0。
-   `#渲染通道数`：用于跟踪多通道渲染的句柄。
-   `#纹理上传数`：上传到硬件的纹理数量。
-   `#渲染三角形数`：渲染的三角形图元总数。在我那台 iPad 1 上，我的测试应用目前每帧渲染 879,978 个三角形，并以 3 FPS 的速度运行。当然，像三角形数量这样的指标只是相对概念，因为它们会根据大小和方向发生巨大变化。
