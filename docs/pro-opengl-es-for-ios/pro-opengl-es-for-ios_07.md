**那着色器呢？**

这里并不是深入讲解着色器设计和语言的地方，但让我们也通过摆弄它们来揭开一些神秘面纱。列表 1-3 是顶点着色器。

列表 1-3. 预处理顶点的`Shader.vsh`

```
attribute vec4 position;
attribute vec3 normal;
varying lowp vec4 colorVarying;
uniform mat4 modelViewProjectionMatrix;
uniform mat3 normalMatrix;
void main()
{
vec3 eyeNormal = normalize(normalMatrix * normal);
vec3 lightPosition = vec3(0.0, 0.0, 1.0); //1
vec4 diffuseColor = vec4(0.4, 0.4, 1.0, 1.0);
float nDotVP = max(0.0, dot(eyeNormal, normalize(lightPosition)));
colorVarying = diffuseColor * nDotVP;
gl_Position = modelViewProjectionMatrix * position; //2
}
```

在这个顶点着色器中，光线是针对这个特定立方体隐藏的；这些值是`x`、`y`和`z`值。将中间的值改为`5.0`，这将会把光线移到场景上方很远，但只会影响蓝色立方体。

在第 2 行，`gl_Position`是一个预定义的对象，它携带当前顶点的位置。

在末尾添加以下一行：`gl_Position.x*=.5;`。图 1-11a 显示了结果。

图 1-11a,b. 左侧是顶点着色器中垂直缩放的变化，右侧是片段着色器中的颜色变化。

第一步是获取描述几何形状的数据，以及如何处理光照、颜色、材质和纹理的信息，并将其送入管线。

接下来，数据被移动和旋转，然后计算每个物体上的光照并存储。场景——例如一个太阳系模型——必须根据你设置的视点进行移动、旋转和缩放。视点呈现为一个视锥体，一种矩形锥体，用来将场景限制在理想情况下可管理的水平。

接着对场景进行裁剪，这意味着只有可能可见的内容才会被实际处理。所有其他内容都会尽早被剔除并丢弃。实时图形学发展的历史在很大程度上与物体剔除技术有关，其中一些技术非常复杂。

让我们回到太阳系的例子。如果你正在看地球，而月球位于视点后方，那么就完全不需要处理月球数据。裁剪层级正是处理这一问题的，既在物体级别上，也在顶点级别上。当然，如果你能在提交到管线之前自己预先剔除物体，那就更好了。或许最简单的方法就是直接判断一个物体是否在你身后，这样它就可以完全跳过。如果物体太远而看不见，或者被其他物体完全遮挡，也可以进行剔除。

剩余的对象现在被投影到“视口”上，这是一种虚拟显示器。

此时，光栅化开始了。光栅化将图像分解成片元，这些片元实际上是单个像素。片元是像素与额外信息（如纹理和雾）的捆绑包，为下一步做好准备。

现在，可以对片元应用纹理和雾效果。例如，如果雾可能遮挡较远的片元，同样可以发生额外的剔除。

最后一个阶段是，幸存的片元被写入帧缓冲，但前提是它们必须满足一些最后的操作。在这里，片元的 Alpha 值用于实现半透明效果，深度测试用于确保最近的片元绘制在较远片元的前面，模板测试则用于渲染到非矩形视口。
