## Alpha 混合

你肯定注意到了 RGBA 颜色四元组。如前所述，`A` 代表 Alpha 通道，传统上用于指定图像的半透明度。在用于纹理的位图中，Alpha 层形成了一种特殊图像：某部分半透明，另一部分完全透明，第三部分则完全不透明。

如果物体不使用纹理，而是通过顶点、光照或全局颜色来指定颜色，Alpha 可以让整个物体或场景具有半透明属性。`1.0` 表示物体或像素完全不透明，而 `0` 则表示完全不可见。

要使 Alpha 生效，与任何混合模型一样，需要同时处理源图像和目标图像。由于通过示例更容易理解，我们现在就开始第一个例子。

**混合函数**



要改变默认混合，我们必须使用`glBlendFunc()`，它带有两个参数。第一个参数指定如何处理源颜色，第二个参数指定如何处理目标颜色。要理解这个过程，请注意最终发生的是：每个 RGBA 源分量与每个目标分量进行加法、减法或其他运算。也就是说，源的红色通道与目标的红色通道混合，源的绿色通道与目标的绿色通道混合，以此类推。这通常用以下方式表示：将源 RGBA 值称为`Rs`、`Gs`、`Bs`和`As`，将目标值称为`Rd`、`Gd`、`Bd`和`Ad`。但我们还需要源和目标混合因子，分别表示为`Sr`、`Sg`、`Sb`和`Sa`，以及`Dr`、`Dg`、`Db`和`Da`。（实际上并没有看起来那么复杂。）以下是最终合成颜色的公式：

( `R`，`G`，`B`) = (( `Rs` * `Sr`) + ( `Rd` * `Dr`)，( `Gs` * `Sg`) + ( `Gd` * `Dg`)，( `Bs` * `Sb`) + ( `Bd` * `Db`))

换句话说，将源颜色乘以其混合因子，然后与乘以自身混合因子的目标颜色相加。

最常见的混合形式之一是将半透明面叠加到已绘制的内容（即目标）之上。如前所述，这可以是一个模拟的窗格、飞行模拟器的平视显示器，或者在与现有图像混合后看起来更好的其他图形。（后者在 Distant Suns 中大量使用，用于星座名称、轮廓等元素。）根据用途，你可能希望叠加层几乎不透明（alpha 接近 1.0），或者非常微弱（alpha 接近 0.0）。

在这个基本混合任务中，源颜色首先乘以 alpha 值（其混合因子）。因此，如果源红色最大值为 1.0 且 alpha 为 0.75，则结果只需将 1.0 乘以 0.75 即可得出。绿色和蓝色也同样处理。另一方面，目标颜色乘以 1.0 减去源 alpha。为什么？这实际上产生了一个永远不会超过最大值 1.0 的合成颜色；否则，可能会出现各种颜色失真。或者这样理解：源的 alpha 值是源允许填充的 1.0 颜色“宽度”的比例。剩余的空间即为 1.0 减去源 alpha。alpha 越大，源颜色可使用的比例越大，为目标颜色保留的比例则越小。因此，当 alpha 接近 1.0 时，复制到帧缓冲区中的源颜色越多，从而替换掉目标颜色。

**注意** 在这些示例中，使用归一化的颜色值，因为这使得跟踪过程更容易，而不是使用无符号字节（后者将颜色范围表示为 0 到 255）。
