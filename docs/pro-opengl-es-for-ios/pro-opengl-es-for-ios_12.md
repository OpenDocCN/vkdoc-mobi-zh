# 第 3 章

## 构建 3D 世界

在前两章中，我们介绍了炫酷的内容和数学内容（这些内容可能既炫酷又枯燥）。我们浏览了 `Xcode` 的 `OpenGL ES 2` 模板，旨在让你初步了解一个“简单”`OpenGL` 应用的结构和设计原则。在第 3 章中，我们将退一步，先审视一个非常简单的 2D 应用，然后逐步将其迁移到 3D 环境（4D 超立方体不在本书讨论范围内）。在此过程中，还会穿插介绍更多关于投影、旋转等 3D 理论。

## 再补充一点理论

请记住，`OpenGL ES` 对象是 3D 空间中点的集合；也就是说，它们的位置由三个值定义。这些值连接起来形成面（即平面表面），而这些面是三角形。三角形再连接起来形成对象或对象的部件。

要将构成顶点的一组数字、构成颜色的另一组数字，以及在屏幕上组合顶点和颜色的其他数字整合起来，必须向系统说明其图形环境。诸如视点位置、接收图像的窗口（或视口）、宽高比以及其他数字杂物等信息，都是完成 3D 回路所必需的。更具体地说，我将介绍 `OpenGL` 的坐标、它们与视锥体的关系、对象如何从场景中被裁剪或剔除，以及如何绘制到设备屏幕。

## OpenGL 坐标

如果你在任何系统上做过任何类型的图形工作，你都会熟悉常规的 X-Y 坐标系。X 始终是水平轴，向右为正；Y 始终是垂直轴，向下为正，原点位于左上角。这被称为屏幕坐标，很容易与数学坐标混淆，后者原点位于左下角，且 Y 轴向上为正。幸运的是，苹果的渲染框架 `Quartz 2D` 打破了传统，使用了数学坐标（尽管可以通过使用一些 `Core Graphics` 变换例程来根据需要调整）。

现在跳到 `OpenGL` 3D 坐标，我们有一个略有不同的系统，使用 `笛卡尔` 坐标（即表达空间中位置的标准方式）。回到数学坐标，`OpenGL` 的原点位于左下角，`+Y` 向上。但现在我们添加了第三个维度，表示为 Z。在这种情况下，`+Z` 指向你，如图 3-1 所示。

**图 3-1.** `OpenGL ES` 3D `笛卡尔`坐标系（图片由 Jorge Stolfi 提供）

实际上，在 `OpenGL` 中有几种坐标系或空间，每个空间都会变换到下一个空间：

- 对象空间，相对于每个对象。
- 摄像机（或视点）空间，局部于你的视点。
- 投影（或裁剪）空间，即显示最终图像的 2D 屏幕或视口。
- 标准化设备坐标（NDCs），表示从 -1 到 1 标准化后的 `xyz` 值。也就是说，一个或一组值被标准化，使其适合一个边长为 2 的立方体。
- 窗口（或屏幕）坐标，即场景在实际屏幕上显示时的最终位置。

对象空间、视点空间和裁剪空间是你通常需要关注的三个空间。例如，对象坐标以局部原点生成，然后移动并旋转到视点空间。比如，如果你有大量用于战斗游戏的飞机，每架飞机都有自己的局部原点。通过仅移动（或平移）原点，并让几何体的其余部分跟随移动，你应该能够将飞机移动到世界中的任何位置。此时，会针对视锥体测试对象的可见性，视锥体是定义虚拟摄像机实际可见范围的空间体积。如果对象位于视锥体之外，则被视为不可见并被裁剪（或剔除），从而不对其进行进一步操作。正如你在第 1 章中可能记得的那样，图形引擎设计中的大部分工作都集中在引擎的裁剪部分，目的是尽可能早地丢弃大量对象，从而产生更快、更高效的系统。

最后，在所有这些步骤之后，面向屏幕的 `OpenGL` 部分准备转换（或投影）剩余的对象。这些对象就是你的飞机、飞艇、导弹、路上的卡车、海上的船只、投石机松鼠，以及你可能会塞进游戏中的任何其他东西。

**注意：** `OpenGL` 并没有真正将任何东西定义为“世界空间”。然而，视点坐标是次优选择，因为你可以根据你的位置来定义所有东西。

## 视点坐标

`OpenGL` 中并没有神奇的视点对象。因此，你并非移动视点，而是相对于你自身移动所有对象。没错，这很容易让人混淆，因为你会发现自己经常需要改变数值的正负号。所以，与其说是你远离某个对象，不如说是该对象实际上正远离你。想象一下，你在拍摄一辆汽车从你身边飞驰而过的视频。在 `OpenGL` 下，汽车是静止的；而你和你周围的一切都在相对于它移动。这在 `OpenGL ES 1` 中主要通过 `glTranslate*()` 和 `glRotate*()` 调用实现，或在 `OpenGL ES 2` 中直接使用矩阵实现，你将在后面看到。正是在这一点上，之前章节中提到的 `OpenGL` 模型视图矩阵开始发挥作用。模型视图矩阵处理基本的 3D 变换（与将 3D 视图投影到屏幕 2D 空间的投影矩阵，或帮助将图像应用到对象上的纹理矩阵相对）。你将经常用到它。

从现在开始，除非另有说明，否则假设我指的是 `OpenGL ES 1`。

## 视锥体与投影矩阵

在几何学中，棱台通常是指棱锥或圆锥被两个平行平面切割后剩余的部分。换句话说，想象一下被削掉顶部三分之一的吉萨大金字塔（这并非我赞同破坏埃及古迹）。

在图形学中，视锥体定义了虚拟摄像机实际可见的世界部分，如图 3-3 所示。

**图 3-3.** 视锥体

与 `OpenGL` 中的许多事物不同，视锥体的定义非常直观，并且紧密遵循概念图形，只需定义一个体积（有时称为“视锥”）即可。任何整体或部分位于视锥体内的对象，最终都可能显示在屏幕上（如果未被更近的对象遮挡的话）。
