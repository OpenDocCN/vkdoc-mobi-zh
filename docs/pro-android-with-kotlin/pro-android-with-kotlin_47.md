# 发布库

到目前为止，我们仅在本地使用该库，这意味着你的开发机器上某处有该库项目，并在同一台机器的其他项目中使用它。你也可以发布库，这意味着在拥有公司仓库的情况下，让库可供公司环境内的其他开发者使用，或者对于你想提供给社区的库，使其真正公开。

不幸的是，发布库的过程相当复杂，涉及在多个地方修改构建文件，并使用第三方插件和仓库网站。这使得发布库成为一项复杂且脆弱的任务，向你详细描述一种可能的发布流程，在你阅读本书时很可能已经过时。因此，我建议你自己进行研究——在你常用的搜索引擎中输入“发布 Android 库”，它会立刻指向能帮助你的在线资源。如果你找到了几个可能满足需求的流程，一般经验法则是选择（A）拥有大型支持社区且（B）尽可能简单的那个。

此外，对于公司项目，如果要使用公共仓库，请确保你得到了许可。如果你无法使用公共仓库，安装公司仓库并非极其复杂的任务。例如，要建立一个公司 Maven 仓库，你可以使用软件套件“Artifactory”。
