# 引言

自本书第一版出版以来，已过去了数年时光。第一版出版时，Android 8 和 API 级别 26 是当时的最新版本，但此后 Android 9、10、11 和 12 版本相继发布。同时，Android Studio 也从 3.1 版本迁移至 4.2（2021 年 6 月），随后又经历了 Arctic Fox、Bumblebee、Chipmunk 和 Dolphin（均从 2021 年中至今）。与此同时，Android Gradle 插件从 3.0 版本迁移至 7.2，Kotlin 从 1.2 版本迁移至 1.5。此外，用于确保跨不同 API 级别版本兼容性的 `android.support` 包现已弃用，并由 Jetpack（ [`https://developer.android.com/jetpack`](https://developer.android.com/jetpack) ）取代。

在复杂的图形渲染和游戏开发方面，尽管 2018 年新型 3D 图形渲染引擎 Vulkan 尚处于起步阶段，因此未包含在第一版中，但它逐渐获得了越来越大的影响力，因此在第 9 章中增加相关章节如今意义重大。

201 年中，谷歌宣布了 *Android Automotive*，这是 Android 操作系统的一个变体，能够管理特定车型的信息娱乐系统。

随着 Jetpack Compose 的出现，一种简化的 UI 开发技术进入了 Android 世界，它通过提供复杂的 API 并应用优雅的声明式构建器式编程模式，显著减少了代码量。Jetpack 于 2018 年推出。

所有这些变化都反映在本书的第二版中，确保您作为开发者能够了解最新的 Android 开发 API 和工具。

## 简洁性与表现力

本书中将要讲解的程序，尽管与 Kotlin 的思维方式有很强的关联，但对于 Java 开发者或任何其他现代计算机语言的开发者来说，也并非完全神秘。Kotlin 的设计目标之一是表现力，因此理解 Kotlin 程序所需的精力更少，即使程序变得更短。但您必须明白，在某些情况下，为了追求极致的简洁性，您不得不牺牲表现力和可读性。

在判断孰优孰劣时，作者倾向于支持表现力而非简洁性，但请放心，冗长的编程风格被认为是不可取的。最终，专业开发者希望编写简洁的应用，因为更少的代码意味着更低的维护成本。
