# 任务

一个*任务*是一组相互交互的活动，最终用户将它们视为一个应用的各个元素。用户启动一个应用，会看到主活动，在那里进行一些操作，打开和关闭子活动，可能切换到另一个应用，再回来，最终关闭应用。

更深入地看，一个*任务*所展现的主要结构是其*返回栈*或简称为*栈*，应用的活动在此栈中堆叠。对于简单的应用，关于这个栈的标准行为是：当你启动一个应用时，第一个活动成为这个栈的*根*，然后从应用内部启动的下一个活动位于其顶部，另一个子活动位于两者之上，以此类推。当你通过导航返回（这就是*返回栈*这个名字的由来）关闭一个活动时，该活动会从栈中移除。当根活动被移除时，整个栈被关闭，你的应用也就被认为关闭了。

在 `AndroidManifest.xml` 的 `<application>` 元素内部，我们看到一些设置可以改变任务栈的标准行为，我们将在第 3 章（关于活动的章节）中看到更多内容。这样，一个定制化的任务栈可以成为帮助你的最终用户理解并流畅使用你的应用的有力工具。但请记住，复杂的栈行为可能让刚开始使用你应用的用户难以理解，因此，你的目标应该是在强大性与易用性之间找到良好的平衡。

注意

关于任务和返回栈运作方式的更多深入见解，可参见 [`https://developer.android.com/guide/components/activities/tasks-and-back-stack.xhtml`](https://developer.android.com/guide/components/activities/tasks-and-back-stack.xhtml)。
