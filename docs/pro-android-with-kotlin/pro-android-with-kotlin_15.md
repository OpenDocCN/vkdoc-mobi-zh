# 后台服务

后台服务在后台运行，也就是说，它们不会在状态栏中显示条目。不过，它们被允许使用 Toast 向用户发送简短的通知消息。与前台服务相比，后台服务更为脆弱，因为 Android 期望它们与用户活动的关联更松散，因此在资源短缺时更容易决定终止它们。

从 Android 8.0（API 级别 26）开始，以旧方式实例化后台服务受到了一些限制，并且建议转向使用 JobScheduler 方法。当以下情况均不成立时，运行在 Android 8.0 或更高版本上的应用被视为以后台服务的形式运行：

*   该应用有一个可见的活动，当前处于活动或暂停状态。
*   该应用有一个前台服务，即在其运行期间调用了 `startForeground()` 的服务。
*   另一个前台应用正在连接到它，无论是通过使用它的某个服务还是将其用作内容提供者。

一旦 Android 8.0（或更高版本）的应用作为后台应用启动或切换为后台应用，在它被视为 *空闲* 之前，会有一个几分钟的时间窗口。一旦 *空闲*，应用的后台服务就会被停止。作为例外，如果后台应用处理用户可见的任务，例如处理 Firebase Cloud Messaging (FCM) 消息、接收广播（如 SMS 或 MMS 消息）、执行来自通知的 `PendingIntent`（一个由其他应用以原始应用权限执行的 Intent）或启动一个 `VpnService`，则该应用会进入 *白名单*，并被允许执行后台服务。

从 Android 8.0 开始，以前通过执行后台作业完成的许多事情，现在被认为适合由 JobScheduler API 处理——请参阅第 8 章。
