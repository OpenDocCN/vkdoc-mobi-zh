# 4. 探索你的开发环境

对于几乎所有的`HomeKit`开发者来说，入门意味着搭建一个新环境，在某些情况下，还意味着一种新的工作方式。如果你是一个熟悉`iOS`的开发者，那么你已经为`HomeKit`做好了充分的准备……只要你对`iOS`最新的一两个版本也很熟悉。自从`iOS`从`iPhone OS`（而`iPhone OS`本身又是继苹果在 1997 年收购`NeXT`之后，从`NeXTSTEP`和`OpenStep`衍生而来的）诞生以来，在（刚刚）过去的十年间，很多事情都发生了变化。

作为一名`HomeKit`开发者，除了要熟悉`iOS`之外，你还需要熟悉`HomeKit`本身。而且，为了让你跟得上节奏，你需要开始使用最新版本的`Xcode`，这是用于`iOS`和 Mac 应用的集成开发环境（`IDE`）。

你可能需要探索一门新的编程语言（`Swift`），并且，也许还需要熟悉事件驱动编程。这看起来要求很高，但事实上，这些是当今开发环境的常见方面，无论你是在为`HomeKit`还是其他现代`API`（应用程序编程接口）（即使是其他平台上的`API`）进行开发，你都可能需要至少了解其中一些内容。

那么，让我们开始吧。



## 获取开发者访问权限

苹果正在让开发者（包括那些平台新手）越来越容易地为 macOS 和 iOS 开发应用。如果你已经是苹果开发者，已经拥有开发者凭证，可以跳过本节，直接阅读“下载工具”部分。如果你从未成为苹果开发者，本节将为你提供概述。如果你曾是苹果开发者但有一段时间（大约自 2015 年起）未使用过你的账户，那么你需要了解一些（简化方面的）变化。

没有改变的是，你需要一个 **Apple ID** 才能访问全套开发者工具。只要你年满 13 岁（年龄可能因所在国家而异）并拥有有效的电子邮件地址（用于确认身份），就可以免费申请一个 Apple ID。你的 Apple ID 可用于在 iTunes、App Store 和 Mac App Store 中进行购买。如果你已经有一个，可以直接用作开发者账户。

然而，请注意，作为开发者，你可能希望拥有一个单独的 Apple ID 账户。事实上，如果你是 iBooks Author，至少目前，你在 iBooks Store 中用于发布的 Apple ID 需要与你的开发者 Apple ID 分开。（请查看 `developer.apple.com` 获取你所在国家的最新信息。）

最重要的是，你需要一个 Apple ID（免费），并且这个 Apple ID 可以让你访问像 `Xcode`（IDE）和 `iOS Simulator` 这样的开发工具，你可以用它们来测试你的 iOS 应用。

凭借免费的 Apple ID，你可以构建应用并在自己的设备上进行测试。（这项功能是从 2015 年的 `Xcode 7` 开始新增的。在此之前，你必须加入开发者计划才能做到这一点。）

说到这个，你可能在某个时候想加入苹果的某个开发者计划。尽管现在即使不是开发者计划的成员，你也可以在自己的设备上安装应用，但如果不注册成为开发者，你就无法将应用提交到 App Store。这意味着你无法将应用分发给其他人。为了能够访问 App Store，你需要加入 `developer.apple.com` 上的某个开发者计划。有几种不同类型的计划，但大多数开发者会选择每年 `$99` 的基础会员资格，这能让你访问 App Store。

**注意**

你的开发者计划会员资格通常每年包含两次开发者技术支持事件（DTI）。你可以以每次 `$99` 的价格购买更多（两个为一组）。DTI 允许你向苹果工程师提问。问题可以很宽泛，比如“我在哪里可以找到关于……的信息”，但大多数开发者更倾向于使用更详细的技术支持。你可以提交一个你遇到问题的应用（或应用的一部分）给开发者支持。（DTI 不涵盖对使用他人应用的支持，但确实涵盖对使用 `Xcode` 的支持。）苹果工程师会审查你的代码，并帮助你定位问题和寻找解决方案。这是一项极其宝贵的资源，特别是对于无法求助于同事的独立开发者（当然，你有朝一日也会乐意回报这份帮助）。

有了你的 Apple ID（无论是个人使用还是作为开发者使用的独立账户），你就可以下载并安装工具，开始构建 HomeKit 应用了。

## 审视工具

正如上一节所述，成为苹果开发者计划的成员，你就能获得所需的工具。你不需要一开始就拥有全部工具，但以下是这些工具的概述以及你需要关注的内容。

不要被这个列表吓到：它的目的是让你了解涉及的内容。`Xcode` 的标准下载包含了所有这些工具的当前版本。`Xcode` 中的选项可以让你在工具发布时自动接收更新。（你也可以仅在需要时接收更新。通常，自动更新是针对小版本——“点”——发布，例如 `8.3.1`，而不是像 `8` 或 `8.3` 这样的大版本。）

此外，请记住，`Xcode` 和这些工具都是免费提供的。某些开发功能可能需要开发者计划会员资格，但基本的开发工具以及在你自己设备上部署的工具都是免费的。

**提示**

完整下载可能需要一段时间。你可能需要前往“系统偏好设置”中的“节能器”面板，以防止在下载完成前系统自动进入休眠状态。你也可以选择在系统和网络环境压力较小的时候进行完整下载——在许多情况下，夜间下载是个不错的选择。

这些工具可分为几个相互关联的类别。

- 语言
- 框架
- 模拟器

### 语言：Swift 和 Objective-C

在苹果收购 NeXT 并推出 OS X（以及后来的 iOS）之前，开发者使用各种编程语言。最初的 Mac 基于 Pascal，操作系统中也包含一些低级代码。随着时间的推移，操作系统还包含了一些 C 和 C++ 代码。收购 NeXT 带来了 Objective-C 编程语言，这种语言是与 NeXT 一起开发的（你可以查阅维基百科上关于这段历史的优秀文章，了解更多信息）。

Objective-C 在很长一段时间内一直是 Mac 和 iOS 的主要语言，并且至今仍然是操作系统和应用编程的主要语言之一。然而，与其他面向对象编程语言一样，硬件和软件的进步带来了新的机遇。Objective-C 主要仍由苹果公司管理（至今依然如此）。2014 年，苹果发布了一门新编程语言的第一个版本：Swift。它充分利用了操作系统和软件开发技术中的许多进步。Swift 现在是一个开源项目，并且在 Linux 和 IBM Swift Sandbox 网站（`https://swiftlang.ng.bluemix.net/?cm_mmc=developerWorks-_-dWdevcenter-_-swift-_-lp&cm_mc_uid=58573134700514602917751&cm_mc_sid_50200000=1460291775#/repl`）上有可用的版本。

**注意**

Objective-C 和 Swift 在苹果产品及第三方应用上仍然广泛使用。尽管苹果今天的演示和示例代码通常使用 Swift，但苹果也经常提供 Objective-C 的替代版本。此外，考虑到庞大的遗留软件数量，在可预见的未来，这两种语言都将继续被使用。

如果你是苹果平台开发的新手，Swift 通常是最容易学习的语言；但最终，在一些特殊情况下，你需要对 Objective-C 有基本的了解（这类似于对自然语言的被动——阅读——知识和主动——说或写——知识之间的区别）。



### 框架

框架历经多年演变。如今，这个词与多年来一样，指的是一组可复用的库。这些库还可以包含图像和字符串等资源。你可能未曾注意到的是框架的范畴。在苹果软件的世界里，当今的大多数框架都可追溯到 NeXTSTEP。由此衍生出两大框架及框架集：核心框架和专注于用户界面的高级框架（这虽是一个极大的简化，但足以为你开展 HomeKit 开发提供背景知识）。

多年来，许多较小的框架作为操作系统的一部分被创建出来。它们的名称清晰地表明了自身用途。在核心层面，有诸如 `CoreLocation`、`CoreGraphics`、`CoreText`、`CoreVideo` 等框架。在界面层面，则有 `AddressBook`、`CloudKit`、`UIKit`（用于视图和控件）、`Security` 框架、`WebKit` 等。

框架被打包成可复用的形式。当你首次在 Xcode 中创建项目时，需选择要创建的项目类型，相关框架便会自动安装到项目中。你可以在开发过程中添加其他框架。（不必担心丢弃未使用的框架：Xcode 的构建过程不会将未使用的框架复制到可运行代码中。）

框架的一个关键特性是它们可以版本化。这意味着它们可以与使用这些框架的代码并行开发和修订，但两者之间的连接相当松散，因为你可以同时在开发环境中拥有一个框架的多个版本。

最近，框架又进一步发展，允许在苹果平台之间共享代码。如果你创建了一个用于管理数据的 API，你可以将其（或它的接口）打包成一个完全平台中立的框架。然后，你可以在 iOS、macOS、watchOS 和 tvOS 的代码中包含这个框架。在这些环境中，你可以编写合适的用户界面，但框架本身则与平台无关。

这意味着，你将使用苹果开发工具包中的框架，并且很可能还会创建一些供自己使用的小型框架。（由于现在需要为 iOS 和 OS X，以及 watchOS 和 tvOS 编写代码，框架的后一种用途在过去几年中变得越来越重要。）

### 模拟器

为还不存在的产品编写代码一直是一项挑战（早期 Macintosh 应用程序代码以及操作系统都是在 Lisa 计算机上编写的，而时至今日，我们仍在 Mac 上为 iOS 设备编写代码）。

尚未问世设备的代码必须以某种方式进行测试，因此我们有模拟器可用来在宿主机上——也就是你的 Mac 上——运行这些代码。如今，你可以下载适用于各种设备和操作系统的模拟器。

通过标准 Xcode 下载，你会获得 iOS、macOS、watchOS 和 tvOS 操作系统的当前版本。通常，你还会自动获得紧挨着的上一版本。如果你想要，还可以下载更早的版本。有时，注册开发者可以下载即将发布操作系统的预发布版或开发者预览版。这些下载内容受你成为注册开发者时所同意的保密协议约束。

除了操作系统版本，你还可以模拟各种设备。操作系统的预发布版本通常会在其公开发布前几个月提供。硬件的预发布版本（例如新版 iPad）通常直到公开发布时才广泛可用。

关于模拟器，最重要的一点要记住：它们仅仅是模拟器：除了实际设备，没有什么能做到完全精确的测试。有些功能（iCloud 同步就是其中之一）完全由你控制：它不会像在实际设备上那样自动发生，而是需要你在 iOS 模拟器上通过命令触发。这能让你更好地管理你的测试。

#### 模拟 HomeKit 配件

HomeKit 引入了一整套需要模拟的新设备：第三方 HomeKit 配件。这对苹果和开发者来说都是一项艰巨的任务，因为这些第三方产品的开发由第三方厂商控制。

苹果公司创建了一个工具，让你可以模拟这些配件，无论它们是否已存在。这个工具本身（即 HomeKit Catalog）可供你使用。其源代码可以从 `developer.apple.com` 下载，并在第 5 章中讨论。如果你想快速入门，可以从 [`https://developer.apple.com/library/content/documentation/NetworkingInternet/Conceptual/HomeKitDeveloperGuide/EnablingHomeKit/EnablingHomeKit.html#//apple_ref/doc/uid/TP40015050-CH2-SW3`](https://developer.apple.com/library/content/documentation/NetworkingInternet/Conceptual/HomeKitDeveloperGuide/EnablingHomeKit/EnablingHomeKit.html#//apple_ref/doc/uid/TP40015050-CH2-SW3) 下载 HomeKit 配件模拟器。

## 准备工作

你可以从 Mac App Store 或开发者网站的 Xcode 页面下载当前版本的 Xcode：[`https://developer.apple.com/xcode/`](https://developer.apple.com/xcode/)。如果你访问 `developer.apple.com` 并注册为开发者，可以点击那里的链接。从开发者网站下载可能会让你获得比仅从 Mac App Store 下载更多的信息，特别是如果你已注册了某个开发计划。
