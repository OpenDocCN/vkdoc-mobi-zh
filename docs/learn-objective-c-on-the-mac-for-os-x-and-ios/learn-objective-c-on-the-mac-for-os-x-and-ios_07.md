# 第 3 章：面向对象编程入门

如果你已经使用和编程计算机一段时间，你可能不止一次听说过"面向对象编程"这个术语。**面向对象编程**，通常简称为 OOP，是一种最初为编写模拟程序而开发的编程技术。OOP 很快被其他类型软件的开发者所接受，例如涉及图形用户界面的软件。不久，"OOP"就成为了一个主要的行业流行语。它承诺成为神奇的银弹，让编程变得简单而愉快。

当然，没有什么能真正达到那种炒作的程度。像大多数追求一样，OOP 需要学习和练习才能熟练掌握，但它确实能让某些类型的编程任务变得更容易，在某些情况下甚至更有趣。在本书中，我们将大量讨论 OOP，主要是因为 Cocoa 基于 OOP 概念，而 Objective-C 是一种设计为面向对象的语言。

那么什么是 OOP？OOP 是一种构建由对象组成的软件的方法。对象就像你计算机内部的小机器，彼此通信以完成工作。在本章中，我们将看一些基本的 OOP 概念。之后，我们将研究导致 OOP 的编程风格，描述一些 OOP 特性背后的动机。最后，我们将详细描述 OOP 的机制。

> **注意：** 像许多"新"技术一样，OOP 的根源可以追溯到远古时代。OOP 从 20 世纪 60 年代的 Simula、20 世纪 70 年代的 Smalltalk、20 世纪 80 年代的 Clascal 以及其他相关语言演变而来。现代语言如 C++、Java、Python，当然还有 Objective-C，都从这些较古老的语言中汲取了灵感。

当我们深入 OOP 时，请在你的耳朵里放一条 Babel 鱼，并准备好沿途遇到一些奇怪的术语。OOP 带有许多听起来花哨的行话，这让它看起来比实际更神秘和困难。你甚至可能认为计算机科学家创造了冗长、听起来令人印象深刻的词语来向每个人展示他们有多聪明，但当然，并非所有科学家都这样做。好吧，别担心。我们会在遇到每个术语时对其进行解释。

在我们进入 OOP 本身之前，让我们先看看 OOP 的一个关键概念：间接性。

### 一切都是间接性



编程界有句老话说：“计算机科学中的任何问题，都可以通过增加一个间接层来解决。”**间接**（Indirection）是一个听起来高大上但含义简单的词——不要在你的代码中直接使用某个值，而是使用指向该值的指针。这里有一个现实世界的例子：你可能不知道最喜欢的披萨店的电话号码，但你知道可以在电话簿上找到它。像这样使用电话簿就是一种间接形式。

间接也可以指，你请另一个人去做某事，而不是自己动手。假设你有一箱书要还给住在镇另一边的朋友 Andrew。你知道你的邻居今晚正好要去拜访 Andrew。与其自己开车穿越小镇，放下书再开回来，不如请你的好邻居帮忙运送这箱书。这是另一种间接：你让别人替你完成工作，而不是你亲自去做。

在编程中，你可以将间接应用到多个层级，编写调用其他代码的代码，而其他代码又去访问另一层代码。你可能有过拨打技术支持热线的经历。你向支持人员解释问题，对方会将你转接到具体的职能部门。那里的工作人员再把你转接到具备相应技能的二线技术人员那里。如果你和我们一样，到了这一步，你可能会发现自己打错了号码，不得不被转接到其他部门寻求帮助。这种“踢皮球”就是一种间接形式。幸运的是，计算机有无限的耐心，可以承受被从一个地方转到另一个地方，去寻找答案。

## 变量与间接

你可能会惊讶地发现，你已经在程序中使用过间接了。不起眼的变量就是间接的一个现实应用。考虑下面这个打印数字 1 到 5 的小程序。你可以在*Learn ObjC Projects*文件夹中的*03.01 Count-1*目录下找到它：

```
#import <Foundation/Foundation.h>

int main (int argc, const char *argv[])
{
    NSLog (@"The numbers from 1 to 5:");
    for (int i = 1; i <= 5; i++) {
        NSLog (@"%d\n", i);
    }
    return (0);
} // main
```

`Count-1`有一个`for`循环，它运行五次，每次循环都使用`NSLog()`显示`i`的值。当你运行这个程序时，会看到类似如下的输出：

```
2012-01-21 11:52:01.940 03.01 Count-1[26429:903] The numbers from 1 to 5:
2012-01-21 11:52:01.943 03.01 Count-1[26429:903] 1
2012-01-21 11:52:01.944 03.01 Count-1[26429:903] 2
2012-01-21 11:52:01.944 03.01 Count-1[26429:903] 3
2012-01-21 11:52:01.945 03.01 Count-1[26429:903] 4
2012-01-21 11:52:01.947 03.01 Count-1[26429:903] 5
```

现在，假设你想升级程序，让它打印数字 1 到 10。你需要修改代码中的两处地方（在下面的列表中已加粗显示），然后重建程序（此版本在*03.02 Count-2*文件夹中）：

```
#import <Foundation/Foundation.h>

int main (int argc, const char * argv[])
{
    NSLog (@"The numbers from 1 to 10:");
    for (int i = 1; i <= 10; i++) {
        NSLog (@"%d\n", i);
    }
    return (0);
} // main
```

`Count-2`的输出如下：

```
2012-01-21 12:03:13.433 03.02 Count-2[26507:903] The numbers from 1 to 10:
2012-01-21 12:03:13.435 03.02 Count-2[26507:903] 1
2012-01-21 12:03:13.436 03.02 Count-2[26507:903] 2
2012-01-21 12:03:13.436 03.02 Count-2[26507:903] 3
2012-01-21 12:03:13.437 03.02 Count-2[26507:903] 4
2012-01-21 12:03:13.437 03.02 Count-2[26507:903] 5
2012-01-21 12:03:13.438 03.02 Count-2[26507:903] 6
2012-01-21 12:03:13.438 03.02 Count-2[26507:903] 7
2012-01-21 12:03:13.438 03.02 Count-2[26507:903] 8
2012-01-21 12:03:13.439 03.02 Count-2[26507:903] 9
2012-01-21 12:03:13.439 03.02 Count-2[26507:903] 10
```

以这种方式修改程序显然不是什么棘手的改动：你可以通过简单的查找替换操作完成，而且只需要修改两处。然而，在由数万行代码组成的大型程序中执行类似的查找替换就会困难得多。我们必须非常小心地对待仅仅将 5 替换为 10 的操作：毫无疑问，有些数字 5 的实例与此无关，因此不应该被改为 10。

变量正是用来解决这个问题的。与其直接将循环上限值（5 或 10）硬编码到代码中，我们可以通过把数字放入一个变量来解决问题，从而增加了一个间接层。当你添加变量后，你不再告诉程序“循环 5 次”，而是告诉它“去查看名为`count`的变量，它会说明要运行多少次循环”。现在，这个程序叫做`Count-3`，如下所示：

```
#import <Foundation/Foundation.h>

int main (int argc, const char * argv[])
{
    int count = 5;
    NSLog (@"The numbers from 1 to %d:", count);
    for (int i = 1; i <= count; i++) {
        NSLog (@"%d\n", i);
    }
    return (0);
} // main
```

程序的输出应该毫无意外：

```
2012-01-21 12:16:51.442 03.03 Count-3[26596:903] The numbers from 1 to 5:
2012-01-21 12:16:51.446 03.03 Count-3[26596:903] 1
2012-01-21 12:16:51.447 03.03 Count-3[26596:903] 2
2012-01-21 12:16:51.448 03.03 Count-3[26596:903] 3
2012-01-21 12:16:51.449 03.03 Count-3[26596:903] 4
2012-01-21 12:16:51.449 03.03 Count-3[26596:903] 5
```

**注意**  `NSLog()`的时间戳和其他信息占用了大量篇幅，为了清晰起见，我们将在后续的代码列表中省略这些信息。

如果你想打印数字 1 到 100，只需在代码中一个显而易见的地方做修改：

```
#import <Foundation/Foundation.h>

int main (int argc, const char * argv[])
{
    int count = 100;
    NSLog (@"The numbers from 1 to %d:", count);
    for (int i = 1; i <= count; i++) {
        NSLog (@"%d\n", i);
    }
    return (0);
} // main
```

通过添加一个变量，我们的代码现在更加清晰，也更易于扩展，尤其是在其他程序员需要修改代码时。要更改循环的值，他们不必仔细检查每个用到数字 5 的地方，以确定是否需要修改。相反，他们只需修改`count`变量即可得到想要的结果。

## 通过文件名的间接

文件提供了另一个间接的例子。考虑`Word-Length-1`，这是一个打印单词列表及其长度的程序；它位于*03.04 Word-Length-1*文件夹中。这个至关重要的程序是你新成立的 Web 2.0 创业公司 Length-o-words.com 的关键技术。以下是代码清单：

```
#import <Foundation/Foundation.h>

int main (int argc, const char * argv[])
{
    const char *words[4] = { "aardvark", "abacus",
                             "allude", "zygote" };
    int wordCount = 4;
    for (int i = 0; i < wordCount; i++) {
        NSLog (@"%s is %lu characters long", words[i], strlen(words[i]));
    }
    return (0);
} // main
```

`for`循环决定在任何时候处理`words`数组中的哪个单词。循环内的`NSLog()`函数使用`%s`格式说明符打印出单词。我们使用`%s`，因为`words`是 C 字符串的数组，而不是`@"NSString"`对象的数组。`%lu`格式说明符接收`strlen()`函数的整数值，该函数计算字符串的长度，并将其与单词本身一起打印出来。

当你运行`Word-Length-1`时，会看到类似下面这样的信息输出：

```
aardvark is 8 characters long
abacus is 6 characters long
allude is 6 characters long
zygote is 6 characters long
```

**注意**  再次强调，我们省略了`NSLog()`在`Word-Length-1`输出中添加的时间戳和进程 ID。

现在，假设投资 Length-o-words.com 的风险资本家希望你使用另一组不同的单词。他们仔细审查了你的商业计划，并得出结论，如果你使用乡村音乐明星的名字，你可以销售给更广阔的市场。
