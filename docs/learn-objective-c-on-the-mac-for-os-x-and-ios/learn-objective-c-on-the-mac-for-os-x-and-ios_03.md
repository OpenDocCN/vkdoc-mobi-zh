## 前言

每当软件开发人员采用一个新平台时，他们都面临着艰巨的任务：需要熟悉新环境中的编程语言、开发工具、设计模式以及可用的标准软件库。

通常情况下，这必须与在压力下尽快交付代码的工作同步进行，而开发人员很容易退回到以往系统中熟悉的做法。这往往会导致代码与平台不匹配，可能重复实现已有的功能，并且日后会造成维护上的麻烦。

在理想情况下，新开发人员能得到已经熟悉该平台的同事的帮助，他们可以指导其入门并朝着正确的方向前进。不幸的是，iOS 平台本身的成功导致很多情况下根本无法实现这一点。

如果你身边没有导师，那么次优的选择是什么呢？
