# 5. 与全息影像交互

在本章中，我们将学习混合现实开发中使用的输入机制。用户可以通过多种方式与应用程序中的全息影像及其他元素进行交互。这些方式包括手势、语音命令、注视和控制器。我们将逐一介绍每种输入方法，并学习如何使用 `MRTK` 中的输入资源。


## 输入方式

每种用于 Windows 混合现实的输入方式都有其优势和局限性。以下列表提供了各输入类别的说明：

- **视觉注视：** 在 Windows 混合现实中，视觉注视是用户聚焦全息图和物体的主要方式。视觉注视对混合现实的重要性，就如同鼠标对个人电脑一样。你用鼠标指向电脑屏幕上的对象。同样地，你用视觉注视（你看的方向）来指向三维空间中的对象。在电脑屏幕上，鼠标的位置由光标或箭头表示。视觉注视光标通常表现为一个小点或甜甜圈形状的物体。目前，视觉注视由头部运动控制，而非眼睛的实际注视。在混合现实中有多种使用视觉注视的方式，例如指向物体、指向远处位置（用于瞬移），以及让物体跟随你的视线。

    **注意：** 某些混合现实头戴设备可能没有支持手势的传感器，而是依赖运动控制器作为其主要输入方式。

- **语音：** 语音输入是指使用语音命令来与混合现实体验进行交互。语音命令在开发混合现实应用时极其有用，因为它无需杂乱的用户界面即可实现高度控制和自定义。用户可以说出单个词或一系列词来选择物体、激活功能以及增强体验。语音还可用于听写单词和句子（语音转文本），以实现快速文本输入，而无需使用键盘。当设备连接到互联网时，语音识别会更准确。这是因为你的语音数据将被发送到微软基于云的语音识别技术。当在线语音识别设置关闭时，语音服务将仅限于使用基于设备的识别功能，如 Windows 语音识别应用。语音识别仍然有效，但微软不会收集任何语音数据。

- **其他硬件：** 除了已列出的主要输入方式外，还有多种硬件选项可与 Windows 混合现实头戴设备配合使用。这些设备包括蓝牙键盘和鼠标、蓝牙游戏手柄、点击器及其他蓝牙配件。

在本章的后续部分，我将带你了解如何在混合现实应用中使用每种输入方式。

### 步骤 3：边界控制

`Bounds Control` 允许您在混合现实中变换对象。您可以更改对象的大小和比例。它相当于缩放选项，但是在 3D 尺度上操作的。

让我们在 `HandInteractionExamples` 场景中尝试此功能，以探索该特性。如图 5-11 所示，在 `Bounds Control` 标题下有五个对象，它们将向您展示 `Bounds Control` 在 HoloLens 1 和 2 中的工作方式。

![../images/441101_2_En_5_Chapter/441101_2_En_5_Fig11_HTML.jpg](img/441101_2_En_5_Fig11_HTML.jpg)

*图 5-11*

面板中的五个对象各自通过增大、减小或旋转来响应 `Bounds Control` 手势。

角落处的控制柄可帮助您缩放、旋转和平移对象。`Bounds Control` 也会响应用户的输入。例如，在 HoloLens 2 中，`Bounds Control` 会响应手指的靠近，并提供视觉反馈以估算对象的距离。如图 5-12 所示，脚本在 `Inspector` 面板中包含几个可自定义的选项。让我们逐一了解：

![../images/441101_2_En_5_Chapter/441101_2_En_5_Fig12_HTML.jpg](img/441101_2_En_5_Fig12_HTML.jpg)

*图 5-12 — Bounds Control 脚本下的各种可自定义元素*

- **Target Object（目标对象）：** `Target Object` 指的是可以使用 `Bounds Control` 脚本进行操控的对象。如果此处未指定任何对象，则它将把所属对象作为目标对象。
- **Behavior（行为）：** `Behavior` 设置规定了 `Bounds Control` 在附加到对象时应如何表现。它包含用于激活 `Bounds Control`、控制是否在某个轴上扁平化控件等设置。
- **Smoothing（平滑处理）：** 它允许您在缩放或旋转对象时缩放平滑行为。
- **Visuals（视觉效果）：** 您还可以通过配置 `Visuals` 部分下的设置来更改 `Bounds Control` 的视觉效果。它们可以是链接的或内联的可脚本对象。
- **Constraints（约束）：** 约束系统有助于控制对象的缩放、旋转和平移。通过配置这些设置，您将能够将移动限制在一定范围内。
- **Events（事件）：** 事件在激活后有助于提供音频反馈。有几个事件可以提供语音或音频反馈，例如 `Rotate Started()`（旋转开始）、`Rotate Stopped()`（旋转停止）、`Scale Started()`（缩放开始）、`Scale Stopped()`（缩放停止）、`Translate Started()`（平移开始）和 `Translate Stopped()`（平移停止）。

`Object Manipulator` 组件允许您使用单手或双手来缩放、旋转和变换对象。此功能以前由 `Manipulation Handler` 组件处理，现在已被 `Object Manipulator` 组件取代。您可以根据各种输入（如手部骨骼、手部射线、凝视和手势）来自定义 `Object Manipulator` 组件。

如果您希望将此功能集成到您的对象中，则需要将 `Object Manipulator (script)` 组件添加到相应的对象上。请确保您也在对象中包含碰撞体组件。为了使您的操作更真实，您可以在对象中包含 `RigidBody` 组件；这样做之后，您的数字对象的行为会像真实物体一样。它们会像自然物体一样与其他物体碰撞；如果没有 `RigidBody` 组件，物体将穿过另一个物体，使其看起来不真实。

- **Host Transform（宿主变换）：** 包含需要被操控的对象变换。
- **Manipulation Type（操纵类型）：** 操纵模式可以是单手或双手。您可以在此字段中选择这两种设置。
- **Allow Far Manipulation（允许远距离操纵）：** 允许您使用指针进行远距离交互。
- **One Handed Manipulation properties（单手操纵属性）：** 单手操纵属性负责管理由单只手执行的操作；可能是远距离或近距离交互。远距离交互使对象能够通过凝视、手部射线和运动控制器射线进行交互。顾名思义，用户可以远距离与对象交互。近距离交互以触摸和抓取的形式呈现。相应的触摸和抓取事件通过指针（如 `PokePointer` 和 `SpherePointer`）调用。
- **Two Handed Manipulation properties（双手操纵属性）：** 双手操纵属性负责管理由双手执行的操作，例如移动、旋转和缩放。
- **Constraints（约束）：** 为对象添加约束有助于限制对象的移动。您可以选择附加到对象的任意约束管理器。
- **Physics（物理）：** 仅当附加了 `RigidBody` 组件时，此字段才会被激活。
- **Manipulation Events（操纵事件）：** 列出了一组事件，例如 `OnHoverStarted`（悬停开始）、`OnHoverEnded`（悬停结束）、`OnManipulationStarted`（操纵开始）和 `OnManipulationEnded`（操纵结束），这些事件在进行交互时按顺序发生。



### 第 6 步：在应用中实现手势

现在您已对手势输入方式有了高层理解，掌握了创建手势功能所需的脚本和对象。让我们回顾一下具体内容：

- **边界控制**：使您能够在混合现实中变换对象。您可以使用对象四角的操作柄来改变对象的大小和比例。
- **`Pressable.cs`脚本**：将事件重定向到`PhysicalPressEventRouter`，同时充当手按压按钮的连接逻辑。
- **`Interactable.cs`脚本**：通过包含此组件使对象具备交互性，让对象能够响应用户输入。
- **近交互抓取**：使对象可感知并准备好接受输入。
- **对象操纵器**：允许您通过缩放、旋转和变换来操纵对象。对象操纵器具有多种属性和字段供您自定义。

您可能还想探索示例场景中附加的其他脚本和对象，以更精确地了解如何在场景中融入手势。我们在本节中一直在探索的`HandInteractionExamples`场景是启动新项目时极佳的模板场景。它包含前面列出的核心预制件，以及多个可供项目修改或扩展的独特示例。随着经验积累，您也可以考虑构建包含常用组件的模板场景。如果您希望从混合现实工具包提供的测试场景入手，修改后另存为模板场景，可以在`层级`中选择场景，然后通过菜单栏选择`文件` ➤ `另存场景`。

### 语音命令最佳实践

语音命令是用户与混合现实应用进行交互的绝佳方式。以下是一些 HoloLens 特有的语音命令供您尝试。请尝试说出：

- `“What can I say?”`
- `“Go to Start”`（可用于打开开始菜单）
- `“Increase Volume”`
- `“Decrease Volume”`
- `“How much battery do I have left”`
- `“Increase Brightness”`
- `“Decrease Brightness”`
- `“Start Recording”`
- `“Stop Recording”`
- `“Shut down the device”`
- `“Restart the device”`
- `“What time is it”`
- `“Go to sleep”`

以下是在应用中实现语音功能时需牢记的一些最佳实践：

- 使用包含两个或更多音节的**关键词**。这有助于识别各种口音的语音。
- 在设计应用时，应考虑用户**误触发**语音命令的情况。在适当的情况下，允许用户撤销操作（例如，当意外删除一个对象时）。
- 确保所有语音命令都是**独特的**。如果两个或多个命令听起来相似，语音识别器可能会激活错误的命令。
- 尽可能确保你的语音命令在各种口音下都能被**轻松识别**。
- 语音命令是快速访问嵌套菜单或其他可能需要多种手势操作场景的绝佳方式。
- 如果用户界面没有显示语音命令列表，请考虑为用户提供一份清单。
- 在可通过点击或语音命令激活的按钮旁边放置**麦克风图标**。如果可能，请使所有按钮和 UI 元素都支持语音启用。

## 其他硬件输入

有各种硬件选项可与 Windows Mixed Reality 头戴式设备配合使用。这些设备包括运动控制器、蓝牙键盘和鼠标、蓝牙游戏手柄、点击器等蓝牙配件。

虽然最佳实践建议避免在混合现实体验中使用传统的 PC 硬件（键盘和鼠标），但在某些应用中，这些输入方法可能是合适的。

对于可能涉及移动第三人称对象（如游戏角色、全息直升机或全息赛车）的应用，游戏手柄是一个绝佳选择。
