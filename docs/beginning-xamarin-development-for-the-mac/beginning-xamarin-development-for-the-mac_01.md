## 引言

近年来，移动应用编程已成为 IT 市场最重要、最激动人心的方面之一，催生了大量应用。同时，由于市场碎片化（表现为不同尺寸设备、操作系统和软件开发工具并存），移动开发极具挑战性。因此，要开始进行移动开发，你需要考虑这些不同因素并决定你的策略。

目前三大主流移动平台是：苹果（包括 iOS、watchOS 和 tvOS）、安卓（Android）以及通用 Windows 平台（UWP），每个平台都提供专用的应用开发工具。每个平台都有特定的硬件要求。显然，iOS SDK 要求你使用本地 Mac 机器或安装在 Mac 上的远程构建代理。UWP 应用可以在 Windows 10 机器上构建。最后，你可以使用 Mac 或 Windows 机器来开发安卓应用。因此，首要问题是选择哪种硬件。对于移动开发，基于 Intel 的 Mac 平台是最佳选择，因为它允许你同时使用 macOS 和 Windows 10。后者可以通过 Boot Camp 原生安装，或使用 Parallels Desktop 虚拟安装。因此，Mac 机器将为你提供最大的灵活性。

选择硬件后，你需要选择软件开发策略。这里有三种传统选项：

* **原生应用** – 在这种情况下，你需使用平台 SDK，这要求你使用特定平台的 IDE 和编程语言。例如，对于 iOS、watchOS 和 tvOS，你需要使用 Swift 或 Objective-C 以及 Xcode。原生工具可以让你完全访问平台 API，但代价是需要学习特定平台的编程语言。
* **混合应用** – 它们使用 Web 技术像网页一样进行编程。这些页面随后通过原生的 WebView 组件进行渲染，该组件可理解为每个平台提供的本地 Web 浏览器。在这种情况下，你在编程工具方面拥有很大的灵活性，并且可以在不同平台间使用相同的代码，但代价是对特定平台 API 的访问权限有所减少。
* **移动 Web 应用** – 它们是为移动设备定制视图的 Web 应用。这种策略最简单易行，但不允许你访问设备组件，并且需要网络连接。

当然，开发策略取决于多种因素，从你的编程偏好到应用的复杂性和目标平台。如果你打算同时针对多个平台，你可以选择一种跨平台移动工具，例如 Xamarin.Forms、Qt Mobile、Embarcadero RAD Studio 或 React Native。它们提供了一个额外的、统一的、平台无关的编程 API 层，该层在编译期间会被转换为特定平台的 API。重要的是，跨平台工具不仅提供对 API 的访问，还提供对可视化控件的访问。例如，如果你创建了一个消息对话框，它在编译期间会被转换为特定平台的消息对话框。因此，在这种“编写一次，到处运行（WORA）”的方法中，你确实可以在不同平台上编译相同的代码，但代价是灵活性有所损失。最后，如果你的目标是广泛利用特定平台的 API，你将选择 Xamarin.iOS 或 Xamarin.Android。这些工具允许你使用 C# 或 F# 开发原生应用。Xamarin.iOS 和 Xamarin.Android 提供了一个额外的薄层，将 C# 或 F# 代码映射到原生平台 API。你通常使用 Xamarin.iOS 和 Xamarin.Android 为每个平台分别实现逻辑和设计 UI。虽然你需要额外工作来创建特定平台的 UI，但你仍然可以在不同应用之间共享平台无关的代码。这种方法为你带来两个重要优势：

* 你的应用是原生的。因此，它看起来像任何其他内置应用，因为它利用了原生可视化控件。
* 你的应用是用现代编程语言编写的。因此，你可以轻松访问众多库和广泛的社区。对于想要开始开发移动应用的 C# 程序员来说，这一点尤其重要。



在本书中，我们将学习如何使用 `Xamarin.iOS` 为苹果设备：iPhone 和 iPad（`iOS`）、Apple Watch（`watchOS`）以及 Apple TV（`tvOS`）开发应用。我们首先会准备开发工具（`Visual Studio for Mac`），并了解可用的项目模板（第 1 章）。随后，我将明确展示 `Xamarin.iOS` 与苹果提供的原生 SDK 之间的关系，并探讨应用的结构与生命周期（第 2 章）。接着，我们将学习如何创建视图（第 3 章）以及实现视图间的导航（第 4 章）。之后，我们将处理触摸手势（第 5 章），研究如何通过自动化测试打造高质量应用（第 6 章），并消耗来自 RESTful Web 服务的数据（第 7 章）。最后，我们将学习如何为 Apple Watch（第 8 章）和 Apple TV（第 9 章）开发应用。
不过，在本书中，我们不会学习如何在不同平台间复用代码，也不会学习如何使用 `Xamarin.Forms` 开发 WORA（一次编写，随处运行）应用。关于这方面的更多信息，请参考 Alessandro Del Sole 所著的 *Beginning Visual Studio for Mac: Build Cross-Platform Apps with Xamarin and .NET Core* 一书。
