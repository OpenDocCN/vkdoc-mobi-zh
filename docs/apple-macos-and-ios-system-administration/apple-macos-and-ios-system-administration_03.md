# 3. macOS 安全

macOS 是否真的比 Windows 更安全尚有争议，但 Apple 已经赢得了 Mac 比其他平台更不易受到恶意软件和隐私泄露影响的声誉。在 macOS 过去几个版本的迭代中，Apple 的许多安全和隐私服务持续演进，Mojave 和 Catalina 更是进一步强调了 `沙盒化` 和 `用户确认`。本章探讨了 Mac 系统管理员必须熟悉的硬件和软件安全与隐私控制。

## Mac 硬件安全

### 基本 EFI 启动安全

每台现代 Mac 都通过使用 `固件密码` 提供了可扩展固件接口（EFI）启动安全。使用 `固件密码实用工具` 或 `启动安全性实用工具`（取决于所使用的 macOS 版本），系统管理员可以锁定用户的能力，使其在未先输入密码的情况下无法将 Mac 启动到预启动环境。当你设置 EFI 密码后，在正常使用系统时通常看不到密码提示。当用户尝试将 Mac 启动到 `恢复模式`、`网络启动` 或 `可移动介质启动` 时，EFI 将在继续之前提示输入密码。这将 Mac 锁定为仅能从内部硬盘的操作系统启动。

要访问固件密码实用工具或启动安全性实用工具，你必须首先按照本书第 2 章的概述将 Mac 启动到恢复模式。一旦进入 macOS 恢复环境，请从 `实用工具` 菜单中选择 `固件密码实用工具` 或 `启动安全性实用工具`，如图 3-1 所示。按照屏幕上的提示设置固件密码，然后重新启动。确保你设置的密码是你会记住的，或者记录在你的 IT 组织内的某个地方。

### Apple T2 安全芯片

自 2017 年底 `iMac Pro` 发布起，Apple 开始为大多数新款 Mac 配备一款名为 `T2` 的下一代定制芯片。T2 芯片集成了多种不同的硬件控制器，旨在通过单一芯片实现硬件级安全。例如，它集成了 `SSD 控制器` 以及一个全新的 `安全隔区` 协处理器，该协处理器负责在硬件层面管理启动安全和存储加密。

截至本文撰写时，以下机型配备了 T2 芯片：

*   2017 年款 iMac Pro（及后续机型）
*   2018 年款 Mac mini（及后续机型）
*   2018 年款 MacBook Air（及后续机型）
*   2018 年款 MacBook Pro（及后续机型）

要确定您的 Mac 是否配备 T2 安全芯片，可以通过 `关于本机` 对话框查看 `系统报告` 并点击 `系统报告` 按钮来查找。在系统信息界面中，依次展开 `硬件` > `控制器 (或 iBridge)`；如果 `控制器型号名称` 为 `Apple T2 芯片`，则您的计算机配备了该芯片。T2 安全芯片启用/影响着以下功能：

*   **触控 ID**（将触控 ID 数据存储在安全隔区中）
*   前置 **FaceTime 高清摄像头** 的信号图像处理，用于面部识别
*   启用 **安全启动**
*   增强的 **加密存储**
*   安全 **系统管理控制器**
*   麦克风安全的 **安全音频控制器**

`安全启动` 是 T2 芯片的一项功能，应特别引起 Mac 系统管理员的关注。此功能仅在配备 T2 芯片的 Mac 上可用，当您启动进入 `macOS 恢复` 模式时会找到它。要配置安全启动功能（如图 3-2 所示），启动进入恢复模式后，从 `实用工具` 菜单中选择 `启动安全性实用工具`。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig2_HTML.jpg](img/492151_1_En_3_Fig2_HTML.jpg)

图 3-2
安全启动功能的配置选项

安全启动提供三种选项：`完整安全性`、`中等安全性` 和 `无`。任何 T2 机型出厂时默认启用 `完整安全性`。此选项需要互联网连接，并会检查您尝试安装的操作系统当前是否受 Apple 信任。`中等安全性` 会检查您尝试安装的操作系统是否曾被 Apple 信任，且无需互联网连接。`无` 则完全不强制要求操作系统签名或信任。

请注意，在启用 T2 的系统上，此实用工具取代了 `固件密码实用工具`。它具备控制外部启动的能力，您可以禁止或允许从 USB 闪存驱动器、Thunderbolt 硬盘等外部介质启动。对于资深的 Mac 系统管理员而言，还需要一提的是，T2 芯片不支持网络启动或 `NetBoot`。T2 Mac 唯一支持的有网络功能的启动选项是使用 Apple 的 `互联网恢复` 服务。如果您正在使用基于 NetBoot 的映像部署工作流来预置硬件，请注意您将无法在启用 T2 的 Mac 上这样做。

## 文件权限

与其他基于 Unix 的操作系统一样，`macOS` 使用 `POSIX` 文件权限。对于任何文件、文件夹或宗卷，可以按用户或组分配四种标准权限。

### POSIX

`只读`：用户可以浏览文件目录并从中打开/复制文件，但不能写入该目录或保存对文件的更改。

`只读与执行`：用户可以读取目录中的文件并执行目录中的应用程序。

`只写`：用户仅能向具有此权限的文件夹写入内容。他们无法打开该文件夹或浏览其内容，只能将文件放入具有只写权限的目录中。

`无访问权限`：用户无权访问该文件或文件夹。

除了文件权限选项外，`macOS` 中的每个文件或目录都具有一个 `所有者`（特定用户）、一个（特定）`群组` 以及一个 `所有人` 群组。"所有人"群组是一个统称，指代任何可能访问该文件或目录但不是特定所有者或群组成员的用户。您可以将此理解为 *其他所有人*。所有者是创建文件或将其复制到当前目录的特定用户。群组通常继承自文件或文件夹所在的父文件夹。

`macOS` 中默认的文件权限群组包括：

*   `staff`：所有本地用户帐户所属的群组，包括标准用户。
*   `admin`：所有具有管理员权限的本地用户帐户所属的群组，拥有一些提升的权限，例如能够写入 `/应用程序` 文件夹。
*   `wheel`：此群组保留给 `超级用户` 或 `root` 帐户。root 帐户是该群组的唯一成员，且应保持如此。许多系统文件和目录受此群组保护。

### 修改文件权限

除非您正在 Mac 上设置特定的局域网或互联网文件共享，否则建议并最好保持文件和文件夹的默认权限。如果您在同一台 Mac 上的多个用户之间本地共享文件，Apple 在每个用户的家目录中都提供了一个 `公共` 文件夹，用于放置旨在同一台机器上跨多个用户共享的文件。

如果您确实需要修改文件权限，可以在访达中选择文件或文件夹，然后从 `文件` 菜单中选择 `显示简介` 或通过 `右键点击` 来操作。图 3-3 概述了如何进行这些更改。

*   `挂锁`：如果已锁定，点击它以使用具有解锁权限的用户进行认证，以便更改文件或文件夹的共享和安全设置。
*   `+/- 按钮`：使用这些按钮来添加或删除用户或群组，并设置额外的权限。

## Gatekeeper

苹果限制 `macOS` 上恶意软件的方法是要求合法的应用程序开发人员对其应用程序进行*签名*，否则未经用户明确同意，这些应用程序将无法运行。*Gatekeeper*是苹果给这个功能起的名字，它通过*安全性与隐私系统偏好设置*进行控制。图 3-10 显示了配置 Gatekeeper 的选项。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig10_HTML.jpg](img/492151_1_En_3_Fig10_HTML.jpg)

图 3-10
安全性与隐私系统偏好设置“通用”标签中的 Gatekeeper 选项

Gatekeeper 为系统管理员提供了两个选项，允许 Mac 运行来自第三方开发者的应用程序。**“App Store 和被认可的开发者”**是默认选项，该选项会阻止未经代码签名或非从*Mac App Store*下载的应用程序运行。**“App Store”**是更安全的选项，它限制用户运行任何从互联网下载的应用程序，只允许从*Mac App Store*下载的应用程序。

作为管理员，您可能仍然需要出于某种原因运行一个未经代码签名或不是从 Mac App Store 下载的应用程序。在早于 `Sierra` 的 `macOS` 版本中，有第三个选项**“任何来源”**，允许您运行应用程序而不会受到 Gatekeeper 的阻拦。尽管此选项在较新的版本中已被移除，但管理员仍然可以通过**右键单击** Gatekeeper 阻止的应用程序，选择**“打开”**，然后在警告对话框中同意，从而覆盖 Gatekeeper 并允许该应用程序运行，即使这样做可能不安全。

`FileVault`是苹果内置的磁盘加密技术，目前版本为 2.0。要启用`FileVault`，请使用`Security & Privacy System Preference`，然后在 FileVault 标签页中点击**Turn on FileVault**…按钮，如图 3-11 所示。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig11_HTML.jpg](img/492151_1_En_3_Fig11_HTML.jpg)

### FileVault 机构恢复密钥

如果您有数百或*数千*台需要加密的 Mac，并且您不想为每个用户或每台机器管理单独的恢复密钥字符串，该怎么办？幸运的是，苹果提供了为组织中的每台 Mac 创建和部署**机构恢复密钥**（IRK）的选项。部署 IRK 涉及三个步骤：在一台 Mac 上创建**主钥匙串**，编辑主钥匙串的副本以**移除私钥**，然后将配置**部署**到所有其他 Mac。

创建`FileVault`密钥需要使用命令行界面。打开`Terminal`，在提示符处输入以下命令来创建主钥匙串：

```
security create-filevaultmaster-keychain ~/Desktop/FileVaultMaster.keychain
```

系统会提示您创建密码。按照提示创建并验证密码。命令执行完毕后，您的桌面上会出现一个名为`FileVaultMaster.keychain`的文件，如图 3-13 所示。这是私钥恢复密钥的原始副本，可用于解锁组织中任何配置为使用 IRK 的 Mac。您需要将其复制到安全位置，并限制仅授权人员可以访问。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig13_HTML.jpg](img/492151_1_En_3_Fig13_HTML.jpg)

图 3-13
该命令已在桌面上生成了一个 FileVaultMaster 钥匙串

接下来，我们需要准备一份用于部署到其他 Mac 的副本：

1.  使用`Keychain Access`应用程序打开`FileVaultMaster.keychain`文件。
2.  在`Keychain Access`应用程序中，单击左侧的`FileVaultMaster`以选中它。如果需要，单击**padlock**图标并进行身份验证以解锁它。密码与您在`Terminal`中创建`FileVaultMaster`文件时使用的密码相同。
3.  在右侧找到名为`FileVault Master Password Key`的*私钥*文件，如图 3-14 所示。单击它以选中，然后从`Edit`菜单中选择`Delete`将其删除。现在您就有了一个缺少私钥的钥匙串文件。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig14_HTML.jpg](img/492151_1_En_3_Fig14_HTML.jpg)

图 3-14
删除 FileVault Master Password Key

现在我们已经准备好将 IRK 部署到其他 Mac。在每台您想要配置使用 IRK 的 Mac 上，请按照以下步骤操作。

4.  将我们修改过的`FileVaultMaster.keychain`文件复制到`/Library/Keychains/`文件夹，如图 3-15 所示。

![../images/492151_1_En_3_Chapter/492151_1_En_3_Fig15_HTML.jpg](img/492151_1_En_3_Fig15_HTML.jpg)

图 3-15
将修改后的钥匙串文件复制到要使用 IRK 的每台客户端 Mac 的`/Library/Keychains/`目录

5.  打开`Terminal`应用程序，输入以下命令来正确设置文件的权限。这些命令将所有者设置为`root`，组设置为`wheel`，然后设置我们想要的正确读/写权限：

```
sudo chown root:wheel /Library/Keychains/FileVaultMaster.keychain
sudo chmod 644 /Library/Keychains/FileVaultMaster.keychain
```

6.  打开`Security & Privacy System Preference`并浏览到 FileVault 标签页。单击按钮以启用`FileVault`。如果操作正确，您将不会收到创建恢复密钥的提示，而是会看到类似图 3-16 中的消息。

图 3-16
FileVault 不会提示输入恢复密钥选项；它会自动使用 IRK

**专业提示**
您可能会遇到这样一种情况：组织中的一些机器已经启用了`FileVault`，而您希望将其转换为使用 IRK。如果是这种情况，在复制钥匙串文件并设置正确的文件权限后，您可以在`Terminal`中输入一个命令，将现有的`FileVault`恢复密钥转换为您机构的恢复密钥。

将现有的`FileVault`实例转换为机构恢复密钥的命令：

```
sudo fdesetup changerecovery -institutional -keychain /Library/Keychains/FileVaultMaster.keychain
```

## 总结

在本章中，我们探讨了保护 Mac 安全的多种方法，了解了 Apple 在核心安装的`macOS`中提供的各种内置安全功能，并认识了`Transparency, Consent, and Control`等新兴的安全框架。在深入研究了 Mac 相关的两个章节后，我们将转换主题，接下来探索`iOS`和`iPadOS`。

### iOS 客户端管理

与`macOS`相比，`iOS`及其衍生的`iPadOS`是相对年轻的操作系统。它于 2007 年随第一代`iPhone`一同发布，至今仅有十多年历史，目前已发展到第十三个主要版本。相较于桌面操作系统，`iOS`的管理要简洁直接得多。其最初被设计为手机的操作系统，并不具备传统计算机操作系统的要求，例如从互联网任意位置安装下载的软件或强大的外围设备支持。虽然`iOS`比真正的桌面级系统简单，但在支持大量`iPhone`和`iPad`时，系统管理员仍需熟悉一些关键功能。本章为理解 Apple 为运行`iOS` 13 和`iPadOS`（13）的设备提供的配置、安全和故障排除能力奠定了基础。

## iOS 硬件支持

`iPhone`和`iPad`的物理按钮或开关非常少。在大多数情况下，iOS 设备不被认为是用户可自行维修的，这意味着只有`Apple Authorized Repair Center`或`Apple Retail Store`可以进行维修。除了标准重启，你还可以尝试使用特殊的按钮组合进行`forced restart`，以解决`iPad`或`iPhone`的常见问题。

### iOS 恢复模式

iOS 设备上的**恢复模式**类似于 `macOS 恢复模式`，它是一个有限的预引导环境，需要在操作系统完全加载前通过特定按键组合进入。这个环境有时被称为 `iBoot`（引导加载程序），它允许设备连接到电脑，以便用户执行擦除和恢复操作。

需要使用 iOS 恢复模式的场景包括：将密码锁定且遗忘密码的设备恢复到出厂设置；解决系统启动时卡在白色苹果标志的故障；或在无需用户交互的情况下，将新设备升级至最新 iOS 版本，使其在部署时直接停留在`设置助手`界面。

当 iOS 设备进入恢复模式后，唯一能进行的操作就是将其连接到运行 `iTunes`（或 macOS Catalina 及更高版本中的 `Finder`）的电脑，并按照屏幕提示恢复或升级操作系统，如图 4-8 所示。请注意，在此模式下恢复系统会完全擦除设备上的所有数据。务必确保手边有最新备份，以便在恢复成功后重新安装应用程序和用户数据。

*   **iPhone 8 系列、iPhone X 系列、iPhone 11 系列**：使用 USB-Lightning 数据线将设备连接到电脑。快速按下并松开`音量增大`键，再快速按下并松开`音量减小`键，然后长按`侧边`按钮，直到看到恢复模式屏幕。

*   **没有实体主屏幕按钮的 iPad**：关闭设备电源。长按顶部的`睡眠/唤醒`按钮，同时使用 USB-C 或 Lightning 数据线将 iPad 连接到电脑。持续按住按钮，直到出现恢复模式屏幕。
*   **有实体主屏幕按钮的 iPad**：关闭设备电源。长按`主屏幕`按钮，同时用 Lightning 数据线将 iPad 连接到电脑。持续按住按钮，直到看到恢复模式屏幕。



## 设备固件更新模式

**设备固件更新**（`DFU`）模式并非许多技术人员经常需要使用到的功能。它实际上在 **iBoot** 引导加载程序之前加载，是一个比 `iOS 恢复模式` 层级更低的环境。请注意，让设备进入此模式可能比较困难，因为它需要精确的时机把握，并且在操作正确时设备不会提供任何反馈。

通常仅在少数几种情况下会使用 `DFU` 模式。最主要的一种是将较新版本的 `iOS` 固件降级到较旧的版本。例如，出于兼容性问题或类似原因，您可能需要从 `iOS 13` 降级到 `iOS 12`。另一个使用它的原因是尝试修复已损坏的固件或 `iOS` 安装，而通过恢复模式进行的恢复未能解决该问题。虽然这种情况已不如过去普遍，但如果您有用户对设备进行了越狱，您也可以使用 `DFU` 模式来撤销越狱操作。

要将您的设备置于 `DFU` 模式，请按照针对您特定设备系列的以下说明操作：

*   **iPhone 8 系列、iPhone X 系列、iPhone 11 系列** **：** 将 USB-Lightning 数据线插入电脑和 iPhone。按下 **调高音量** 按钮，然后按下 **调低音量** 按钮，接着同时按住 **侧边** 按钮和 **调低音量** 按钮 5 秒钟，之后松开 **侧边** 按钮，但继续按住 **调低音量** 按钮。您会在电脑上收到设备已进入恢复模式的消息，但 iPhone 屏幕上不会显示任何内容。

*   **iPhone 7 系列** **：** 关闭设备。同时按住 **侧边** 按钮和 **调低音量** 按钮 8 秒钟，然后松开 **侧边** 按钮，但在通过 USB 数据线将设备连接到电脑时，继续按住 **调低音量** 按钮。您会在电脑上收到设备已进入恢复模式的消息，但 iPhone 屏幕上不会显示任何内容。

*   **iPhone 6 或 iPhone SE** **：** 关闭设备。同时按住 **电源** 按钮和 **主屏幕** 按钮 10 秒钟，然后松开 **电源** 按钮，但在通过 USB 数据线将设备连接到电脑时，继续按住 **主屏幕** 按钮。您会在电脑上收到设备已进入恢复模式的消息，但 iPhone 屏幕上不会显示任何内容。

*   **强制重启不带实体主屏幕按钮的 iPad** **：** 将 USB-Lightning 数据线插入电脑和 iPhone。按下 **调高音量** 按钮，然后按下 **调低音量** 按钮，接着同时按住 **睡眠/唤醒** 按钮和 **调低音量** 按钮 5 秒钟，之后松开 **睡眠/唤醒** 按钮，但继续按住 **调低音量** 按钮。您会在电脑上收到设备已进入恢复模式的消息，但 iPad 屏幕上不会显示任何内容。

*   **强制重启带实体主屏幕按钮的 iPad** **：** 关闭设备。同时按住 **睡眠/唤醒** 按钮和 **主屏幕** 按钮 10 秒钟，然后松开 **睡眠/唤醒** 按钮，但在通过 USB 数据线将设备连接到电脑时，继续按住 **主屏幕** 按钮。您会在电脑上收到设备已进入恢复模式的消息，但 iPad 屏幕上不会显示任何内容。

## iOS 备份

既然我们正在讨论恢复 `iOS` 的话题，或许应该回顾一下创建 **备份** 的过程。Apple 提供了两种备份 iOS 设备的方法。您可以通过 `iTunes`（或如前所述，在 `macOS Catalina` 中使用 `访达`）将设备备份到电脑，也可以备份到 `iCloud`。选择哪种方法取决于您的具体需求以及您的 iCloud 账户中的可用存储空间。

### 电脑备份

## iOS 设备安全

当苹果公司设计`iOS`时，他们从头开始就融入了现代化的安全功能。磁盘加密默认启用，并且在标准的`iOS`安装中不是可选项。作为其`透明度、同意和控制倡议`的一部分，苹果不断加强`iOS`的安全和隐私功能。`Touch ID`、`Face ID`和`激活锁`都是有助于保护用户数据和隐私、阻止设备被盗的下一代功能。

### 密码锁/Touch ID/Face ID

每个 iOS 设备都具备`密码`锁定选项。这使用户可以设置一个解锁设备和访问主屏幕、用户数据及`App`所需的密码。它既可以是一个简单的四位数字，也可以是一个完整的短语或密码。苹果早些时候推出了生物指纹传感器`Touch ID`，它是带有物理主屏幕按钮的设备上的标准功能。下一代面部识别系统`Face ID`是 iOS 设备上（无物理主屏幕按钮）的标准功能。这两项技术都与密码设置协同工作以保护您的设备。这些功能可以在 iOS 13 中使用`Face ID`的设备的`设置`➤`面容 ID 与密码`中，以及使用`Touch ID`的设备的`设置`➤`触控 ID 与密码`中进行配置。

> 专业提示
> 为了增强数据安全性，特别是在政府或企业环境中，您可能希望设置设备在若干次无效密码尝试后重置/恢复到出厂默认设置。这可以防止对丢失或被盗设备进行暴力破解攻击。

### 激活锁

`激活锁`是苹果几年前实施的一项防盗功能，旨在为`iOS`用户提供另一层设备安全。`激活锁`与您的`Apple ID`和`iCloud`帐户配合工作，确保在重置或恢复后能够重新激活`iOS`设备的唯一人员，是在该设备上登录的`iCloud`帐户的所有者。如果设备进入 iOS 恢复模式，被抹掉，然后在没有先注销现有`iCloud`帐户的情况下，使用全新的 iOS 13 副本进行恢复，则在尝试使用设置助理激活设备时，您将看到如图 4-22 所示的信息。

图 4-22

要继续激活过程，您必须拥有之前在该设备上登录的帐户的`Apple ID`或`iCloud`密码。没有该密码，您将永远无法完成设备设置，因此它不会成为一个可用的系统。为了避免激活锁带来的麻烦，您应始终让最后一位用户在重置或恢复操作系统之前`退出`登录`iCloud`。在由于设备故障需要恢复的情况下，请确保`iCloud`密码可用，以便您可以通过`激活锁`快速完成身份验证，并完成系统的配置以供使用。

如果您因某些原因必须恢复设备，但没有`iCloud`帐户来绕过`激活锁`，则必须通过`AppleCare`或`Apple 零售店`联系苹果支持。通过提供购买证明，一旦证实设备所有权，苹果员工可以重置设备的激活锁状态。

## 用户帐户

所有 iOS 设备都设计为个人、单用户体验。每台设备只有一个本地用户帐户；所有其他帐户都用于访问各种服务，如`App Store`或`iCloud`，或第三方电子邮件服务，如`Gmail`或`Microsoft Exchange`。虽然建议您为苹果的所有服务使用同一帐户，但也可以分别为`App Store`和`iCloud`使用不同的帐户。



## Apple ID

`Apple ID` 是使用 iOS 设备所需的主要账户。`Apple ID` 是免费的，可以使用任何电子邮件地址进行配置，甚至可以来自微软或谷歌等竞争对手的邮箱。要创建 `Apple ID`，您的用户只需访问 [`http://appleid.apple.com`](http://appleid.apple.com) 并注册即可。您也可以在 `iOS` 中许多需要登录才能启用服务的区域注册 `Apple ID`，例如 `App Store`。教育或商业客户还可以创建**机构 Apple ID**，我们将在第 8 章中详细介绍。`Apple ID` 用于启用多种 Apple 服务，包括 `iMessage`、`FaceTime` 和 `App Store`。

## iCloud 账户

`iCloud` 账户经常与 `Apple ID` 混淆，因为您可以使用 `Apple ID` 的用户名注册 `iCloud` 服务，并且通常也会这样做。然而，这实际上是两个独立的账户。`iCloud` 服务通常涉及在包括 `文稿`、`信息`、`通讯录`、`日历`、`Safari` 书签等各种 Apple 设备之间同步数据。`iCloud` 还启用了诸如 `查找我的 iPhone`、`活动共享`（与 `Apple Watch` 配对时）和 `iCloud 钥匙串`等服务。

## App Store 账户

第三种账户是 `App Store` 账户，这是一个已注册**付款方式**的 `Apple ID`，可用于从 `App Store` 购买和安装应用程序。虽然 `iCloud` 账户是个人账户，但用于 `App Store` 的 `Apple ID` 可以与您的家庭或组织中的其他人共享。您可以通过 `设置` 应用程序 ➤ `iTunes 与 App Store` 手动使用 `Apple ID` 登录 `App Store`，然后点击**登录**按钮登录，如图 4-23 所示。

![../images/492151_1_En_4_Chapter/492151_1_En_4_Fig23_HTML.jpg](img/492151_1_En_4_Fig23_HTML.jpg)

图 4-23

请注意，除非您计划与组织中的所有用户共享账户凭据，否则共享的 `App Store` 账户可能不是一个好主意。当您使用共享的 `App Store` 账户在 iOS 设备上购买并安装应用程序时，下次该应用程序需要更新时，它会提示您的用户输入共享账户的密码。如果他们不知道密码，可能会锁定 `App Store` 账户或请求重置密码，这可能会给其他同样配置了共享账户的设备带来问题。如果您将密码透露给最终用户，他们可能会进行未经授权的购买，并记在您的账户上。

幸运的是，Apple 已实施两种解决方案来解决 `App Store` 的这些问题，同时允许组织集中购买应用程序并在多台设备上进行管理。第一种解决方案针对消费者，称为**家人共享**。**家人共享**允许每个家庭成员拥有自己的 `App Store` 账户，但可以访问家庭中任何人购买的共享应用程序库。另一种解决方案针对教育和商业客户，是使用 Apple 的**批量购买计划**（`VPP`），其中应用程序被购买并分配给与组织成员相关联的任意数量的唯一 `Apple ID`。`VPP` 在第 8 章中有更详细的介绍。

## iPadOS

当 `iPad` 首次发布时，该设备受到的一个批评是它本质上是一个大号的 `iPhone`，因为它运行的是 `iOS` 而不是 `macOS`（或其他类型的桌面操作系统）。在过去的 `iOS` 版本中，Apple 继续提供特殊的 `iPad 专用` 功能，如**并排应用多任务处理**和对 `Apple Pencil` 手写笔的支持。从 iOS 13 开始，Apple 决定将所有专属于 iPad 的功能整合在一个统一的名称下——`iPadOS`。

虽然底层操作系统仍然是 `iOS`，但以下功能是 `iPad` 独有的，仅在 iOS 13 的 `iPadOS` 子集中找到：

### 共享 iPad

最后，讨论 `iPadOS` 不能不提 **共享 iPad** 功能，它允许多个用户在共享设备上拥有唯一的登录账户。这与在 `macOS` 上拥有多个用户账户非常相似，但有一个巨大的例外——此功能仅限于 Apple 的**教育客户**。如果您在学术环境中支持一推车共享的 `iPad`，启用**共享 iPad** 可能会带来巨大的优势。

要使用**共享 iPad**，学术组织必须能够访问 `Apple 校园教务管理`。使用 `Apple 校园教务管理`，系统管理员将为每个学生创建特殊的**受管理的 Apple ID**。使用 `Apple 校园教务管理`，您可以将设备准备为**共享 iPad**，从而在设备上启用远程监控和多用户支持。所有学生的数据都存储在机构账户下的**受管理的 Apple ID** 内的 `iCloud` 中。

> 专业提示
> `Apple Classroom app` 的各种功能和 `Shared iPad` 超出了本书的范围。如果您是教育工作者，对 `Apple School Manager` 和 `Shared iPad` 感兴趣，请咨询您所在机构的 Apple 代表以获取更多信息。

## 总结

在本章中，我们通过配置限制、网络服务和 Apple 账户，为管理 iOS 设备奠定了基础。正如您所注意到的，逐个设置 iOS 设备配置可能非常耗时且容易出错。在下一章中，我们将探索 `Apple Configurator` 工具，并学习如何使用 `蓝图` 和 `描述文件` 来配置数百或数千台设备。
