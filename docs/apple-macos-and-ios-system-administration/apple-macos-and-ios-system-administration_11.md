### 使用脚本将 Mac 绑定到 Active Directory

Apple 还提供了一个名为`dsconfigad`的命令行工具，用于编写 Active Directory 配置脚本。在 Apple Remote Desktop 的`发送 Unix`任务中使用此命令，可以有效地一次性将多台 Mac 加入域，并进一步自动化该过程。如果您所在的组织没有使用 MDM 解决方案，这尤其有用。

开始此练习前，您应该在一台测试 Mac 上从全新安装`macOS`开始。您应该为其命名，并在 Active Directory 中创建一个匹配的计算机记录。我已经创建了一台名为`101-2013MPRO`的测试 Mac 并相应命名，但这次我将主机名留为`unset`，以便在本练习中与您分享一个快捷方式。我已经在 AD 中为其创建了计算机记录，但尚未将其加入域。

1.  现在我们的计算机名称和主机名一致了，我们可以准备使用`dsconfigad`命令加入域了。您可以查看`dsconfigad`的手册以了解更多关于此命令功能和额外选项的信息，但在本练习中，我们将只介绍基础知识。我们需要定义四个参数：
    1.  `客户端 ID` (`-c`) 代表我们要用来将 Mac 加入域的名称。
    2.  `管理员账户` (`-u`) 代表域管理员账户的用户名。



### 3. 管理员密码

`-p` 参数代表域管理员账户的密码。

### 4. 域

`-domain` 参数代表我们要将 Mac 加入的域。

![../images/492151_1_En_11_Chapter/492151_1_En_11_Fig17_HTML.jpg](img/492151_1_En_11_Fig17_HTML.jpg)
*图 11-17：使用脚本将计算机名称复制到主机名*

1.  我们将使用 `Terminal` 应用程序在测试 Mac 上交互式地开发此脚本，但一旦您完善了它，就可以在 Apple Remote Desktop 中创建一个模板并将其应用于整个 Mac 机群。在 `101-2013MPRO` 上打开 `终端` 以开始操作。

2.  我们将编写一个简短的小脚本，用于捕获 Mac 的计算机名称并将其保存为变量，然后使用 `scutil --set` 命令相应地设置主机名。因此，在终端命令行提示符处输入：

```
HOSTNAME=$(networksetup -getcomputername)
scutil --set HostName $HOSTNAME
```

按下 **Enter** 键，然后在提示 `HostName [计算机名称]` 输出时再次按下 **Enter** 键，接着在提示时输入本地管理员密码。一旦完成更改并再次回到命令行提示符，输入 `scutil --get HostName`，现在它应该与其他设置匹配为 `101-2013MPRO`。如果您在 Apple Remote Desktop 中使用此脚本，它将始终将 Mac 客户端的主机名更改为与计算机名称匹配。此时，您的终端窗口应类似于图 11-17。

**专业提示：** 您还可以在此处定义一个额外选项，即您希望将 Mac 客户端分配到的 OU。但是，由于我们已经在我们想要的位置创建了计算机记录，所以可以跳过此参数。如果您想手动定义放置此 Mac 的 OU，可以使用 `-ou` 参数，后跟 `"OU=, DC=, DC="` 格式的路径，例如 `"OU=macOS PCs, OU=Main Office Computers, DC=MyCompany, DC=local"`。

![../images/492151_1_En_11_Chapter/492151_1_En_11_Fig18_HTML.jpg](img/492151_1_En_11_Fig18_HTML.jpg)
*图 11-18：重命名计算机并将其加入域的完整脚本*

1.  让我们开始输入带有客户端 ID 参数的 `dsconfigad` 命令。因为我们将在多台计算机上使用相同的脚本，所以我们将使用一个令牌，而不是为此计算机指定确切的名称。我们可以使用 `"hostname -s"` 令牌，以便在脚本中自动应用此 Mac 的主机名作为客户端 ID。这将模仿我们通过*用户与群组系统偏好设置*将 Mac 加入域时的 GUI 行为。

2.  接下来，我们将为 `MyCompany.local` 域指定域管理员账户的用户名和密码。最后，我们将添加指定域名的参数。包含所有参数的完整脚本应类似于图 11-18。

![../images/492151_1_En_11_Chapter/492151_1_En_11_Fig19_HTML.jpg](img/492151_1_En_11_Fig19_HTML.jpg)
*图 11-19：脚本运行后，我们可以在"用户与群组系统偏好设置"中确认 Mac 已在域中*

1.  它将提示您输入本地管理员账户的密码。一两分钟后，它将成功完成。打开*用户与群组系统偏好设置*，单击 **登录选项** 按钮，确认现在列出了 Windows 域，并且**网络账户服务器**旁边有一个绿色指示器图标，如图 11-19 所示。

```
dsconfigad -c "'hostname -s'" -u Administrator -p noAcce$$ -domain MyCompany.local
```

**专业提示：** 您可以将其他命令与 `dsconfigad` 结合使用来设置其他设置，例如 `-mobile enable` 以支持移动账户选项，以及 `-passinterval` 来调整域计算机密码更改的频率，以便长时间脱网的系统不会脱离域。如果您对此有疑问，请咨询您的 Windows 系统管理员，以确定是否需要此选项或其他选项。

我们已经成功配置了 Active Directory，并使用三种不同的方法将我们的 Mac 加入了域。现在我们的 Mac 已在域上，我们可以访问 Windows 服务器上的其他资源。

## Microsoft Exchange 集成

Apple 在 `macOS` 和 `iOS` 的操作系统层面都提供了 `Microsoft Exchange` 集成。你可以使用描述文件来为邮件、联系人、日历和任务配置 Exchange 账户设置。对于 Don 的新款 `iPhone 11`，我们将直接使用描述文件管理器远程为他的设备配置 Exchange 设置。对于他的 Mac，他想要使用 Microsoft Outlook，因此我们将在下一节下载并激活 Office 365 应用程序后进行手动配置。

### 使用有效载荷变量配置 Microsoft Exchange 账户

在开始此练习之前，请务必在描述文件管理器中为 Don President 创建一个新用户，用户名使用 `dpresident`。注册一台测试用的 iOS 设备并将其命名为 `Don's iPhone`，这样你就可以修改并向该设备推送新的描述文件。既然我们已经创建了一个账户，我们将创建一个 Exchange 有效载荷，该有效载荷将使用他的用户名作为 `payload variable`（有效载荷变量）来配置其设备上的账户。

1.  打开 `Profile Manager`（描述文件管理器），浏览 `Devices`（设备）部分，找到 `Don's iPhone`。点击选择它，然后点击 `Settings`（设置）标签页并 `Edit`（编辑）描述文件。
2.  在描述文件编辑器中，向下滚动到 `iOS` 部分，选择 `Exchange` 有效载荷。点击 `Configure`（配置）按钮以指定设置。
3.  如图 11-32 所示，根据 Exchange 有效载荷的要求，在各个字段中输入 Exchange 服务器信息。对于用户名，我们将使用 `%short_name%` 而不是 `dpresident`。这样做可以让我们将此有效载荷应用于我们组织中的其他用户，并且在应用描述文件时，`%short_name%` 变量将解析为当前用户的用户名，在此例中为 `dpresident`。

![../images/492151_1_En_11_Chapter/492151_1_En_11_Fig32_HTML.jpg](img/492151_1_En_11_Fig32_HTML.jpg)

图 11-32: 使用有效载荷变量设置 Exchange 配置

4.  向下滚动到有效载荷的 `Enabled Services`（已启用的服务）部分，确保所有这些选项都已勾选，如图 11-33 所示。这将使整个 `iOS` 系统能够访问来自 Microsoft Exchange 的数据，包括“电话”应用的通讯录。

5.  点击 `OK`（确定）按钮保存配置有效载荷，然后点击 `Save`（保存）进行应用。在描述文件应用时，系统会提示用户输入密码，之后 Exchange 配置将完全完成。

**专业提示**：请注意，如果你的公司使用了 ActiveSync 中包含的一些 MDM 功能，这些功能也将应用于设备。例如，最小密码长度要求等将适用于设备，并且限制性更强的有效载荷将优先于其他。例如，如果你在描述文件管理器中要求最小密码长度为四个字符，但 ActiveSync 设置为要求至少六个字符，那么在 Exchange 数据开始同步到设备之前，设备将提示用户设置一个新的六位数密码。

无论你使用的是**自带设备**（BYOD），还是在描述文件管理器中管理公司拥有的设备，你都可以看到使用描述文件来应用特定设置（如 Microsoft Exchange）比手动配置每台单独的设备要好得多。

## 将用户从 Windows 迁移到 macOS

如果 Don 之前使用 OneDrive 存储所有数据，并且在他的 Windows PC 上没有使用 `*.pst` 文件，那么可能没有太多需要迁移的内容。然而，许多用户可能仍在其旧 PC 上存有需要转移到 Mac 的数据。为了完成 Don President 从 PC 到 Mac 迁移的最后一个步骤，我们需要将任何剩余的数据迁移到他的 Mac。

幸运的是，苹果提供了一个相当有用的迁移工具来协助此过程。首先，确保他的新 Mac 和旧 PC 连接到同一网络，然后在 Don 的 Mac 上浏览到 `/Applications/Utilities/` 文件夹，并打开 **迁移助手** 实用程序。

### 使用迁移助手

启动迁移助手后，系统会警告你将关闭所有打开的应用程序，点击 **继续** 以开始。系统会提示输入本地管理员凭据以继续。认证成功后，包括 Finder 在内的所有应用程序都将关闭，迁移助手将开始运行。当提示选择要从中迁移的计算机时，应选择 **从 Windows PC**，如图 11-37 所示。

![../images/492151_1_En_11_Chapter/492151_1_En_11_Fig37_HTML.jpg](img/492151_1_En_11_Fig37_HTML.jpg)
图 11-37 迁移助手使用“从 Windows PC”选项

在 Don 的旧 Windows PC 上，打开浏览器并访问 [`www.apple.com/migrate-to-mac`](http://www.apple.com/migrate-to-mac)，下载 **Windows 迁移助手**。当出现提示时，继续 **运行** 下载的 `setup.exe` 文件，按照安装向导的步骤完成安装以及所需的任何附加组件。安装完成后，从“开始”菜单或 Windows 桌面打开 **Windows 迁移助手** 应用程序。按照提示操作，直到你看到 PC 的名称出现在 Mac 上，如图 11-38 所示。

当提示选择各种文件和用户帐户时，**选择相关信息**，如图 11-39 所示。选择要迁移的信息后，点击 **继续** 按钮。

## 总结

在本章中，我们学习了如何将 Apple 解决方案集成到由 Microsoft 解决方案构成的企业网络中。我们将一位高级用户从 Windows PC 迁移到 `MacBook Pro`，同时让他们能够继续使用他们习惯的打印机、文件共享和应用程序，而不会损失任何生产力。运用本章中学到的相同工具和策略，您应该能够轻松地将 Mac、iPhone 或 iPad 集成到您现有的 Microsoft 环境中，只需付出最少的努力或进行广泛的基础设施变更。
