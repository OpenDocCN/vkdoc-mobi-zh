# 7. macOS Server

自 1990 年代末该软件的第一个版本问世以来，Mac 服务器的概念已经发生了巨大变化。在本章中，我们将探讨 Apple 在`macOS`客户端操作系统中内置的服务器功能、安装`macOS Server`应用程序时添加的服务、如何确定您是否需要一台专用的 Mac 服务器，以及如果需要，如何配置和管理该服务器。

## macOS Server 简史

最初的 Mac OS Server 代号为"Rhapsody"，它最初被设计为苹果公司在 1996 年收购 NeXT 后开始开发的新一代 Mac 操作系统的第一个版本。`Mac OS X`是最终在 2000 年代初发布的客户端操作系统，与此同时，苹果开始销售`OS X Server`作为一个在客户端操作系统之上运行的应用程序，为其提供服务器级功能。多年来，`Server`应用程序的功能已被缩减或迁移到`macOS`的基本功能集中。如今，只有少数服务仍然专属于`macOS Server`。

### 服务

最初亮相时，`OS X Server`提供了许多用于管理 Mac 客户端的有用功能。苹果包括了其自己的轻量级目录访问协议(LDAP)，称为`Open Directory (OD)`，旨在管理用户和组，类似于微软（更流行）的`Active Directory (AD)`。除了 LDAP，它还包括文件共享、打印机共享和基于`Apache`的 Web 服务器。作为 Mac 系统管理员的热门解决方案，OS X Server 还包括用于管理 Mac 机群的`NetBoot`和`Workgroup Manager`。

除了各种管理工具外，`OS X Server`还提供`CalDAV`、即时消息、邮件服务器和许多其他与生产力相关的服务。然而，随着更多此类解决方案开始迁移到各种云托管模型，苹果将其中的大多数转移到了 iCloud，或者决定完全停止这些服务。

随着苹果开始更多地转向针对 iOS 设备（以及后来的 Mac）采用移动优先的管理原则，他们开始将`macOS Server`转变为移动设备管理工具。许多核心"服务器"服务被移入了`macOS`的基础安装中。如今，`macOS Server`保留了`Open Directory` LDAP 服务，并且`Workgroup Manager`已被弃用，取而代之的是`Profile Manager`，它允许您为`iOS`和`macOS`客户端创建和部署配置描述文件。



## 硬件

苹果服务器硬件多年来也发生了巨大变化。最初运行`OS X Server`的硬件包括`Power Mac G3`和`Power Mac G4`工作站。苹果曾一度非常重视专用服务器硬件，并发布了`Xserve`和`Xserve RAID`产品，但最终还是停产了这些型号，转而支持`Mac Pro`工作站。随着对专用服务器操作系统需求的减弱，苹果继续开发和制造专用服务器硬件已不再具有意义。

## macOS Server 简介

在本章中，我们将探讨服务器操作系统最常见的功能，以及如何配置`macOS`或`macOS Server`来实现这些目的。其中一些功能在客户端操作系统的标准安装中即可用，而其他功能则需要您购买并安装`macOS Server`应用程序后才能使用。

### 我需要一台 Mac 服务器吗？

在我们开始构建 Mac 服务器之前，首先考虑使用场景以及是否真的需要一台专用的 Mac 服务器。随着苹果移除了 Mac 服务器提供的许多服务，他们也努力在`macOS`中构建了更多跨平台工具，使其能够作为客户端完全融入其他常见网络，特别是基于 Microsoft 和 Linux 技术构建的网络。很有可能，您的组织已经拥有某种 LDAP、打印机共享和文件共享解决方案，您可以非常轻松地在`macOS`客户端上使用它们。

如果您是一位经验丰富的 Mac 系统管理员，您可能熟悉`黄金三角`这个术语。这指的是您使用运行`Open Directory`的 Mac 服务器与运行 Microsoft `Active Directory`的`Windows`服务器同步，以管理用户账户，然后使用`Workgroup Manager`来设置 Mac 特定的权限或设置。如今，您可以直接将 Mac 绑定到 Active Directory 域，而无需再专门为您的`macOS`端点运行第二个 LDAP。用于限制设备和锁定特定设置的`配置描述文件`取代了`Workgroup Manager`组件。

文件共享在最近的版本中也发生了变化。苹果过去使用他们的`AppleTalk 文件协议`作为`macOS`客户端的首选文件共享技术。如今，`Samba`是默认的文件共享协议，可以与`Windows`和 Linux 客户端无缝协作。这里的要点是，如果您已经在运行各种基于`Windows`或 Linux 的服务器，那么在开始构建 Mac 服务器来创建这些现有文件共享的冗余`仅限 Mac`版本之前，请考虑如何利用这些现有技术与您的`macOS`客户端配合使用。

除了目录服务或文件/打印共享之外，还有许多基于云的解决方案可能更适合您的组织。网站托管是立刻想到的一个例子。几年前，使用 Mac 服务器托管网站非常普遍，而且在许多方面，它是比使用 Microsoft 的`IIS`更优的解决方案。虽然`Apache`在`macOS`中仍然可以作为附加服务使用，但您同样可以轻松地使用众多低成本的云托管解决方案来托管您的网站。

以下是如果您选择部署一台 Mac 服务器，可能希望在上面运行的一些更常见的服务。我们将在本章中详细介绍其中的许多服务：

*   **文件共享：** 从`macOS High Sierra`开始，苹果已将`文件共享服务器`集成到操作系统的标准安装中。您可以通过`共享系统偏好设置`启用此服务。此服务允许您通过网络共享文件和文件夹。
*   **打印机共享：** 从`macOS High Sierra`开始，苹果已将`打印机共享服务器`集成到操作系统的标准安装中。您可以通过`共享系统偏好设置`启用此服务。此服务允许您通过网络共享打印机。
*   **Apple Remote Desktop：** 您可以以完整控制台模式安装 Apple Remote Desktop，或按照本书第 6 章所述将其作为`远程任务服务器`安装。
*   **内容缓存：** 从`macOS High Sierra`开始，苹果已将`内容缓存服务器`集成到操作系统的标准安装中。您可以通过`共享系统偏好设置`启用此服务。此服务会缓存来自`App Store`和其他苹果托管服务（如`软件更新`）的内容副本，以便在多个设备上安装更新或内容时节省带宽。
*   **Web 服务器：** `Apache`已集成到 Mac 操作系统的标准安装中。您可以通过命令行启用它。
*   **Open Directory：** 包含在`macOS Server`应用程序的安装中，`Open Directory`是一个 LDAP，允许您配置基于网络的用户和组。
*   **描述文件管理器：** 一个基本的移动设备管理解决方案，允许为`iOS`和`macOS`客户端创建和分配`配置描述文件`——包含在`macOS Server`应用程序中。
*   **Xsan：** 包含在`macOS Server`应用程序的安装中，用于创建和管理存储区域网络（SAN）解决方案。
*   **DNS：** 从`macOS High Sierra`开始，苹果已从`macOS Server`的安装中移除了 DNS。您可以使用其他各种解决方案，包括`BIND`或`Knot DNS`。
*   **DHCP：** 从`macOS High Sierra`开始，苹果已将`DHCP 服务器`集成到操作系统的标准安装中。您可以通过命令行启用和管理此服务。
*   **NetBoot/NetInstall：** 苹果通过使用 T2 安全芯片以及在`macOS Server`的较新版本中移除这些功能，继续降低了此服务的地位。核心技术在`Mojave`之前的`macOS`版本中仍然可用。如果您计划使用此工作流，我建议您将`DeployStudio`作为一个解决方案来考察（`www.deploystudio.com`）。

那么，您需要一台 Mac 服务器吗？要回答这个问题，视情况而定，但很可能不需要。如果您是一家小型企业，没有任何现有的目录服务，也不需要本地文件/打印服务器，那么部署一台专用的 Mac 服务器可能是有意义的。

如果您是一家拥有大量现有服务器解决方案的大型组织，请尝试将您的 Mac 整合到该生态系统中，而不是构建一个独立的仅限 Mac 的环境。对您来说更容易，对您的最终用户也更好，能够在现有的企业网络中作为一等公民参与。越来越普遍的情况是，想要一台 Mac 服务器的主要原因是运行`描述文件管理器`，除非您的组织已经拥有来自第三方供应商（如`JAMF`、`Mosyle`或`Addigy`）的有效移动设备管理解决方案。

## 部署 Mac 服务器

如果您确定在您的环境中专用 Mac 服务器会很有用，我们就可以开始配置一台 Mac 来充当服务器。



### 硬件考量

根据你计划如何使用 Mac 服务器的不同，你需要为这项工作选择合适的硬件。这通常涉及在 `Mac Pro` 和 `Mac mini` 之间做出选择。在撰写本文时，苹果公司售价低至 800 美元的 `Mac mini` 具备足够的原始性能，能够胜任大多数服务器角色。但其内置存储相对有限，仅为 128 GB SSD 空间。因此，如果你计划用它进行大量的文件共享，则应该考虑购买某种外置雷雳存储解决方案。

为了获得最高级别的性能，新款 2019 年 `Mac Pro` 可以配置惊人的内存容量、内部磁盘空间以及多达 28 个处理器核心！它的价格也可能高达一辆新车的价格，因此请确保你的需求与正确的规格相匹配。大多数组织可以使用某种型号的 `Mac mini` 满足需求。即使你选择定制配置，并将内部磁盘升级到 2 TB SSD，其成本也仅仅是 Mac Pro 的一小部分。

还有一点你需要考虑，特别是如果你计划将 Mac 主要用作 `描述文件管理器` 服务器时，那就是云托管模型。市场上已有几家供应商提供云端 Mac 服务器。最受欢迎的是 `MacStadium`，你可以按月支付较低的费用来访问一台裸机 Mac 服务器。你可以在 [`www.macstadium.com`](http://www.macstadium.com) 探索这些选项。

### 备份

正如我们在第 2 章和第 3 章中所讨论的，你可以选择基于云的备份解决方案，也可以使用 `时间机器` 将你的 Mac 服务器备份到连接的外置磁盘。为你的服务器选择备份解决方案时，唯一的注意事项是它需要备份整个磁盘以及所有连接的存储设备。你应该备份所有用户数据、配置以及运行在 Mac 服务器上的任何应用程序。因此，应避免使用像 `iCloud`、`OneDrive` 或 `Dropbox` 这样的解决方案，因为它们仅同步用户的个人目录，而非整个磁盘。

**专业提示**

如果你打算使用 `时间机器` 驱动器或其他类型的连接存储来备份服务器，请务必制定某种规程，将这些驱动器轮换到安全的异地存储设施中。许多组织都设有数据恢复站点或使用第三方服务来异地存储备份驱动器，以防发生火灾或其他自然灾害。你也应该定期对 `时间机器` 驱动器执行相同的操作。

### 初始配置

至此，你应该已经为你的 Mac 服务器、存储和备份选择了合适的解决方案。接下来我们需要做的是在服务器上安装一个全新的 `macOS Catalina` 副本。因此，请启动进入 `互联网恢复` 模式或使用可启动的 USB 安装程序来启动你的 Mac，使用 `磁盘工具` 对驱动器进行分区和/或格式化，然后安装 `macOS`。

操作系统安装完成后，请按步骤完成 `设置助理`。在电脑上创建一个 `本地管理员` 账户并登录该账户。进入本地管理员用户的桌面后，将墙纸更改为能表明这是服务器的样式。我喜欢使用某种灰色调，不过有些系统管理员喜欢创建自定义的服务器墙纸。你的服务器看起来应该类似于图 7-1 中的样子。

#### 网络设置

我们首先要做的是为服务器配置名称和静态 IP 地址。使用`共享系统偏好设置`来命名您的服务器。我将我的服务器命名为`MacServer`。如果您有多台现有服务器，可能会有标准化的服务器命名规则可以使用。此配置假设您只是在内部局域网中共享一些数据。

**专业提示**

请仔细检查本地主机名和 Unix 主机名，确保它们一致。如图 7-2 所示，我已经将我的三个服务器名称配置为相匹配。

#### 服务器安全考量

在配置服务器时，确实需要尝试禁用所有不会用到的服务。通过这种方式，你可以限制可能存在的、进而可能被利用的漏洞数量。这并不是一份用于确保服务器安全而必须锁定的详尽清单，但它为你提供了一个起点。没有任何东西能 100% 免受黑客或其他安全威胁。

以下是部署 Mac 服务器时建议考虑的一些更改：

*   **禁用访客用户：** 打开`用户与群组系统偏好设置`，在左侧窗格的用户列表中点击`访客用户`。确保`允许访客登录这台电脑`和`允许访客用户连接共享文件夹`均未勾选。
*   **禁用登录屏幕上的电源控制：** 在`用户与群组系统偏好设置`中，点击`登录选项`按钮。取消勾选`显示睡眠、重新启动和关机按钮`旁边的复选框。
*   **禁用自动登录：** 在`登录选项`中，确保`自动登录`设置为`关闭`。
*   **禁用用户列表：** 在`登录选项`中，选择将登录窗口显示为`名称和密码`。
*   **仅启用软件更新以获取安全补丁：** 打开`软件更新系统偏好设置`并点击`高级…`按钮。是否要自动安装所有更新由您决定，但至少确保勾选`安装系统数据文件和安全更新`框。这样，你的 Mac 服务器将自动获得重要的安全更新补丁。
*   **启用屏幕保护程序密码：** 打开`安全性与隐私系统偏好设置`。点击`通用`标签页，并勾选`睡眠或屏幕保护程序开始后立即要求输入密码`旁边的复选框。将下拉菜单设置为`立即`。
*   **启用 FileVault：** 点击`FileVault`标签页并启用`FileVault`加密。
*   **启用防火墙：** 点击`防火墙`标签页。打开防火墙，然后按照本书第 3 章详述的防火墙选项来配置所需的安全级别和任何应用程序例外。
*   **禁用定位服务：** 点击`隐私`标签页，然后点击左侧窗格中的`定位服务`。取消勾选`启用定位服务`旁边的复选框。
*   **禁用分析：** 当仍处于`隐私`标签页时，点击左侧窗格中的`分析`按钮。取消勾选`共享 Mac 分析`和`与 App 开发者共享`。
*   **禁用蓝牙：** 打开`蓝牙系统偏好设置`。点击`关闭蓝牙`按钮以禁用它。
*   **关闭 Wi-Fi：** 假设你使用以太网将服务器连接到网络，我们应该关闭 Wi-Fi 无线网络。打开`网络系统偏好设置`，点击左侧窗格中的`Wi-Fi`按钮。点击`关闭 Wi-Fi`按钮以禁用它。
*   **退出 iCloud：** 如果在设置助理过程中登录了 iCloud，请前往`iCloud 系统偏好设置`并退出 iCloud。

## 配置基本服务器服务

现在我们的 Mac 服务器已准备就绪，可以开始启用一些服务了。在本节中，我们将探讨随核心`macOS`客户端操作系统捆绑提供的服务器级功能。这些是 Mac 服务器上最常见的服务，包括内容缓存、文件服务、打印服务和 Web 服务。

### 内容缓存

`内容缓存`是之前包含在`macOS Server`中，但随后被移到核心`macOS`安装中的最受欢迎的功能之一。此服务在你网络上的设备上运行，监听从 Apple 或 Apple 托管服务（如`iBooks`、`App Store`、`Mac App Store`等）下载内容的客户端。然后，当第一个客户端下载内容时，运行内容缓存服务的计算机会下载一份副本并保留它。当同一子网上的第二个、第三个或任何其他设备尝试下载该内容时，它会从缓存服务器复制，而不是再次通过互联网下载。

这种解决方案在拥有多个共享互联网连接的客户端的企业和学校中极为流行。我曾将其用于 1:1 iPad 部署和需要下载并安装相同安全更新或`macOS`版本的 Mac 实验室，它确实加快了安装速度并节省了带宽使用。

要启用内容缓存服务，请打开`共享系统偏好设置`，并勾选`内容缓存`旁边的复选框，如图 7-8 所示。你可以选择要缓存哪些内容，我通常坚持使用默认的`所有内容`选项。如果你只希望通过 USB 线物理连接到 Apple Configurator 的 iOS 设备使用内容缓存，请勾选`互联网连接`旁边的框。

### 文件共享

Mac 服务器上运行的另一个最常见的服务是文件共享。要启用本地网络文件共享，请打开`共享系统偏好设置`并勾选`文件共享`旁边的复选框。默认情况下，`macOS`使用 Samba 在网络共享文件。如果出于某些原因你想使用旧的 AppleTalk 协议，可以点击`选项…`按钮并勾选`使用 AFP 共享文件和文件夹`旁边的框。我们不勾选此项，因为我们不想使用 AppleTalk。

## 证书

当您完成将 Mac 客户端绑定到 Open Directory 的步骤时，您可能收到过一条消息，提示 `此服务器不提供安全（SSL）连接。是否继续？` 在这种情况下，由于我们是在设置测试服务器，因此选择忽略此消息并继续。但是，在生产环境中，您需要使用 SSL 来保护客户端与服务器之间的通信，以防止中间人攻击等。即使您不使用 Open Directory，并且仅部署服务器以配合 Profile Manager 使用，您也需要对证书以及在 macOS Server 中更改它们的方式有基本的了解。

### 回退 SSL 证书

Apple 提供的最基本的证书是 `回退证书`。它在您安装 Server 应用程序时创建，虽然可用但很少使用。您可以在 Mac 服务器上打开 `钥匙串访问` 实用工具，然后点击 `系统` 钥匙串下的 `证书` 来找到此证书。

### 自签名证书

就 Open Directory 而言，Apple 提供了一个自签名证书，您可以在安装 Open Directory 服务时使用。自签名证书会提示用户信任它们，通常不是生产安装的最佳选择。如果我们希望用受信任的第三方证书替换它，可以调整用于 Open Directory 的证书。

### 管理证书

我们可以随时更改用于任何服务的证书。在对这些证书执行任何操作之前，我们需要停止使用它们的服务。在本例中，我们需要暂时关闭 Open Directory 服务，方法是点击 Open Directory 窗口右上角的开/关切换开关，如图 7-40 所示。

## 描述文件管理器

我们将在 `macOS Server` 中配置的最后一项服务，实际上也是如今拥有一台 Mac 服务器的主要原因——`描述文件管理器`。`描述文件管理器` 是一个移动设备管理（MDM）解决方案，它与 Apple 的云服务（如我们在本书中已提及的 `设备注册计划（DEP）` 和 `批量购买计划（VPP）`）集成。我们将在第 8 章和第 9 章中使用 `描述文件管理器` 来创建配置描述文件，以 *无线* 方式供应和管理 iOS 设备及 Mac 客户端。

在我们开始之前，使用 `描述文件管理器` 时需要注意几个前提条件。首先，服务器必须能从互联网访问。对于我的测试服务器，我临时启用了 `Apache` 并确保我能从网络内外都访问到 "It works!" 页面。我还创建了一个域名，并确保我可以通过 DNS 名称（而不仅仅是 IP 地址）在公共互联网上访问到服务器。测试完成后，您可以再次禁用 `Apache` 服务。其次，一旦您拥有一个可用的 Web 服务器，它还需要启用 SSL 加密。因此，您需要从您选择的证书颁发机构获取一个签名的证书。

### 专业提示

我曾使用 [namecheap.​com](http://namecheap.com/) 或 [godaddy.​com](http://godaddy.com/) 作为我的 SSL 证书供应商，费用不高。还有一些替代方案，例如 [letsencrypt.​org](https://letsencrypt.org/)，您可以用它来获取免费的签名 SSL 证书。

我们将开始配置 `描述文件管理器` 服务，点击侧边栏中的 `描述文件管理器` 按钮，然后点击 `开/关` 开关将其 **开启**，如图 7-46 所示。

## 总结

在本章中，我们讨论了专用 Mac 服务器的必要性、内置在核心`macOS`客户端操作系统中的服务器级解决方案，以及通过安装`macOS Server`应用程序提供的进阶服务。虽然每个组织的需求会有所不同，但很明显，只要可能，Apple 都提倡使用现有的行业标准目录和文件共享以及打印共享技术，而不是专有的仅限 Mac 的解决方案。

正如您所看到的，`macOS Server`主要已成为通过配置文件管理器工具启用移动设备管理（MDM）的工具。可以说，除非您想使用配置文件管理器，否则您可能根本不需要 Mac 服务器。如果您确实需要配置文件管理器，它可能是您需要在服务器上运行的唯一服务，并且它不依赖于 Open Directory 或任何其他服务。

在下一章中，我们将探讨 MDM 概念，包括`User Approved MDM`、`Device Enrollment Program`和`Volume Purchase Program`。这些解决方案构成了为 Apple 品牌设备开发下一代部署和支持模式的基础。
