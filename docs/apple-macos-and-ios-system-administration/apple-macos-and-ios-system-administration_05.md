# 使用 Apple 远程桌面管理 macOS 客户端

Apple 远程桌面 (ARD) 是一个基本的客户端管理工具，Mac 系统管理员已经使用了超过 15 年。在本章中，我们将回顾 ARD 的四个主要组件——资产管理、软件分发、客户端管理和远程支持。如果您需要管理超过几个 `macOS` 客户端，ARD 是必备工具。



## Apple Remote Desktop 简介

当前版本的 Apple Remote Desktop 可在 `Mac App Store` 获取，版本为 3.9，价格为 79.99 美元，如图 6-1 所示。虽然它比 `Apple Configurator` 贵得多，但就商业桌面管理套件而言，它的价格相当实惠。如果你的组织还没有针对 `macOS` 客户端的**管理解决方案**，那么使用 `ARD` 是一个不错的选择。

### 安装 Apple Remote Desktop

在规划使用 Apple Remote Desktop 时，你需要确定将其安装在何处。对此有几种不同的看法。如果你是组织中唯一的 Mac 技术员，你可以将其安装在你主要的 Mac 台式机或笔记本电脑上。就远程控制任务的响应速度而言，这将为你提供最佳体验。你可以将所有脚本和安装文件存储在主机上，并且可以随主机数据一起备份整个 ARD 系统。

这种方案的缺点是，当你需要在稍后的时间或日期安排安装时，特别是当你的计算机可能未在线或不可用时。例如，假设你想在今晚凌晨 3 点在公司计算机上安排软件安装，但你需要将笔记本电脑带回家，届时它将不在公司网络上，无法推送安装文件和运行任务。在这种情况下，你可能希望设置一个 `Remote Task Server` 或将 `ARD` 直接安装在专用的 Mac 服务器上，然后从你的工作站使用 `Screen Sharing` 或 `ARD` 访问该服务器。

### 远程任务服务器

在第 7 章中，我们将更详细地介绍 `macOS Server`，但目前你需要知道的是，与 `Windows` 有特定的服务器版和客户端版不同，任何 Mac 都可以作为服务器。`ARD` 的 `Remote Task Server` 指的是一台计算机，它按照指定计划对各类任务（如软件安装）进行排队或执行。默认情况下，Apple Remote Desktop 会将安装有 `ARD` 应用程序的本地计算机设置为 `Task Server`。

如安装部分所述，将 Apple Remote Desktop 安装在你主要的工作站上存在一些明显的缺点。其中之一是，当待处理的任务计划正在进行时，你需要保持 Mac 在线。你可以使用 `Remote Task Server` 来缓解这个问题。通过指定一个 `Remote Task Server`，你可以在工作站上使用 `ARD`，但所有待处理计划任务都将在远程服务器上排队。

要配置远程任务服务器，你需要在一台配备 `静态 IP 地址` 的 Mac 上安装 Apple Remote Desktop。打开 `ARD`，从 `Remote Desktop` 菜单中选择 `Preferences…`。图 6-2 显示了 `Task Server` 选项卡和 `Use Task Server on this computer` 选项，并且 `Allow remote connections to this server` 的复选框已被勾选。

### 服务器安装

我首选的方案是在一台配备 `静态 IP 地址` 的 Mac 服务器上安装 `ARD`。这允许我团队中的任何成员访问 `ARD` 控制台并共享所有安装文件或脚本。安装过程与在单台工作站上安装非常相似，只是需要对 `ARD Preferences` 进行一项更改。

默认情况下，Apple 设置了一个安全选项，即无法使用 Apple Remote Desktop 远程控制另一台运行 Apple Remote Desktop 的 Mac。要禁用此功能，请从 `Remote Desktop` 菜单中选择 `Preferences…`，然后点击 `Security` 选项卡，如图 6-4 所示。勾选 `Allow control of this computer when this application is running` 选项旁边的复选框。

## 在 ARD 控制台中组织客户端

控制台中只有一台 Mac 时，仅使用 `所有计算机` 分组可能没问题。但是，你可能在 ARD 控制台中有 `数十台` 甚至 `数百台` Mac，你会希望它们有一定的组织方式，以便轻松找到并有效管理它们。

让我们用一些 `组` 和 `列表` 来进行组织。使用 `添加组/列表 + 按钮`，创建两个 `组`，一个名为 `开发`，另一个名为 `生产`。然后创建两个 `列表`，一个叫 `测试 Mac`，另一个叫 `服务器`。使用 `拖放` 操作，将 `服务器` 列表移动到 `生产` 文件夹中，并将 `测试 Mac` 列表移动到 `开发` 文件夹中。最后，将测试 Mac `拖放` 到 `测试 Mac 列表` 中，如图 6-12 所示，我操作的是 `dwsmith’s MacBook Pro`。完成后，你的控制台应该与我的类似，包含计算机、列表和组。

## Mac 远程支持

顾名思义，Apple 远程桌面提供了一套强大的工具集来远程控制 Mac。远程控制 Mac 客户端时主要有三种选项：`观察` 模式、`幕帘` 模式和 `控制` 模式。你可以在主 ARD 控制台窗口中启动这三种选项。选择一台计算机，然后点击工具栏上的 `观察`、`控制` 或 `幕帘` 按钮，即可打开远程控制窗口，如图 6-14 所示。

你可以 `截取` 远程屏幕的 `屏幕截图`，并将其作为文件保存到你的计算机上，或者在远程计算机和运行 ARD 控制台的工作站之间 `复制和粘贴`。最后，如果高分辨率图像有延迟，你可以在网络连接较慢时，使用窗口右上角的 `滑块` 调整图像质量以提高性能。



### 观察模式

观察模式是最简单的远程模式。你可以在不与被控计算机进行任何交互的情况下查看一个或多个系统。这在您与最终用户交谈并希望观察他们的 Mac，让他们展示遇到的问题以便您更好地进行故障排除时非常有用。这个功能也常用于教室或计算机实验室，实验室管理员或讲师希望在作业或测试期间关注课堂上学生使用计算机的情况。

### 控制模式

默认情况下，使用 ARD 控制台选择远程控制另一台 Mac 时，你将处于*共享*控制模式。在此模式下，你的最终用户可以继续控制他们的系统，而你作为远程管理员与他们共享控制权。这可能会导致鼠标和键盘的“争夺”，因此，如果你需要完全控制系统并锁定用户，使其无法控制自己的系统，可以点击另一个控制按钮切换到*绝对*控制模式，如图[6-15]所示。

### 幕帘模式

幕帘模式是第三种远程控制形式，类似于绝对控制和共享控制模式，作为远程管理员，你可以控制用户的桌面。然而，幕帘模式的主要区别在于它会遮蔽远程屏幕，使最终用户看不到你的操作。这在需要使用任何安全程序解决问题，且不想向用户披露具体操作时非常有用。例如，映射一个安全的网络驱动器并浏览你不想让用户看到的敏感数据。

**专业提示**

当使用**幕帘**模式时，ARD 会提示你在接管用户屏幕时输入一些内容到对话框中，该对话框会显示给最终用户。这有助于告知用户正在他们的系统上进行工作，并显示远程任务何时完成。图[6-16]展示了最终用户在此过程中看到的示例。

### 聊天

除了远程控制，你还可以使用 ARD 内置的即时消息功能远程与用户聊天。你可以选择想要与之交互的用户所在的计算机，然后从**交互**菜单中选择**聊天**来开始聊天会话。这将允许你在帮助用户排查问题时与他们实时来回交谈。

### 锁定屏幕

**交互**菜单下的**锁定**和**解锁屏幕**功能在学校和公共实验室环境中非常有用，在这些环境中可能会发现用户违反了学校的可接受使用策略。例如，如果一名学生做了违反政策的事情并且被远程观察到，你可以锁定他们的屏幕，从而在需要向学校管理层或有关部门报告违规行为时，能够“当场抓获”。屏幕被锁定时，看起来与*幕帘*模式非常相似，除了关闭系统电源外，用户无法进行任何操作。

### 软件分发

Apple Remote Desktop 最后，也可能是最有用的一个功能，是作为软件分发的工具。软件分发的目的是为了简化在整个`macOS`客户端群中安装和升级软件的过程。是的，你可以使用 USB 驱动器或网络共享，然后逐台 Mac 安装软件，但这确实非常耗时且容易出错。Apple Remote Desktop 提供了几个功能，用于通过网络向所有`Macintosh`端点批量安装应用程序。

### 专业提示

为什么我们要将安装程序复制到`/tmp`？`tmp`文件夹是复制这些文件的好地方，这样我们就不必担心事后清理。我们可以在脚本末尾使用`rm`命令来删除安装程序，但我们也可以让操作系统在一两次重启后替我们清理。
