# 2. macOS 客户端管理

在本章中，我们将探讨单台 Mac 的管理。主题将包括管理本地用户账户、探索 `macOS` 的文件结构、学习命令行界面以及执行一些基本的硬件故障排除。

## Mac 硬件故障排除

与大多数 PC 硬件制造商不同，苹果公司对于允许谁修理其硬件一直相当严格。除了通过自己的 Apple 零售店提供硬件维修服务外，他们还为希望进行维修的第三方公司制定了全面的授权维修中心流程。话虽如此，大多数技术人员在不使保修失效的情况下，自己能安全进行的硬件维修是有限的。作为一名 Mac 技术员，在预约 `Genius Bar` 并带着 Mac 前往当地 Apple 零售店之前，了解一些基本的硬件故障排除技巧非常重要。

### 非易失性随机存取存储器

`非易失性随机存取存储器`，或有时被长期 Mac 用户称为 `参数 RAM`，基本上是逻辑板上一小部分内存，用于存储计算机关机时不会擦除的信息。它存储音量级别、时区、显示器分辨率和启动磁盘的设置。虽然这种情况不常发生，但有时该内存可能会损坏；通过使用图 2-1 中找到的键盘组合键*清除它*，可以将这些设置重置为出厂默认值。这通常是故障排除一台行为异常的 Mac 的第一步。它通常可以解决与时区相关的问题、开机后需要很长时间才显示苹果标志的慢启动问题，或与音频或内置扬声器相关的问题。

专业提示

如果您正在使用 `可扩展固件接口密码`，如第 3 章所述，需要注意的是，在您移除该密码之前，您将无法清除 `NVRAM`。清除 `NVRAM` 完成后，别忘了重新启用您的密码。



### 系统管理控制器 (SMC)

`系统管理控制器` (SMC) 是 Macintosh Intel 硬件架构的一个组件。这是 Mac 硬件独有的，除了强制执行 `macOS` 最终用户许可协议（这是您无法轻易在非苹果公司生产的基于 Intel 的个人电脑上运行 `macOS` 的原因之一），它还是一个管理众多集成硬件组件的“万能”部件。苹果在其支持网站上发布了一份详尽的列表，但总的来说，SMC 在不同程度上参与了从开启/关闭风扇到逻辑板上 LED 灯的功能、声音、蓝牙、睡眠/唤醒等多种事项。

在排查 Mac 硬件问题的过程中，您可能需要重置 SMC。苹果建议除非您已用尽所有其他选项（包括标准重启和清除 `NVRAM`），否则请等待后再进行此操作。重置 SMC 最常见的一些原因包括：即使在处理器负载较轻时 Mac 的风扇也持续运转，或者您的 MacBook 电池无法充电。它并不能解决所有问题，但在将 Mac 送到维修中心进行可能昂贵的诊断之前，这绝对值得一试。

一般来说，重置 Mac 上 SMC 的过程取决于您拥有的具体型号。台式机与笔记本电脑、以及带可拆卸电池的笔记本电脑与不带可拆卸电池的笔记本电脑，其重置步骤各不相同。最近，随着 `Apple T2 安全芯片` 的出现，重置配备了该协处理器的 Mac 的 SMC 又有另一种不同的方法。建议您参考苹果的支持页面，获取有关如何重置您正在排查的特定 Mac 的 SMC 的说明。

苹果关于 SMC 的知识库文章可以在这里找到：
*   [`https://support.apple.com/en-us/HT201295`](https://support.apple.com/en-us/HT201295)

### 启动前键盘组合键

在排查 Mac 故障时，您应该熟悉许多键盘组合键。图 2-1 展示了各种启动前键盘快捷键，其中许多可用于选择除内置硬盘之外的其他启动磁盘。您可以启动进入 `安全模式`、在启动时启用详细日志记录、进入 `恢复模式`，甚至可以使用 `目标磁盘模式` 将您的 Mac 变成一块巨大的外部 Thunderbolt 硬盘。

专业提示
`目标磁盘模式` 可以是排查无法从内置硬盘启动的 Mac 问题的有效方法。通过使用目标磁盘模式，您可以尝试将内部驱动器挂载到另一台 Mac 上，使其看起来像通过 Thunderbolt 或 Firewire 连接的外部硬盘驱动器。如果驱动器拒绝挂载，这可能表明存在硬件故障。如果驱动器确实挂载了，但 Mac 仍无法从其启动，您或许可以在擦除驱动器并安装全新的 macOS 副本之前，检索数据或创建驱动器的备份。

## macOS 恢复

在 `macOS` 的标准安装中，苹果提供了一个隐藏分区，其中包含操作系统的有限副本和一些应用程序，如 `磁盘工具`、`终端` 和 `macOS 安装器` 包。这个区域实际上被称为 `macOS 恢复`，正如您所猜测的那样，它在操作系统出现问题时用于尝试恢复您的 Mac。

如果由于某种原因，您的 Mac 没有本地恢复分区，您也可以利用 `互联网恢复` 服务。互联网恢复是苹果托管的基于云的可启动网络卷。从互联网恢复启动需要有效的互联网连接，并且启动时间可能较长，具体取决于您的连接速度。

一旦您启动进入恢复模式，可以执行以下几项操作：
*   从时间机器备份恢复。
*   重新安装或全新安装 `macOS`。
*   通过 Safari 浏览苹果支持网站（如果您有活动的互联网连接）。
*   使用磁盘工具执行磁盘急救或其他与驱动器相关的任务。
*   访问终端。
*   管理硬件安全设置（第 3 章将详细介绍）。

### 磁盘工具

虽然其中大多数选项不言自明，但 `磁盘工具` 包含许多在 `macOS 恢复` 模式下特别有用的工具。因为我们是从恢复分区启动，而不是从包含完整 macOS 系统文件夹的分区（通常称为 `启动磁盘`）启动，所以我们可以对启动卷运行实用程序和命令。虽然当您从 Mac 的启动磁盘启动后，可以在 `/Applications/Utilities` 内的磁盘工具应用程序中访问许多相同的选项，但您可能在系统当前运行的同一卷上执行它们时遇到问题。这使得 `macOS 恢复`，特别是互联网恢复，成为直接处理 Mac 内置硬盘的最佳选择。图 2-2 显示了磁盘工具的默认视图，并标出了一些重要选项。

专业提示
在 `macOS 恢复` 模式下使用磁盘工具时要小心。很容易意外删除您的启动分区甚至整个内置硬盘。在对 Mac 上的磁盘工具进行操作之前，请确保已备份所有重要数据。

*   **容器面板：** 卷、驱动器、分区、容器和已挂载磁盘的列表。

### 时光机器备份

作为系统管理员，您应该部署某种备份机制，以便在发生灾难性系统故障时存储关键数据的副本。时光机器是在安全数据中心备份个人 Mac 或 Mac 服务器的一种简单而有效的方式。在办公室或教室环境中，这可能不是最佳解决方案，因为外置驱动器很容易丢失。虽然时光机器驱动器可以加密，但从成本角度考虑，为每个用户配备外置驱动器可能并不可行。在这些情况下，请考虑使用基于云的文件备份解决方案，例如 iCloud Drive 或 Microsoft OneDrive for Business。

> **专业提示**
> 如果您使用时光机器备份 Mac，并且其格式为 HFS+，请不要让 macOS 将其转换为 APFS，因为在此过程中可能会损坏现有的备份数据。如果您计划更换为 APFS 格式的磁盘并归档 HFS+ 格式的驱动器，最好从一个新格式化的驱动器开始。

## 文件系统与 macOS

每个操作系统都有其偏好的文件系统。如果您有 Microsoft Windows 系统管理背景，您一定听说过 NTFS 或 FAT32，但您可能只知道苹果使用的文件系统称为 Mac 格式或苹果格式。实际上，Mac 有几种常见的文件系统。分层文件系统（HFS）及其较新的版本 HFS+（对旧 HFS 格式的增强）现在已是苹果的旧文件系统。相对较新的苹果文件系统（APFS）随 macOS High Sierra 发布，具有一系列现代功能和增强的安全性。除了 macOS，APFS 也是 iPhone 和 iPad 等 iOS 设备的文件系统。

### 分层文件系统增强版（HFS+）

Mac 的传统文件系统源于 Mac OS Classic，在操作系统的较新版本中被称为 Mac OS 扩展。它缺乏许多现代功能，如 ZFS、NTFS 和 APFS 中存在的校验和、快照以及多进程同时访问。您仍然可以将驱动器格式化为 HFS+，但 APFS 现在被认为是 macOS 的标准，特别是对于配置为启动卷的固态硬盘（SSD）。

### 苹果文件系统（APFS）

APFS 是 macOS 和 iOS 文件系统的现代标准。它提供了许多现代功能，包括空间共享容器、快照、更强的加密以及比 HFS+ 更快的性能。容器概念对 macOS 来说是新的，在 APFS 中大量使用。容器包含一个或多个单独的卷，这些卷显示为独立的分区，但共享相同的空间，并能根据需求动态增长或缩小。这意味着，当特定卷需要更多空间时，数据可以被分配给该卷，而无需用户交互或知晓。

这种空间分配方案在新的 macOS Catalina 中特别有用，因为系统文件与用户数据位于不同的只读卷上。如果您通过“磁盘工具”查看安装了 macOS Catalina 的驱动器，如图 2-3 所示，您会看到两个不同的卷。Macintosh HD 在一个卷上提供基础操作系统和应用程序，而 Macintosh HD – Data 存储用户目录和用户安装的应用程序。这些都在一个单独的容器中（显示为 Macintosh HD），并且卷会根据需要增长。

### 分区方案与驱动器安全选项

在 macOS 中选择格式化驱动器时，您会注意到有很多选项，包括*区分大小写、加密、日志式*等的各种组合。苹果提供了一篇知识库文章来定义这些选项。

知识库文章可以在此处找到：
* [`https://support.apple.com/guide/disk-utility/file-system-formats-available-in-disk-utility-dsku19ed921c/mac`](https://support.apple.com/guide/disk-utility/file-system-formats-available-in-disk-utility-dsku19ed921c/mac)

除了各种格式化选项外，您可能还会被提示选择驱动器分区方案。有三种选项：
*   **GUID：** 这是基于 Intel 的 macOS 启动卷的标准分区方案选项。
*   **Apple 分区方案：** 现在很少使用，它是 Mac OS Classic 和早期在 PowerPC 架构上运行的 Mac OS X 版本的旧分区方案。
*   **主引导记录：** 这是 Windows 的分区方案，用于 FAT 和 ExFAT 格式。

最后，当您抹掉卷并选择格式时，可能也注意到了“安全选项...”按钮。这些安全选项让您可以选择磁盘上数据被抹除的彻底程度。一些组织要求对所有要停用的磁盘进行完全安全的清理。默认情况下，macOS 使用最不安全（但最快）的单次抹除选项。如果您需要更安全的东西，例如符合国防部 5220.22-M 协议的要求，您可以通过将安全抹除滑块一直移动到最右侧来选择该选项。

### APFS 快照

我们将在下一节更详细地讨论 APFS，但值得一提的是，在系统恢复方面，苹果新文件系统的功能之一就是能够对计算机状态进行文件级快照。如果驱动器格式化为 APFS，当要安装主要的 macOS 更新时，会创建一个 APFS 快照并保存为一个恢复点。通常，操作系统会决定何时适合进行快照，并且这个过程是自动的，因此不需要用户干预。

有时您可能希望强制创建快照。苹果确实通过终端中的命令行工具提供了这种功能。我们将在本章后面更详细地介绍终端，但在这里，只需通过“访达”浏览到 `/应用程序/实用工具` 并启动“终端”应用程序。您将看到一个命令提示符。要创建快照，我们将使用 `tmutil` 命令。在终端的命令提示符下输入以下命令并按回车键以创建快照：

```
tmutil snapshot
```

如果一切顺利，终端会显示一条消息，告知您已创建了一个带有今天日期/时间戳的快照。如果您好奇是否存在任何快照，也可以使用 `tmutil` 命令列出所有可用的快照。快照存储在文件系统级别，仅在操作系统需要时（例如，使用时光机器时）才被挂载。输入以下命令并按回车键：

```
tmutil listlocalsnapshots /
```

如果您需要从快照恢复 Mac，只需启动进入 macOS 恢复，然后选择“从时光机器备份恢复”选项。选择存储快照的驱动器（很可能是您的内置启动磁盘）。您将看到一个可用快照列表（如果存在）。只需选择您想要恢复的快照，然后点击“继续”。恢复过程完成后，您的 Mac 将重新启动，并应恢复到选定快照时的状态。

> **专业提示**
> APFS 快照更多是恢复工具，而非备份工具。我建议将快照用作在重大更改前恢复到某个时间点的一种方式，并继续将整个磁盘备份到另一个驱动器或云端。快照在创建 24 小时后会被删除，因此它们不是可靠的长期备份方法。



## 安装与升级 macOS

从当前 macOS 版本升级至下一个版本，通常的做法是使用 Mac App Store 或“系统偏好设置”中的“软件更新”来下载并安装下一个主要版本。如果你的 Mac 状态良好，只是需要最新的操作系统，这种方法很方便。但如果需要在新格式化的驱动器上全新安装 macOS，我们就得多做一些准备工作。

### 从互联网恢复安装

如果具备网络连接，一个常见的选择是通过“互联网恢复”选项启动至 macOS 恢复模式。在此模式下，你可以抹掉并格式化内置硬盘，然后通过互联网从 Apple 安装全新的 macOS。这种方法虽然有效，但可能耗时较长，并且在校园或大型企业中，如果大量 Mac 共享一个网络连接，则可能不可行。

### 创建可引导的 USB 安装盘

如果你没有可用的网络连接，或者需要为大量 Mac 抹掉内置驱动器并全新安装 macOS，你将需要创建一个或多个可引导的 USB 安装盘。为此，你需要一个至少 8 GB 空间的 USB 闪存盘、一份 macOS 安装器应用程序，并能访问“终端”应用。

第一步是从 Mac App Store 或通过“软件偏好设置”中的“软件更新”下载 macOS 安装器。下载完成后，安装器通常会自行启动并提示你开始安装。此时，取消安装器并退出安装应用程序。使用“访达”浏览到 `/应用程序` 文件夹，确认 `安装 macOS Catalina` 应用程序存在。

如果还未插入 USB 闪存盘，请现在插入并打开“磁盘工具”。将驱动器格式化为 APFS 格式并为其命名。记下这个名称。在我使用的脚本中，磁盘名称为 `Untitled`。接着，浏览到 `/应用程序/实用工具`，然后打开“终端”应用。在提示符处输入以下命令：

```
sudo /Applications/Install\ macOS\ Catalina.app/Contents/Resources/createinstallmedia --volume /Volumes/Untitled --nointeraction
```

如果脚本输入正确，它会提示你输入管理员密码。假设你拥有本地管理员权限，此密码与你登录 Mac 时使用的密码相同。它可能还会要求授予“终端”应用访问已连接的 USB 磁盘的权限，如果是这样，你可以允许此操作。验证通过后，你将在终端中看到输出信息，显示其正在格式化并创建可引导 USB 磁盘，如图 2-4 所示。

### 创建可引导的技术员硬盘

可引导 USB 安装器很有用，但它的局限性同样明显，就像使用 macOS 恢复模式一样，你只能使用有限的实用工具和“终端”应用。如果能拥有一个完整的 Mac 界面，包括桌面、众多应用程序和访达访问权限，那不是更好吗？是的！每位优秀的 Mac 技术人员都应拥有一个包含完整 macOS 副本的可引导外置硬盘，用于故障排除。我推荐使用新型的外置固态硬盘，因为它们更便于携带，且在放入笔记本电脑包中被晃动时更不易磨损。

这类硬盘的目的是将你所有的 macOS 故障排除工具集中在一处。我通常会在我的硬盘上保存各种软件安装器的副本、磁盘急救工具以及此类文件。这对于诊断涉及内置启动宗卷的问题、在没有近期备份的情况下恢复用户数据，或者在不依赖命令行界面的情况下对启动宗卷进行任何其他修改，都极其有帮助。

**专业提示**

从这类外置驱动器启动 Mac 时，特别是由于技术问题无法登录时，请在启动时使用 `option` 键组合，这样你就可以选择该驱动器作为启动盘，而无需启动到操作系统并通过“系统偏好设置”进行设置。

创建这类硬盘非常简单明了。只需将其连接到 Mac，使用 GUID 分区图将其格式化为 APFS 宗卷，然后从 `/应用程序` 文件夹运行 macOS 安装器，但在安装界面步骤中指定外置驱动器。请注意，你可能需要点击“显示所有磁盘”按钮才能看到外置驱动器。一旦 macOS 安装到该驱动器上，你只需按步骤完成“设置助理”并创建本地用户帐户，就像在内部磁盘上安装一样。

**专业提示**

作为安装过程的一部分，你的 Mac 很可能会将外置驱动器设置为新的首选启动宗卷。请务必使用“启动磁盘”系统偏好设置将其改回内置驱动器，否则可能导致 Mac 在启动时寻找外置驱动器，从而在从内部启动磁盘启动前产生不必要的延迟。




## 终端简介

您选择 Mac 是因为它*易于使用*，而且不像 `Windows` 或 `Linux` 那样需要那些命令行界面，对吧？那为什么我们在这本书的第二章就开始讨论命令行界面了呢？！答案很简单——因为您会遇到这样的情况：强大的实用程序*只能*通过终端使用，就像我们刚才为了创建可启动的 USB 安装盘而必须使用的命令所证明的那样。除此之外，使用脚本可以成为一种有效的工具，通过像 Apple Remote Desktop 这样的工具，一次性自动化处理多台 Mac 客户端的各种日常任务。本节将介绍许多您在使用终端或编写自己的脚本时会经常用到的常用命令和概念。

### 提升权限

如果您之前接触过终端，可能已经输入过一两条以 `sudo` 开头的命令。这是一个特殊的命令，用于调用*提升权限*。即使您在 Mac 上属于 `admin` 组，您仍然不是系统上权限最高的用户。那个权力属于 `root` 用户或 `super` 用户。

有些命令只能以超级用户身份运行；在这种情况下，我们需要使用 `sudo` 命令，它是*super user do*的缩写。当您使用以 `sudo` 开头的命令时，系统会提示您输入密码，然后命令才会运行。如果您是 Mac 上的本地管理员，该密码将与您本地用户帐户使用的密码相同。例如，就我而言，我的 `dwsmith` 帐户密码与超级用户的密码相同。



### Shells

关于电脑名称，我们还有另一个潜在问题。仔细看图 2-16 中的终端窗口。您看到命令提示符处的 Unix 主机名是 `dwsmith-MBP` 了吗？为什么我有两个不同的主机名？因为 macOS 有 *三种* 不同的计算机名称——`电脑名称`、`本地主机名` 和 `Unix 主机名`。作为一个最佳实践，我建议将 Unix 主机名和本地主机名设置为一致。在某些情况下这可能不是什么大问题，但以后在远程系统上运行脚本时，这绝对可以为您避免一些潜在问题。我见过一些“恐怖故事”案例，网络上的每台 Mac 的电脑名称都*不同*，而它们的 Unix 主机名却都*相同*。

```

```

现在，您的 Mac 的三个名称将趋于一致，这应该能简化在网络中识别它的任务。

### 应用程序捆绑包

`应用程序捆绑包`，有时也称为`应用程序包`，实际上是一种文件夹或容器，对最终用户来说看起来像一个单一文件，但实际上它是一个包含支持数据的自包含可执行程序包。我们可以`右键单击`应用程序图标，选择`显示包内容`来深入查看此捆绑包中包含的支持文件。这在日常工作中很少使用，但值得了解的是，应用程序不仅仅是表面看起来那么简单。

### 内核扩展

内核扩展，简称`kext`，是一小段加载到操作系统内核中并提供对硬件或其他组件支持的代码。您通常会看到第三方 kext 用于外部硬件，例如通过 Thunderbolt 或 USB 连接的专业音频混音器。您可以通过浏览`/System/Library/Extensions`文件夹或如图 2-43 所示通过终端运行`kextstat`命令来查看这些 kext 文件。

![../images/492151_1_En_2_Chapter/492151_1_En_2_Fig43_HTML.jpg](img/492151_1_En_2_Fig43_HTML.jpg)

**图 2-43: kextstat 命令的输出，显示所有正在运行的内核扩展**

从`macOS High Sierra`开始，苹果引入了一项新的安全功能，要求用户在内核扩展首次需要加载时*批准*它们。系统会通过对话框提示用户批准，然后将用户重定向到*安全性与隐私系统偏好设置*，在`通用`标签页下点击`批准`按钮。苹果在`Mojave`和`Catalina`中继续加强了 kext 的安全性，这两者都要求 kext 必须由开发者进行*代码签名*。同样重要的是要注意，如果您清除了`NVRAM`，所有已授权的 kext 都必须重新授权。

> **专业提示**
>
> 如果您使用的应用程序或工作流程涉及 kext，最佳做法是确保您使用的是最新的应用程序版本，以避免由于苹果不断调整的安全要求而导致问题。在批准客户端系统上的操作系统更新之前，您还应咨询那些广泛使用 kext 的应用程序的开发者，以确保兼容性。即使是像从`10.15.0`升级到`10.15.1`这样的小更新，也存在破坏软件兼容性的风险。

#### 钥匙串访问

任何关于 macOS 客户端管理的章节，如果不讨论令人头疼的 `钥匙串`，都是不完整的！如果你进行过任何一段时间的 Mac 支持工作，很可能遇到过关于 `登录` 钥匙串的晦涩信息或其他恼人的与钥匙串相关的错误。钥匙串是什么，为什么它给我的用户带来这么多麻烦？

钥匙串可以通过位于 `/Applications/Utilities` 的 `钥匙串访问` 实用程序进行管理，它存储了 Mac 上用于访问各种网站、安全应用程序、服务和无线网络的所有不同用户名和密码组合。它还可以存储证书和加密密钥。你的 Mac 上有多个钥匙串，包括 `系统`、`本地项目` 和 `登录`，如果你已登录并激活了它，`iCloud 钥匙串` 还可以通过在多个 Apple 设备间同步项目来发挥作用。图 2-45 展示了钥匙串访问应用程序以及我的 Mac 上的各种钥匙串、密码、证书和密钥。

![../images/492151_1_En_2_Chapter/492151_1_En_2_Fig45_HTML.jpg](img/492151_1_En_2_Fig45_HTML.jpg)

图 2-45

在登录钥匙串中找到的钥匙串访问数据

虽然这看起来相当简单和无害，但钥匙串的问题通常始于用户密码被重置或更改时。密码更改后，用户下次登录时，可能会看到一个对话框，提示他们使用新密码更新登录钥匙串或创建新的登录钥匙串。用户毕竟是用户，他们可能会选择错误的选项、跳过此步骤，或者不知道他们的旧密码而无法有效更新它，现在我们遇到了臭名昭著的登录钥匙串问题。

如果用户知道旧密码，这个问题相对容易解决。只需打开 `钥匙串访问`，选择 `登录` 钥匙串，然后从 `编辑` 菜单中选择 `更改钥匙串“登录”的密码`。它会提示输入旧密码和新密码，然后相应地更新钥匙串。

但如果用户不知道旧密码呢？在这种情况下，我们必须创建一个新的登录钥匙串。为此，请打开 `钥匙串访问`，然后从 `钥匙串访问` 菜单中选择 `偏好设置…`。在偏好设置对话框中，点击 `重置我的默认钥匙串` 按钮，如图 2-46 所示。请注意，这将删除 `登录` 和 `本地项目` 钥匙串，并用新的*默认*钥匙串替换它们。这意味着存储在这些钥匙串中的任何密码或条目都将被删除，用户将需要重新登录各种网站和服务以重新建立这些密钥。
